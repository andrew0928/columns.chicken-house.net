# 敗家: HTPC

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 什麼是 HTPC？
- A簡: HTPC 是以電腦為核心的客廳影音系統，整合影片、音樂、相片與錄影，透過大螢幕與遙控器提供沙發化操作。
- A詳: HTPC（Home Theater PC）指以個人電腦組建的家庭劇院中樞，連接電視與音響，透過如 Windows Media Center（MCE）等 10-foot UI 操作介面，整合播放 DVD、MPEG4/DivX、MP3 與相片瀏覽，亦可搭配 TV 卡錄影與看節目表。其核心價值在於彈性高、整合既有數位媒體，並藉由遙控器或手機遠端控制，實現客廳友善的使用體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, A-Q7, B-Q2

Q2: 為何需要專用 HTPC，而不是沿用桌機？
- A簡: 多螢幕與操作混亂讓體驗不佳，MCE 常開在錯誤螢幕；專用機可簡化流程與提升穩定性。
- A詳: 以既有桌機兼做 HTPC 易遇多顯示器管理、視窗定位與輸出介面限制，例如同時驅動雙 DVI 螢幕與 HDMI 電視需兩張顯卡，且 MCE 有時會在桌面螢幕而非電視開啟，增加操作成本。專用 HTPC 可針對客廳情境調校輸出、音訊與電源管理，搭配遙控器與 Sleep 快速喚醒，避免鍵鼠與桌機工作干擾，帶來一致、低摩擦的體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q13, C-Q9, D-Q1

Q3: HTPC 與傳統桌上型電腦在客廳使用的差異？
- A簡: HTPC強調10-foot介面、遙控操作、低噪低耗與影音輸出；桌機偏向鍵鼠與近距離工作。
- A詳: 傳統桌機以鍵盤滑鼠、近距離（2-foot UI）操作為主，並非針對遠距離大螢幕互動；HTPC需面向沙發情境（10-foot UI），提供大字體、簡化選單、遙控器支援與睡眠秒喚醒。硬體上，HTPC偏向低功耗、安靜與影音輸出完整（HDMI/DVI），並整合影音庫與錄影功能。這些差異決定了選購組件、作業系統設定與日常操作方式的不同。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, A-Q7, B-Q14

Q4: HDMI 與 DVI 有何差異？如何選擇？
- A簡: 兩者皆為數位影像；HDMI常用於電視並可傳音訊，DVI多見於電腦螢幕，只傳影像。
- A詳: HDMI 與 DVI 均傳輸 TMDS 數位視訊，畫質在規範內無本質差異。實務上，客廳電視多以 HDMI 為主（並可承載音訊），電腦螢幕以 DVI 常見。若電視僅提供 HDMI/色差，優先用 HDMI 確保全數位串接；雙螢幕桌面則多用 DVI。三輸出（兩 DVI 螢幕 + 一 HDMI 電視）時，選擇具足夠數位輸出的顯示方案尤為關鍵。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q6, D-Q2

Q5: 什麼是 Windows Media Center（MCE）？
- A簡: MCE 是面向客廳的媒體介面，整合影片、音樂、相片與電視錄影，支援遙控器與大螢幕。
- A詳: Windows Media Center 是 Windows 平台的 10-foot 使用者介面，強調遠距離操作。它整合影音播放（DVD、MPEG-2、DivX/MPEG4 等）、音樂與相片瀏覽、媒體庫管理與節目表/錄影（搭配 TV 卡）。配合 MCE 專用遙控器與 IR 接收器，可用「綠色按鈕」快速啟動、以電源鍵控制 Sleep/喚醒，並將客廳操作完全鍵鼠化為遙控器體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, B-Q2, C-Q5

Q6: 什麼是 Windows SideShow？
- A簡: SideShow 讓外部裝置（如手機）顯示與控制 PC 的資訊，例如 MCE 曲目與節目表。
- A詳: Windows SideShow 是一項將電腦資訊推送至輔助顯示裝置的技術。透過安裝 SideShow 裝置驅動與相對應 Gadget，外部設備（如 WM5/WM6 手機）可在不打擾主螢幕下，顯示 MCE 的節目表、正在播放曲目，並執行播放/暫停等控制。此機制十分適合不開電視時控制音樂，或在播放不中斷情況下進行媒體瀏覽。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q12, C-Q4

Q7: 什麼是 MCE 遙控器？有哪些特點？
- A簡: MCE 遙控器具綠色鍵啟動、媒體快捷鍵與電源鍵可控 Sleep/喚醒，免鍵鼠操作。
- A詳: MCE 遙控器搭配 eHome IR 接收器使用，提供直接啟動 Media Center 的綠色按鍵、播放/暫停、音量、導航等專用鍵。其電源鍵預設對應系統 Sleep 與喚醒，使用者可實現「按一下就開、再按就睡」的客廳體驗。安裝上通常免驅動（Vista x64 亦可即插即用），適合全面以遙控器取代鍵盤滑鼠操作影音內容。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q2, C-Q2, D-Q3

Q8: 為何本文未配置 TV 卡？
- A簡: 因已有兩台電腦具 TV 錄影功能，且少在客廳看 Live TV，考量成本暫不添購。
- A詳: 作者的使用情境以播放既有影音檔與音樂為主，看電視多以錄影回看而非即時收看；家中其他桌機已安裝 MCE + TV 卡可負責錄影。因此第三張 TV 卡的效益偏低，加上市售卡價位較高，遂選擇不在客廳 HTPC 安裝，將資源集中於輸出、遙控體驗與低功耗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q7, D-Q8

Q9: 低耗電 HTPC 的核心價值是什麼？
- A簡: 低耗與快喚醒帶來長時間待命、即開即用與靜音體驗，兼顧省電與便利。
- A詳: 以 45W 等級 CPU、整合顯示與高效率電源打造的 HTPC，開機後閒置僅約 50–55W，播放影音約 60–65W，重負載 75–80W；進入 Sleep 更只需 1–2W。這使其能長時間待命，按遙控電源即時喚醒觀看，結束後再睡眠，兼顧省電、安靜與使用便利，特別適合客廳的零等待操作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, C-Q3, D-Q6

Q10: AMD 780G 平台對 HTPC 有何優勢？
- A簡: 成本效益佳、整合顯示輸出齊全、搭配低功耗 CPU，整體功耗與預算友善。
- A詳: 文中評估以 780G 為核心的主機板方案，較更換可插雙顯卡主機板與中高階顯卡便宜。整合顯示平台通常提供足夠的數位輸出與影音處理能力，支援日常 DVD/DivX 播放，且與低功耗處理器（如 45W TDP 之 4850e）搭配，可實現低待機與低播放功耗，符合 HTPC 對安靜、節能與足夠影音性能的需求。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, B-Q22, C-Q1

Q11: 什麼是電腦的 Sleep 模式？在 HTPC 為何重要？
- A簡: Sleep 將狀態保存在記憶體、降至極低功耗，遙控可瞬間喚醒，提升客廳體驗。
- A詳: Sleep（待命）會將系統狀態維持於記憶體並關閉大多數元件，功耗可降至 1–2W 等級，遙控器或 USB 事件可喚醒回到原狀。相較完整關機/開機需較長等待，Sleep 在 HTPC 場景能「一按即開，一按即睡」，滿足頻繁短時使用。要發揮效果，需正確設定 BIOS 的 USB 喚醒與系統電源管理策略。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q3, D-Q3, D-Q6

Q12: 為什麼選擇 45W 的 AMD 4850e 處理器？
- A簡: 它在影音負載足夠的同時大幅降低功耗與發熱，利於靜音與小電源配置。
- A詳: HTPC 的主要負載是影音解碼與介面操作，屬中低至中等運算需求。45W TDP 的 4850e 提供足夠效能播放 DVD、MPEG4 等，同時降低平台整體功耗，讓整機在播放時約 60–65W、待機僅 1–2W，並可搭配較小功率、效率佳的電源與靜音散熱，提升客廳環境的舒適度與成本效益。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, B-Q17, C-Q1

Q13: 為何需要三個數位輸出（DVI×2 + HDMI）？
- A簡: 兩台桌面螢幕用 DVI、電視需 HDMI，為保持全數位畫質與簡化連接所需。
- A詳: 作者原想以桌機同時驅動兩台 DVI 螢幕與一台僅提供 HDMI 的 SHARP 電視，為避免類比色差或 D-Sub 造成畫質損失，堅持全數位輸出。多數顯卡最多兩路同時輸出，於是需要主機板支援雙顯卡或另找整合輸出方案，這成為採購與設計 HTPC 的關鍵考量之一。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q22, C-Q6, D-Q1

Q14: DVI/HDMI 與 D-Sub/色差有何本質差異？
- A簡: DVI/HDMI 為數位傳輸、無轉換損失；D-Sub/色差為類比，易受線材與干擾影響。
- A詳: 數位（DVI/HDMI）可點對點傳遞像素資料，畫質穩定且抗干擾，並支援 EDID 協商；類比（VGA D-Sub/色差）則需 DAC 轉換，訊號會受線材與距離影響，細節與銳利度較不穩。對於僅提供 HDMI/色差的電視，選用 HDMI 能維持畫質一致性，也簡化音訊路徑的整合可能。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q6, D-Q2

Q15: 便宜的 HDMI 線是否可用於 HTPC？
- A簡: 在合規距離與品質下便宜線可用；本文 3 米線表現正常，畫質無差異。
- A詳: HDMI 為數位傳輸，只要線材達規格且距離內信號完整，畫質即為「到或不到」。文中使用 3 公尺、價格低廉的 HDMI 線連接電視，實際播放 DVD/MPEG4 均表現良好。除非超長距離或高干擾環境，無須迷信昂貴線材；更應關注連接器接觸、線材合規與佈線可靠性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q23, C-Q6, D-Q2

Q16: MCE 綠色按鈕的功能是什麼？
- A簡: 按下即啟動 Media Center 主介面，提供快速進入 10-foot UI 的入口。
- A詳: MCE 遙控器上的綠色按鈕為系統層級快捷鍵，對應啟動（或帶到前景）Media Center。它讓使用者不需鍵鼠即可一鍵進入媒體主選單，搭配電源鍵的 Sleep/喚醒，可形成「喚醒→綠鍵→播放」的流暢操作鏈，強化客廳的即時性與易用性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q19, C-Q2, D-Q1

Q17: 為何在客廳操作要避免鍵盤與滑鼠？
- A簡: 鍵鼠不利遠距離與家人共用；遙控器/手機更符合沙發化、簡單、一致的體驗。
- A詳: 客廳場景講求遠距離、多人共享與低學習成本。鍵鼠需近距離擺放、佔空間且操作繁複，不利家人使用。遙控器提供直覺按鍵與導航，手機/SideShow 則能在不開電視下完成控制。以遙控為中心可簡化流程、降低錯誤與干擾，提升 HTPC 的日常可用性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, A-Q19, C-Q4

Q18: 在 MCE 中播放 MP3 與相片有何好處？
- A簡: 單一介面整合媒體庫，遙控即可操作，免切換應用程式，客廳體驗一致。
- A詳: 相較以 Media Player/Photo Gallery 分別處理，MCE 將音樂、相片與影片置於同一 10-foot 介面，遙控器可一鍵切換並瀏覽。這減少應用切換與鍵鼠依賴，讓家庭成員以相同流程操作，並能結合播放清單、投影片等功能，打造一致的客廳娛樂入口。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, B-Q7, D-Q5

Q19: SideShow 與傳統 MCE 遙控器差異是什麼？
- A簡: 遙控器盲控直覺；SideShow 可顯示節目表與曲目，適合不開電視與不中斷播放。
- A詳: 傳統遙控器提供快速、盲控的播放與導航，適合大多數情境；SideShow 裝置（如 WM5/6 手機）透過 Gadget 顯示節目表、當前曲目等資訊，且不會中斷電視畫面，尤其適合僅聽音樂、不開電視的情境。兩者互補：一般操作用遙控，資訊瀏覽與隱性控制用 SideShow。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q20, C-Q4, D-Q5

Q20: 為何「二合一」的 SideShow + MCE 遙控器價格高？
- A簡: 其整合顯示與遙控硬體，需支援 SideShow 規格與 MCE 控制，成本較高。
- A詳: 市面上曾有符合 SideShow 規格且同時為 MCE 遙控器的產品，內建顯示螢幕與相關通訊模組，能顯示節目表/曲目並進行 MCE 控制。然而顯示元件、通訊與韌體開發使 BOM 與研發成本上升，價格約 USD 199。相較之下，「平價 MCE 遙控器 + 現有手機」是更具成本效益的替代方案。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q19, C-Q4

Q21: 為何選用 330W 高品質電源而非更大瓦數？
- A簡: 整機滿載不到 100W，高品質 330W 足夠且效率高，提升穩定與靜音。
- A詳: 電源最佳效率區通常落在額定功率的中低段。本文實測整機重載僅 75–80W，選擇 330W 等級已提供足夠餘裕，且在日常負載下處於高效率區，減少熱與噪音。相較名不見經傳的大瓦數電源，選擇信賴品牌可提升穩定性與壽命，符合 HTPC 的長時待命與靜音需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q10, D-Q6, C-Q10

Q22: HTPC 與機上盒（Set-top Box）有何差異？
- A簡: HTPC 可高度自訂與整合個人媒體；機上盒易用但彈性低、擴充受限。
- A詳: 機上盒主打開箱即用、服務綁定與固定功能；HTPC 作為 PC 平台，可彈性配置硬體、儲存與輸出，整合個人媒體庫與錄影，並支援多種控制方式（遙控、手機）。代價是初期組裝與設定較複雜。若重視個人媒體整合與可塑性，HTPC 更合適；若追求極簡即用，機上盒更省事。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q5, C-Q1

Q23: 為何不考慮 USB/PCI 顯示輸出解決方案？
- A簡: 這類方案效能與相容性常不足，難以滿足多媒體播放與三輸出需求。
- A詳: 文章指出 USB/PCI 類延伸顯示卡效能偏弱，帶寬與延遲限制使其不適合作為主要影音輸出，更遑論同時三路數位輸出與流暢播放。對 HTPC 而言，穩定、高品質的視訊輸出與解碼能力更重要，因此倾向使用整合度高、輸出完整的顯示平台或獨顯，而非外掛式低效解法。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q9, B-Q22, D-Q7

Q24: 在 HTPC 中，睡眠/喚醒與完整開關機如何取捨？
- A簡: 睡眠秒喚醒、耗電 1–2W；完整開關機更省電但等待久，客廳建議以睡眠為主。

Q3: 如何設定 HTPC 的睡眠與喚醒？

### Q&A 類別 D: 問題解決類（10題）

Q1: MCE 有時開在錯誤螢幕怎麼辦？
- A簡: 症狀為 MCE 不在電視上；設電視為主顯示器、固定全屏或改用專用 HTPC。
- A詳: 症狀：按綠鍵後 MCE 出現在桌面螢幕。原因：多螢幕位置記憶、主顯示器切換與 EDID/驅動差異。解法：1) 設電視為主顯示器；2) 全屏啟動 MCE；3) 簡化為單顯示模式觀影；4) 考慮建立專用 HTPC。預防：固定接線、避免頻繁切換主顯示器，保持解析度一致。

- 學習階段: 核心
- 關聯概念: B-Q13, C-Q9, A-Q2

Q2: 連接電視 HDMI 無訊號或畫面異常怎麼辦？
- A簡: 症狀為黑屏/錯解析；確認線材、解析度與輸入源，必要時改用 DVI→HDMI。
- A詳: 症狀：無訊號/超出範圍。原因：線材不良、輸入源選錯、解析度不支援、HDCP/EDID 協商失敗。解法：1) 確認電視輸入源；2) 減短或更換 HDMI 線；3) 設定電視原生解析度與刷新率；4) 嘗試 DVI→HDMI 串接。預防：使用合規線材、固定接頭，避免熱插拔頻繁導致握手失敗。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q4, C-Q6, B-Q8

Q3: 遙控器無法喚醒電腦怎麼辦？
- A簡: 症狀為睡眠中無反應；啟用 USB Wake、允許 IR 裝置喚醒，檢查睡眠狀態。
- A詳: 症狀：按遙控電源鍵無反應。原因：BIOS 未啟用 USB Wake、IR 裝置未允許喚醒、處於關機非睡眠。解法：1) BIOS 啟用 USB Wake；2) 裝置管理員勾選允許喚醒；3) 確認 Windows 電源鍵行為為睡眠；4) 測試其他 USB 埠。預防：固定睡眠策略、避免 USB 集線器影響供電。

- 學習階段: 核心
- 關聯概念: B-Q5, C-Q3, C-Q8

Q4: 手機 SideShow 連不上或不穩定怎麼辦？
- A簡: 症狀為配對失敗/掉線；重配對、更新驅動、靠近裝置並重啟 Gadget。
- A詳: 症狀：無法配對或操作延遲。原因：藍牙堆疊相容性、Gadget Beta 不穩、干擾。解法：1) 解除配對重建；2) 更新藍牙驅動/韌體；3) 重新啟動 SideShow 服務與 Gadget；4) 減少干擾、縮短距離。預防：固定配對的裝置名稱與 PIN，維持系統更新，避免同時多藍牙設備干擾。

- A詳: 症狀：需開電視才看得到媒體庫。原因：傳統遙控無顯示。解法：1) 配置 SideShow 手機，直接選專輯並播放；2) 在 MCE 建立播放清單，熟悉按鍵盲控。預防：規劃清晰的媒體庫結構與封面，常用歌單置頂，減少找尋成本。

Q6: HTPC 耗電高或過熱怎麼辦？

- A詳: 症狀：閒置功耗高、溫度大。原因：高 TDP CPU、顯示驅動設定不當、背景程式、低效率電源。解法：1) 設定省電計劃；2) 更新驅動、關閉特效；3) 整理風道；4) 以低 TDP CPU/整合顯示與高效率電源。預防：定期清理灰塵、檢視待機功耗，維持 Sleep 工作流。

- A詳: 症狀：畫面超出或未滿版。原因：EDID 重新讀取導致縮放變更、過掃設定不匹配。解法：1) 固定輸出解析度與刷新率；2) 在 GPU 控制台設定縮放/過掃補償；3) 儘量避免在播放中切換電視電源。預防：使用穩定 HDMI 線材與固定輸入源，減少握手重新協商。
