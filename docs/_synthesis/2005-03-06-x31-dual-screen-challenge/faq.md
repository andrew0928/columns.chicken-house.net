# X31 + 雙螢幕的挑戰 @_@

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是雙螢幕（延伸桌面）？
- A簡: 雙螢幕是用兩台顯示器擴展同一作業系統桌面，非鏡像。可同時顯示更多視窗以提升效率。
- A詳: 雙螢幕（延伸桌面）指把作業系統的工作區域擴展到兩台顯示器上，使滑鼠與視窗可跨螢幕移動，不同於「鏡像」僅顯示相同內容。它能在一邊放主要工作視窗，另一邊放參考資料、通訊或監控面板，減少切換視窗的頻率，提升專注與效率。以 X31 筆電搭配外接 17 吋 LCD 為例，可將原 1024x768 筆電畫面與 1280x1024 外接螢幕組成更大的工作空間。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q1, C-Q1

A-Q2: 使用雙螢幕的核心價值是什麼？
- A簡: 提升生產力、減少視窗切換、改善資訊可見性，特別適合文件閱讀、程式開發與多工。
- A詳: 雙螢幕的價值在於增加有效視野，讓多個資訊面板同時可見。例如程式開發時左側畫程式碼、右側看文件或偵錯；影像工作可分離工具欄與畫布；日常辦公可同時看郵件、文件與即時通訊。相較頻繁 Alt+Tab 切換，延伸桌面更符合人類視覺的並行處理。搭配直立（Pivot）螢幕，閱讀長文、程式碼與網頁更有效率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q3, C-Q9

A-Q3: 什麼是直立（Pivot）顯示？
- A簡: 將顯示器物理旋轉 90 度，並在系統中旋轉輸出，形成縱向畫面，利於閱讀長內容。
- A詳: 直立顯示（Pivot）是把顯示器旋轉 90 度並讓系統輸出相對應的旋轉影像，使顯示的有效高度增加。這對閱讀文件、程式碼、網站、PDF 等縱向資訊非常實用。以 1280x1024 面板為例，旋成 1024x1280 後可同屏顯示更多行數。Pivot 可由顯示卡驅動原生支援，或用第三方軟體（如 Pivot Pro）達成，後者與顯示卡品牌無關。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q5, B-Q2, C-Q2

A-Q4: 為何直立使用時「上下視角」格外重要？
- A簡: 直立後原本左右視角變成上下方向，TN 面板上下視角窄，易出現色偏與亮度不均。
- A詳: 多數 TN 面板水平方向視角較寬、垂直較窄。當螢幕直立後，原本的「垂直視角」成為你觀看的「上下」方向，容易出現上暗下亮或色偏現象，影響閱讀舒適度。因此挑選可 Pivot 的顯示器時，需重視上下視角規格（或選擇視角更寬的 IPS/VA），並搭配可調整高度與傾角的腳座以對齊視線，減少視角引起的失真。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q6, D-Q7

A-Q5: 顯示卡「驅動旋轉」與「軟體旋轉」有何差異？
- A簡: 驅動旋轉由 GPU 執行，效能佳；軟體旋轉由程式轉換畫面，通用但較吃 CPU。
- A詳: 驅動旋轉是顯示卡在掃描輸出前於硬體中完成座標與像素轉換，延遲低、相容性與影片播放體驗較佳，但受限於驅動功能與模式支援。軟體旋轉（如 Pivot Pro）在軟體層拋轉畫面，幾乎不挑顯示卡，能在驅動不支援時解決問題，但會耗用額外 CPU/GPU 資源，可能影響滑鼠游標流暢度與影片播放表現。選擇依需求與相容性決定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q3, D-Q2

A-Q6: 什麼是色彩深度？16 位元與 32 位元差在哪裡？
- A簡: 色深指每個像素可表達的位元數。16 位元較省記憶體但色階少，32 位元色彩細緻。
- A詳: 色彩深度（Color Depth）是每像素使用的位元數，常見有 16 位元（High Color）與 32 位元（True Color，含 8 位元透明度）。位元數越高，色彩階數、漸層與細節越佳，但 VRAM 占用也越高。在雙螢幕或旋轉情境下，若 VRAM 限制或驅動相容性不足，可能需將其中一個螢幕降至 16 位元以確保穩定運作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, B-Q5, D-Q3

A-Q7: 什麼是視訊記憶體（VRAM），與雙螢幕有何關係？
- A簡: VRAM 儲存畫面像素與圖形資源。雙螢幕與旋轉會增加記憶體需求與佔用。
- A詳: VRAM 是顯示卡用來存放畫面緩衝區（Framebuffer）、圖形表面與相關資源的記憶體。解析度越高、色深越大、畫面越多（雙螢幕）、旋轉或特效越多，對 VRAM 的需求越高。即使表面計算看似足夠，驅動還需額外緩衝與離屏表面，實際占用可能超出額定容量，導致模式受限、關閉旋轉或強制降色深。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q5, D-Q3, C-Q4

A-Q8: DVI 與 VGA 有何差異？
- A簡: DVI 為數位訊號，抗干擾佳；VGA 為類比訊號，線長與品質影響畫質。
- A詳: VGA 使用類比 RGB 訊號，訊號衰減、噪聲與線材品質會影響圖像清晰度與銳利度。DVI（或後來的 HDMI/DP）以數位傳輸像素資料，能維持像素級對位與清晰度。對 1280x1024 的 17 吋 LCD，VGA 多數仍可接受，但長線路或劣質線材可能模糊。若硬體不支援 DVI 輸出（如 X31 機型），即使 Dock 有 DVI 端子也無法使用。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, B-Q12, D-Q8

A-Q9: 為什麼 ThinkPad X31 的 Dock 雖有 DVI 孔卻不能輸出？
- A簡: 因 X31 顯示核心不提供 DVI 訊號，Dock 的 DVI 端子無法啟用，僅具外觀。
- A詳: 某些 Dock 的 DVI 端子是「直通」或「由顯示核心提供 TMDS 訊號」的設計，若主機（X31）顯示晶片未輸出對應數位訊號，就無實際功能。此時只能使用 VGA 類比輸出。選購或使用 Dock 時需確認機型相容表。若需數位輸出，只能更換支援 DVI/DP 的機種或外接顯示轉接器（當年選項有限）。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, C-Q5, D-Q4

A-Q10: 17 吋 1280x1024（5:4）LCD 有何特性？
- A簡: 典型商用規格，文字清晰、成本低，旋轉為 1024x1280 適合閱讀與編輯。
- A詳: 17 吋 1280x1024 面板屬 5:4 比例，像素密度適中，當年廣泛用於辦公。橫向使用顯示區域均衡；直立後成 1024x1280，縱向顯示空間大幅提升，適合文檔、程式碼與網頁。多採 TN 面板，需留意上下視角。對不玩遊戲的使用者，反應時間與高刷新率非首要，反而是視角、支架可旋轉性與價格更重要。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q4, C-Q6

A-Q11: CRT 與 LCD 在雙螢幕上的差異？
- A簡: CRT 體積大耗電高但無固有解析度；LCD 薄省電有原生解析度，直立更易用。
- A詳: CRT 可在多解析度下維持幾何比例，但體積與耗電大且會老化亮度下降。LCD 有原生解析度，該解析度最清晰，偏離會插值模糊；但重量輕、支架多支援 Pivot，適合直立使用。對雙螢幕與長時間辦公，LCD 的眼睛負擔、桌面空間與可旋轉性更有優勢。本文案例即由 19 吋 CRT 退役改為 17 吋 LCD。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q10, B-Q15

A-Q12: 不玩遊戲為何對「反應時間」不敏感？
- A簡: 文書與瀏覽對畫面殘影容忍高，重點在清晰度與舒適度，反應時間非關鍵。
- A詳: 反應時間影響動態畫面的殘影與拖影，對高速遊戲、影片運動場景較關鍵。辦公、程式與網頁主要呈現靜態或低速變化內容，對殘影較不敏感，因此更應關注字體清晰、視角、亮度均勻與護眼設定。對直立閱讀，視角與可調支架的影響往往大於毫秒級反應時間差異。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q10, B-Q15

A-Q13: 為何 32MB VRAM 可能無法同時支援兩個 32 位元顯示？
- A簡: 除畫面緩衝外，驅動與旋轉需額外表面記憶體，實際占用超過可用容量。
- A詳: 表面估算僅計算 Framebuffer，但實際還包含離屏表面、指針、位移與旋轉緩衝、硬體加速資源等。雙螢幕、不同解析度與色深、旋轉與疊加（Overlay）會使驅動保留額外空間。32MB 名義容量在某些驅動組合下不足，因此需把其中一個顯示降至 16 位元（Hicolor）或調降解析度，或改用軟體旋轉以迴避驅動限制。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, A-Q7, B-Q5, D-Q3

A-Q14: Pivot Pro 是什麼？解決了什麼問題？
- A簡: 第三方旋轉軟體，與顯示卡無關，能在驅動不支援下提供 90 度旋轉。
- A詳: Pivot Pro 是用戶層或過濾驅動層實作的桌面旋轉工具，支援 90/180/270 度旋轉、熱鍵與每螢幕獨立設定。當顯示卡驅動缺少旋轉功能、或在特定雙螢幕色深/解析度組合下禁用旋轉時，Pivot Pro 可繞過限制。代價是可能增加 CPU 使用與延遲，對影片播放有影響，但對辦公閱讀與程式開發十分實用。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, B-Q3, C-Q2

A-Q15: 為何某些驅動在混合解析度與色深下禁用旋轉？
- A簡: 驅動內部能力矩陣受限，混合像素格式與多顯示輸出時，旋轉路徑被關閉。
- A詳: 顯示驅動會根據顯示引擎、像素格式、記憶體盤點與輸出通道能力決定可用功能。混用 32/16 位元、不同解析度與雙輸出時，部分硬體路徑（含旋轉與疊加）無法同時成立，驅動會隱藏或禁用選項。此時可改用軟體旋轉、統一色深、調整解析度，或更新驅動版本測試是否改善相容矩陣。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, D-Q1, D-Q3

### Q&A 類別 B: 技術原理類

B-Q1: 雙螢幕在顯示管線中如何運作？
- A簡: GPU 以多顯示輸出引擎讀取各自緩衝，經定時器與轉換後輸出至兩台螢幕。
- A詳: 技術原理說明：現代 GPU 具多個掃描輸出（CRTC/Timing Generator），各自讀取對應的 Framebuffer，經縮放與色彩轉換後送到 DAC（VGA）或 TMDS（DVI）。關鍵步驟或流程：EDID 讀取→模式決策→建立每螢幕表面→啟用延伸桌面。核心組件介紹：CRTC、Overlay/Scaler、Color LUT、DAC/TMDS 及記憶體控制器。X31 雙螢幕即利用內建 VGA 輸出與內部顯示面板共同運作。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q7, B-Q13

B-Q2: 螢幕旋轉（90°）背後的機制是什麼？
- A簡: 透過座標轉換與記憶體重取樣，將原橫向像素映射至直向輸出。
- A詳: 技術原理說明：旋轉需要將畫素位置 (x,y) 轉換為 (y,h−1−x) 等對應，並以行/列交換的掃描順序輸出。關鍵步驟或流程：建立旋轉表面→座標映射→像素格式處理→輸出同步。核心組件介紹：硬體旋轉單元（若有）、顯示引擎的縮放/轉向模組、或軟體層的圖形 API 拋轉。硬體旋轉延遲低；軟體旋轉須額外緩衝並可能佔用 CPU。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q3, A-Q5, B-Q3

B-Q3: 軟體旋轉（如 Pivot Pro）如何運作？
- A簡: 在用戶或驅動過濾層攔截桌面輸出，重排像素並送回顯示管線。
- A詳: 技術原理說明：軟體旋轉可透過 GDI/DirectDraw/Direct3D 的表面攔截或虛擬顯示驅動，將原始桌面渲染至離屏表面後再旋轉送出。關鍵步驟或流程：建立虛擬畫面→旋轉與合成→輸出至實體顯示。核心組件介紹：過濾驅動（Filter Driver）、用戶態服務、圖形 API 封裝層。優點是相容性高；缺點是增加延遲與 CPU/GPU 使用。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, A-Q14, D-Q2

B-Q4: 為何旋轉會增加記憶體與計算負載？
- A簡: 旋轉需額外緩衝與像素搬移，導致更多 VRAM 與記憶體頻寬消耗。
- A詳: 技術原理說明：旋轉時原始 Framebuffer 不再與掃描順序一致，需建立中介表面或使用硬體轉向單元，增加一次讀寫。關鍵步驟或流程：渲染→中介緩衝→轉向輸出。核心組件介紹：離屏表面、旋轉單元、記憶體控制器。多螢幕時此額外成本乘以顯示數量，若 VRAM 吃緊或頻寬不足，驅動可能停用旋轉以維持穩定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q7, A-Q13, B-Q5

B-Q5: 色深與 VRAM 需求如何計算與規劃？
- A簡: VRAM≈寬×高×位元/8×顯示數，加上離屏緩衝；保留裕度以免超限。
- A詳: 技術原理說明：單一畫面 VRAM 需求為 W×H×(bpp/8)。雙螢幕為兩者和，另需額外離屏表面與旋轉緩衝。關鍵步驟或流程：估算每螢幕需求→加總→加上 30–100% 裕度（依驅動）。範例：1024×768×32b≈3MB；1280×1024×32b≈5MB；名義總和約 8MB，但加上離屏、旋轉與疊加後可能接近或超過 32MB 限制。核心組件介紹：Framebuffer、Offscreen Surfaces、Overlay。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, A-Q7, A-Q13

B-Q6: 為何混合 16/32 位元色深會影響驅動功能？
- A簡: 不同像素格式需不同處理管線，部分硬體與驅動難以同時支援旋轉等特性。
- A詳: 技術原理說明：混合像素格式會導致渲染與輸出管線需同時維持多種表面，含色彩轉換與 Alpha 處理。關鍵步驟或流程：建立異格式表面→同步顯示引擎設定→協調硬體資源。核心組件介紹：像素格式轉換器、顯示引擎能力矩陣。為簡化與穩定，驅動常在異格式組合下關閉進階功能（如旋轉/Overlay），以避免資源衝突或超額占用。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q6, A-Q15, D-Q1

B-Q7: Dock 上 DVI 端子的硬體架構與限制？
- A簡: 多為顯示核心直通或必要信號轉出。若主機無 TMDS 輸出，端子即形同虛設。
- A詳: 技術原理說明：部分 Dock 僅把主機內顯的 DVI 訊號引出，並非自行產生數位輸出。關鍵步驟或流程：GPU 產生 TMDS→Dock 連接器引出→DVI 端子輸出；若 GPU 無 TMDS，流程中斷。核心組件介紹：TMDS 發射器、Dock 背板、連接器匯流。X31 顯示核心不輸出 DVI，故 Dock 的 DVI 端子不可用，只能改接 VGA。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q9, C-Q5, D-Q4

B-Q8: TN 面板上下視角狹窄的原因與影響？
- A簡: 液晶排列造成垂直方向灰階與色度變化敏感，直立後更易顯示不均。
- A詳: 技術原理說明：TN（Twisted Nematic）因分子偏轉特性在垂直視角下光穿透變化大，導致 Gamma shift。關鍵步驟或流程：視角偏移→亮度/對比變化→色偏。核心組件介紹：偏光片、液晶盒、背光模組。直立使用時原垂直視角成為觀看主方向，影響更大。建議選 IPS/VA 或評測上下視角，並用可調腳座與適當高度校正視線。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, C-Q6, D-Q7

B-Q9: 1280x1024（5:4）解析度與像素密度的影響？
- A簡: 5:4 比例在直立為 1024x1280，縱向資訊密度提升，需適當 DPI 與縮放。
- A詳: 技術原理說明：物理像素固定，改變方向僅變座標；直立後垂直像素變多。關鍵步驟或流程：設定正確原生解析度→啟用 ClearType→調整縮放/DPI。核心組件介紹：EDID、縮放引擎、字型平滑。對 17 吋 1280×1024，標準 96DPI 下文字清楚；直立後建議適度放大（110–120%）以舒適閱讀並避免過小字。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q7, C-Q2, A-Q10

B-Q10: ATI 驅動中的旋轉參數（登錄檔）如何運作？
- A簡: 透過登錄檔開關顯示旋轉功能，啟用後面板與選單出現旋轉選項。
- A詳: 技術原理說明：驅動以登錄檔旗標控制 UI 與功能顯示。關鍵步驟或流程：定位 HKEY_LOCAL_MACHINE\SOFTWARE\ATI Technologies\Desktop\{GUID}\Rotation→值 00→01→重新載入驅動/重開機。核心組件介紹：登錄檔配置、ATI 控制面板、Tray Icon。啟用後可見旋轉選單，但仍受模式組合限制而可能無法實際旋轉。

B-Q11: 為何某些模式下 ATI 隱藏旋轉選項？

- A詳: 技術原理說明：驅動初始化時檢查可用硬體資源（VRAM、Overlay、Scaler）及模式相容性，若組合不滿足旋轉需求即停用。關鍵步驟或流程：模式切換→能力重評估→UI 變更。核心組件介紹：能力矩陣（Caps）、模式管理器。解法是改變色深/解析度、關閉一顯示器或用第三方軟體旋轉。

B-Q12: VGA 訊號品質受哪些因素影響？
- A簡: 線材品質/長度、連接器、干擾與電路設計會影響清晰度與抖動。
- A詳: 技術原理說明：VGA 為類比訊號，對阻抗匹配、頻寬與外部電磁干擾敏感。關鍵步驟或流程：使用屏蔽良好短線→確保接頭緊固→避免與電源線並行。核心組件介紹：DAC、VGA 線、終端阻抗。高解析度下劣質線材會導致邊緣模糊與鬼影，影響文字銳利度。可換更好的線或改用數位介面（若支援）。

- A詳: 技術原理說明：雙輸出共用同一記憶體與加速單元，需在表面、疊加與特效間取捨。關鍵步驟或流程：列出需求→盤點資源→裁剪（如關閉 Overlay、降色深）。核心組件介紹：Overlay、合成器、記憶體控制器。若資源不足，驅動會限制某些組合（如旋轉、硬解影片）以維持穩定。

B-Q14: 硬體旋轉與軟體旋轉對游標/影片的影響？

- A詳: 技術原理說明：硬體旋轉在掃描前轉向，游標與 Overlay 直接輸出；軟體旋轉需經中介表面再輸出。關鍵步驟或流程：計算轉向→合成游標→輸出。核心組件介紹：硬體游標、Overlay、轉向單元。結果是硬體旋轉更平滑；軟體旋轉適合文書閱讀，但播放高位元率影片時 CPU/頻寬吃緊可能掉幀。

B-Q15: LCD 刷新率的角色為何？與 CRT 有何不同？
- A簡: LCD 固定為逐點保持，常見 60Hz 即可；CRT 需更高刷新避免閃爍。
- A詳: 技術原理說明：LCD 像素採取樣保持（Sample-and-Hold），不依賴高刷新避免閃爍；CRT 以掃描光束激發磷光體，刷新不足易閃爍。關鍵步驟或流程：設定原生解析度與適當刷新（60–75Hz）。核心組件介紹：面板驅動、背光掃描/保持。辦公使用 LCD 以原生解析度與 60Hz 運作即可，無需執著高刷新。

D-Q1: 遇到 ATI 旋轉選項消失怎麼辦？

D-Q5: 修改登錄檔後無法開機或藍畫面如何處理？
- A簡: 進入安全模式還原備份，移除/重裝驅動，恢復原 Rotation 值。
- A詳: 問題症狀描述：改鍵值後重啟出現異常。可能原因分析：驅動載入異常或鍵值衝突。解決步驟：1) F8 進安全模式。2) regedit 匯入事前備份或手動將 Rotation 改回 00。3) 移除顯示驅動並重開機後重裝。預防措施：事前備份機碼與建立系統還原點；測試環境先驗證；避免同時改多處設定。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q10, C-Q10, C-Q3

D-Q6: Pivot Pro 與顯示卡驅動衝突怎麼辦？
- A簡: 更新至相容版本、關閉重疊功能、改變色深/解析度或改回驅動旋轉。
- A詳: 問題症狀描述：安裝後閃退、黑屏、無法旋轉。可能原因分析：過濾驅動與顯示卡驅動互斥、模式不相容。解決步驟：1) 更新 Pivot Pro 與顯示驅動。2) 停用硬體疊加或特效。3) 調整色深或解析度。4) 若持續衝突，卸載 Pivot Pro 改回驅動旋轉。預防措施：安裝前檢查相容清單與版本；變更模式逐一測試。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, B-Q3, C-Q2

D-Q7: 直立後上下視角偏色嚴重怎麼解？
- A簡: 調整螢幕高度/仰角，降低亮度，若仍嚴重改用視角較好的面板。
- A詳: 問題症狀描述：直立後上暗下亮或色偏。可能原因分析：TN 面板垂直視角窄。解決步驟：1) 降低亮度與對比。2) 調整腳座高度與仰角，使視線垂直中心。3) 開啟字型平滑與放大縮放減少仰視角度。4) 若需求嚴苛，改用 IPS/VA 面板。預防措施：選購前檢視視角參數與實測，避免將直立螢幕放太高。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q8, C-Q6

D-Q8: VGA 畫面模糊或鬼影的原因與解法？
- A簡: 線材/接頭不良或過長，改用短且屏蔽佳的線，避免干擾並固定連接。
- A詳: 問題症狀描述：文字不銳利、邊緣有拖影。可能原因分析：VGA 類比訊號受線材品質、阻抗不匹配與 EMI 影響。解決步驟：1) 更換高品質短線。2) 緊固接頭，避免與電源線平行。3) 降低解析度測試。4) 若可行，改用數位介面。預防措施：預留良好線材與走線，定期檢查接點鬆動。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, B-Q12, C-Q5

D-Q9: Windows 偶爾不記住雙螢幕排列怎麼辦？
- A簡: 鎖定主顯示器、固定連接順序，更新驅動並清理快取後重設排列。
- A詳: 問題症狀描述：重啟或重連後位置錯亂。可能原因分析：EDID 變動、驅動快取錯誤、接線順序不同。解決步驟：1) 固定線材與接孔位置。2) 設定主顯示器並應用。3) 更新顯示驅動。4) 刪除顯示器快取（需謹慎）後重新設定。預防措施：避免頻繁更換接孔；使用穩定驅動版本。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, B-Q1, C-Q9

D-Q10: 旋轉後影片播放掉幀或 CPU 飆高如何處理？
- A簡: 於未旋轉螢幕播放或啟用硬體加速，降低解析度/色深，關閉多餘特效。
- A詳: 問題症狀描述：影片卡頓、CPU 高使用。可能原因分析：軟體旋轉導致拋轉與縮放負載大、Overlay 不可用。解決步驟：1) 在未旋轉的螢幕播放影片。2) 影片播放器啟用硬體加速。3) 降低外接螢幕解析度/色深。4) 關閉桌面特效。預防措施：將高動態任務安排在硬體旋轉或未旋轉的顯示輸出上。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q14, B-Q13, C-Q2
