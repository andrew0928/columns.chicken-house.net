# 新硬碟... Hitachi Travelstar 5K160...

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 2.5 吋筆電硬碟？
- A簡: 筆電用 2.5 吋機械硬碟，體積小、耗電低，常見 PATA/SATA 介面，兼顧容量與攜帶性。
- A詳: 2.5 吋筆電硬碟是以直徑 2.5 吋碟片為基礎的機械式磁碟，專為行動裝置設計。相較 3.5 吋桌機碟，體積與重量更小、耗電更低、發熱較少，抗震設計更著重。介面以 PATA（IDE）與 SATA 為主，轉速常見 4200、5400、7200 rpm。適合筆電、外接隨身碟盒與嵌入式系統，依需求在容量、效能與可靠性間取捨。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q12, A-Q4, A-Q13

A-Q2: 什麼是 Hitachi Travelstar 5K160？
- A簡: 日立 Travelstar 5K160 為 2.5 吋 5400rpm 筆電硬碟，容量 160GB，採垂直寫入。
- A詳: Hitachi Travelstar 5K160 是日立於 2.5 吋筆電市場推出的機械硬碟系列，標稱 5400 rpm 轉速與 160GB 容量，採用垂直磁記錄（PMR）提升單碟密度。提供 PATA 與 SATA 版本以適配不同世代筆電。相較同時期 7200 rpm 小容量碟款，5K160 以更大容量與較低發熱、噪音、耗電見長，適合需要大空間儲存但仍重視日常順暢性的使用者。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, A-Q3, B-Q1

A-Q3: 5K160 與 7K60 的差異是什麼？
- A簡: 5K160 容量大 160GB、轉速 5400rpm；7K60 轉速快 7200rpm、容量僅 60GB，熱與耗電較高。
- A詳: 5K160 與 7K60 代表兩種取向。5K160 採 5400 rpm 與 PMR，容量達 160GB，主打空間、穩定與續航；7K60 屬 7200 rpm 小容量（60GB），主打隨機存取與傳輸率較快，但相對發熱、噪音與耗電較高。實務上，5K160 更適合需要大量檔案（如多媒體、虛擬機）的筆電；7K60 則適合追求反應速度且資料量有限的工作負載。選擇需依用途、熱管理與電池續航綜合評估。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q23, B-Q2

A-Q4: 5400 rpm 與 7200 rpm 的差異？
- A簡: 7200rpm 傳輸與隨機存取較快；5400rpm 更省電、安靜、溫度較低，續航較佳。
- A詳: 轉速決定磁片線速與平均延遲。7200 rpm 有較低的旋轉延遲與較高順序吞吐，對隨機 I/O 與大型傳輸有利；但因摩擦與轉動損失增加，普遍帶來更高溫度、震動與功耗。5400 rpm 則在續航、靜音與可靠度上更友善，適合行動使用。實務上，系統體感還受快取、固件演算法、磁密與作業系統快取影響，因此需綜合評估。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q2, A-Q23, A-Q26

A-Q5: 什麼是垂直寫入（PMR）？
- A簡: PMR 將磁化方向垂直於碟面，提高記錄密度與容量，並維持讀寫可靠度。
- A詳: 垂直磁記錄（Perpendicular Magnetic Recording）將磁區磁化方向由水平改為垂直於碟面，並在底層加上軟磁回路層，使磁通更集中。幾何與材料設計改善超順磁效應，讓磁區更小仍穩定，提升單位面積記錄密度與總容量，同時保持錯誤率在可接受範圍。PMR 是 2.5 吋硬碟容量大躍進的重要技術基礎。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q13, A-Q23

A-Q6: 為什麼需要更換出現壞軌的硬碟？
- A簡: 壞軌致讀寫失敗與卡頓，易擴散且風險升高，應立刻備份並更換。
- A詳: 壞軌代表磁碟表面或磁區信號劣化，ECC 無法校正。系統讀到壞軌時會反覆重試，造成長時間 I/O 等待，整機凍結或藍畫面。壞軌通常會擴散，風險逐步上升。建議立即多重備份，監控 SMART 指標，並在可控時間窗口更換新碟，避免在工作關鍵時刻崩潰。延遲更換不僅增加資料遺失機率，也使遷移與救援成本更高。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, A-Q11, D-Q1

A-Q7: 什麼是壞軌（Bad Sector）？
- A簡: 壞軌是無法可靠讀寫的扇區，可能源於物理損傷或磁信號衰退。
- A詳: 壞軌是硬碟中無法正確讀寫的最小儲存單元（扇區）集合。可能由頭碟接觸、顆粒污染、介質老化、振動衝擊或製程瑕疵導致。韌體會重試與使用 ECC 校正，仍失敗時會標記並（可行時）重映射到備用區。邏輯壞軌則可能由掉電或傳輸錯誤導致校驗失敗。大量或增加中的壞軌是逐步失效的警訊。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q3, A-Q18, D-Q1

A-Q8: 什麼是硬碟備份？為什麼重要？
- A簡: 備份是將資料副本另存於不同媒介或位置，用於災後快速復原。
- A詳: 備份指把重要資料複製至另一載體（外接碟、NAS、雲端）或映像檔，並定期更新與驗證。當硬碟壞軌、刪除失誤、勒索軟體或系統崩潰時，可用備份快速還原。良好策略包含 3-2-1 原則（三份拷貝、兩種媒介、一份異地），配合全備、增量與差異備份週期，並定期演練復原。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q28, C-Q7, D-Q10

A-Q9: 什麼是磁碟複製（Cloning）與映像（Imaging）？
- A簡: 複製直接把來源磁碟拷到目標；映像封裝成檔案，日後再還原。
- A詳: 磁碟複製把來源磁碟或分割區逐區塊複製到目標磁碟，完成即能開機，適合硬體直換。映像則將內容封裝為映像檔，支援壓縮、分割與驗證，利於長期備份或批量部署。工具可採扇區層級或僅複製已用區塊。選擇取決於是否要立即啟用、是否跨容量、與容錯需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q10, B-Q6, C-Q1

A-Q10: 什麼是 Acronis True Image？與 Ghost 有何不同？
- A簡: True Image 支援熱備份、映像與複製，容錯佳；Ghost 傾向離線作業與傳統流程。
- A詳: Acronis True Image 是主流映像與備份軟體，支援在 Windows 內建立快照、製作全備/增量映像、磁碟複製、異硬體還原與校驗。相較經典 Ghost，True Image 更著重圖形化操作、VSS 熱備份與錯誤忽略、自動分割調整等。來源碟有壞軌時，True Image 的容錯與跳過策略常較易成功完成遷移。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, B-Q7, C-Q1

A-Q11: 什麼是 S.M.A.R.T.？如何用於預警？
- A簡: SMART 是硬碟自我監測技術，透過屬性趨勢預估故障風險並告警。
- A詳: S.M.A.R.T. 在韌體中追蹤讀寫錯誤率、重映射扇區、啟動次數、溫度等健康指標。管理工具可讀取並評估門檻與趨勢，提供預警。當 Reallocated、Pending、Uncorrectable 等值上升，代表媒體劣化，應立即備份與更換。雖非完美預測，但對風險評估與維護排程極具參考價值。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q10, C-Q5, D-Q1

A-Q12: PATA 與 SATA 筆電硬碟的差異？
- A簡: PATA 為 44pin 平行傳輸；SATA 為序列介面，線材小、熱插拔與效率佳。
- A詳: PATA（IDE）使用 44 針排線，需跳線設定主從，線材寬且影響走線與散熱。SATA 為序列傳輸，7 針接頭、小線材、點對點連線，支援熱插拔、較高速率與 NCQ。選購新碟需確認主機板介面與轉接相容性，避免買到無法安裝的版本。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q15, C-Q8, D-Q8

A-Q13: 選購筆電硬碟時應考慮哪些因素？
- A簡: 容量、轉速、介面、可靠度、功耗/溫度、相容性與保固服務需綜合評估。
- A詳: 選購時先確認介面（PATA/SATA）與實體高度。依工作負載取捨容量與轉速：大量檔案與續航重視者偏大容量 5400rpm；追求反應速度者可選 7200rpm。評估品牌可靠度、保固年限與 RMA 便利性。關注噪音與溫度表現以免過熱。考慮備援與備份成本，決定是否加購外接盒、NAS 或雲端儲存。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q26, A-Q16

A-Q14: 為何容量會影響使用體驗？
- A簡: 大容量減少空間焦慮與碎片化，外圈線速高，整體更穩定順暢。
- A詳: 充足容量讓系統與應用有足夠暫存與更新緩衝，避免長期滿盤。磁碟接近滿時，碎片化與放置限制導致尋道增多與延遲上升，體感變慢；清理空間也耗時。更大容量意味外圈線速更高，提高順序讀寫表現。對多媒體與大量檔案工作者，容量升級帶來立即可感的順暢。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q23, B-Q2, C-Q3

A-Q15: 什麼是資料移轉？筆電單槽有何限制？
- A簡: 資料移轉把系統與檔案搬到新碟；單槽需借助外接或網路方式。
- A詳: 資料移轉包含系統、開機記錄、分割區與檔案的完整複製或還原。單槽筆電無法同時接兩顆內碟，需用 USB/SATA 外接盒、光碟/網路開機或接桌機對拷。過程要注意供電與線材品質、錯誤忽略設定，完成後驗證開機與資料完整性，再正式上線。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q2, C-Q1

A-Q16: 什麼是硬碟保固/RMA？
- A簡: 保固/RMA 是原廠對故障硬碟的維修/更換服務，依序號與年限判定。
- A詳: 硬碟保固多以序號對應出貨日期計算。期間內符合條件可走 RMA：先備份抹除資料、妥善包裝並附故障描述，寄至授權中心。原廠測試後維修或更換同級良品。RMA 排除人為損壞或拆封，寄送規範依地區為準。保留購買憑證與測試報告可提升處理效率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q9, D-Q9, A-Q30

A-Q17: 買硬碟遇缺貨時有哪些管道可找？
- A簡: 可詢問授權經銷、實體商圈、網購與拍賣通路，並設定到貨通知。
- A詳: 熱門型號缺貨時，多管齊下：詢問授權經銷、連鎖通路與在地電腦商圈；關注網路商城、拍賣平台與二手市場。向賣家留下聯絡方式以獲到貨通知，或評估規格接近的替代型號。留意介面與保固來源差異，避免相容性不符或保固不明的情形。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q12, A-Q13, C-Q10

A-Q18: 什麼是磁區重配置（Sector Remapping）？
- A簡: 韌體將不良扇區映射到備用區，對上層呈現為健康區塊，數量記錄於 SMART。
- A詳: 當扇區重試仍無法可靠讀寫時，韌體把該邏輯區塊映射到預留備用區，稱重配置。成功重配置數量呈現於 SMART Reallocated Sector Count。若 Pending/Uncorrectable 持續增加且備用區用罄，代表介質惡化。重配置僅是暫時維持可用的機制，不應取代更換硬碟的決策。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q10, B-Q3, D-Q1

A-Q19: 為什麼複製過程會被壞軌卡住？
- A簡: 壞軌觸發多次重試耗時，若未啟用忽略/跳過策略，整體進度停滯或失敗。
- A詳: 壞軌導致 ECC 失敗，韌體會多次讀取重試，每次耗費數秒至數十秒。複製工具預設常逐扇區讀取，遇錯即停或反覆嘗試，使總時間暴增甚至失敗。啟用忽略錯誤、跳過策略或調降重試次數，可先完成大部分資料，再針對問題區域多輪掃描或標記不可讀，降低停機風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q1, D-Q2

A-Q20: 何時該選擇重灌系統而不是複製？
- A簡: 系統嚴重毀損、壞軌遍布或硬體差異大時，重灌更乾淨可靠並利於後續維護。
- A詳: 若來源碟 SMART 惡化、檔案系統錯誤頻仍、惡意程式或驅動殘留難解，或有跨平台差異（IDE→SATA、換控制器），重灌能獲得更乾淨狀態與較少遺留問題。事前盤點授權、備份資料與設定，列出驅動與更新清單，評估停機成本，綜合決定採複製或重灌。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q3, C-Q7, A-Q9

A-Q21: 什麼是 MBR 與 GPT 的差異？
- A簡: MBR 最多 2TB 且四主分割；GPT 支援超大容量、多分割與備援校驗。
- A詳: MBR 使用 32 位 LBA，單碟上限 2TB，最多四主分割（或三主一延伸）。開機程式位於首扇區。GPT 為 UEFI 時代標準，支援極大容量與更多分割，具備備援分割表與 CRC 校驗提升可靠度。Windows XP 多為 MBR；新系統建議 GPT 以便擴充與可靠性。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q18, C-Q3, D-Q4

A-Q22: 磁碟分割對效能與使用有何影響？
- A簡: 合理分割利於管理與還原；外圈分割順序讀寫更快，過度分割反增負擔。
- A詳: 將系統與資料分割可降低維修風險，系統毀損時僅重灌系統分割。外圈區域線速較高，將高 I/O 工作放外圈可得較佳吞吐。過度或不當分割可能造成零碎空間與管理負擔。應考量對齊、未來擴充與備份策略，避免後續調整帶來風險。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q3, B-Q9, D-Q4

A-Q23: 為什麼較大容量但較低轉速仍然划算？
- A簡: 面密度高帶來外圈高線速，大容量減少滿盤退化，體感常不輸高轉速小碟。
- A詳: PMR 提升面密度讓外圈實際資料通過率成長，即使 5400 rpm，順序讀寫也能接近或追平舊世代 7200 rpm 小容量碟。更大容量提供暫存空間，降低滿盤導致的效能退化與維護成本。對多數筆電工作負載，容量帶來的便利與穩定性，常比小幅延遲差更重要。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q2, A-Q14, A-Q4

A-Q24: 什麼是磁碟快取（Cache）？
- A簡: 硬碟內建 DRAM 快取，用於預取與寫入合併，平滑短期效能波動。
- A詳: 控制器配備 DRAM 做快取。讀取時進行預取（read-ahead），將鄰近區塊提前載入；寫入時暫存小寫入並合併為連續寫，減少尋道。快取搭配命令排序（如 NCQ）與固件演算法運作。快取改善短期體感，卻不能取代長期容量與磁密帶來的吞吐優勢，亦需防範掉電風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q14, B-Q15, A-Q4

A-Q25: 什麼是 I/O 等待導致的系統停滯？
- A簡: 儲存長時間無回應使執行緒阻塞，視窗顯示未回應，似當機實為卡 I/O。
- A詳: 應用對磁碟發出同步 I/O 後，若硬碟因壞軌或重試而遲未回應，執行緒會在核心態等待，滑鼠可動但視窗未回應。Windows 會標示「未回應」。這是 I/O 等待導致的假性凍結。透過事件檢視器、SMART、效能監視器與磁碟指示燈行為，可辨識瓶頸並採取備份與更換行動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, D-Q1, D-Q2

A-Q26: 轉速與溫度、噪音的關係？
- A簡: 轉速越高，摩擦與震動越大，通常更熱、更吵且更耗電。
- A詳: 轉速提升，軸承與碟片空氣剪切增強，產生更多熱與聲噪，電機耗能上升。高轉速機種需更嚴謹動平衡與散熱設計。筆電空間有限，導風與散熱受限，高轉速碟易推高整機溫度影響舒適與壽命。選型需在效能與熱設計取得平衡。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q19, D-Q6

A-Q27: 什麼是硬碟健康檢查與報告？
- A簡: 藉由 SMART、表面掃描與錯誤日誌，定期評估健康與風險。
- A詳: 健康檢查包含讀取 SMART 指標、執行短/長測試、表面掃描（讀/寫驗證），並檢閱系統日誌中之磁碟錯誤。工具如 CrystalDiskInfo、smartctl、廠商診斷程式可視覺化結果。定期檢查能早期發現 Reallocated、Pending、CRC Error 等異常，及時備份與安排更換，降低停機與資料損失。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, B-Q10, D-Q1

A-Q28: 什麼是增量備份與差異備份？
- A簡: 增量備份保存自上次任意備份變更；差異備份自上次全備後變更。
- A詳: 全備完整複製所有資料。增量備份以最近一次任意備份為基準，只保存新增/變更部分，體積最小但還原需串接多檔。差異備份以最近一次全備為基準，保存自那之後全部變更，體積較大但還原較簡單。選擇策略時需在存儲成本、還原速度與風險間權衡。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q7, D-Q10, A-Q8

A-Q29: 為什麼在換 OS 前仍先換硬碟？
- A簡: 先排除硬體風險保資料安全，新碟亦為日後升級留足空間與回退彈性。
- A詳: 既有硬碟有壞軌或容量不足時，先換新碟可降低 OS 升級過程中失敗與資料遺失風險。新碟提供更大空間承載安裝檔與回復點，也能先完整複製現有系統，日後平滑升級並保留回退選項。把不可控因素拆開，能縮短停機時間並提升成功率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, C-Q3, D-Q3

A-Q30: 什麼是收貨與開箱驗收流程？
- A簡: 核對型號序號、外觀與介面，並完成 SMART 與容量檢測，以便即時退換。
- A詳: 收貨先檢查外箱與防震包材是否完好，核對型號、介面（PATA/SATA）與序號。檢視接頭與外殼是否撞傷。接上測試主機，讀取 SMART、執行快速自檢與容量/壞軌掃描。紀錄序號、購買憑證與測試報告，便於保固。提早發現問題可立即退換，避免安裝後再拆裝的成本。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, C-Q10, A-Q16


### Q&A 類別 B: 技術原理類

B-Q1: 垂直寫入（PMR）如何運作？
- A簡: PMR 以單極寫頭與軟磁回路集中磁通，垂直磁化提高面密度與穩定性。
- A詳: 原理說明：PMR 改以垂直磁化，搭配軟磁回路（SUL）提供回磁路，減少超順磁影響。關鍵流程：1) 伺服定位磁頭；2) 單極寫頭施加強磁場垂直寫入；3) SUL 導引回磁通閉合；4) 讀頭（GMR/TMR）感應微弱訊號。核心組件：單極寫頭、SUL 層、讀通道與伺服控制。此設計讓更小磁域仍可穩定儲存，提升容量。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, B-Q13, A-Q23

B-Q2: 轉速如何影響效能與延遲？
- A簡: 轉速決定旋轉延遲與線速；7200rpm 延遲更低、吞吐更高但熱與功耗上升。
- A詳: 原理說明：平均旋轉延遲約為 60/(2×RPM) 毫秒，7200rpm 約 4.17ms、5400rpm 約 5.56ms。關鍵流程：總 I/O 時間=尋道+旋轉延遲+傳輸時間。面密度提升亦增加線速。核心組件：主軸馬達、致動器、讀寫通道。實務上，快取與 NCQ 也影響體感，需與容量、溫度及功耗一併考量。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, A-Q26, A-Q23

B-Q3: 壞軌的形成與檢測機制是什麼？
- A簡: 介質劣化或機械衝擊致扇區不可讀，韌體透過重試、ECC 與重映射處理。
- A詳: 原理說明：介質磁訊號衰退、頭碟接觸或污染導致位元錯誤超過 ECC 能力。關鍵流程：1) 讀取錯誤觸發重試；2) ECC 嘗試校正；3) 失敗則標記並重映射到備用區；4) SMART 記錄屬性變化。核心組件：讀寫通道、ECC 引擎、韌體映射表（P-list/G-list）。大量壞軌代表硬體失效風險加劇。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q7, A-Q18, D-Q1

B-Q4: Windows 為何會因 I/O 錯誤而「卡住」？
- A簡: 同步 I/O 在儲存堆疊阻塞，IRP 等待裝置回應，使用者看到應用「未回應」。
- A詳: 原理說明：應用發出同步 I/O，I/O 請求封包（IRP）經文件系統、類別/埠驅動傳至硬碟，壞軌造成長時間重試。關鍵流程：1) IRP 入列；2) 裝置重試逾時；3) 上層執行緒等待完成；4) UI 卡頓。核心組件：NTFS、磁碟類別驅動、ATAPI/SATA 埠驅動、硬碟韌體。調整逾時或使用能跳過錯誤的工具可緩解。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q25, D-Q1, D-Q2

B-Q5: True Image 的映像與還原流程？
- A簡: 透過 VSS 建立快照，讀取已用區塊建立映像，驗證後再寫回目標磁碟。
- A詳: 原理說明：使用 VSS（Volume Shadow Copy）凍結一致性狀態，進行線上備份。關鍵流程：1) 觸發 VSS 快照；2) 掃描檔案系統已用區；3) 壓縮/分卷並計算校驗；4) 儲存於目標或網路；5) 還原時重建分割與開機結構。核心組件：VSS 提供者、Acronis 篩選驅動、映像引擎與驗證模組。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, B-Q6, C-Q1

B-Q6: 扇區對拷與僅複製已用區塊的差異？
- A簡: 扇區對拷逐扇區複製；已用區塊模式僅複製檔案系統使用部分。
- A詳: 原理說明：扇區對拷無視檔案系統，完整複製，包含空白與壞區嘗試。已用區塊則解析檔案系統結構以節省時間。關鍵流程：1) 偵測檔案系統；2) 決定模式；3) 遇錯誤時採跳過/重試策略。核心組件：檔案系統解析器、位元圖（bitmap）、錯誤處理器。來源盤有壞軌時，優先使用能跳過的已用區塊模式。

- 學習階段: 核心
- 關聯概念: A-Q9, A-Q19, C-Q1

B-Q7: Ghost 與 True Image 的架構差異？
- A簡: Ghost 偏離線扇區鏡像；True Image 支援線上快照、容錯與彈性還原。
- A詳: 原理說明：Ghost 傳統以 DOS/WinPE 啟動進行離線扇區鏡像；True Image 內建 VSS 熱備份與圖形化精靈。關鍵流程：Ghost 需關機開機媒體；True Image 可線上備份與離線還原。核心組件：Boot 媒體、映像引擎、VSS/快照驅動、分割調整器。True Image 在壞軌處理與跨容量調整上更友善。

B-Q8: 單槽筆電資料移轉的技術選項？

- A詳: 原理說明：以外部通道串接第二顆磁碟或映像儲存。關鍵流程：1) 以 USB/SATA 外接盒連接新碟；2) 以軟體於系統內或開機媒體進行鏡像/對拷；3) 抽換硬碟；4) 驗證開機。核心組件：USB-SATA 橋接器、開機媒體（WinPE/Acronis）、網路堆疊（SMB/NFS）。亦可經由網路 PXE 啟動還原環境。

B-Q9: 擴容時分割區調整的原理？

- A詳: 原理說明：擴容需移動/擴展分割與檔案系統，更新 MBR/GPT 表與 VBR。關鍵流程：1) 鎖定分割；2) 移動尾端資料；3) 擴展檔案系統（NTFS 擴展 MFT 與位元圖）；4) 校驗。核心組件：分割管理器、檔案系統驅動、對齊計算器。注意 XP 下系統分割擴展限制，常需離線工具。

B-Q10: SMART 監測項目如何工作？
- A簡: 韌體累積屬性計數與趨勢，離線自檢與門檻判斷提供預警。
- A詳: 原理說明：SMART 屬性由韌體維護，包含錯誤率、重映射、溫度等，並支援短/長/傳輸自檢。關鍵流程：1) 屬性更新；2) 門檻比較；3) 日誌記錄；4) 查詢與告警。核心組件：屬性表、SMART 日誌、自檢例程。監控 Reallocated、Pending、UDMA CRC Error 對風險評估特別關鍵。

B-Q11: 壞軌情況下的資料回復流程？
- A簡: 先鏡像後修復：優先可讀區塊，反覆掃掠疑難區，再修復檔案系統。
- A詳: 原理說明：降低對故障盤壓力，優先擷取可讀資料。關鍵流程：1) 以容錯工具建立磁碟映像（先快掃過可讀區）；2) 多輪針對不可讀區小塊重試；3) 在映像上修復檔案系統或恢復檔；4) 驗證完整性。核心組件：映像工具（如 ddrescue 類）、錯誤重試策略、檔案系統修復器（chkdsk/ntfsfix）。
- 難度: 高級
- 學習階段: 進階

B-Q12: 硬碟 RMA 的序號與驗證機制？

- A詳: 原理說明：原廠以序號映射出貨與保固期。關鍵流程：1) 於官網輸入序號查詢；2) 送出故障描述與聯絡資訊；3) 收到 RMA 編號與寄回指引；4) 廠測並更換或維修；5) 回寄。核心組件：序號資料庫、RMA 系統、測試工站。需遵守包裝與資料抹除規範。
- 難度: 初級

B-Q13: PMR 如何提升單位面積記錄密度？
- A簡: 垂直磁化減少相鄰位元干擾，搭配材料與結構讓位元更小仍穩定。
- A詳: 原理說明：垂直磁化搭配 SUL 使磁通閉合，降低位元間相互干擾，抑制超順磁效應。關鍵流程：1) 優化磁性薄膜顆粒與結晶方向；2) 採單極寫頭提高局部磁場；3) 控制熱磁穩定性。核心組件：SUL、單極寫頭、先進讀通道。整體讓面密度顯著提升。

- 學習階段: 進階

B-Q14: 磁碟快取與預取/回寫機制是什麼？
- A簡: DRAM 快取結合預取與寫入合併，透過命令排序提升短期效能。
- A詳: 原理說明：快取以時間/空間區域性預測資料需求。關鍵流程：1) 讀取後預取鄰近區；2) 小寫入先入快取合併為連續寫；3) 接收 FUA/Flush 命令維持一致性。核心組件：控制器 MCU、DRAM、固件演算法（如 NCQ 排序）。改善體感但需平衡掉電風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q24, B-Q2, D-Q6

B-Q15: PATA 與 SATA 在電氣與協定上的差異？
- A簡: PATA 平行傳輸與主從架構；SATA 序列點對點，採 FIS 與 NCQ 協定。
- A詳: 原理說明：PATA 使用多位平行信號，受串音限制速率；SATA 以差動序列傳輸，抗噪且易升速。關鍵流程：PATA 需跳線主從；SATA 為點對點，支援熱插拔與電源管理。核心組件：PATA 控制器、SATA 主機匯流排介面（HBA）、連接器與線纜。SATA 還引入 FIS 框架與 NCQ 命令排序。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, C-Q8, D-Q8

B-Q16: 檔案系統如何處理壞扇區（以 NTFS 為例）？
- A簡: NTFS 透過 $BadClus 標記壞簇，chkdsk 可掃描並隔離不可用區域。
- A詳: 原理說明：NTFS 維護 $BadClus 檔案記錄壞簇，使配置器避免再使用。關鍵流程：1) chkdsk /r 掃描讀取錯誤；2) 標記壞簇並嘗試移動可恢復資料；3) 更新位元圖。核心組件：$MFT、$Bitmap、$BadClus、chkdsk 工具。這屬治標不治本措施，仍需更換硬碟。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q1, C-Q1, A-Q7

B-Q17: 映像檔校驗與完整性驗證如何進行？
- A簡: 建立與還原時計算校驗碼（CRC/SHA），驗證映像未損毀且可還原。
- A詳: 原理說明：將映像分段計算校驗碼儲存於目錄或尾端。關鍵流程：1) 建立映像時計算 CRC/SHA；2) 儲存驗證資訊；3) 還原前執行 Verify；4) 錯誤則重取或重建。核心組件：校驗模組、映像目錄、錯誤處理器。可降低媒體錯誤或傳輸損毀造成的還原失敗。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q1, C-Q7, D-Q2

B-Q18: 引導（MBR）與磁碟簽章如何影響開機？
- A簡: MBR 載入分割開機記錄；磁碟簽章對應已裝置路徑，克隆後需一致性處理。
- A詳: 原理說明：MBR 含開機碼與分割表，將控制權交給活動分割 VBR，再由 NTLDR/bootmgr 啟動系統。磁碟簽章映射到 Windows 註冊表 MountedDevices。關鍵流程：1) 複製 MBR/VBR；2) 調整分割與簽章；3) 檢查 boot.ini/BCD。核心組件：MBR、VBR、磁碟簽章與註冊表。衝突會導致 0x7B 或無法開機。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: D-Q3, C-Q6, A-Q21

B-Q19: 電源管理如何影響硬碟耗電與壽命？
- A簡: APM/ACPI 控制待機與停轉，降低耗電；過度停轉可能增磨損。
- A詳: 原理說明：OS 透過 ACPI 策略、APM 指令調整硬碟省電（停轉、降速、快取策略）。關鍵流程：1) 空閒計時；2) 進入低功耗或停轉；3) I/O 喚醒。核心組件：ACPI 策略、HDD 韌體、省電參數。適度降耗延長續航，但頻繁停轉/啟動會增加機械負荷，需平衡設定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q26, D-Q6, C-Q7

B-Q20: 速度、容量與可靠度的權衡機制？
- A簡: 高轉速與高密度提效能，伴隨熱與錯誤率風險；設計需在三者間取捨。
- A詳: 原理說明：效能取決於轉速與面密度；容量仰賴面密度與碟片數；可靠度受熱、震動與寫密度影響。關鍵流程：1) 設定設計目標；2) 選擇轉速/密度/碟片組合；3) 驗證熱穩定與錯誤率。核心組件：主軸、碟片/磁層、控制器/韌體。用戶選購時同樣需在速度、容量與可靠度之間平衡。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q13, A-Q23, A-Q26

### Q&A 類別 D: 問題解決類（10題）

D-Q1: 遇到硬碟壞軌導致卡頓怎麼辦？
- A簡: 立刻備份，執行 SMART 與 chkdsk，啟用容錯複製，儘速更換硬碟。
- A詳: 問題症狀：磁碟燈長亮、系統未回應、存取特定檔案即卡住。可能原因：物理壞扇區、介質劣化、傳輸錯誤。解決步驟：1) 先備份關鍵資料；2) 查 SMART（Reallocated/Pending）；3) chkdsk /r 標記壞簇；4) 以可忽略錯誤的工具（True Image）複製至新碟；5) 更換硬碟。預防措施：定期備份與健康檢查，避免滿盤與劇烈震動。

- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q7, C-Q1

D-Q2: True Image 複製過程遇到讀取錯誤如何處理？

- A詳: 問題症狀：Clone 停滯、錯誤代碼、進度長時間不前。可能原因：壞扇區、USB 連線不穩、來源檔案系統損毀。解決步驟：1) 勾選 Ignore bad sectors；2) 選擇僅複製已用區塊；3) 先製作映像檔並驗證，再還原到新碟；4) 改用開機媒體執行；5) 檢查線材與供電。預防措施：定期 chkdsk、早期備份與更換。

- 學習階段: 核心
- 關聯概念: B-Q6, C-Q1, B-Q17

D-Q3: 新硬碟裝上後無法開機怎麼診斷？

- A詳: 問題症狀：黑畫面出現「No bootable device」或重啟循環。可能原因：引導程式缺失、分割未標記活動、磁碟簽章衝突。解決步驟：1) 設內碟為第一開機；2) 用修復主控台執行 fixmbr、fixboot、bootcfg /rebuild；3) 檢查分割是否為活動；4) 移除重複磁碟簽章。預防措施：Clone 時包含 MBR/VBR，完成後先以外接啟動測試。

D-Q4: 複製後分割區大小沒變怎麼擴充？

D-Q10: 複製後軟體授權或啟用問題如何解決？
- A簡: 預先解除綁定，複製後重新啟用或聯繫廠商，保留授權文件。
- A詳: 問題症狀：應用要求重新啟用或報授權錯誤。可能原因：磁碟簽章/硬體 ID 變更。解決步驟：1) 複製前解除授權（如 Adobe/MS）；2) 複製後重新啟用；3) 無法啟用則聯繫廠商客服；4) 檢查日期時間與網路。預防措施：整理授權清單、序號與帳號，設定備援與文件保存。
