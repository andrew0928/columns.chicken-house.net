## 資訊整理

### 知識架構圖
1. 前置知識：
   - 了解軟體開發基本流程（需求、設計、開發、測試、部署、維運）
   - 熟悉前後端分離架構與版本控制（Git）
   - 基本的 LLM 使用與 Prompt 概念
   - 專案管理與效益衡量（KPI/OKR、ROI）

2. 核心概念：
   - Prompt 到 Product：將「互動式指令」系統化、可重複、可驗證，轉化為可交付的產品能力
   - Vibe Coding：以生成式 AI 驅動的迭代式協作開發方式，快速探索與成型
   - 可觀測與量化指標：用數據證明導入 AI 帶來的效益（交付時間、缺陷率、覆蓋率等）
   - Domain Knowledge 適配：在複雜產業場景中建立知識、上下文、測試基線以提升 AI 可用性
   - 前後端整合與 Prompt 資產化：將需求、介面、測試與 Prompt 文件化、版本化，納入開發生命週期

3. 技術依賴：
   - 大語言模型與工具鏈（模型選型、API、提示模板）
   - 資料與知識管理（RAG/檔案庫、Context 管理、術語表）
   - 軟體工程基礎設施（CI/CD、測試自動化、版本控制）
   - 可觀測性與評估（AI 評測指標、A/B、使用分析、錯誤回報）
   - 部署與維運（環境管理、權限合規、成本監控）

4. 應用場景：
   - 新功能快速原型與迭代（Vibe Coding 驅動）
   - 既有功能小改與文件/測試更新的半自動化
   - 企業級複雜系統的知識注入與流程輔助
   - 前後端整合開發的協同、契約生成與端到端測試
   - 維運與客服輔助（Runbook 生成、故障排除、知識庫更新）

### 學習路徑建議
1. 入門者路徑：
   - 學會撰寫清晰 Prompt：角色、目標、輸出格式、範例
   - 嘗試以 AI 生成小型前後端範例，並手動修正與測試
   - 練習將成果以 README、Issue、Test Case 的形式保存與版本控制

2. 進階者路徑：
   - 引入 Prompt 模板化與檔案化，建立可重複的開發工作流
   - 結合 RAG/術語表處理特定 Domain 知識，提升輸出一致性
   - 建立評估指標（品質、速度、成本）與 A/B 實驗，追蹤導入效益

3. 實戰路徑：
   - 在實際專案中為新功能採用 Vibe Coding，定義整合測試與契約
   - 將既有功能的小改動納入 Prompt 資產（差異化指令、回歸測試）
   - 於維運導入 AI：自動生成 Runbook、事故摘要、工單分類與知識庫同步

### 關鍵要點清單
- Prompt 到 Product：把一次性對話轉化為可重複、可測試、可維護的產出流程（優先級: 高）
- Vibe Coding：用 AI 驅動的快速迭代來探索與成形，並配合工程紀律落地（優先級: 高）
- 效益量化指標：以交付時間、缺陷率、測試覆蓋率、成本等 KPI 驗證 AI 導入（優先級: 高）
- 前後端整合策略：在 AI 生成的基礎上以契約/測試串接，縮短整合成本（優先級: 高）
- Prompt 資產化：建立 Prompt 檔、模板、版本與變更紀錄，納入 repo（優先級: 高）
- 既有功能小改流程：用差異化 Prompt 與回歸測試保障穩定性（優先級: 中）
- Domain 知識注入：以術語表、RAG、範例庫提升在複雜產業場景的可用性（優先級: 高）
- 模型與工具選型：依任務特性選擇模型、向量庫、代理框架等（優先級: 中）
- 安全與合規：資料保護、權限控管、審計與模型使用界線（優先級: 高）
- 成本治理：監測推理成本、快取策略與本地/雲端平衡（優先級: 中）
- 測試自動化：從單元到端到端測試，搭配 AI 生成用例與契約（優先級: 高）
- 可觀測性與評估：建立 AI 行為監控、誤差分析與持續改進回路（優先級: 中）
- 維運場景應用：AI 協助故障定位、Runbook 生成與知識庫更新（優先級: 中）
- 人才轉型與分工：工程師與 PM 強化溝通、需求釐清與系統性思維（優先級: 中）
- 初學者學習重點：工程基礎、清晰表達、測試思維與版本化習慣（優先級: 高）
