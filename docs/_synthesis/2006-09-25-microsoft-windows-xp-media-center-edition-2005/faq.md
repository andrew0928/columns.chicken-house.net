# Microsoft Windows XP Media Center Edition 2005

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Microsoft Windows XP Media Center Edition 2005（MCE 2005）？
- A簡: 基於 Windows XP 的家庭劇院版，整合看電視、錄影、節目表與遙控器操作。
- A詳: MCE 2005 是 Windows XP 的特殊版本，提供客廳娛樂體驗的「10-foot」使用介面。它整合電視調諧卡、電子節目表（EPG）、預約錄影、時光平移（Time Shift）、媒體庫與遙控器操作，讓 PC 具備類似數位錄影機（DVR）的能力。使用者可用 LCD/電視螢幕與遙控器操作，無須鍵盤滑鼠，即可看電視、按節目表一鍵錄影、管理已錄影片，並支援待機自動喚醒以準時錄影。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q5, B-Q1

A-Q2: MCE 2005 的核心價值是什麼？
- A簡: 用軟體把 PC 變成 DVR，中控電視內容、錄影與媒體播放，免月費更彈性。
- A詳: 核心價值在於把通用 PC 轉化為高易用性的家庭娛樂中心：以 EPG 快速找節目並一鍵預約、以 Time Shift 隨時暫停/倒轉直播、錄到硬碟靈活管理空間、並透過待機喚醒省電不漏錄。相較專用機上盒，MCE 不需月租費、可升級硬碟/解碼器/顯示卡，軟硬整合彈性高，兼具影音播放與一般電腦用途。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q12, B-Q3

A-Q3: MCE 2005 與一般電視卡附帶軟體有何差異？
- A簡: MCE 以一致介面、完整 EPG、強大錄影排程與遙控器支援勝出。
- A詳: 多數電視卡軟體僅能看電視與簡單錄影，介面各自為政。MCE 提供統一 10-foot UI、線上 EPG、頻道對應、整季錄影、Time Shift、PIP 預覽，以及待機喚醒與空間管理。其驅動與解碼以 Windows 標準（BDA、DirectShow）為基礎，兼容性佳，並支援 eHome 遙控器、IR Blaster 控制機上盒，整體體驗更像一體化 DVR。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, B-Q1, B-Q5

A-Q4: MCE 2005 與 TiVo 有何不同？
- A簡: MCE 是 PC 軟體方案，通常免月費；TiVo 是專用硬體加服務，常需訂閱。
- A詳: TiVo 是整合硬體與雲端服務的專用 DVR，裝機即用、以訂閱服務提供 EPG 與智慧錄影。MCE 則是安裝於 PC 的作業系統版本，搭配電視卡、解碼器與網路即可使用 EPG、錄影與 Time Shift。MCE 優勢是硬體可升級、無月租、用途廣；TiVo 優勢是開箱即用、服務一致性高。兩者都提供節目表、預約與暫停倒轉，但生態與維護模式截然不同。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q5, B-Q2

A-Q5: 什麼是電子節目表（EPG）？為什麼重要？
- A簡: EPG 是可更新的節目時間與資訊來源，支援選台、搜尋與預約錄影。
- A詳: EPG 是由服務提供者定期更新的頻道與節目資料，包含節目名稱、時段、簡介、分類等。MCE 依 EPG 決定節目表介面呈現、頻道對應與「整季錄影」規則。使用者能直接在節目表選節目按錄影，系統自動處理開始/結束時間、衝突與喚醒。EPG 的品質與覆蓋率直接影響選台便利、搜尋精準度與錄影成功率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q2, C-Q2, D-Q1

A-Q6: 什麼是時光平移（Time Shift）？
- A簡: 直播暫停與倒轉技術，透過緩衝錄影隨時停看、回放或快轉。
- A詳: Time Shift 將直播串流即時寫入硬碟環狀緩衝區，讓你隨時按暫停上廁所、倒轉回看剛剛錯過的片段，或跳過廣告。MCE 以後台服務持續維護緩衝，並確保快取與錄影互不干擾。其體驗取決於硬碟效能、解碼器穩定度與驅動相容性，設定得當能大幅提升觀影自由度。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q4, C-Q5, D-Q5

A-Q7: 什麼是預約錄影與整季錄影（Series Recording）？
- A簡: 依節目表預先安排錄影；整季錄影自動抓取同節目所有集數。
- A詳: 預約錄影讓你在 EPG 選取節目時段，MCE 會於開始時間自動喚醒錄製並於結束時停止。整季錄影則以節目識別與規則（新集數/任何時段/保留集數上限）自動捕捉每一集，並處理衝突優先序。這大幅降低手動管理成本，特別適合連續劇與體育賽事。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q3, B-Q12, C-Q4

A-Q8: 什麼是「10-foot UI」？對 MCE 有何意義？
- A簡: 為遠距離與電視螢幕設計的大字體、高對比、遙控器友善介面。
- A詳: 10-foot UI 指使用者距離螢幕約三公尺觀看時仍清晰易操作的介面設計。MCE 將字體、按鈕、焦點高亮、方向鍵導航與簡化層級深度最佳化，配合遙控器即可完成多數操作。這確保在客廳環境、傳統電視的解析度與對比下仍可讀，達到「像機上盒般」的體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, B-Q11, C-Q9

A-Q9: MCE 2005 的顯示與操作設計有哪些特點？
- A簡: 大字體高對比、焦點明確、方向鍵導航、可搭配遙控器與 PIP 預覽。
- A詳: MCE 採用高對比配色、放大圖示與動態焦點，並以遙控器方向鍵/OK/返回為主。節目表左下角可顯示 PIP 預覽目前播放畫面，降低情境切換成本。設計目標是去滑鼠鍵盤化，兼顧客廳視距、LCD/電視顯示與家庭使用情境，讓找節目、錄影與播放一氣呵成。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, A-Q11, B-Q10

A-Q10: 用 LCD 螢幕看電視與連接傳統電視有何差異？
- A簡: LCD 像素對映精準、無過掃；傳統電視有過掃與銳利度差異，體驗不同。
- A詳: LCD 螢幕常以逐行顯示與原生解析度 1:1 對映，文字銳利、幾何精準。傳統電視可能有過掃（裁切邊緣）、交錯掃描、色彩與銳利度差異，影響 UI 可見區。MCE 的 10-foot UI 兼顧兩者，但最佳化設定（縮放、過掃補償、色彩空間）會影響體驗；若習慣用 LCD 看電視，操作精細度較高。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q9, B-Q11, D-Q10

A-Q11: 什麼是 PIP（畫中畫）在節目表中的應用？
- A簡: 節目表同時顯示小窗即時預覽目前播放的畫面，降低切換成本。
- A詳: MCE 在節目表介面左下角提供即時預覽視窗，持續播放目前收看的頻道畫面。這不是真正雙調諧器 PIP，而是同頻道的縮圖預覽，讓你在瀏覽節目表時不錯過現場畫面。其設計提升情境連續性與選台效率，對遙控器操作尤為便利。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q10, A-Q9, D-Q5

A-Q12: 為何錄到硬碟更方便？
- A簡: 硬碟存取快、管理彈性大，易刪改、分類與轉存，節省實體媒材。
- A詳: 錄影檔存放硬碟可隨時播放、快轉、刪除與重新命名，亦能依空間自動清理舊節目。相較錄影帶/光碟，硬碟可搜尋、排序、標籤，並透過網路備份或轉檔（遵循權限）到其他裝置。MCE 也會根據可用空間與保留規則自動管理錄影清單，降低維護負擔。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q6, C-Q7, D-Q9

A-Q13: 待機與自動喚醒對 MCE 的意義是什麼？
- A簡: 節能不漏錄；電腦閒置睡眠，排程到時自動醒來錄影再回睡眠。
- A詳: MCE 利用作業系統 ACPI 機制在待機/休眠狀態下維持低功耗，錄影排程到時由排程服務設置的喚醒計時器喚醒系統進行錄影。完成後可依設定自動回到待機，兼顧省電與可靠性。必要前提是硬體與 BIOS 支援 S3/S4 與喚醒計時，並正確設定驅動與電源選項。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q9, C-Q6, D-Q2

A-Q14: MCE 2005 的軟硬體需求為何？
- A簡: 相容電視調諧卡、MPEG-2 解碼器、足夠硬碟、支援 ACPI 的主機板與遙控器。
- A詳: 需一張 MCE 相容的電視卡（BDA 驅動）、安裝 MCE 相容的 MPEG-2 視訊解碼器與音訊解碼器、具備足夠硬碟空間（錄影/Time Shift）、穩定顯示卡與音效卡驅動。主機板需支援 ACPI S3/S4 與 RTC 喚醒。建議搭配 eHome 紅外線接收器與 MCE 遙控器；若使用機上盒，需 IR Blaster 控制換台。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q7, C-Q1

A-Q15: MCE 2005 常見的編解碼與檔案格式是什麼？
- A簡: 直播/錄影多採 MPEG-2，檔案以 DVR-MS 儲存，透過 DirectShow 解碼播放。
- A詳: MCE 時代的電視錄影以 MPEG-2 為主，容器常為 DVR-MS（基於 ASF，含節目中繼資料）。播放與錄影管線採 DirectShow 濾鏡，並可透過 DXVA 硬體加速降低 CPU 負載。音訊多為 MPEG-1 Layer II 或 AC-3。轉出或轉檔須考慮 DRM 保護與相容性，建議使用相容轉檔工具。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, B-Q7, C-Q10

A-Q16: MCE 2005 如何支援遙控器操作？
- A簡: 透過 eHome 紅外線接收器支援 RC6 遙控碼，提供完整導覽與控制。
- A詳: MCE 搭配 eHome IR 接收器，支援 RC6 協定的 MCE 遙控器，涵蓋方向、OK、返回、播放/暫停、錄影、資訊等鍵。系統 UI 針對遙控器最佳化，並支援 IR Blaster 發射控制外部機上盒換台。裝置驅動與遙控器對碼正確後，即可無鍵盤滑鼠操作大部分功能。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q8, D-Q6

A-Q17: MCE 2005 的限制與典型適用情境有哪些？
- A簡: 依賴相容硬體與地區 EPG；適合客廳 DVR 化、喜愛自訂與升級的用戶。
- A詳: 限制包含：地區 EPG 覆蓋度不一、需相容的 BDA 調諧卡與解碼器、DVR-MS 相容性與 DRM 限制、驅動與睡眠喚醒可靠性。典型情境如：想以 PC 取代機上盒錄影、整合媒體庫、低成本擴充硬碟、使用遙控器在客廳操作。願意調校與維護者可獲得高度彈性與效能。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q1, D-Q2, C-Q7

A-Q18: 有哪些替代方案？與 MCE 2005 差異是什麼？
- A簡: 專用 DVR、智慧電視、他牌 HTPC 軟體；差異在成本、易用與可塑性。
- A詳: 替代方案包括：TiVo 等專用 DVR（易用但需服務費）、智慧電視/機上盒 DVR 功能（整合度高但擴充有限）、其他 HTPC 軟體（功能類似但生態不同）。MCE 優勢是 Windows 生態、硬體廣泛、功能完整；劣勢是需自行安裝調校。選擇取決於你偏好「開箱即用」或「高度自訂」。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q17, B-Q1


### Q&A 類別 B: 技術原理類

B-Q1: MCE 2005 的整體架構如何運作？
- A簡: 以 ehshell UI、DirectShow 媒體管線、BDA 調諧卡、後台服務協同完成 DVR。
- A詳: 原理說明：前端 ehshell.exe 提供 10-foot UI；後端以 DirectShow 構建收視/錄影圖形，透過 BDA 驅動接管調諧卡；ehsched/ehrecvr 服務負責排程、錄影與喚醒。關鍵流程：啟動電視→建立濾鏡圖→連接調諧器/解碼→播放；預約錄影→寫入 DVR-MS→更新媒資。核心組件：eHome 服務（ehsched、ehrecvr）、BDA Tuner、MPEG-2 解碼器、DXVA、DVR-MS Writer。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, B-Q6, B-Q7

B-Q2: EPG 更新機制與工作流程是什麼？
- A簡: 定期從線上服務抓取節目資料，映射頻道，更新節目表供搜尋與預約。
- A詳: 原理說明：系統按排程連線至 EPG 服務下載頻道與節目 JSON/XML 資料。關鍵步驟：設定地區與提供者→掃描頻道→頻道號碼映射→下載/合併節目→建立索引。核心組件：EPG 資料提供端、索引器、頻道映射器、ehsched 同步器。更新影響選台顯示、整季錄影規則與衝突判定，需保持時間/時區正確。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, C-Q2, D-Q1

B-Q3: 預約錄影與排程如何實現？
- A簡: 由 ehsched 設置錄影任務與喚醒計時，ehrecvr 負責錄影寫檔與狀態。
- A詳: 原理說明：使用者在 EPG 設定錄影→ehsched 轉為具體時間與頻道的任務，設置系統喚醒計時（RTC）。關鍵流程：任務建立→前置喚醒→鎖定調諧器→ehrecvr 開檔寫入 DVR-MS→後置延長→收尾。核心組件：ehsched（排程/喚醒）、ehrecvr（錄影）、DVR-MS Writer、衝突管理器（優先序/整季規則）。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q7, B-Q12, D-Q2

B-Q4: 時光平移（Time Shift）背後機制是什麼？
- A簡: 以環狀緩衝檔即時寫入串流，播放端控制讀指標實現暫停與倒轉。
- A詳: 原理說明：直播串流經解複用後寫入硬碟緩衝（固定長度），播放器可移動讀取位置。關鍵流程：接收→寫入環狀快取→播放/快轉/倒轉→自動清退舊片段。核心組件：File Source（緩衝讀寫）、Demux（解複用）、AV 解碼器、時間基準控制（時戳）。硬碟 I/O 與解碼器品質決定平順度。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, C-Q5, D-Q5

B-Q5: 通道掃描與頻道映射如何運作？
- A簡: 掃描射頻找到頻道，配對 EPG 頻道號與名稱，建立可選清單。
- A詳: 原理說明：調諧卡逐步掃描頻段，探測視音頻 PID 以識別頻道。關鍵流程：硬體掃描→生成頻道清單→人工/自動映射至 EPG 頻道號→隱藏無效台→測試播放。核心組件：BDA 調諧器、頻道偵測器、EPG 映射器。映射品質影響選台顯示與預約準確度。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q3, D-Q8, B-Q2

B-Q6: 錄影檔案格式與寫入流程是什麼？
- A簡: 以 DVR-MS 容器包裝 MPEG-2 視訊與音訊，寫入同時嵌入節目中繼資料。

D-Q8: 頻道號碼錯亂或對不上節目表如何處理？
- A簡: 重新映射頻道至正確 EPG 台號，隱藏重複台並測試每台播放。
- A詳: 症狀：選台顯示名稱錯誤、預約錄錯台。可能原因：掃描頻率表差異、EPG 對應錯誤、重複台。解決步驟：在頻道清單手動對映台號→改名→隱藏無效台；重新掃描並測試。預防：固定供應商設定；變更訊號源後重跑精靈。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, C-Q3, D-Q1

D-Q9: DVR-MS 檔案無法播放或分享怎麼辦？
- A簡: 裝對應解碼器或使用相容播放器，必要時合法轉檔，留意 DRM 限制。
- A詳: 症狀：在他機播放失敗。可能原因：缺解碼器、DRM 限制、檔案毀損。解決步驟：於目標機安裝 MPEG-2/AC-3 解碼器或使用支援 DVR-MS 的播放器；若無 DRM，可轉檔為更通用格式。預防：保留原檔備份、維持相同解碼器環境，注意僅於合法範圍分享。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q15, C-Q10, A-Q12

D-Q10: 待機喚醒後無畫面或 HDMI 手握問題怎麼辦？
- A簡: 重新協商顯示輸出、更新驅動，調整電源策略避免關閉顯示輸出。
- A詳: 症狀：喚醒後黑屏、解析度錯亂。可能原因：顯卡驅動 bug、HDMI/HDCP handshaking 失敗、顯示輸出關閉。解決步驟：切換輸入或重插線促成 handshaking；更新顯示卡驅動；設定喚醒後重新初始化顯示；關閉「關閉顯示器」計時。預防：穩定驅動、使用可靠線材與固定 EDID。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q9, B-Q7, C-Q6
