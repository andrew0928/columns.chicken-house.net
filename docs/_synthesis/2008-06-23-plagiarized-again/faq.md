# 又被盜文了... :@

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是「盜文」？
- A簡: 未經授權複製或轉載他人文章，且未依規範署名、連結或標明來源的侵權行為。
- A詳: 盜文指未得作者同意擅自重製、散布他人文字內容。常見樣態包含全文抄貼、去頭去尾、改成簡繁體、調整格式或標題後發布，且不標示原作者與來源連結。與正當引用不同，盜文侵害作者重製、公開傳輸與署名權，傷害創作者動機與社群信任，常見於部落格、聚合站與知識平台。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, B-Q1

A-Q2: 合法轉載與盜文的差異是什麼？
- A簡: 合法轉載遵守授權條款並署名連結；盜文未授權、未署名或超出授權範圍。
- A詳: 合法轉載需符合兩要件：取得授權（明示授權如 CC 條款，或個別同意）與完整署名（作者、原文標題、連結、授權種類）。若僅複製內文、移除情境或改寫而不標明來源，即屬盜文。部分授權允許摘要與引用，但限制商用或改作，逾越即侵權。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q4, A-Q9

A-Q3: 為什麼需要註明出處與作者？
- A簡: 署名維護作者權利、資訊可追溯、尊重勞動，且利於讀者回到來源延伸學習。
- A詳: 署名是著作人格權的核心，讓讀者辨識原創者、評估可信度並追溯文脈；同時讓創作者獲得正當曝光，累積聲譽。技術文特別仰賴原文脈絡（環境、版本、限制），連回原文可避免斷章取義與誤用，也鼓勵分享文化持續運作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q13, C-Q1

A-Q4: 什麼是「署名」（Attribution）？
- A簡: 明確標示作者、原文標題、連結與授權，必要時附日期與變更說明的做法。
- A詳: 署名是對原作者與來源的清楚標註，包括作者名、原文標題、原文永久連結、使用的授權條款（如 CC BY 4.0），並建議附上發佈日期與改作註記。署名應置於顯著位置，避免僅以「來源：網路」含糊帶過。良好署名提升透明度與檢索性，便於權利核對。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, A-Q2, A-Q9

A-Q5: 什麼是「合理使用」（Fair Use）？
- A簡: 在特定目的下有限度引用他作，不影響市場且符合四要件衡量時，屬合理使用。
- A詳: 合理使用通常依四要素衡量：目的與性質（教育、評論、非商用）、著作性質（事實性較寬）、使用量與質（少量且非作品精華）、對市場影響（不取代原作需求）。簡短摘錄、評論批評與教學示例較可能合理；全文複製、移除脈絡或替代原作價值，多不構成合理使用。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q2, A-Q10, D-Q4

A-Q6: 簡繁體轉換或翻譯算不算新作品？
- A簡: 翻譯與轉換屬改作，仍依附原作權利；需授權並保留署名與來源。
- A詳: 翻譯（含簡繁轉換）屬衍生著作，必須取得原作授權方可公開散布。即便語系改變、用詞調整，內容核心仍源於原作，需保留署名與連結。除非原作採允許改作的授權條款（如 CC BY-SA），否則擅自翻譯散布屬侵權。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q9, D-Q2, C-Q10

A-Q7: 摘要、改寫與抄襲的界線？
- A簡: 摘要應自述重點並連回原文；改寫需重構觀點；抄襲則大量重製未署名。
- A詳: 摘要是用自己的話濃縮主要觀點，應附上來源連結；改寫需加入新分析、結構或觀點，具明顯轉化性；抄襲則是大量沿用原文結構與句子，僅做微幅改動（同義替換、段落刪改）且不署名，屬侵權。判斷時看相似度、用量與是否取代原作功能。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, B-Q11, D-Q9

A-Q8: 在知識問答平台轉貼解答要注意什麼？
- A簡: 引用應標註作者與連結，僅摘錄必要段落，並補充自身說明與適用條件。
- A詳: 問答平台鼓勵分享，但應以引用而非全文搬運為原則：標明作者、原文連結與段落引用標示；僅摘錄解題關鍵，並補充環境版本、限制與你的理解。若需大量內容，先徵得授權或改用連結導回原文，避免替代原作價值。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q10, A-Q4, D-Q3

A-Q9: 常見授權類型（All rights reserved 與 CC）的差異？
- A簡: All rights reserved 須個別授權；CC 事先授權，依條款署名、商用、改作等限制。
- A詳: All rights reserved 表示所有權利保留，轉載需逐一取得同意。Creative Commons 提供事前授權組合：BY（署名）、NC（非商用）、ND（禁止改作）、SA（相同方式分享）。讀者可依條款轉載與改作，但必須完整署名並遵守限制。未明示授權時，預設為權利保留。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q2, C-Q1, B-Q15

A-Q10: 什麼是「引用」與「重製」的差異？
- A簡: 引用為少量、必要且帶出處的摘錄；重製為全部或實質部分的複製散布。
- A詳: 引用以目的正當（評論、說明）為前提，摘錄少量必要內容並標明出處，且不影響原作市場；重製則是以任何方式複製作品的全部或實質部分並公開傳播。技術文章多以引用段落或程式片段為宜，全文重貼通常需授權。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, C-Q10, D-Q1

A-Q11: 連結（Link）與複製（Copy）有何不同？
- A簡: 連結導向原站不涉及重製；複製將內容搬至他處公開，通常需授權。
- A詳: 超連結僅建立導覽關係，內容仍在原站提供；複製則把文本或資源移至新頁面提供存取，觸及重製與公開傳輸權。雖連結一般合法，但若以嵌入方式規避原站脈絡或廣告，也可能引發爭議。尊重原站流量與脈絡是基本網路禮儀。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q10, A-Q12, D-Q4

A-Q12: 為何作者不貼侵權頁面的連結？
- A簡: 避免替侵權頁導流與累積權重，減少其 SEO 與曝光收益，是一種無聲抗議。
- A詳: 連出侵權頁面會增加對方流量與反向連結，可能提升其搜尋排名與可信訊號。作者可改以截圖、打碼或敘述方式取證與告知，避免不必要的導流。同時透過正式通知與申訴機制處理，兼顧證據保全與降低擴散。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q4, B-Q10, C-Q3

A-Q13: 轉貼的網路禮儀（Netiquette）是什麼？
- A簡: 先求授權、清楚署名、保留脈絡、連回原文、適量引用並尊重創作者意願。
- A詳: 良好轉貼禮儀包含：遵循授權條款或先徵求同意；完整署名與連回原文；保留關鍵脈絡（環境、版本、註解）；只引用必要段落並加入你的價值；在問答平台以摘要與連結代替全文貼；尊重作者的下架或更正要求。這些守則維繫社群信任。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, C-Q10, D-Q1

A-Q14: 盜文對創作者與社群的影響為何？
- A簡: 打擊創作動機、誤導讀者、扭曲回饋機制，並使錯誤資訊擴散。
- A詳: 盜文剝奪署名與導流，讓創作者失去回饋與成長養分；脫離原文脈絡的抄貼易造成版本錯置與誤解，讀者也無法追溯更新。搜尋排序可能錯置原創與抄本，扭曲生態。長期結果是分享意願下降、知識品質下滑，社群信任受損。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q8, D-Q4, A-Q3

A-Q15: 合法改作與二創需要哪些條件？
- A簡: 需有授權或合理使用基礎、具轉化性、完整署名與標示改動，且不替代原作。
- A詳: 合法二創建立於授權（如 CC BY-SA 允許改作）或合理使用；內容需具明顯轉化性（新目的與分析），避免僅同義改寫。必須署名原作者、標示改動，並在可行時附上差異與原文連結。發佈時應避免取代原作市場與主要功能。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, A-Q7, B-Q15

### Q&A 類別 B: 技術原理類

B-Q1: 內容盜用常見技術手法有哪些？
- A簡: 網頁抓取、RSS 全文聚合、內容拼接、同義改寫與自動轉簡繁等機制。
- A詳: 原理：以爬蟲抓取 HTML/RSS，抽取正文再發佈。流程：1) 解析頁面結構；2) 去樣板取正文；3) 轉簡繁或同義替換；4) 自站發布。組件：爬蟲（requests）、解析器（XPath/CSS）、NLP 改寫器、排程器與發布器。防範依賴結構變化、署名檢測與監控告警。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, B-Q5, C-Q7

B-Q2: 搜尋引擎如何處理重複內容（Duplicate）？
- A簡: 以指紋/相似度比對找出重複，選定權威頁做主要索引，其他降權或折疊。
- A詳: 原理：SimHash、Shingling 等文字指紋衡量相似度，配合連結分析與時序。流程：1) 建立內容指紋；2) 對比索引庫；3) 選定 Canonical；4) 折疊其他版本。組件：爬蟲、索引器、相似度模組、排名器。作者可用 rel=canonical、站內結構與外部信號輔助正本排序。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q10, C-Q6, D-Q4

B-Q3: 如何用搜尋發現被盜文的原理與方法？
- A簡: 以關鍵片段精確搜尋、設定警示與比對快照，鎖定抄貼頁面來源與時間。
- A詳: 原理：長字串匹配能高精度定位相同文本。步驟：1) 選取具體句子加引號搜尋；2) 比對結果時間與域名；3) 檢視快照/存檔；4) 記錄證據。組件：Google/Bing、快照、Wayback、RSS 追蹤。可搭配告警工具自動化監控關鍵段落。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q8, C-Q3, D-Q1

B-Q4: rel=canonical 的原理與作用是什麼？
- A簡: 指示首選版本，協助搜尋引擎將權重集中於原文，降低重複內容風險。
- A詳: 原理：在頁頭標記 rel="canonical" 指向原文 URL，告知索引系統此頁是副本或變體。步驟：1) 決定正本 URL；2) 各變體頁加入 canonical；3) 測試收錄。組件：HTML 頁頭、站內連結策略、站長工具。對 RSS/聚合引入者可引導正本排名，但無法強制抄站遵守。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q6, B-Q2, D-Q4

B-Q5: RSS 全文與摘要的風險與機制差異？
- A簡: 全文 RSS 易被抓文抄貼；摘要降低再散布價值並導流至原站詳文。
- A詳: 原理：抓文者偏好可直接複製的全文 Feed。流程：1) 產生全文/摘要；2) 客端抓取；3) 重發布。組件：Feed 產生器、摘要長度設定、導流連結。實務上以摘要＋「閱讀全文」導回原站，並插入署名/版權聲明，可降低盜用收益。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q7, B-Q1, A-Q12

B-Q6: DMCA 通知移除的處理流程是什麼？
- A簡: 權利人通知→服務商轉知→下架或反通知→爭議解決，具時限與責任分配。
- A詳: 原理：避風港機制要求平台在知悉侵權後迅速處置。步驟：1) 蒐證；2) 向平台/主機商送 DMCA 通知；3) 平台移除或轉知；4) 對方可反通知；5) 期間可能提訴訟。組件：通知格式、法務窗口、工單系統。非美國法域平台常有等效機制。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q4, C-Q5, D-Q5

B-Q7: 網路取證與證據保全如何進行？
- A簡: 保留截圖、原始碼、HTTP 標頭與時間戳，並用第三方存證確保完整性。
- A詳: 原理：固定證據狀態與時間以利舉證。步驟：1) 全頁截圖與錄影；2) 另存 HTML/CSS/JS；3) 儲存 HTTP 標頭；4) 以 WebArchive/公證服務存證；5) 記錄 WHOIS。組件：存證平台、哈希校驗、時間戳。確保可重現性與鏈結清晰。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q3, D-Q1, B-Q12

B-Q8: 版權聲明與授權標記如何被機器理解？
- A簡: 以機器可讀的標記（RDFa/JSON-LD/HTML rel）標示 CC 與作者資訊供抓取辨識。
- A詳: 原理：在頁面嵌入結構化資料宣告作者與授權。步驟：1) 選擇 CC 條款；2) 以 RDFa/JSON-LD 加入 license、creator、sameAs；3) 驗證標記。組件：schema.org/CreativeWork、rel="license"、驗證工具。利於聚合器與搜尋了解授權範圍。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q1, C-Q6, B-Q15

B-Q9: 伺服器日誌如何輔助盜文取證？
- A簡: 透過日誌追蹤爬蟲 UA、來源 IP、抓取頻率與時間，輔助還原事件時序。
- A詳: 原理：存取記錄揭示誰在何時如何抓取。步驟：1) 啟用並保留 access log；2) 過濾異常 UA/IP；3) 對照發文與抄貼時間；4) 匯出保全。組件：Nginx/Apache 日誌、Log 分析器、存證流程。可支持先發主張與抓文來源判定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, D-Q10, C-Q9

B-Q10: 連結信號與 SEO 如何影響正本排序？
- A簡: 原創頁的外部連結、內部結構與時序信號，有助於壓過抄貼頁排名。
- A詳: 原理：排名結合內容相似度、權威與連結圖。步驟：1) 建立清晰站內結構；2) 爭取高品質外鏈；3) 使用 canonical；4) 監控重複訊號。組件：Sitemap、內鏈、站長工具。避免連向抄貼頁，並集中權重到正本可提高可見度。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, A-Q12, C-Q6

B-Q11: 內容指紋（SimHash/Shingling）比對的原理？
- A簡: 以雜湊近似技術衡量文本相似度，快速偵測改寫或部分抄貼的重複內容。
- A詳: 原理：Shingling 將文本切成 k-gram，比對重合比例；SimHash 將特徵壓縮成指紋，漢明距離小代表相似。步驟：1) 正規化文本；2) 抽特徵/切片；3) 產生指紋；4) 近似檢索。組件：分詞器、Hash 函式、倒排索引。可偵測同義改寫與簡繁轉換的抄襲。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q7, B-Q2, C-Q8

B-Q12: WHOIS/IP 查詢如何定位主機商與責任方？
- A簡: 透過域名與 IP WHOIS 查詢得知註冊者、DNS 與 ASN，判斷可通知的主機商。
- A詳: 原理：登記資料揭示資源持有人與營運單位。步驟：1) 查域名 WHOIS；2) 解析 A 記錄得 IP；3) 查 IP WHOIS/ASN；4) 鎖定主機商 Abuse 聯絡。組件：RDAP、路由資料庫、反查工具。為寄送侵權通知與跨境處理提供依據。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q5, B-Q7, D-Q6

B-Q13: robots.txt 與 noindex 能防止盜文嗎？
- A簡: 不能。它們指示搜尋引擎索引行為，無法阻止惡意抓取與複製。
- A詳: 原理：robots.txt/noindex 屬自律協議，正規爬蟲會遵守，但惡意抓取可無視。步驟：1) 正確標記敏感路徑；2) 理解其僅影響收錄非存取；3) 輔以其他防護。組件：robots.txt、meta robots、驗證工具。防盜需結合監控、法務與技術跡證。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q1, C-Q7, D-Q10

B-Q14: 版權監控工具的架構與運作？
- A簡: 以關鍵片段與指紋監測全網，觸發告警與工單，支援半自動申訴。
- A詳: 原理：定期搜尋與指紋比對偵測相似頁。流程：1) 抽取特徵；2) 訂閱警示；3) 匹配命中；4) 產出報告；5) 申訴工單。組件：Google Alerts、Copyscape、API、任務排程。可將法務模板與通報流程串接，提高反應效率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q8, C-Q4, D-Q1

B-Q15: 授權相容性與轉授權驗證如何進行？
- A簡: 檢查授權條款、相容矩陣與再授權權限，確保改作與散布合法。
- A詳: 原理：授權決定可否改作/商用/再授權。步驟：1) 確認原作授權；2) 查相容矩陣（如 CC BY-SA 與其他條款）；3) 套用相容授權；4) 標示來源與改動。組件：授權文件、機器可讀標記、審核清單。避免因相容性錯誤導致侵權。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q9, A-Q15, C-Q1

### Q&A 類別 C: 實作應用類（10題）

C-Q1: 如何撰寫清楚的授權與引用規範？
- A簡: 明確列出授權條款、署名格式、可否商用/改作與引用範例，置於顯著位置。
- A詳: 步驟：1) 選擇授權（如 CC BY-NC-SA）；2) 撰寫「如何引用」段落，包含作者、標題、原文連結、授權；3) 提供正確與錯誤範例；4) 加入機器可讀標記；5) 置於關於頁與文末。程式碼：加入 <link rel="license"> 與 JSON-LD schema。注意：簡潔、易複製、全站一致，避免模糊字眼。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q9, B-Q8

C-Q2: 如何禮貌要求侵權方註明出處或下架？
- A簡: 以友善口吻提供證據與正確引用方式，設期限回應，保留後續法務空間。
- A詳: 步驟：1) 收集證據（截圖、連結、時間）；2) 發送郵件/私訊：說明侵權、給出正確署名示例與期限（如 3–5 日）；3) 記錄送達；4) 逾期轉寄主機商或平台。範本：主旨、侵權頁、原文、請求動作、期限、聯絡方式。注意：理性、中立、避免人身攻擊。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q1, C-Q5, B-Q6

C-Q3: 如何保全被盜文的證據？
- A簡: 全頁截圖與錄影、保存原始碼與標頭、第三方存證與時間戳，確保可驗證。
- A詳: 步驟：1) 使用全頁截圖與螢幕錄影；2) 另存 HTML/CSS/JS、下載圖片；3) 儲存 HTTP 標頭（curl -I）；4) 上傳 WebArchive/Notary 掛簽；5) 記錄時間線。注意：避免修改檔案、保留原檔哈希；多地備份。最佳實務：附 WHOIS 與快照比對。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, B-Q12, D-Q5

C-Q4: 如何向搜尋引擎提交 DMCA 申訴？
- A簡: 在權利移除中心填表上傳證據與聲明，提交後追蹤工單與處理結果。
- A詳: 步驟：1) 準備證據與聲明（善意、真實）；2) 前往 Google Legal Removal 或 Bing Copyright；3) 填原文/侵權網址、描述、簽名；4) 上傳截圖與存證；5) 追蹤狀態。注意：陳述具體、避免群發濫訴；保留回覆紀錄。最佳實務：同步通知平台與主機商。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, D-Q4, C-Q5

C-Q5: 如何向主機商寄發侵權通知？
- A簡: 找出主機商 Abuse 信箱，附證據與請求，引用適用法規並要求下架或限制。
- A詳: 步驟：1) 用 WHOIS/RDAP 找 IP 持有人與 Abuse 聯絡；2) 寄送詳述通知：侵權頁、原文、證據、請求動作、期限；3) 附身分與授權證明；4) 跟進工單。注意：尊重法域差異與時區；保留往返。最佳實務：同時告知域名註冊商與 CDN。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q12, B-Q6, D-Q6

C-Q6: 如何在頁面加上 canonical 與授權標記？
- A簡: 在 <head> 加 rel=canonical 指向正本，並加入 rel=license 與 JSON-LD 作者/授權。
- A詳: 步驟：1) 在正本與變體頁：<link rel="canonical" href="正本URL">；2) 授權：<link rel="license" href="授權URL">；3) JSON-LD CreativeWork：author、datePublished、license；4) 驗證。注意：避免重複 canonical、確保 URL 正規化。最佳實務：站內所有頁一致。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q8, B-Q10

C-Q7: 如何設計 RSS 降低抓文？
- A簡: 提供摘要、加導流連結與署名提示，於 Feed 中嵌入版權與原文 URL。
- A詳: 步驟：1) 設定摘要長度 100–200 字；2) 在 description 末加「閱讀全文」連原文；3) 插入作者、授權、版權宣告；4) 加上原文永久鏈；5) 監控異常抓取。注意：摘要保留關鍵脈絡；避免輸出全文。最佳實務：用 Feed Proxy 插入追蹤參數。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, B-Q13, C-Q8

C-Q8: 如何用 Google Alerts 監控盜文？
- A簡: 以引號包關鍵句建立警示，設定頻率與語言，命中後核對來源與存證。
- A詳: 步驟：1) 挑選不易改寫的關鍵句；2) 建立警示：「"關鍵句"」；3) 設週期與來源；4) 命中檢視頁面、快照與日期；5) 入庫證據。注意：多語與簡繁變體分別設定；結合 Copyscape/自建指紋。最佳實務：建立回應 SOP 與工單流。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q14, B-Q11, C-Q3

C-Q9: 如何嵌入隱性水印或 Web Beacon 追蹤？
- A簡: 在圖片/內嵌資源加入唯一標記或追蹤像素，記錄外站載入行為與時間。
- A詳: 步驟：1) 圖片加不可見水印或 EXIF 訊息；2) 內嵌 1x1 追蹤像素；3) 設唯一查詢參數；4) 伺服器記錄 Referrer/IP/時間；5) 異常告警。注意：符合法規與隱私；避免影響讀者體驗。最佳實務：僅用於取證輔助，不作唯一依據。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q9, B-Q7, D-Q10

C-Q10: 在問答平台正確引用與回覆的方法？
- A簡: 以摘要+連結為主，引用必要段落並標作者與授權，補充個人環境與說明。
- A詳: 步驟：1) 摘要原文重點；2) 用引言標示關鍵段落；3) 標作者、標題、原文連結與授權；4) 補充你環境版本與注意事項；5) 若需大量內容先徵授權。注意：避免全文貼。最佳實務：附上你實測結果與替代解法，提升價值。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, A-Q10, D-Q3

### Q&A 類別 D: 問題解決類（10題）

D-Q1: 遇到未署名轉貼怎麼辦？
- A簡: 先取證，再友善通知對方修正與署名；逾期未處理再向平台/主機商申訴。
- A詳: 症狀：你內容出現在他站且無來源。原因：不懂授權或故意抄貼。解法：1) 截圖/存證；2) 發出修正/下架請求；3) 設期限；4) 未果提報平台與主機商或 DMCA；5) 視情況公開說明。預防：清楚授權、RSS 摘要、監控告警。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q2, C-Q3, C-Q4

D-Q2: 對方去頭去尾並轉簡體以規避相似度，怎麼處理？
- A簡: 強調核心內容相同，提供原始與抄本比對證據，依流程申訴與要求署名或下架。
- A詳: 症狀：段落重排、簡繁轉換。原因：試圖躲避檢測。解法：1) 用關鍵句比對與指紋工具；2) 存證簡繁兩版本；3) 通知對方與平台；4) 申訴附上相似比對。預防：在段落內嵌可辨識標識與署名提示，維持先發時間證據。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q11, C-Q3

D-Q3: 內容被貼到問答平台賺點數，怎麼回應？
- A簡: 以平台檢舉機制申訴，並留言提供正確引用與原文連結，要求修正或移除。
- A詳: 症狀：他人以你文當解答未署名。原因：急於得分或不知授權。解法：1) 平台檢舉侵權；2) 留言要求署名連結或改為摘要；3) 向平台說明你為權利人；4) 必要時 DMCA。預防：提供可複製的正確引用格式，教育社群。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, C-Q10, C-Q4

D-Q4: 搜尋引擎把抄貼頁排在原文前面怎麼辦？
- A簡: 強化正本信號（canonical、內外鏈）、提交移除與回報抄貼，避免導流給抄站。
- A詳: 症狀：SERP 抄本優先。原因：權重、時序與連結圖失真。解法：1) 加 canonical 與結構化資料；2) 爭取外鏈與內鏈集中；3) 對抄本提移除；4) 回報錯誤歸屬。預防：發布同步推送索引用、站長工具監控、RSS 摘要化。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q4, C-Q6

D-Q5: 對方拒絕下架也不署名，後續如何處置？
- A簡: 升級至平台/主機商與搜尋引擎法務通道，必要時委託律師採取法律行動。
- A詳: 症狀：溝通無效。原因：惡意或牟利。解法：1) 向主機商/域名註冊商寄發通知；2) 搜尋引擎 DMCA 下架；3) 評估訴訟風險與成本；4) 公開聲明與社群協助。預防：標準化 SOP、範本與證據庫，縮短反應時間。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q5, C-Q4, B-Q6

D-Q6: 海外網站或不適用 DMCA 的情況怎麼辦？
- A簡: 使用當地等效機制與主機商政策，或透過搜尋引擎下架與社群壓力處理。
- A詳: 症狀：跨境無法直接 DMCA。原因：法域差異。解法：1) 查當地法律與平台政策；2) 仍通知主機商（多數有濫用政策）；3) 向搜尋引擎申訴；4) 社群揭露與品牌溝通。預防：多語授權聲明、國際化存證。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q12, C-Q5, C-Q4

D-Q7: 過度反擊引發公關反噬如何拿捏？
- A簡: 就事論事、聚焦行為不攻擊個人，保留溝通紀錄並公開透明說明處置。
- A詳: 症狀：社群爭議擴大。原因：語氣強硬、指控過頭。解法：1) 使用中性語言與事實陳述；2) 公開證據與流程；3) 先私下溝通再公開；4) 必要時第三方調解。預防：制定溝通準則與對外罐頭回應，避免情緒化。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q2, D-Q1, A-Q13

D-Q8: 頻繁盜文打擊創作動機，如何自救？
- A簡: 建立監控與快速應對機制，轉向社群平台深耕、內容差異化與會員化。
- A詳: 症狀：疲於維權、意興闌珊。原因：回饋失衡。解法：1) 自動監控與外包處理；2) 內容服務差異化（互動、社群、課程）；3) 會員專區與郵件直達；4) 心理與時間管理。預防：明確授權、提高原站價值與連結網。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q14, B-Q14, C-Q8

D-Q9: 被反指控我抄襲，如何澄清？
- A簡: 以先發證據、原始檔與版本控制記錄，對照時序與變更歷程自證清白。
- A詳: 症狀：遭抄者反咬。原因：時序混淆。解法：1) 出示發布時間、快照、Git 紀錄；2) 提供草稿/原檔與哈希；3) 比對內容差異與先發證明；4) 平台申訴。預防：固定發佈時間戳與第三方存證、保持原檔與草稿鏈。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, B-Q2, C-Q3

D-Q10: 整站被鏡像或大量搬運的對策？
- A簡: 以技術封鎖、法務通知與搜尋引擎處置並行，輔以追蹤與黑名單策略。
- A詳: 症狀：域名鏡像、大量同步。原因：自動化抓取。解法：1) 層級封鎖：IP/ASN、速率限制、WAF；2) 追蹤：Beacon/日誌；3) 法務：主機商/註冊商/DMCA；4) SEO：canonical/移除索引；5) 公開揭露。預防：CDN/WAF 規則、RSS 摘要、速率控管。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q1, B-Q9, C-Q9
