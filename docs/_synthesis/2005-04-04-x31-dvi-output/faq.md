# ThinkPad X31 以 PCI 顯示卡啟用 DVI 輸出：概念、原理、實作與故障排除 FAQ

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 什麼是 DVI？
- A簡: DVI 是數位視訊介面，提供數位端對端傳輸，減少轉換失真。常見 DVI-D 與 DVI-I，對 LCD 螢幕尤為清晰穩定。
- A詳: DVI（Digital Visual Interface）是一種以數位訊號傳輸視訊的介面標準，避免了 VGA 類比訊號的數位/類比轉換損失。DVI-D 傳數位，DVI-I 同時帶類比與數位訊號，便於相容舊裝置。其優點包含影像銳利、色彩準確、抗雜訊與對高解析度支援良好，並能透過 EDID 取得螢幕資訊實現即插即用。對 LCD 這類原生數位顯示器尤其能提升畫質。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q17, A-Q18, B-Q5, B-Q10

Q2: 為什麼 LCD 螢幕用 DVI 通常比 VGA 更好？
- A簡: DVI 免去數位/類比轉換，降低畫質損失；LCD 原生數位，DVI 可提供更銳利文字與穩定顏色。
- A詳: LCD 的面板內部為數位矩陣，VGA 類比輸入需經 DAC/ADC 兩次轉換，易導致幾何與細節失真。DVI 以數位訊號直接傳遞像素資料，避免時序飄移與電磁干擾，文字邊緣更銳利、灰階過渡更穩定，並能精確對應原生解析度。對長時間閱讀或細節檢視工作而言，使用 DVI 會有明顯舒適度與準確性提升。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q17, B-Q5, B-Q6

Q3: 什麼是 ThinkPad X31 的 Dock PCI 擴充？
- A簡: X31 可搭配具單一 PCI 插槽的 Dock，透過底座匯流排外接傳統 PCI 擴充卡。
- A詳: X31 搭配的 Dock（底座）提供擴充埠與一個傳統 32-bit PCI 插槽。筆電底座連接器將主機板的 PCI 匯流排訊號延伸到 Dock，讓使用者能插入桌機用的 PCI 周邊卡（如顯示卡、1394 卡等）。此設計提升了老筆電的擴充性，但受限於單一插槽、匯流排頻寬與電力、以及多數 Dock 世代對 PCI 熱插拔支持不足的現實。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q1, B-Q13, D-Q2, C-Q10

Q4: 什麼是外接 PCI 顯示卡？
- A簡: 外接 PCI 顯示卡是插在 Dock 的 PCI 插槽，提供額外顯示輸出（如 DVI）的 GPU 方案。
- A詳: 外接 PCI 顯示卡是將桌機用 PCI 介面的顯示晶片卡安裝於筆電 Dock 的 PCI 插槽，以補足內建顯示輸出的限制（例如缺乏 DVI）。它具備獨立的 VRAM 與輸出埠（DVI、VGA、S-Video 等），可作為主要或次要顯示配接器。優點是畫質與輸出介面擴充，但在舊型 Dock 環境下常無法熱插拔，且可能占用唯一插槽。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, B-Q3, B-Q19, C-Q1

Q5: 為什麼舊款內顯常沒有 DVI 輸出？
- A簡: 早期行動 GPU 著重低功耗與成本，常僅提供 VGA；DVI 需額外晶片與佈線。
- A詳: 早期筆電設計受限於成本、主機板空間與功耗預算，行動 GPU 多以 VGA 類比輸出為主。DVI 需要 TMDS 傳輸對、連接器與額外的訊號整合，對主機板與屏線要求更高。且當時 LCD 普及程度不足、解析度較低，DVI 的邊際價值較小，因此許多舊款機種不提供 DVI，促成外接顯卡或 Dock 擴充的需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q2, B-Q5, C-Q1

Q6: 什麼是 IBM Presentation Director？
- A簡: IBM 提供的顯示情境切換工具，整合快捷鍵切換內外螢幕、投影等情境。
- A詳: Presentation Director 是 IBM（後 Lenovo）為 ThinkPad 開發的顯示配置管理工具，可定義「情境」以快速切換內建面板、外接螢幕、複製/延伸模式與解析度/色彩深度設定。其優勢在與特定機型驅動與 ACPI 的整合。但早期版本通常只對內建 GPU 與支援名單內的輸出介面最佳化，對外插第三方顯卡的支援有限。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q4, C-Q5, D-Q3, C-Q3

Q7: Presentation Director 與顯示卡驅動的關係是什麼？
- A簡: 它透過驅動與 ACPI 介面套用顯示配置；支援度受限於驅動相容性與機型整合。
- A詳: 該工具本質是高階自動化層，呼叫顯示卡驅動提供的 API（如解析度、螢幕拓撲）與作業系統顯示設定，再配合機型 ACPI/熱鍵事件達成一鍵切換。因此若外接顯卡驅動未暴露必要介面、或非 ThinkPad 驅動套件的支援範圍，工具就難以正確辨識與管理，導致僅能操作內建 GPU 的情況。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q12, D-Q3, C-Q5

Q8: 什麼是 IEEE 1394（FireWire）？
- A簡: 1394 是高速序列匯流排，適合影像擷取、硬碟與音訊設備，具即時與等時性傳輸。
- A詳: IEEE 1394（又稱 FireWire、i.LINK）提供高效能、低延遲與等時性（Isochronous）傳輸，常見於 DV 攝影機、音訊介面與外接儲存。其直連拓撲與裝置自供電能力，讓多媒體工作流程穩定。筆電可透過 1394 卡擴充此介面，但需要 PCI 或 PCMCIA 插槽資源，與顯示卡等擴充需求形成取捨。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, C-Q6, D-Q8, C-Q10

Q9: PCI 與 PCMCIA（CardBus）的差異是什麼？
- A簡: PCI 是桌機內部匯流排；PCMCIA/CardBus 是筆電外插擴充卡，頻寬與電力較低。
- A詳: PCI 是主機板內部並行匯流排（32-bit/33MHz 常見），可用於顯示卡、網卡等高帶寬裝置。PCMCIA/CardBus 為筆電外接擴充規格，CardBus 本質接近 32-bit PCI，但受限外形與供電，實際可用頻寬與功耗較低，適合 I/O 類卡（如 1394、網卡）。兩者在物理介面、功耗能力、熱插拔性與可用驅動生態不同。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, C-Q6, C-Q10, D-Q9

Q10: 休眠與關機有何差異？
- A簡: 休眠將記憶體狀態寫入磁碟後斷電；關機則完全結束系統，不保留執行狀態。
- A詳: 休眠（Hibernate）把 RAM 內容存成映像檔，之後關閉電源；開機時再將映像載回，恢復應用程式狀態。關機（Shutdown）則正常結束進程、卸載驅動並關閉電源，不保留會話。休眠適合快速恢復長工作階段；但遇到硬體拓撲改變（如顯示卡移除），恢復可能失敗，這時需改以完全關機避免不一致。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q11, D-Q4, D-Q2, B-Q14

Q11: 休眠與睡眠（待命）差異為何？
- A簡: 睡眠保留 RAM 通電可快速喚醒；休眠寫入磁碟可斷電更省電但恢復較慢。
- A詳: 睡眠（Sleep/Standby）將系統置於低功耗，RAM 維持通電，喚醒迅速但耗電較休眠高；休眠將 RAM 寫入磁碟後完全斷電，恢復較慢但不耗電。兩者遇到硬體變更的韌體/驅動相容性各異，外接顯卡的在位與否，可能使喚醒或恢復時顯示拓撲錯亂或失敗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q10, D-Q4, B-Q14, B-Q12

Q12: 什麼是熱插拔與熱抽換？PCI 顯卡能熱插拔嗎？
- A簡: 熱插拔允許通電插拔裝置；傳統桌機 PCI 顯卡多不支援，Dock 多數亦不行。
- A詳: 熱插拔（Hot Plug）允許通電更換裝置並由系統動態管理資源；熱抽換（Hot Swap）強調更換後不中斷服務。傳統 32-bit PCI 與舊世代 Dock 多不支援對顯示卡的熱插拔，OS/BIOS/ACPI/電源序列亦未針對 GPU 完整設計。因此外接 PCI 顯卡通常需關機或至少卸載至非使用狀態方可移除。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q13, D-Q2, C-Q7

Q13: 外接顯卡與內顯的切換概念是什麼？
- A簡: 由 BIOS/OS 與驅動決定主/次配接器，輸出與模式由驅動與設定檔協同控制。
- A詳: 系統開機時會依 BIOS 設定與硬體偵測選定主顯示配接器（Primary Adapter），OS 載入對應驅動後，再決定是延伸桌面、複製或單一輸出。外接顯卡可能被設為主顯示，影響登錄畫面輸出與 Dock/Undock 行為。切換需驅動支援、OS 的螢幕拓撲設定與工具（如 GPU 控制台或 Presentation Director）協同。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q12, C-Q2, D-Q5

Q14: 什麼是顯示模式切換（複製/延伸/僅外接）？
- A簡: 複製同畫面，延伸擴大桌面，僅外接僅用外螢幕；由 OS 與驅動共同控制。
- A詳: 顯示模式包含鏡像（Clone/鏡射）、延伸（Extend）與單螢幕（Internal Only/External Only）。鏡像便於簡報，但解析度受限於最低面板；延伸可提升工作區域；僅外接常用於桌面替代。模式切換需考量主顯示、EDID 解析度、色彩深度與更新率，由 OS 顯示設定與 GPU 控制面板協作完成。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q12, C-Q5, D-Q5, A-Q6

Q15: 為什麼外接 PCI 顯卡常「不能停用」就拔除？
- A簡: 因作業系統將其作為活躍顯示配接器，且 PCI 不支援 GPU 熱拔除。
- A詳: 外接顯卡若正提供桌面或作為主顯示，OS 與驅動綁定視窗管理器、GDI/DirectX 資源在該配接器上，無法即時卸載；加上傳統 PCI 無 GPU 熱插拔流程，拔除會導致系統當機或黑屏。因此需要切換回內顯、登出甚至關機，確保驅動卸載與資源釋放後，方能安全移除 Dock。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q14, D-Q2, C-Q7

Q16: 什麼是 EDID？
- A簡: EDID 是螢幕提供的顯示能力描述，讓主機讀取支援解析度與時序以正確輸出。
- A詳: EDID（Extended Display Identification Data）由顯示器透過 DDC 通道提供，包含廠牌、型號、建議解析度、支援時序、色域等資訊。GPU 驅動讀取 EDID 後決定可用解析度與最佳化輸出，並支援熱插拔偵測。若 EDID 讀取失敗，可能造成「無訊號」或不正確解析度，需要重讀、EDID 覆寫或強制自訂時序。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q10, D-Q1, D-Q7, C-Q8

Q17: DVI 與 VGA 的訊號差異是什麼？
- A簡: DVI 傳數位 TMDS，抗干擾佳；VGA 為類比 RGB，易受線材與時序影響。
- A詳: DVI 以 TMDS 編碼數位傳輸像素資料，訊號完整度高、雜訊低，線長受限下仍保真。VGA 以類比電壓表示顏色，易受阻抗不匹配、電磁干擾、時鐘抖動影響，導致模糊與彩邊。對原生數位的 LCD，DVI 可像素對齊顯示，文字清晰；VGA 需 ADC 重建像素，對準與相位調校成為關鍵。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q2, B-Q5, C-Q8

Q18: DVI-D 與 DVI-I 有何不同？
- A簡: DVI-D 僅數位訊號；DVI-I 同時帶數位與類比腳位，更具相容性。
- A詳: DVI-D（Digital）只提供 TMDS 數位通道，適合純數位顯示器。DVI-I（Integrated）在相同連接器上同時提供 TMDS 與類比 RGB 腳位，可透過轉接頭輸出 VGA。選購顯示卡與線材時需確認端口型別與通道（單/雙通道），以對應顯示器需求與解析度帶寬要求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, B-Q17, C-Q1, C-Q8

Q19: nVidia GeForce FX5200 屬於什麼級別的顯卡？
- A簡: FX5200 為入門級 GPU，支援基本 2D/多螢幕與 DVI，足以驅動辦公與一般顯示。
- A詳: GeForce FX5200 是早期入門級 GPU，雖 3D 效能有限，但具備獨立 VRAM 與常見的 DVI/VGA 輸出，對於 2D 桌面、雙螢幕與高解析度 LCD 駕馭足夠。其低功耗、低發熱特性適合 Dock 的電力與散熱條件，用於為舊款筆電補上 DVI 是務實選擇。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, C-Q9, B-Q19, D-Q7

Q20: 使用 Dock 的核心價值是什麼？
- A簡: Dock 提供 I/O 與插槽擴充，讓筆電具桌面級連接與工作流，提升生產力。
- A詳: Dock 透過專用連接介面延伸多種 I/O（視訊、網路、USB、音訊）與插槽（如 PCI），讓筆電在辦公桌上具桌機般的擴充性與便利切換，回到行動時再脫離。這特別適合需要外接高品質顯示（DVI）、高速 I/O（1394）等需求，同時集中線材管理，改善日常使用體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, C-Q10, B-Q13, D-Q10

Q21: 為什麼有人選擇外接顯卡來獲得 DVI？
- A簡: 內建顯示不支援 DVI 時，外接卡能提升畫質、解析度與輸出彈性。
- A詳: 舊款筆電常僅提供 VGA，無法滿足 LCD 的數位輸出需求。外接顯卡透過 Dock 的 PCI 插槽加入 DVI 端口，提供更清晰穩定的影像、支援更高解析度與多螢幕。雖需取捨插槽與熱插拔限制，但對長時使用外接顯示器的使用者，整體觀感與效率提升顯著。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, C-Q1, C-Q2, D-Q1

Q22: 何謂主顯示配接器（Primary Adapter）？
- A簡: 系統作為主要輸出的顯示卡，影響登入畫面、工作列與視窗管理之歸屬。
- A詳: Primary Adapter 是 OS 與韌體在啟動時選定的主要顯示設備。其決定了開機畫面輸出位置、登入 UI、預設工作列與全螢幕應用的投影目標。當外接顯卡經 Dock 連接時，可能被選為 Primary，影響 Undock 流程與顯示切換策略，需在設定中明確規劃。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, C-Q2, D-Q5, A-Q13

Q23: 為什麼 Presentation Director 可能無法管理外接顯卡？
- A簡: 工具與內顯整合深，對第三方卡驅動支援有限，無法取得必要控制介面。
- A詳: 該工具依賴特定 ThinkPad 驅動與 ACPI 事件，並呼叫受支援的顯示驅動 API 完成切換。若外接顯卡的驅動未暴露或未被工具識別，或未納入支援名單，則情境切換無法涵蓋該卡。此時需改用 GPU 原廠控制台或 OS 顯示設定與指令工具自建情境。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q4, C-Q5, D-Q3

Q24: 為什麼 Dock 常只有一個 PCI 插槽，該如何取捨？
- A簡: 體積、熱與成本限制，使 Dock 僅配單槽；需在顯卡、1394 等間取捨或改走 PCMCIA。
- A詳: Dock 的工業設計受限尺寸、散熱、電力供應與成本，多僅提供單一 PCI 插槽。當需求同時包含 DVI 與 1394 時，必須權衡：顯示卡可用 DVI 改善日常視覺；1394 可改採 PCMCIA/USB 方案。規劃時依使用頻率與效益排序，並留意驅動與供電兼容性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, A-Q9, C-Q6, C-Q10

### Q&A 類別 B: 技術原理類

Q1: X31 的 Dock PCI 匯流排是如何連到筆電主機板的？
- A簡: 透過專用底座連接器延伸 PCI 匯流排訊號至 Dock 的單一 PCI 插槽。
- A詳: Dock 底部的高密度連接器將主機板上的 PCI 匯流排與 I/O 控制訊號引出，Dock 內部再經橋接與供電管理後提供標準 32-bit PCI 插槽。此路徑受限於線長、阻抗與電源序列，通常不設計為熱插拔；當插入顯卡時，系統在開機階段分配 PCI 資源與中斷，驅動於 OS 載入後接管裝置。

Q2: 為何 Windows 對傳統 PCI 顯卡不支援熱拔除？
- A簡: 缺乏 GPU 熱插拔協定與資源釋放機制，拔除會破壞圖形子系統。
- A詳: 傳統 PCI 無統一的 GPU Hot Plug 標準，Windows 圖形堆疊（GDI/DirectX）與視窗管理依賴顯示配接器持續存在。若活躍顯示卡被移除，資源與中斷映射中斷，導致黑屏或藍屏。僅特定伺服器平台上的 PCIe 與專用熱插拔控制器、配合驅動設計，才具備有限的熱拔除能力。

Q3: 主顯示配接器的選定流程是什麼？
- A簡: BIOS 偵測裝置順序與設定，OS 載入驅動後確立主/次顯示與輸出拓撲。
- A詳: 開機階段，BIOS 依據裝置枚舉與設定（如「Init Display First」）挑選初始化的 VGA 裝置。進入 OS 後，圖形驅動載入，根據可用連線與使用者設定決定主顯示、延伸或鏡像。Dock 中外接顯卡若優先初始化，可能成為主顯示，影響登入畫面與 Undock 行為。

Q4: Presentation Director 的切換機制與限制為何？
- A簡: 透過 ACPI 熱鍵事件搭配顯示驅動 API 套用情境；受限於驅動支援清單。
- A詳: 工具監聽熱鍵與 Dock 狀態變更，呼叫系統與顯示卡驅動 API（變更主要螢幕、解析度、輸出啟停）並套用預存情境。若第三方外接 GPU 不在支援矩陣，或驅動未提供一致 API，則無法納管其輸出，只能管理內建顯示，需以替代方案補足。

Q5: DVI 影像品質提升的技術機制是什麼？
- A簡: TMDS 數位傳輸避免 ADC/DAC 誤差，像素對齊與 EDID 協助正確時序。
- A詳: DVI 以 TMDS 封裝像素資料，經差動對傳輸，抗干擾與穩定性高。LCD 端直接以數位驅動像素，不需類比重建，消除相位/取樣誤差。EDID 提供建議時序，驅動選擇最佳解析度與刷新率，確保像素對齊與色彩準確，最終呈現更銳利的文字與穩定的灰階。

Q6: LCD 與 CRT 對訊號的需求有何不同？
- A簡: LCD 原生數位、像素對齊；CRT 原生類比、需連續波形驅動。
- A詳: CRT 以電子束掃描，需要穩定的類比波形（VGA）與同步訊號；分辨率彈性大。LCD 是固定像素矩陣，最佳於原生解析度顯示，數位訊號（DVI）能直接對應像素網格。當以 VGA 驅動 LCD，需 ADC 重建，易產生模糊，並仰賴相位/時鐘校準。

Q7: IEEE 1394 的傳輸原理與優勢？
- A簡: 採串列總線與等時通道，提供穩定低延遲，適合影音串流與擷取。
- A詳: 1394 以分散式仲裁與等時性通道確保頻寬與時間準確性，避免丟幀。其點對點拓撲、即插即用與裝置供電讓多媒體裝置鏈接簡單。相對 USB，1394 更重視持續吞吐與等時傳輸，故在 DV 影像與音訊工作流表現佳。缺點是生態逐漸萎縮與介面普及度下降。

Q8: PCMCIA/CardBus 的工作方式是什麼？
- A簡: 以類 PCI 的 32-bit 通道提供熱插拔 I/O 擴充，受限於外形與供電。
- A詳: CardBus 是 PCMCIA 的 32-bit 規格，透過控制器橋接至系統匯流排，支援熱插拔與即插即用。其頻寬足以支援網卡、1394、USB 控制器等 I/O，但對高帶寬/高功耗裝置（如顯卡）不適合。驅動透過標準匯流排介面曝光，利於快速部署。

- A詳: 傳統 PCI 沒有一致的熱插拔設計，僅特定伺服器支援；PCIe 規範提供 Hot Plug 能力（Slot Power Control、Presence Detect），不過仍仰賴主板、韌體與驅動協作。對 GPU，此能力常限於特定平台（如伺服器/工作站），一般 Dock 難以達成。

Q17: DVI 單通道與雙通道帶寬如何影響解析度？

- A詳: DVI 單通道（Single-Link）具 165 MHz TMDS 時脈，典型可支援 1920×1200@60Hz；雙通道（Dual-Link）將資料線倍增以提高帶寬，可達 2560×1600 等高解析度。實際可行還受限於顯示卡輸出能力、線材品質與顯示器支援。

Q18: 多螢幕的延遲與幀緩衝如何影響體驗？

Q19: 驅動如何處理不同廠牌 GPU 同時存在？

- A詳: OS 會為每張 GPU 載入對應驅動，管理各自的輸出埠與 EDID。若不同供應商驅動在圖形堆疊處理存在衝突，可能導致不穩定或黑屏。實務上可避免混用、或選擇一張為主顯示，另一張僅做副輸出；必要時使用基本顯示驅動。

- A詳: 休眠映像包含顯示狀態與記憶體內容，若恢復時外接顯卡不在或成為不同主/次關係，驅動需重建資源。舊平台/舊驅動處理不完善，導致失敗、黑屏或還原緩慢，體驗劣化。因此常建議以關機或在 Dock 狀態固定時使用休眠。

Q23: 以外接顯卡驅動 DVI，對電源與散熱有何要求？

- A詳: 外接顯卡雖為入門款，仍需連續穩定的供電與基本散熱。Dock 的電源模組與通風設計有限，長時間高解析輸出（高像素時脈）會放大電壓與熱的壓力。選低功耗卡、確保 Dock 通風與避免阻塞散熱孔，有助穩定。

Q1: 如何在 X31 的 Dock 安裝 PCI 顯示卡以啟用 DVI？
- A簡: 關機斷電，裝入低功耗 PCI 顯卡，接 DVI 線與螢幕，開機安裝驅動並測試。
- A詳: 實作步驟：1) 完全關機並拔除電源；2) 打開 Dock 槽，插入低功耗單槽 PCI 顯示卡（如 FX5200），固定螺絲；3) 連接 DVI 線至 LCD；4) 接回電源並開機；5) 安裝 nVidia 驅動；6) 於顯示設定確認外接螢幕。注意：選擇 DVI-D 線材、確保 Dock 通風，並避免與其他高耗電卡併用。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q4, B-Q1, C-Q9, D-Q1

Q2: 如何設定 Windows 以外接 DVI 為主要螢幕？
- A簡: 進入顯示設定選擇外接螢幕為主顯示，套用解析度並關閉內建面板或延伸。
- A詳: 步驟：1) 右鍵桌面→顯示設定；2) 點選外接螢幕（DVI）→「設為主要顯示」；3) 套用原生解析度/縮放；4) 視需求關閉內建面板或設為延伸。nVidia 控制台可進一步設定拓撲。注意：登入畫面可能仍受 Primary 選擇影響，Dock 狀態請保持一致。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q13, B-Q3, B-Q12, D-Q5

Q3: 如何建立「回家/外出」兩種顯示配置快速切換？
- A簡: 建立兩套顯示設定檔，透過 GPU 控制台或腳本快捷鍵一鍵套用。
- A詳: 作法：1) 在 GPU 控制台建立「家用 DVI」與「外出內建」兩套配置；2) 啟用快捷鍵切換；3) 或用 Windows 顯示設定與第三方工具儲存配置；4) 舊版 Windows 可用硬體設定檔配合 Dock 狀態套用。注意：切換前先存檔，避免在全螢幕程式中切換。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q11, B-Q24, D-Q10

Q4: 如何用 devcon 指令啟用/停用外接顯卡以輔助切換？
- A簡: 安裝 devcon，取得裝置硬體 ID，編寫批次檔以 disable/enable 外接 GPU。
- A詳: 步驟：1) 安裝 Microsoft devcon；2) 在裝置管理員查外接顯卡硬體 ID（如 PCI\VEN_10DE…）；3) 建立批次檔：devcon disable "PCI\VEN_10DE*"；devcon enable "PCI\VEN_10DE*"；4) 配合登出/登入或重啟 Explorer。注意：請勿在其為主顯示時停用，可能黑屏；先切回內顯。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q2, B-Q11, D-Q2, D-Q3

Q5: 沒有 Presentation Director 時，如何切換顯示模式？
- A簡: 使用 OS 顯示設定或 GPU 控制台，設定主顯示與延伸/鏡像模式。
- A詳: 方法：1) 右鍵桌面→顯示設定，選擇顯示器，設定「主顯示」、「延伸/複製」；2) 以 nVidia 控制台套用拓撲；3) 可用快捷鍵或小工具（如顯示切換器）快速套用。注意：保留安全回退方案（計時自動還原），避免錯誤設定造成黑屏。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q12, D-Q5, C-Q3

Q6: 如何選購 PCMCIA（CardBus）1394 擴充卡？
- A簡: 選 OHCI 相容晶片、足夠供電與需要的端口型式（4/6pin），留意驅動支援。
- A詳: 重點：1) 晶片挑選 TI/NEC OHCI 相容；2) 端口數量與型式（4-pin 無電供、6-pin 供電）；3) 外接電源選項以帶動高耗電裝置；4) 驅動支援 OS 版本；5) 隨附纜線。安裝後測 DV/音訊裝置穩定性。若需求降低，可考慮轉向 USB 介面之替代品。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q8, A-Q9, B-Q8, D-Q8

Q7: 如何降低「必須關機才能脫離 Dock」的不便與風險？
- A簡: 制定固定流程：先切回內顯、存檔、停用外顯，再關機脫離，縮短停機時間。
- A詳: 建議：1) 建立「外出」配置（主顯示內建）；2) 關閉應用與存檔；3) 用 devcon 停用外顯（若非主顯示）；4) 關機而非休眠；5) 脫離 Dock；6) 啟動後以內顯工作。回家時再 Dock、開機、套用「家用 DVI」。維持流程一致性，可大幅降低錯誤率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, B-Q13, B-Q14, D-Q2

Q8: 如何最佳化 DVI 輸出的畫質與縮放？
- A簡: 使用螢幕原生解析度、正確縮放模式、關閉過度銳化並校正顏色。
- A詳: 步驟：1) 設定螢幕原生解析度與 60Hz；2) 在顯示器選單選 1:1 或「僅中心」縮放；3) 關閉銳化/動態對比；4) 顯卡面板設定正確色域與完整 RGB 範圍；5) 進行基本校色。注意線材品質與長度，避免 EDID 讀取失敗導致錯誤模式。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q2, A-Q16, B-Q10, D-Q7

Q9: 如何更新 FX5200 驅動以提升相容性？
- A簡: 下載對應版本的驅動，乾淨移除舊版後安裝，測試多螢幕與休眠恢復。
- A詳: 實作：1) 取得適用 OS 的 nVidia 驅動（或穩定舊版）；2) 以 DDU/控制台移除舊驅動；3) 重新開機後安裝新版；4) 測試 DVI 輸出、延伸/鏡像模式與解析度；5) 驗證休眠/喚醒；6) 若不穩，回退已知穩定版。注意避免與內顯驅動衝突。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q19, D-Q6, D-Q4, C-Q5

Q10: 如何規劃 Dock 唯一 PCI 槽的使用？
- A簡: 盤點需求，優先畫質或 I/O，替代以 PCMCIA/USB，並評估功耗散熱。
- A詳: 流程：1) 列出常用情境（長時外接顯示/影音擷取/儲存）；2) 決定 PCI 槽用於顯卡或 1394；3) 若選顯卡，1394 改 PCMCIA；4) 驗證供電與散熱；5) 設定切換流程與備用方案。記錄驅動版本與設定，降低日後維護成本。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q20, A-Q24, A-Q8, D-Q9

### Q&A 類別 D: 問題解決類（10題）

Q1: 遇到外接 DVI「無訊號」怎麼辦？
- A簡: 檢查線材/輸入源、驅動與解析度；重讀 EDID 或以安全模式回退設定。
- A詳: 症狀：螢幕顯示無訊號或待機。原因：EDID 讀取失敗、解析度超出、輸入源錯誤、鬆脫或驅動故障。解法：1) 確認螢幕輸入為 DVI；2) 換線/縮短線；3) 以 OS 設定選擇可用較低解析度；4) 重新安裝驅動；5) 必要時安全模式清除顯示設定。預防：使用優質線材與固定配置檔。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q10, A-Q16, C-Q8, D-Q7

Q2: Dock 拔除後系統當機或藍屏，怎麼處理？
- A簡: 不要熱拔顯卡；先切回內顯、停用外顯再關機脫離，或使用硬體設定檔。
- A詳: 症狀：拔 Dock 當下黑屏或 BSOD。原因：外顯為主顯示且活躍，PCI 無熱拔流程。解法：1) 建立脫離流程（切回內顯→停用外顯→關機→拔除）；2) 使用 devcon 與硬體設定檔幫助切換；3) 驅動更新。預防：嚴禁熱拔顯示卡，統一流程。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, B-Q2, C-Q7, C-Q4

Q3: 外接顯卡無法被 Presentation Director 管理怎麼辦？
- A簡: 改用 GPU 控制台或 OS 顯示設定，建立自動化腳本與快捷鍵。
- A詳: 症狀：情境中看不到外顯或套用失敗。原因：工具不支援第三方驅動。解法：1) 使用 nVidia 控制台/nView 管理；2) Windows 顯示設定儲存配置；3) 以 devcon/腳本輔助切換；4) 固定 Dock 狀態對應配置。預防：避免依賴單一工具，建立多層回退方案。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q4, C-Q3, C-Q5

Q4: 休眠後外接螢幕不亮或配置錯亂，如何修復？
- A簡: 重新偵測顯示、切回內顯再切回外顯，必要時重啟驅動或改用關機。
- A詳: 症狀：喚醒後外接不亮、解析度錯亂。原因：拓撲變更、驅動未妥善還原。解法：1) 顯示設定中偵測螢幕；2) 切換主顯示；3) 重啟 Explorer 或重載驅動；4) 驅動更新或回退；5) 改以關機流程。預防：休眠前確保 Dock 狀態固定且使用穩定驅動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, B-Q14, C-Q7, C-Q9

Q5: 無法將外接 DVI 設為主螢幕怎麼辦？
- A簡: 在顯示設定指定主顯示，或調整 BIOS/驅動優先順序，關閉鏡像再嘗試。
- A詳: 症狀：外接無法成為主顯示。原因：鏡像限制、登入前主顯示被內顯鎖定、驅動 Bug。解法：1) 關閉鏡像改延伸；2) 在顯示設定勾選設為主顯示；3) 檢查 BIOS 顯示初始化選項；4) 更新驅動。預防：固定 Dock 狀態與配置檔，避免頻繁改變拓撲。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q12, C-Q2, C-Q5

Q6: 顯示卡驅動衝突導致黑屏，如何排查？
- A簡: 進入安全模式卸除新驅動，以穩定版重裝並逐步測試多螢幕。
- A詳: 症狀：安裝/更新驅動後黑屏或不穩。原因：內外顯驅動衝突、設定遺留。解法：1) 安全模式以 DDU 移除驅動；2) 安裝已知穩定版本；3) 先單螢幕測試，再加外接；4) 關閉不必要的顯示增強功能。預防：避免同時升級多組件，保留可回復的映像。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q19, C-Q9, B-Q16, D-Q1

Q7: 高解析度套用失敗或顯示「超出範圍」怎麼辦？
- A簡: 降低解析度/刷新率，確認 DVI 通道與線材，必要時自訂時序或雙通道。
- A詳: 症狀：畫面黑屏或提示超出。原因：單通道 DVI 帶寬不足、線材/驅動限制、EDID 誤判。解法：1) 選擇原生解析度與 60Hz；2) 確認單/雙通道需求；3) 更換高品質線；4) 以驅動自訂時序。預防：事先查規格與測試。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q17, B-Q10, C-Q8, A-Q18

Q8: 1394 裝置時常消失或速率不穩，如何解決？
- A簡: 檢查卡座與供電、更新 OHCI 驅動、改用外接電源或更換卡/線。
- A詳: 症狀：裝置斷線、掉幀。原因：供電不足、線材品質差、驅動不穩。解法：1) 使用 6-pin 並供電或外接電源；2) 更新 OHCI 驅動；3) 短線與優質線；4) 更換 CardBus 卡。預防：避免與高耗電裝置併用 Dock 電力。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q8, B-Q7, C-Q6, B-Q20

Q9: Dock 中 PCI 卡疑似電力不足怎麼辦？
- A簡: 換低功耗卡、改善散熱、減少同時負載或加裝外部供電配件（若支援）。
- A詳: 症狀：不穩、畫面閃爍、當機。原因：Dock 供電/散熱不足。解法：1) 選低功耗顯卡；2) 確保通風；3) 減少高耗電 USB 裝置同時供電；4) 檢查 Dock 韌體更新。預防：規劃負載並留餘裕。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q20, C-Q1, C-Q10, B-Q23

Q10: 切換情境步驟繁瑣，如何簡化？
- A簡: 用配置檔、快捷鍵與腳本自動化，並建立一致的 Dock/Undock 標準流程。
- A詳: 症狀：每次回家/外出需多步手動。解法：1) 建立家/外出配置檔；2) 設定快捷鍵；3) devcon 停用/啟用外顯腳本；4) 固定流程（先切換→關機→脫離）；5) 文件化步驟供自用。預防：定期測試與更新設定檔，避免版本漂移。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q3, C-Q4, B-Q24, A-Q25
