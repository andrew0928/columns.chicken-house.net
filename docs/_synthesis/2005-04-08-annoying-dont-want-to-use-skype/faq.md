# 真可惡, 不想用 skype 了..... :@

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 什麼是 Skype？
- A簡: Skype 是 VoIP 通訊服務，提供語音、視訊、即時訊息與檔案傳輸；跨平台免費互打，付費可撥市話手機與申辦號碼。
- A詳: Skype 是一款以網際網路為基礎的語音通話（VoIP）與視訊會議工具，支援 Windows、macOS、Linux、iOS、Android 等平台。其核心功能包含：Skype 對 Skype 免費語音/視訊、多人群組會議、即時訊息、檔案傳輸、螢幕分享；付費項目如撥打市話與手機（Skype Credit/訂閱）及取得線上號碼（Skype Number）。適用於跨國家庭通訊、小型團隊協作與個人日常使用。優點是跨平台與普及性，缺點可能是資訊安全疑慮、廣告與品質受網路環境影響。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, A-Q8

Q2: 什麼是 VoIP？
- A簡: VoIP 是以網路封包傳送語音的通訊技術，以軟體或網路電話取代傳統電信交換。
- A詳: VoIP（Voice over IP）是透過網際網路傳輸語音的技術。它將語音類比訊號經編解碼轉為數位封包，透過 IP 網路送達對端，再還原播放。與傳統 PSTN 相比，VoIP 成本較低、擴充彈性好、能整合多媒體（視訊、訊息、檔案）。然而其品質受網路延遲、抖動、封包遺失影響，需要編解碼、回音消除、抖動緩衝與適應性位元率等技術。Skype、Teams、Zoom、SIP 話機都屬 VoIP 應用。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, B-Q3, B-Q11

Q3: Skype 的核心功能有哪些？
- A簡: 語音通話、視訊會議、即時訊息、檔案傳輸、螢幕分享、付費撥市話手機、申請 Skype 號碼。
- A詳: Skype 的功能分為免費與付費。免費包含：一對一與群組語音、視訊通話、聊天、表情與貼圖、檔案與圖片傳送、螢幕分享、通話錄製。付費包含：Skype Credit/訂閱撥打市話與手機、Skype Number 取得在地來電號碼、國際通話方案。另支援來電顯示、來電轉接、語音信箱等。跨裝置同步，適用家庭聯繫、遠距面談、遠端說明與教學等情境。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, C-Q1, C-Q10

Q4: 為什麼有人不想用 Skype？
- A簡: 資安與隱私疑慮、品質不穩、廣告干擾、資源占用、更新與相容性問題，促使使用者轉向替代工具。
- A詳: 常見不使用 Skype 的原因包括：非預設端到端加密（需啟用「私人對話」）、資料蒐集與廣告個人化疑慮；網路波動導致品質不穩；程式資源占用較高致效能下降；版本更新導致介面/功能變動與相容性問題；企業環境中與防火牆、代理設定相衝；與其他工具（Teams、Zoom、Google Meet、Signal）相比在會議管理、錄影、協作整合上不如意。這些痛點會驅動遷移。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q16, A-Q17, D-Q8

Q5: Skype 與傳統電話的差異？
- A簡: Skype 走網路封包、功能多媒體、成本低；傳統電話走電信交換、品質穩但功能有限與資費較高。
- A詳: 傳統電話透過 PSTN 電路交換，語音品質穩定、延遲低，但跨國昂貴且只支援語音。Skype 利用網路封包承載語音與視訊，能提供聊天、檔案、螢幕分享等多媒體功能，跨國成本顯著降低。缺點是品質依賴網路（延遲、抖動、封包遺失），並可能受防火牆限制。對成本敏感與跨境使用者，Skype 具吸引力；對高可靠性的商務語音，傳統或企業級 VoIP/PBX 仍有優勢。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q11

Q6: Skype 與 Zoom/Teams 有何差異？
- A簡: Skype 偏個人與小型會議；Zoom/Teams 在大型會議、管理、整合與企業功能更完整。
- A詳: Skype 聚焦個人與小團隊溝通，介面簡潔、功能涵蓋聊天/通話/分享；Zoom 強於大規模會議、網路研討會、分組討論室與雲端錄影；Microsoft Teams 整合 Office 365、團隊頻道、檔案協作與企業管理。安全預設、治理與稽核能力上，Teams/Zoom 企業方案更完善；Skype 成本較低、上手快。選擇依場景：家庭與輕量會議用 Skype；正式教育、企業協作用 Zoom/Teams。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q17, C-Q9

Q7: Skype 個人版與 Skype for Business/Teams 差異？
- A簡: 個人版重輕量通訊；商務版/Teams 強化企業管理、合規、整合與可擴充性。
- A詳: Skype 個人版提供基本通訊功能與付費撥打；Skype for Business（已被 Teams 取代）與 Teams 則提供企業級身分管理、單一登入、通話錄音與合規保存、會議控管、電話系統（PBX）、通話佈建、保安與治理、API 擴充與 App 整合。對企業，Teams/商務方案可確保合規與可管理性；對個人，Skype 成本低、使用簡單。遷移時須考量資料保留與使用政策。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, C-Q9

Q8: Skype 的收費模式與免費項目？
- A簡: Skype 對 Skype 免費；付費含 Skype Credit/訂閱撥市話手機、Skype Number、加值服務。
- A詳: 免費項目：Skype 對 Skype 語音/視訊/聊天、檔案傳輸、螢幕分享、通話錄製。付費：以 Skype Credit 按量計費或訂閱包月撥市話/手機（依國別費率），以及購買 Skype Number 取得在地號碼接聽來電。另有來電顯示、語音信箱等服務。費率受目的地與方案影響，官方網站提供即時查詢。預付金需注意有效期限與自動續訂設定。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q10, D-Q9

Q9: Skype 的隱私與資料蒐集重點？
- A簡: 會蒐集帳戶、裝置、使用與診斷資料；可在設定限制廣告與隱私選項。
- A詳: Skype（隸屬 Microsoft）可能收集帳戶資訊、聯絡人、裝置識別、IP、使用與診斷資料，用於服務運作、安全、防詐與改良；廣告與建議可能基於個人化資料。可於設定中調整隱私如：只允許聯絡人聯繫、關閉個人化廣告、控制讀取回條、位置分享與資料匯出/刪除請求。敏感通訊可用「私人對話」（Signal 協議）實現端到端加密。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, C-Q7

Q10: Skype 的安全性與加密方式是什麼？
- A簡: 傳輸加密；預設非端到端。啟用「私人對話」可用 Signal 協議端到端加密。
- A詳: Skype 會加密傳輸中的語音、視訊與訊息以防竊聽，但預設通話並非端到端加密，伺服端可能處理路由與媒體。2018 起提供「私人對話」（Private Conversations），採用 Signal Protocol 實現端到端加密，支援訊息與部分通話。為安全建議：啟用雙步驗證、使用強密碼、定期檢視登入裝置與連線應用、避免可疑連結與檔案、調整誰可聯繫你之權限。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q8, C-Q7

Q11: Skype 通話品質受哪些因素影響？
- A簡: 網路頻寬、延遲抖動、封包遺失、裝置效能、麥克風耳機與環境噪音。
- A詳: 影響因素包含：上/下行頻寬不足、延遲高（>150ms）、抖動大、封包遺失（>1-3%）、Wi‑Fi 雜訊與共用擁塞；裝置 CPU/GPU/記憶體不足；麥克風回音、環境噪音與不當音量；防火牆/代理限制造成轉封包或改走 TCP。改善方式：使用有線網路、關閉高占用程式、選用耳麥、調整裝置取樣率、允許 Skype 通過防火牆、更新驅動與應用程式。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, C-Q3, D-Q6

Q12: 為什麼國際通話常選擇 Skype？
- A簡: 成本低、跨平台易用、可購買在地號碼與訂閱，適合跨國聯繫與遠距家庭。
- A詳: 相較傳統跨國電話費率，Skype 透過網路將成本大幅降低，提供信用點數或包月訂閱；Skype Number 讓海外親友以在地費率撥入。跨平台與全球普及性讓雙方容易上手。限制包括：品質受網路影響、需對方裝置與連線、法規或企業網路限制。對經常跨國聯繫者，整體成本效益高，但重要會議需預備備援通道。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q8, C-Q9

Q13: Skype 的 P2P 與雲端架構差異？
- A簡: 早期採 P2P 超級節點分散轉接；近年轉雲端集中服務，管理與穩定性提升。
- A詳: 早期 Skype 使用分散式 P2P 架構，部分客戶端充當超級節點，協助註冊、路由與 NAT 穿越，提高彈性但有資源與管控議題。Microsoft 收購後逐步轉向雲端伺服器架構，集中處理登入、訊號與媒體轉接，提升管理、安全、行動推播與企業相容性。差異在於路由決策位置、可靠性、維運與法規遵循能力。對用戶來說，體驗更一致，但端對端特性減弱。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q15, B-Q7

Q14: NAT/防火牆對 Skype 有何影響？
- A簡: 可能阻擋或改走次佳路徑，導致延遲、丟包、無法視訊或分享。
- A詳: 嚴格的 NAT/防火牆會限制入站/出站 UDP 封包，Skype 可能改走 TCP 或透過中繼伺服器轉發，增加延遲與抖動，影響語音/視訊與螢幕分享。企業代理可能破壞連線保活或 DPI 導致封鎖。建議允許 Skype 應用透過必要連接阜與網域、優先使用 UDP、有條件地允許 STUN/TURN/ICE 類流量；無線環境則需優化 QoS 與 5GHz 頻段。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q6, D-Q7

Q15: 什麼是 Skype 的超級節點（歷史）？
- A簡: P2P 架構中高可用節點，協助註冊、路由與 NAT 穿越之中繼角色。
- A詳: 在早期 Skype P2P 設計裡，超級節點（Supernode）是具公網位址與良好資源的客戶端，被動選任來承擔網路中控樣的功能：協助新用戶註冊、使用者搜尋、建立會話、在 NAT 限制下進行中繼。它提升可擴充性與去中心化，但也帶來未知端佔用資源、安全治理與不可預測性。轉雲端後，這些職能主要由伺服器承擔。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q13, B-Q7

Q16: Skype 的替代方案有哪些？
- A簡: Teams、Zoom、Google Meet、FaceTime、Signal、WhatsApp、Discord 等。
- A詳: 替代工具可依場景選擇：企業協作偏向 Microsoft Teams、Google Meet；大型會議與活動偏向 Zoom；Apple 生態圈可用 FaceTime；隱私導向的端到端加密可選 Signal、WhatsApp；社群與遊戲場景常用 Discord。比較時考量：安全與加密、價格、會議與管理功能、跨平台支援、整合與擴充性。遷移應規劃聯絡人、歷史與培訓。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, C-Q9

Q17: 為什麼需要從 Skype 遷移？
- A簡: 因需求變更、資安與合規、品質管理、整合或成本考量而轉向他工具。
- A詳: 遷移常因：企業合規與治理需求（審計、DLP、合規留存）；更佳的會議管理與錄影；更強的團隊協作與檔案整合；隱私與端到端加密預設；跨平台效能或資源占用；統一到現有的生產力套件（如 M365/Google Workspace）。制訂遷移計畫需盤點使用情境、匯出資料、聯絡人同步、教育訓練與並行期備援。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q9, D-Q10

Q18: Skype 的核心價值是什麼？
- A簡: 低成本跨境通訊、普及易用、功能夠用，適合個人與小型團隊日常溝通。
- A詳: Skype 的價值在於以低成本提供全球可達的語音/視訊與訊息，跨平台且上手快速。對家庭與小團隊，它提供「夠用」的多人通話、螢幕分享、檔案傳送。對偶發性國際通話，用訂閱/點數可控成本。雖在企業級管理與安全上不及 Teams/Zoom，但對非關鍵正式場合與日常連線仍具吸引力。若需求升級，再評估遷移即可。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q16

### Q&A 類別 B: 技術原理類

Q1: Skype 通話連線如何運作？
- A簡: 客戶端登入雲端，交換訊號，協商媒體路徑與編解碼，建立加密的語音視訊通道。
- A詳: 原理包含三段：1) 登入與存在：客戶端連線至 Skype/Microsoft 雲端，完成身分驗證並同步連絡人與在綫狀態；2) 訊號交換：發起呼叫時，雙方透過伺服器進行邀請、能力協商（支援的編解碼、解析度）；3) 媒體協商與傳輸：優先嘗試點對點 UDP，受限時改經伺服器中繼，並建立加密的 RTP/類 RTP 媒體流。核心組件含：客戶端、訊號伺服器、媒體中繼、身分與推播服務。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q7, A-Q14

Q2: Skype 的訊號與媒體傳輸流程為何？
- A簡: 訊號走伺服器控管會話；媒體優先 UDP 端對端，失敗則走雲端中繼轉發。
- A詳: 流程步驟：1) 呼叫建立：客戶端向訊號伺服器送出 INVITE 類訊息（專有協定），對端同意後回復能力資訊；2) 能力協商：雙方交換支援的編解碼（SILK/Opus、H.264/VP8）與網路候選位址；3) 路徑選擇：測試直連（NAT 穿越），若不可行改用中繼；4) 媒體傳輸：建立加密的傳輸通道，啟用回音消除、抖動緩衝與帶寬自適應；5) 結束：釋放資源並上傳品質統計。組件：訊號層、ICE 類模組、媒體引擎、QoS。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q6, B-Q11

Q3: Skype 使用哪些音訊編解碼？如何影響品質？
- A簡: 主要採 SILK、Opus。自適應位元率與降噪讓低頻寬下維持可懂度。
- A詳: SILK 是 Skype 開發的寬頻語音編解碼，針對語音最佳化，具高壓縮與抗丟包；Opus 則結合 SILK 與 CELT 優勢，涵蓋窄到全頻、低延遲與彈性位元率。Skype 依網路狀態自動切換位元率與取樣率，配合丟包隱藏（PLC）、回音消除（AEC）與降噪（NR），在 30–100kbps 即可提供清晰語音。低延遲與穩定網路能進一步提升通話體驗。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q11, A-Q11

Q4: Skype 的視訊編解碼與自適應如何運作？
- A簡: 常用 H.264/VP8，依頻寬與 CPU 調整解析度、幀率與位元率動態適配。
- A詳: 視訊使用 H.264 或 VP8 等編碼，透過即時網路回饋調整關鍵參數：解析度（如 360p~1080p）、幀率（15~30fps）、位元率（數百 kbps 至數 Mbps）。在封包遺失時採用 FEC、重傳與錯誤隱藏；在 CPU 壓力時降幀或降解析度。畫面採用動態多流（選講者高畫質，其他低畫質縮圖）以節省頻寬。核心組件：視訊編碼器、網路適配器、統計回饋。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q11, D-Q5

Q5: Skype 如何做回音消除、雜訊抑制與抖動緩衝？
- A簡: 以 AEC 抑制回音、NR 降噪、AGC 穩定音量，抖動緩衝自適應平滑延遲。
- A詳: 音訊處理鏈包含：AEC（Acoustic Echo Cancellation）藉由揚聲器到麥克風的回授路徑估計，動態消回音；NR（Noise Reduction）以頻譜減法或深度模型降低環境噪音；AGC（Automatic Gain Control）自動調整輸入音量；VAD（Voice Activity Detection）辨識語音段落；Jitter Buffer 根據封包到達變異動態調整緩衝，平衡延遲與抖動。這些 DSP 模組協同確保語音清晰與穩定。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q11, D-Q3

Q6: Skype 的 NAT 穿越機制是什麼？
- A簡: 採近似 ICE 的候選位址探測，優先 UDP 直連，失敗則走中繼伺服器。
- A詳: 雖協定細節專有，整體思路類似 ICE：雙方交換本地、公網與候選位址，透過連通性測試（類 STUN）尋找可行的 UDP 路徑；若遇嚴格對稱 NAT 或防火牆，則改用中繼（類 TURN）穿越。步驟：候選收集、優先序排序、連通測試、連線維持（Keep-alive）。此機制提升端對端成功率並降低延遲，但在受限網路中仍需依賴中繼。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q14, B-Q2

Q7: P2P 超級節點與雲端服務在路由上有何差異？
- A簡: P2P 依分散節點協助路由；雲端由集中伺服器控管與中繼，管理與可預期性更佳。
- A詳: P2P 模式下，部分客戶端充當超級節點，協助註冊、搜尋與路由，優點是去中心化與彈性，缺點是節點品質不可控與資安治理困難。雲端架構改以資料中心伺服器統一負責訊號與媒體中繼，便於故障切換、監控與合規，亦利於行動推播與離線訊息。對使用者而言，雲端提升穩定與預期性，但可能減少純端對端特性。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q13, A-Q15

Q8: Skype 的加密與鑑別流程如何設計？
- A簡: 以 TLS 保護訊號、SRTP 類保護媒體；私人對話使用 Signal 端到端加密。
- A詳: 訊號層通常使用 TLS 建立安全通道進行會話控制與身分鑑別，媒體層採用類 SRTP 的機制對音視訊封包加密。當使用「私人對話」時，透過 Signal Protocol 的密鑰交換（X3DH）與雙棘輪（Double Ratchet）實現端到端加密與前向保密。金鑰管理與憑證由服務端支援。建議搭配 2FA 與安全裝置管理以提升整體保障。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q10, C-Q7

Q9: 在線狀態與推播如何同步到裝置？
- A簡: 伺服器維持連線與裝置訂閱，透過推播服務（APNS/FCM）與同步資料更新。
- A詳: 用戶登入後與 Skype 雲端建立長連線或使用平台推播渠道。手機端經 Apple APNS 或 Google FCM 收到來電與訊息通知，喚醒 App 建立媒體連線；桌機端維持 WebSocket/長連線同步狀態與訊息。伺服器端負責會話狀態分發、離線訊息緩存與既讀回條協調。核心組件：Presence/Sync 服務、Push Gateway、裝置訂閱管理。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, D-Q1

Q10: 檔案傳輸與螢幕分享背後機制是什麼？
- A簡: 經安全通道傳輸；可點對點或經伺服器中繼，視網路與安全策略而定。
- A詳: 檔案傳輸會先經訊號層建立會話，根據網路可達性決定 P2P 或伺服器中繼，並在傳輸層加密保護；可支援斷點續傳與防毒掃描。螢幕分享則以編碼桌面畫面為視訊流（類 H.264），與音訊流同步傳送，並根據頻寬自適應畫質。企業策略可能強制走中繼以便於 DLP 與審計。組件：媒體編碼器、傳輸管理、權限控制。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q5, C-Q3

Q11: 帶寬自適應與封包遺失隱藏如何工作？
- A簡: 透過回饋調整位元率、解析度與冗餘，PLC 補償遺失以維持語音可懂度。
- A詳: 客戶端持續量測 RTT、丟包、抖動，並上報接收端回饋（如 RTCP 類統計）。編碼器根據回饋降低位元率、幀率或切換檢視；採用 FEC/冗餘與有限重傳降低畫質劣化。音訊端啟用 PLC 以譜估計與時間軸延伸填補缺包，避免斷裂。目標是在不過度增加延遲下達成「穩定可懂」的品質。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q3, B-Q4

Q12: Skype 與企業防火牆/代理如何協作？
- A簡: 以允許清單開放網域與埠、優先 UDP、必要時透過中繼並設定 QoS。
- A詳: 企業可配置：1) 允許清單（FQDN）與必要 TCP/UDP 連接阜；2) 優先 UDP，避免深入封包檢測破壞媒體；3) 啟用 QoS 標記（DSCP）保證語音；4) 代理繞過或為媒體流例外；5) Wi‑Fi 規劃（5GHz、頻寬足夠、AP 負載均衡）。必要時強制走中繼以利審計。監控端到端品質以主動調整策略。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q14, D-Q7

### Q&A 類別 D: 問題解決類（10題）

Q1: Skype 登入失敗或無法驗證怎麼辦？

- A詳: 症狀：提示帳密錯誤、無限載入、2FA 不到。可能原因：密碼錯誤、帳號被鎖、網路/代理阻擋、版本過舊、快取毀損。解法：1) web 端驗證可否登入；2) 重設密碼並檢查 2FA；3) 關閉 VPN/代理測試；4) 更新 Skype；5) 移除並重裝；6) 檢查 Microsoft 服務狀態。預防：使用密碼管理器與 2FA、避免重複密碼。

- 學習階段: 基礎
- 關聯概念: B-Q9, C-Q7

Q2: 對方聽不見或你聽不到聲音怎麼辦？

- A詳: 症狀：無聲或音量極低。原因：裝置選錯、系統/Skype 靜音、麥克風權限被拒、驅動或硬體故障。解法：1) 設定 > 音訊與視訊選對裝置；2) 檢查系統音量與輸入輸出；3) 作業系統隱私設定允許麥克風；4) 測試 Echo 服務；5) 更新音效驅動與重啟。預防：固定使用耳麥、避免同時開啟多會議軟體。

- 學習階段: 基礎

Q3: 有回音、噪音或啞音問題如何處理？

- A詳: 症狀：對方回音、環境雜音、間歇性斷音。原因：外放與麥克風迴授、風扇或空調噪音、VAD/降噪過度。解法：1) 改用耳麥；2) 降低喇叭音量、遠離麥克風；3) 關閉噪音源；4) 於設定調整降噪等級；5) 檢查網路丟包。預防：固定安靜環境、使用指向性麥克風、保持一定網路品質。

- 學習階段: 基礎

Q4: 視訊鏡頭無法使用或黑屏怎麼辦？

- A詳: 症狀：黑屏、找不到相機。原因：權限被拒、相機被他程式占用、驅動過舊或不相容、USB 供電不足。解法：1) 作業系統隱私允許相機；2) 關閉其他用相機的 App；3) 更新相機驅動；4) 更換 USB 埠或使用供電集線器；5) 於 Skype 切換相機來源。預防：安裝官方驅動、避免同時多程式占用相機。

Q5: 螢幕分享模糊或無法開啟怎麼辦？
- A簡: 降低解析度與幀率、改用有線網路、檢查權限與企業政策限制。
- A詳: 症狀：對方看不清或無法開始分享。原因：頻寬不足、系統圖形加速問題、權限未授權（macOS 需錄影權限）、企業限制。解法：1) 分享前降低顯示器解析度或只分享視窗；2) 開啟硬體加速；3) 在系統隱私授權螢幕錄製；4) 有線網路；5) 管理端放行策略。預防：會前測試、準備靜態投影片替代方案。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q10, C-Q3

Q6: 通話頻繁斷線或延遲高怎麼辦？
- A簡: 減少併發流量、改用 5GHz/有線、優先 UDP、啟用路由器 QoS。
- A詳: 症狀：通話中斷、聲畫不同步。原因：Wi‑Fi 擁擠、寬頻不足、ISP 擁塞、走 TCP/代理。解法：1) 有線或 5GHz；2) 關閉雲端備份/串流下載；3) 路由器啟用 QoS；4) 防火牆允許 UDP；5) 變更 DNS 或測試替代 ISP。預防：會前測速、保留頻寬、更新韌體與驅動。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, C-Q6

Q7: 被公司防火牆或代理阻擋如何處理？
- A簡: 申請允許清單與埠、優先 UDP 與 QoS，必要時走中繼並配合安全政策。
- A詳: 症狀：無法登入、無媒體或品質差。原因：DPI 封鎖、UDP 被阻、代理需要認證。解法：1) 提供 IT 需開放的 FQDN/埠與白名單；2) 啟用 DSCP 與媒體例外；3) 設定認證代理；4) 強制走中繼並監控。預防：制定通訊政策、容量規劃、定期驗證連通性與品質報表。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q12, C-Q6

Q8: Skype CPU/記憶體占用高導致卡頓怎麼辦？
- A簡: 關閉不必要特效與背景程式、更新版本與驅動、檢查外掛與硬體。
- A詳: 症狀：風扇轉、延遲、畫面卡。原因：高解析視訊、虛擬背景、同時多程式、版本 Bug。解法：1) 關閉虛擬背景與影片預覽；2) 降低視訊解析度；3) 關閉其他會議/瀏覽器分頁；4) 更新 Skype 與顯示卡驅動；5) 重新安裝或清 cache。預防：定期清理開機程式、升級記憶體、用硬體加速。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q4, C-Q3

Q9: 帳號被濫用或出現異常扣款怎麼辦？
- A簡: 立即改密碼與 2FA、停用自動儲值、檢查交易與聯絡客服追查。
- A詳: 症狀：陌生通話紀錄、費用扣款。原因：帳密外洩、釣魚、裝置中毒、自動儲值未控。解法：1) 立刻變更密碼並啟用 2FA；2) 停止自動儲值與取消訂閱；3) 檢查近期登入裝置並登出所有；4) 下載交易明細；5) 聯絡客服申訴。預防：密碼管理器、避免重複密碼、警惕釣魚連結、定期審視付款設定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q8, C-Q7

Q10: 無法永久刪除帳號或資料怎麼辦？
- A簡: 了解關閉 Microsoft 帳號影響，先匯出/清理，再依流程關閉並等待寬限期。
- A詳: 症狀：刪除選項受限或資料仍存在。原因：Skype 與 Microsoft 帳號綁定、寬限期未過、備份/法規保留。解法：1) 匯出資料（C-Q5）；2) 移除公開資訊與頭像、清理聊天；3) 停用訂閱；4) 透過 Microsoft 帳號關閉流程，等待 60 天完成；5) 若合規需要，提交刪除請求。預防：使用前評估與分離使用不同帳號管理。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q10, A-Q17
