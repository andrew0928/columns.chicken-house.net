# 該來的終於來了 :~~~

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Lenovo 收購 IBM PC 業務事件？
- A簡: 2005年聯想接手IBM個人電腦部門，ThinkPad品牌轉移，支援與保固亦逐步聯想化。
- A詳: 2004年12月聯想宣布收購IBM個人電腦事業部（PCD），於2005年5月交易完成。IBM取得聯想近19%股權，並授權聯想在過渡期共用IBM品牌。此後，ThinkPad等品牌的研發、生產與全球支援逐步由聯想主導，官網、驅動、保固體系也隨之轉移。該事件影響深遠，象徵全球PC產業重心與供應鏈結構的變化。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, A-Q9, B-Q1

A-Q2: 什麼是 ThinkPad？
- A簡: 1992年起源於IBM的商務筆電品牌，以耐用性、鍵盤手感與企業功能著稱。
- A詳: ThinkPad是IBM在1992年推出的旗艦商務筆電品牌，強調耐用結構（如鎂合金/碳纖維、滾籠式設計）、優異鍵盤與TrackPoint、嚴謹測試（含軍規項目），並提供企業級維護、保固與部署工具。收購後，聯想延續此定位，維持T/X/L/P等系列線，持續在可靠性與可維修性上投入，成為企業與專業人士常見選擇。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q8, B-Q22

A-Q3: IBM ThinkPad 與 Lenovo ThinkPad 有何差異？
- A簡: 核心定位延續；品牌與支援轉為聯想，設計語言逐步演進，工具由ThinkVantage整合。
- A詳: 收購前機身多為IBM標，過渡期共用標識，後期改為Lenovo標。設計語言延續方正黑色、TrackPoint、商務導向與維修友善，但材質、螢幕比例（16:10到16:9）與介面更新隨時代演進。驅動/工具由IBM頁面轉至聯想（如System Update），保固系統也以聯想為主。總體定位不變，工程細節與供應鏈有所調整。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q8, B-Q3

A-Q4: 什麼是品牌轉移（Brand Transition）？
- A簡: 收購後將品牌使用權、資產與支援體系，從原公司移交至新東家的過程。
- A詳: 品牌轉移是併購中常見的整合任務，包括商標授權與終止計畫、網站與服務重定向、客服與保固資料遷移、軟硬體命名與包裝更新，以及供應鏈文件（FRU/MTM）與設計規範交接。目標是平滑移轉客戶體驗、避免服務中斷，最終完成品牌一體化。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q16, A-Q9

A-Q5: 為什麼 IBM 出售 PC 業務給聯想？
- A簡: PC利潤薄、聚焦企業服務與高附加價值業務；聯想則擴大全球版圖與技術能力。
- A詳: 2000年代初PC市場價格競爭激烈、利潤空間受壓。IBM調整策略，聚焦企業服務、軟體、中高端伺服器與諮詢。聯想藉收購獲取全球品牌、通路與研發能力，迅速躍升全球PC領導者。雙方取長補短，達成策略互補與資源重配置。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q20

A-Q6: 為什麼有人說「筆電變成匪貨」？
- A簡: 口語俗稱指「改由中國大陸品牌/製造」之轉變，帶有價值判斷，不建議使用。
- A詳: 「匪貨」源於歷史政治語境的貶稱，指涉來自中國大陸的商品。ThinkPad品牌轉移至聯想後，部分使用者以此詞表達情緒或身份認同的衝擊。然而該詞具歧視與偏見色彩，不利理性討論。技術層面應聚焦於產品品質、支援政策與實際體驗，而非產地標籤。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q19, A-Q21

A-Q7: 「匪貨」一詞的歷史與爭議是什麼？
- A簡: 帶有政治貶抑的歷史口語，易引發偏見與對立，科技討論中應避免使用。
- A詳: 該詞源自冷戰與兩岸對立語境，以貶低方式稱呼中方或其商品。隨全球供應鏈深化，產品常跨國研發與製造，以該詞貼標籤既不精準也不尊重。於專業領域，應以客觀指標如品質、可靠性、保固與支援評估產品，避免語言傷害與偏見擴散。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q19

A-Q8: ThinkPad 的核心價值是什麼？
- A簡: 耐用可靠、鍵盤與TrackPoint體驗、企業維運友善、長期驅動與保固支援。
- A詳: ThinkPad以結構強度、嚴格測試、優異鍵盤與紅點（TrackPoint）著稱。面向企業的設計包括易維修模組化、詳細硬體維修手冊（HMM）、穩定驅動策略與長期供應、System Update等工具支援，以及國際保固。這些特性使其在商務與專業領域長期受青睞。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q22, C-Q4

A-Q9: 收購後對使用者保固有何影響？
- A簡: 聯想承接保固；國際保固政策延續，但查詢與申請流程改走聯想體系。
- A詳: 收購完成後，聯想承接ThinkPad保固與維修。原IBM保固在有效期內繼續生效，國際保固（IWS）依型號/地區判定。使用者需在聯想官網以機型/序號查詢保固，維修通路與零件供應由聯想網絡提供，體驗以聯想流程為準。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q5, B-Q6, C-Q1

A-Q10: 收購後驅動與軟體支援如何變化？
- A簡: 下載與更新遷移至聯想平台，工具整合為System Update與ThinkVantage套件。
- A詳: 原IBM驅動頁面逐步重定向到聯想支援網站。更新方式轉為Lenovo System Update（TVSU），整合驅動、BIOS、應用工具（如Power Manager、Access Connections）。封裝命名、版本註記與安裝器也依聯想標準統一，方便企業部署與個人維護。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q4, C-Q2

A-Q11: 什麼是 FRU、PN 與 MTM？
- A簡: FRU為可替換零件料號，PN為零件編號，MTM為機型型號碼，便於維修管理。
- A詳: FRU（Field Replaceable Unit）是官方可替換零件料號，確保相容性與保固一致；PN（Part Number）是製造端零件編號；MTM（Machine Type Model）是機型識別碼（如 20KGS0XXXX）。維修、訂件與保固判定皆依這些編碼，搭配序號（S/N）實現精準追蹤。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, B-Q8, C-Q6

A-Q12: 什麼是 BIOS 開機畫面品牌化（Splash Branding）？
- A簡: 開機時顯示品牌標誌的BIOS設定/資源，收購後常以韌體更新完成變更。
- A詳: BIOS Splash是開機初期顯示的圖像資源，反映裝置品牌與型號。品牌轉移時，OEM會透過BIOS/EC更新替換圖像與廠商代碼，同步調整SLIC表（舊款）與廠商資訊，確保一致體驗。此變更不影響硬體規格，但涉及韌體安全與相容性。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q20, C-Q3

A-Q13: 為什麼需要在品牌轉移時更新 BIOS/EC？
- A簡: 同步品牌識別、修補相容性與安全修正，並統一維護與部署機制。
- A詳: 品牌轉移牽涉廠商代碼、簽章、SLIC/ACPI表與韌體資源變更。更新BIOS/EC可：1) 套用新品牌與代碼；2) 修正相容性與已知問題；3) 統一驅動/工具偵測；4) 奠定企業大規模部署基線。更新需遵循官方流程，確保供電穩定與風險管控。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, C-Q3, D-Q1

A-Q14: 什麼是 ThinkVantage 工具套件？
- A簡: IBM/聯想提供的企業維運工具群，如System Update、Rescue and Recovery等。
- A詳: ThinkVantage包含System Update（更新管理）、Rescue and Recovery（備份還原）、Access Connections（網路管理）、Power Manager（電源）、Hotkey整合等。其目標是簡化維護、提升穩定性與安全性，便於IT集中管理與端點標準化。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, C-Q5, D-Q8

A-Q15: IBM 與聯想在過渡期如何協作？
- A簡: 透過品牌授權、工程移轉與服務轉接協議，逐步完成產品與支援體系交接。
- A詳: 過渡協議涵蓋品牌共用時程、設計文件與測試規範交接、網站與客服重定向、零件供應與保固承接、授權軟體與商標使用等。目的在降低客戶衝擊，維持服務連續性，並為新平台建立一致標準。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q16, A-Q4

A-Q16: 什麼是國際保固（IWS）？
- A簡: 依型號與地區適用的跨國保固政策，聯想承接ThinkPad的IWS服務。
- A詳: IWS允許使用者在多國獲得維修支援，但是否適用取決於機型、零件供應與當地政策。查詢透過聯想支援網站輸入序號/MTM判定。企業跨區部署時須依IWS清單採購，以確保服務可用性與時效。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q1, D-Q4

A-Q17: ThinkPad 系列命名（T/X/L/P/E）有何差異？
- A簡: T旗艦商務、X輕薄、P專業工作站、L/E主流商務，各自平衡性能與便攜。
- A詳: T系列定位全面的商務旗艦；X系列強調輕薄與行動力；P系列為ISV認證的行動工作站；L/E系列面向成本敏感的商務需求。收購後命名延續並隨代際演進（如T4xx、X1），以滿足多樣客群。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q22

A-Q18: ThinkPad 設計理念是什麼？
- A簡: 耐用結構、維修友善、長期穩定性與一致的人機介面體驗。
- A詳: 核心理念包含堅固機構（滾籠、金屬骨架）、優質鍵盤/TrackPoint、一致佈局、可維修模組化、企業級測試與長供應期。這些特質確保大規模部署穩定、降低總持有成本（TCO），也是品牌長期價值所在。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q22, C-Q6

A-Q19: 中國製造與品牌所有權有何不同？
- A簡: 製造地是產地，品牌所有權是商標與決策權；兩者常跨國分工。
- A詳: 供應鏈全球化使研發、設計、製造、品牌行銷可分屬不同國家與企業。製造地不等於品質；品質取決於設計規範、品管體系、驗證與售後支援。評估產品應以客觀指標而非產地刻板印象。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q7, A-Q20

A-Q20: 全球供應鏈如何影響品牌形象？
- A簡: 供應鏈決定品質穩定、供貨與售後能力，進而塑造品牌可信度。
- A詳: 品牌形象不僅是標識，更是供應鏈的總體表現：設計驗證、零件一致性、製造良率、服務網絡與備件供應。併購後若能維持或提升這些能力，品牌形象可持續甚至增強；反之則受損。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q7, A-Q5

A-Q21: 為什麼品牌更名會引發情緒反應？
- A簡: 消費者對既有信任與身份認同受撼動，產生不確定感與風險知覺。
- A詳: 品牌是承諾與信任的集合。當熟悉的IBM改為聯想，使用者擔憂品質、保固與價值延續。情緒反應源於資訊不對稱與風險評估。透明的支援政策、穩定的產品體驗與清晰溝通可降低焦慮。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q9, A-Q10

A-Q22: 什麼是 OEM 與 ODM？
- A簡: OEM貼牌生產，ODM設計代工；品牌商可同時自研與委外。
- A詳: OEM（原廠委託製造）依品牌商設計生產；ODM（設計代工）由代工方提供設計方案再貼牌。PC產業常混合模式，品牌主導規範與驗證。收購不改變基本分工，但影響誰負責標準與最終品質。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q1, B-Q22

A-Q23: 機身Logo與標識經歷什麼變化？
- A簡: 從IBM標到過渡雙標，再到Lenovo標；韌體與包裝同步更新。
- A詳: 早期機身與開機畫面多為IBM，過渡期可能雙標或軟體先行更改，後續新機全面Lenovo標。包裝、序號標籤、驅動/工具名稱依序更新，以維持一致體驗與法務合規。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q12, B-Q2, B-Q16

A-Q24: 收購後軟體授權與商標如何處理？
- A簡: 依協議授權過渡、再逐步替换商標與軟體簽章，確保合規與用戶連續性。
- A詳: 協議明定IBM商標授權期限、使用範圍與終止條件。軟體與韌體的簽章、著作權、第三方授權亦需重簽或替換。最終以聯想商標與CA簽章為準，確保法務與供應鏈一致。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q14, B-Q16

A-Q25: 什麼是「Legacy ThinkPad」？
- A簡: 多指收購前或早期設計語言延續的經典機種，象徵傳統精神。
- A詳: 社群常以「Legacy ThinkPad」稱呼傳統方正黑、7列鍵盤、厚實結構與經典介面的機種（如T4x、T60/T61、X60/X61）。這些機器承載品牌歷史，也常被愛好者維修升級。聯想在新設計中保留部分DNA，同時導入新材質與輕薄趨勢。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q18

### Q&A 類別 B: 技術原理類

B-Q1: 品牌轉移在供應鏈如何運作？
- A簡: 以協議與計畫驅動，涵蓋商標、網站、客服、零件與文件完整交接。
- A詳: 技術原理說明：品牌轉移是多系統同步變更，包括PLM/BOM、ERP、CRM、支援與網站。關鍵步驟或流程：1) 商標與法務核可；2) 網站與下載重定向；3) 零件編碼（FRU/PN）與標籤更新；4) 保固與客戶資料遷移；5) 維修網絡切換。核心組件介紹：支援入口（support site）、序號資料庫、零件目錄、知識庫與票務系統。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, A-Q15, B-Q16

B-Q2: BIOS 品牌化變更如何運作？
- A簡: 透過韌體映像更新替換開機圖、廠商代碼與ACPI/SLIC資訊，確保一致。
- A詳: 技術原理：BIOS包含顯示資源、DMI/SMBIOS、SLIC（舊款）與ACPI表。關鍵步驟：1) 發布簽章的BIOS/EC映像；2) 安全環境下刷寫；3) 驗證DMI與品牌字段；4) 回復/回滾方案。核心組件：BIOS主體、EC、刷寫工具、簽章與校驗機制。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, A-Q13, C-Q3

B-Q3: 驅動程式包如何從IBM轉至聯想？

- A詳: 具體實作步驟：1) 下載Lenovo Driver Packs；2) SCCM建立包與驅動分類；3) 任務序列判斷機型注入；4) 部署後TVSU補充。關鍵程式碼片段或設定：WMI查機型：SELECT Model FROM Win32_ComputerSystem。注意事項與最佳實踐：固定基線版本；分環境漸進推送；記錄版本可追溯。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q19, B-Q23, D-Q10

C-Q8: 如何校正電池並延長壽命？
- A簡: 以完整充放校準燃料計，設定充電上限與避免高溫環境。
- A詳: 具體實作步驟：1) 使用電源工具執行Battery Gauge Reset；2) 設充電上限80-90%；3) 避免滿電高溫長置。關鍵程式碼片段或設定：Lenovo Vantage/Power Manager設定門檻。注意事項與最佳實踐：定期校正；老化嚴重則更換原廠電池。

- 學習階段: 基礎

C-Q9: 如何設定Access Connections或改用Windows原生管理？

- A詳: 具體實作步驟：1) 安裝Access Connections與驅動；2) 建立情境檔與優先順序；或3) 企業以Windows原生WLAN設定/GPO。關鍵程式碼片段或設定：netsh wlan add profile。注意事項與最佳實踐：避免重複管理導致衝突；逐步汰換舊工具。

### Q&A 類別 D: 問題解決類（10題）

D-Q1: BIOS更新後黑屏或無法開機怎麼辦？
- A簡: 先嘗試危機復原與回退版本，必要時送修以重刷SPI/EC。
- A詳: 問題症狀描述：更新中斷或不相容導致黑屏、循環重啟。可能原因分析：斷電、韌體損毀、版本相依未滿足。解決步驟：1) 參照機型指南進行BIOS Crisis Recovery（USB復原映像）；2) 清CMOS與載入預設；3) 若仍失敗送原廠重刷。預防措施：使用AC電源、遵循指示、先更新相依（如ME）。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q14, C-Q3

D-Q2: 安裝後熱鍵與OSD不工作如何處理？
- A簡: 重新安裝Hotkey/OSD套件與依賴驅動，確認服務與EC版本。
- A詳: 問題症狀描述：音量/亮度無顯示或按鍵無效。可能原因分析：缺少Hotkey驅動、服務未啟動、EC/BIOS版本不匹配。解決步驟：1) 安裝Hotkey Features Integration；2) 檢查服務啟動；3) 更新EC/BIOS；4) 重新開機。預防措施：重裝系統後按驅動順序安裝，建立映像基線。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q18, C-Q4

D-Q3: TrackPoint/觸控板無反應怎麼辦？
- A簡: 檢查BIOS設定、重新安裝UltraNav驅動，更新EC與清潔硬體。
- A詳: 問題症狀描述：游標不動、按鍵失效。可能原因分析：BIOS停用、驅動缺失/衝突、硬體鬆脫。解決步驟：1) BIOS啟用內建指標裝置；2) 安裝正確驅動；3) 重新插接排線（維修）；4) 更新EC。預防措施：避免液體入侵；系統映像含正確驅動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q10, B-Q24, C-Q4

D-Q4: 聯想官網查不到保固或IWS？
- A簡: 核對序號/MTM與國別，若資料缺失聯繫客服補登或提供購買證明。
- A詳: 問題症狀描述：查詢無結果或國別不符。可能原因分析：資料同步延遲、平行輸入錯誤、非授權通路。解決步驟：1) 再核對序號；2) 提交發票與購買資訊；3) 請客服校正主檔；4) 確認IWS機型清單。預防措施：向授權經銷商購買，保存證明。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, B-Q6, C-Q10

D-Q5: 電池校準失敗或續航驟降的原因？
- A簡: 老化、控制器錯估或高溫使用，需校正或更換原廠電池。
- A詳: 問題症狀描述：百分比不準、突然關機。可能原因分析：循環老化、燃料計偏移、高溫損耗。解決步驟：1) 執行Battery Gauge Reset；2) 更新電源與BIOS；3) 更換老化電池。預防措施：避免長期滿電高溫；設定充電上限。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q13, C-Q8

D-Q6: 指紋辨識無法使用怎麼排查？
- A簡: 重新登錄指紋、更新驅動/韌體，檢查TPM與Windows生物識別設定。
- A詳: 問題症狀描述：無法辨識或登入失敗。可能原因分析：驅動損壞、TPM狀態異常、感測器硬體問題。解決步驟：1) 解除綁定並重註冊；2) 更新驅動與BIOS；3) 檢查Device Manager；4) 硬體維修。預防措施：定期更新；保留PIN/密碼備援。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q25, C-Q4

D-Q7: 更換Wi‑Fi卡出現未授權（1802等）錯誤？
- A簡: BIOS白名單限制導致，需使用核准FRU卡或官方BIOS版本。
- A詳: 問題症狀描述：開機錯誤碼阻止啟動。可能原因分析：Wi‑Fi卡ID不在白名單。解決步驟：1) 使用核准FRU清單之卡；2) 更新BIOS；3) 如仍失敗改回原卡。預防措施：更換前查HMM與FRU相容表，避免非核准零件。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q17, C-Q6

D-Q8: System Update 無法連線或卡住怎麼辦？
- A簡: 檢查網路與代理、更新TVSU版本、改用離線套件或手動下載。
- A詳: 問題症狀描述：掃描/下載停滯或錯誤。可能原因分析：代理限制、舊版Agent、清單異常。解決步驟：1) 更新TVSU；2) 設定代理白名單；3) 使用離線封存；4) 手動下載關鍵更新。預防措施：企業維護本地鏡像與版本基線。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, C-Q5

D-Q9: 顯示或音效驅動不穩定如何處理？
- A簡: 回退至穩定版本或使用OEM封裝，清除殘留並依序安裝。
- A詳: 問題症狀描述：藍屏、畫面閃爍、無聲。可能原因分析：版本衝突、未依序安裝、相依未滿足。解決步驟：1) 移除驅動（DDU建議顯示）；2) 安裝OEM封裝與建議版本；3) 更新晶片組與BIOS。預防措施：鎖定企業基線；延後導入新驅動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q4, B-Q3

D-Q10: 企業映像部署後裝置管理員有未知裝置？
- A簡: 依PNP ID匹配正確驅動，更新Driver Packs並調整注入邏輯。
- A詳: 問題症狀描述：黃色驚嘆號或未知裝置。可能原因分析：驅動包缺漏、機型誤判、順序錯誤。解決步驟：1) 取得PNP ID；2) 檢索Lenovo Driver Packs；3) 調整任務序列/條件；4) 重建映像。預防措施：維護驅動目錄、版本控管與機型偵測腳本。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q23, C-Q7, B-Q7
