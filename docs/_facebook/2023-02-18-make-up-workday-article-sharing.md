今天補班日，下班後讓大家充充電 (咦?)， 就趁今晚貼這篇好了 😃

這篇是我以 .NET Conf 2022 我講的場次，寫成文字稿的內容。雖然是文字稿，但是我補上了當時因為時間限制沒辦法分享的內容，主要是我在思考這些 API 設計時，心裡其實想的是 OOP，只是最後才把物件與類別的設計，對應成 API 的規格而已。

想通這層的對應，你就會發現一切都很簡單 & 自然了。物件之間的耦合夠低，你換成 API 來處理服務與服務之間的通訊，自然也會是低耦合的設計。中間 workshop 的案例就跟演講內容一致，只是這種要照著做的 workshop, 文字形式其實還是比影片適合..
不過，礙於篇幅，第三段講 security 的部分還是被我切到下一篇了，寫太長應該沒有人想要一次看完吧~

今天先貼這篇，實做的 workshop 相信大家會有不少案例想要套著做看看，有任何案例想要討論怎麼做的，歡迎在底下留言給我 😃
