幫比爾叔 Bill Chung 工商一下, C# Source Generator 的課程..

這是 C# 進階功能, 簡單的講就是能做到已前只能靠 Reflection, 在 Runtime 處理的任務, 既麻煩又沒效率, 現在可以往前提到 Compiler 處理階段, 靠 Source Generator 處理的技巧, 兼顧了效率, 安全跟優雅..

這種學工具或是語法的課, 現在越來越少見了, 可能大家都覺得以後靠 AI 寫就好了, 不用自己學... 不過我覺得這是斷了自己未來發展的機會啊, 看看那些 AI 用的好的人, 都是有基礎能力的, 你以後也許不用自己寫 source generator 這類很硬的 code, 不過你得要了解他才能要求 AI 照你的 "想法" 做..

前一篇我在聊的 Gemini SDK for .NET , 我一開始沒有堅持要用 SDK, 結果 AI 就幫我刻了一大包 code, 都是自己用 HttpClient 打 API, 這沒不好, 但是不是我要的, 因為我知道未來發展下去, 我得花一堆額外的成本 ( token ) 來繼續維護這堆 code, 為何我不一開始就明確的用 SDK 呢? 這是你對系統的品味, 對架構的選擇, 而 AI 是你的幫手, 你要先能做對決策, AI 才會放大你的能力

所以, 有不錯的基礎課程, 也有興趣的話可以參考看看, 比爾叔講 OOP, 講 C# 算是老字號了, 大推~

報名連結我放在留言, 我沒抽成, 純推薦 :D
