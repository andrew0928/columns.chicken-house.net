#水電工日誌, 翻了一下，上一篇是 2019/12 貼的，中間又陸陸續續替換了不少設備啊，會選擇現在再貼一篇，是因為我覺得我理想中的架構大致都成形了，是個適合紀錄一下的時間點 :D

就算雲端服務再發達，終究有些運算或服務，還是適合擺在離你最近的地方的。有人說是 #邊緣運算 ( Edge Computing ), 我是蠻認同這想法的。光是看看每隔一陣子一些大型服務要收起來時，就會在各自領域掀起一陣 "逃難潮" , 我就會很慶幸這些對我都沒有很大的影響...

不論技術的好壞，如果你終究有些東西是要擺在身邊，要有高度自主控制權的特性的話，你該怎麼做? 我自從 15 年前開始在家買了一個 15U 的小機櫃之後，就一直有些小型服務 (尤其是給自己用的) 養在自家的設備內。不過這 15 年來，世界改變很多，我的設備也不斷的在演化.. 這篇文章就剛好記錄這些過程。

硬體跟網路環境，大致上就越來越趨向整合成一套了。硬體部份我我挑選了在 controller 的軟體上有不錯表現的 #UniFi 為主，從 router, managed switch, dvr + camera, wifi ap 通通都換成他們家的了，整合的成效也沒讓我失望，原本滿滿一櫃的設備, 現在就精簡成這牌子的 router + switch, 共 2U 就搞定了

其他的軟體或儲存，包含 storage (file, photo), 休閒 (music, video), 以及其他的服務 (reverse proxy + ssl certification, password manager), 以及我自己在試用各種服務的 labs, 通通都容器化了, 最後我只需要有可靠的網路環境及頻寬 ( 10G, VLAN, firewall ) 以及可靠的儲存與運散設備 (我直接挑規格較好的 NAS) 搭配 docker 就搞定了，我以 #Synology 為主，之前掛了一台，換了 8bays 的機種, 原本舊的 4bays 就當作備份機

果然技術的進步，在整合方面最看的到成效了。現在我的機櫃內已經沒有啥特別的東西了，就 router / switch / NAS , 線路也只剩下網路了 (電話跟第四台都廢掉了)，時至今日都運作良好，而且我想要的服務一個都不少啊~

這篇沒有太多架構設計，也沒有 code 分享, 單純就是阿宅在家的設備展示而已, 分享我的規劃方式給同好們參考, 歡迎留言討論或是分享你的做法 :D
