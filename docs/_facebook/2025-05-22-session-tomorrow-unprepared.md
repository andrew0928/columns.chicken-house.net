沒什麼時間練習，結果不知不覺明天就要上場了 @@

明天Generative AI 開發者年會, 主題是 "從 Prompt 到 Product"，意思是我們從內部大家都在用 AI 各種工具，寫各種 prompt 改善我們自己的效率，逐步擴大到把這些經驗用在我們自己的 product 身上，也希望讓客戶享受到 AI 帶來的好處。

光是 prompt -> product 的過程，就發現有不少觀念跟做法是完全反過來的，執行一次的 prompt，考慮的環節跟執行 100 萬次的 prompt 完全不同。明天只有短短的 15 分鐘，我想聊聊跟著團隊跑這專案的收穫跟心得。

這一個月，我有三場公開的演講，我都會用同系列的主題: 從 Prompt 到 Product，不過不用擔心，我不會講重複的內容的... 

* 05/23, Gen AI Developer 年會, 只有 15m, 我會講開發思維的改變

* 06/05, DevOpsDays Taipei 2025, 40m, 我會講 AI APP 開始傳遞不一樣的價值了， DevOps 的文化在這年代會有什麼改變 

* 06/28, STUDY4LOVE, 我會聊聊 RD 的技能這件事。當大家隨手都能寫 code 的年代，什麼能力才能讓你夠格稱為 "工程師" ?
