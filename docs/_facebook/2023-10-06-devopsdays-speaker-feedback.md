一早起床就收到大會寄來的講師滿意度回饋，太感謝了，當講師期待的就是看到大家給的 feedback 啊，有人肯定你的付出的成就感是無價的，再次感謝 09/25 當天早上聽我囉嗦了一小時的各位聽眾..

這次本來想了很久，都在猶豫要不要講這題目，結果被臨門一腳推了一下坑，就交出去了。抽象建模，是我認為很關鍵的事情，但是也因為太抽象，其實很難講，甚至很多人都不重視.. 

不過，模型的設計，然後把它真的做出來，才是軟體的靈魂。沒抓好這些東西，你會發現很多規格，資料表，API，格式，流程等等都會怪怪的。偏偏這些東西都各有各的方法論，然後還分前後端 DBA 等等不同角色，背後沒有用模型來貫穿所有設計，你就會覺得這沒有一體性，不是個成熟的軟體，只是個堆積起來的工具..

最近這兩年，因為 API 的關係，重新花了些時間在面對這問題，也感謝我的老闆之一 Happy 給我一個很棒的範例，才有足夠素材講這一場。

還好尺度拿捏的還可以，沒有抽象到大家看不懂。不過我也因為決定要講好抽象化設計，就把降級實做的部分內容都砍了 (我光講這部分就要花上 1hr 啊..) 結果果然有收到一些回饋是在反映這些缺點的... 包含難懂，缺少案例等等。

最後看到問卷回饋才放心，看來大家是能接受這種題目的 (放連假前開獎... 主辦單位一定要這麼刺激嗎 XDD)。既然是回饋，好的壞的都歡迎，願意寫給我看得我都很感謝 :D

回饋的內容我就貼在下方了，另外，這次的大會共筆，其實也有很多聽眾的意見在上面。這次我維持習慣，歡迎大家在共筆上面跟我互動，大會後的一個禮拜我也持續上去回答問題，結果有別的講師就敲我，說都是我的關係，現在每個講師講完都還要上去補充 XDDD (啊這是好事吧)
