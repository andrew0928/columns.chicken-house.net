感謝 高見龍 的邀請, 第一次參加 #webconf ​... 有收穫的是會後的交流都會聽到更多經驗的分享, 可惜的是我準備的題目感覺沒抓到聽眾族群的期待..

我分享的題目, 談的是 SaaS 以後該怎麼往 Agent 發展, AI 相關的應用風險最高的兩件事, 就是會不會隨著模型的發展就被吃掉了? 或是過度樂觀, 很快做到 80 分, 結果就沒辦法在往下了?

關鍵還是在商業問題的流程要清楚 (這要找領域專家)，另外怎麼善用 AI 來解題的技術專家也要到位 (這是我平日在負責的角色)，拆出對的流程，盤點清楚到底哪些資訊要放進 context, 藉此找出有哪些必要的 tools 需要準備, 這些都是設計良好的 Agent 必備的過程基本功

這些路線其實我們都踩過了, 最好的方法就是盡快做出 MVP, 所有你擔心的事情都在 MVP 確認過後, 再投入資源把需要大規模建設的工程環節做出來..

到目前為止 (2025年末), 把這些題目擺在一起談的人還不多, 一個 session 也很難講到透, 不過拆開來看我就看到好幾場不錯的內容, 像是鈦坦開發自己的 agent 解決維運問題, 也有好幾場談產品設計, 怎樣做對的事情都是同樣的思考角度..

我覺得這是很重要的課題, 不過談得太早了, 主題我沒拿捏好就顯得頭重腳輕.. 感謝有來聽我場次的朋友, 簡報我放在留言, 也歡迎給我你的看法... 這次簡報我放在留言, 有興趣的請自由取用..

--
以下是我的參加心得, 包含老人說教 XDD, TL;DR..

參加活動最大的收穫 (對我而言), 都不是站在台上那 50 分鐘 (那都是結果了), 最有收穫的兩個部分, 一個是事前的準備, 因為要上台, 被迫把自己自以為已經懂得東西重新梳理一次, 就跟寫部落格文章一樣, 這過程會意外的得到很多經驗, 這是單向的 "學習" 得不到的能力.. 

另一個收穫是, 因為你拋出了一個主題, 有相同主題想法的人自然會來跟你交流, 這是第二個收穫. 我是個超級 I 人, 沒這種機會的話就只能自己默默地上網學習了 XDD

這次會後的講師對談, 有幾位聽眾會後來找我聊了 prototype 該怎麼拿捏? AI 該用在哪裡, 該不該訓練自己的模型等等, 看起來大家都是有備而來參加活動的, 雖然這些人不是站在台上的講師, 不過我也搞清楚了原來大家都在做什麼, 都在面對什麼問題, 這些經驗也蠻值得的

另外特別要提一下的, 就是這次在 B 區也有幸跟  聊了一下, 這次活動的後台, 龍哥自己一人靠 vibe coding 的威力, 能搞定 30 萬行 code, 包含 CI/CD 等部署工作, 還有活動前幾個小時還上版做架構調整... 這樣的案例目前只有龍哥做得到, 於是我也抓住機會問了: 

"這麼大規模的專案, 有碰到你非得親自用 debugger 去 trace code 的情況嗎?"

這問題的背景是, 我的工作屬性, 會碰到很多 core 層級的元件, 大家看我 2020 那陣子寫的文章主題大概就能理解, 這類服務都有關鍵的元件, 是絕對不能出錯的, 你要用盡一切技巧抓對介面, 演算法, 測試各種邊界行為... 面對 core code, 我絕對不會放掉的動作是: 親自用 debugger 跑過核心元件, 確認他完全按照我的意思執行... 

然而現在開發工具的發展趨勢, 已經從全功能的 IDE (代表: visual studio) 逐步轉移到 IDE + Agent (代表: cursor), 再轉移到 CLI + Agent (代表: claude code).. 到 CLI 的這個階段, 等於完全切掉了 開發者 跟 code 的直接連結. 即使是 Agent, 他的除錯方式也都是把 code 當作黑箱, 產生各種測試驗證方法, 或是輸出各種 log, 沒看過任何一個 agent 現在有能力操作 debugger 來 trace code..

不過, 聽完龍哥的分享後, 我想通這件事了... 在他這次開發過程中, 其實真的沒有動用到 開 debugger 來 trace code 這操作, 而事實證明他能維護好這種規模的 code project, 而且效能跟品質都能維持很好的水準.. 因為在前面的設計跟規劃, 就已經做得很到位, 後面會出狀況的機率本來就不大. 

不過他也跟我提到, 這次活動網站終究是終端應用, 我聊到有沒有那些環節 (類似金融交易, 絕對不能錯的), 這些部份其實都漂亮的切到外部服務去了 (例如金流服務), 活動網站本身重視的是流程的自動化, 而不是關鍵元件的開發與設計, 因此會有這些差別

像這種案例的討論與心得交換, 平常應該沒什麼機會可以碰的到吧! 其他還聊了很多老人說教的議題, 這我就不貼了 XDD! 其實能聊這些, 就遠遠超過我準備演講的付出了, 完全是值回票價, 再次感謝龍哥 & WebConf 這次的邀請, 獲益良多 :D
