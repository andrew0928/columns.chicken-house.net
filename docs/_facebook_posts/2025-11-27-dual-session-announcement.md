兩個場次一起宣傳好了 :D

年底接了兩場公開的演講, 其實我談的是同一個題目的兩個視角: SaaS 以後到了 Agentic 的時代會變成什麼樣貌... 我覺得這是很重要的題目啊, 因為這會是以後軟體工程師主要發展的方向。我自己這樣看:

現在的服務講究整合, 因此 SaaS 有沒有提供夠好的 API 往往是技術指標之一, 未來 SaaS 講的應該是賦能 Agent, 有沒有提供夠好的 MCP 則是未來的技術指標了。

軟體開發人員站在 AI 海嘯的第一排, 我們是最能親眼目睹這變化過程的一群人，同時也是最有資格替其他領域做這樣改變的人 (總要有人替其他領域把軟體都改寫成 AI 原生的版本吧)。再怎麼說，軟體工程師要失業，也至少還會經歷最後一波高峰，替全世界的產業都把系統翻過一輪才能失業吧，現在就是這波的起點。

因此, 這兩場我想談的分別是:

.NET Conf 2025 Taiwan ( 售票只到今天 23:50 喔 ) - MCP-First Design Concept, 談怎樣才能設計出到位的 MCP server ( 呼應三年前我談 API First Workshop ), 雖然是技術場次, 但是這次沒有講 code, 就是講設計概念, 對 MCP 設計有興趣的別錯過這場。

再來是兩週後的 2025 WebConf, 這是我第一次參加 WebConf, 先前受邀的時候主辦人還跟我說, 畢竟是 "WebConf", 可不可以多講一點開發不要只講 AI XDDD ( 我已經被認為只是蹭 AI 話題的角色了嗎 XD ), 所以這場的主題我就定調為: SaaS 的轉型。

這場我從一樣的出發點, SaaS 到以後 Agent 時代的變化, 但是我會從服務的設計，開發，到驗證的角度來談。很多人都用很聳動的標題，說 "SaaS 已死", 我沒這麼悲觀, 我覺得還是會活得好好的, 就跟前面講的一樣，軟體工程師還是有很多任務非他不可的，只是必須更往前站一點。

這兩場都還來的及買票喔，有來參加的歡迎來聽我的場次 :D
