感謝 Muyueh Lee 的邀請，參加了 11 月份的 #GenAI小聚 , 挑戰 10 min 介紹我的 Gen AI 開發與使用的經驗..

我藉這機會, 重新談了一次 "從 API First 到 AI First" 這主題。雖然題目一樣，不過時間只有十分鐘，來的聽眾都是以應用為主，不是過去我常接觸的純開發技術的領域...。因此，整個論述的方法，強調的重點，全部都重新準備了一次.. 只有案例 (安德魯小舖GPTs)，以及投影片 (其實只有標題) 是相同的...

這次我完全放掉 API 部分的設計心得了，我把重點轉移到 Agent 的設計心得。Agent = AI + API, 最厲害的是中間的 " + "，也就是串起一切可能性的 " function calling ", 我覺得這真的是讓軟體設計開發往前跨越一個世代的突破。

因為 LLM 有推理能力，從兩年前能回答問題，能寫 code, 到現在已經能自己挑 API 來執行了。也許你認為 AI 能寫 code 就很厲害了，不過這還不是終點，客戶改變需求，你終究還是要 "改" code 重新上版..

而 AI 能自己決定怎麼呼叫 API 之後，開始連 "客戶改變需求" 這件事都能應付了，只要你的基礎 API 做的夠好，AI 就能按照使用者的需求，在當下組合出新功能給使用者用，這我已經在這案例 POC 成功了..

這次把 10 分鐘都壓在這主軸上，其實應該換個標題才對，不過，我就繼續沿用這標題吧，感謝今天有來現場的朋友耐心聽完我這主題，沒機會在當下交流 QA 的，都歡迎在底下留言給我。簡報我放在第一則留言
