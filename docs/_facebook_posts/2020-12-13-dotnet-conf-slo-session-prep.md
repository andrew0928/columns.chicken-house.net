就是下禮拜了，講師當久了就是到最後一刻還會改投影片 (這是壞習慣)...  XDD

這次 .NET Conf 我們包了四個 session, 我自己就負責了兩場，一場我自己講 SLO, 另一場跟同事一起講 "刻意練習"，說明我們內部怎麼有系統地讓大家練習基礎技能。

這兩場的內容，我都不再單獨談特定的技術，例如 container, 或是 .net 5 等等, 我反而想讓大家知道, 用 .net 的團隊也是能做出很多水準之上的服務的, 做到這層次, 要面對的都不是單純技術問題了, 因此我們這次挑的實戰系列都是在: 

當你已經有很熟悉的技術作為基礎的前題, 你怎麼充分運用他來改善你的服務? 所以才有這四個 session:

1. Ruddy 老師: DevOps 三步工作法
2. Andrew: 非同步系統的 SLO 設計
3. Steven Tsai: Process Pool
4. Andrew, Fion: 刻意練習, 抽象化能力的鍛鍊

其實這四個 session 是有關係的, 舉例來說, 先有 DevOps 的核心精神，不斷改善，不斷回饋，才有今日的服務基礎。就是因為非同步的體系內我們搞定了 (3) process pool, 我們才有能力在效能之上繼續要求服務水準, 這背後的系統其實是同一套。我們的團隊必須能掌握像是 process pool 這樣複雜的機制，就必須要有良好的基礎能力，過程就是靠 (4) 的練習方式不斷累積起來的基礎能力。

有興趣的話歡迎一次四場都接著聽 XDD, 真的跟你其他想聽的議題衝堂也沒關係，大會 (應該?) 會有錄影吧? 或是直接到我們的攤位找我們聊聊都很歡迎。
