後面本來有些想寫的，不過我最前面還是先來貼一下這次活動收到的回饋吧 (免得大家沒看完就跳過了)

這次 .NET Conf 2020 總算順利結束了，這次包含 Ruddy Lee 老師，我們安排了四個 session, 我的目的是希望把我們 91APP 如何兼顧開發與維運的實戰經驗分享。比起像淘寶那像直接告訴你怎麼在一秒鐘之內完成上萬筆訂單的技術作法，台灣 (包含我們) 應該沒這樣的市場跟需求吧，因此我決定把主題轉向大家更需要的軟技能，團隊怎樣同時顧好開發與維運這件事身上。

因此主題的安排，首先就是 Ruddy 老師講的 DevOps 三步工作法，是整個 DevOps 團隊背後運作的心法。我負責如何顧好服務水準 (SLO)，第三場則是唯一講比較多 code 的 process pool, 最後一場則是跟我的 team member - fion 講如何訓練團隊抽象化思考的能力。

這安排同時涵蓋技術管理，架構設計，以及團隊成員訓練主題的安排。其實我一直很擔心大家沒看到 code 會不會就覺得講師只是出一張嘴講講大道理而已 XDD, 直到最後看到現場連著四場都座無虛席，還有看到事後的問券統計我才放心下來 XDD

這邊我就貼一下我有上台的這兩個場次的問卷回饋。回饋有點多，多到活動總召還私下跟我抱怨，說他這樣都不知道怎麼排版..  XDD, 有幾條回饋看來填錯場次了，也有幾條看來就不是寫給我的 (ex: 姐就是漂亮)，我就厚著臉皮跟主辦單位要了文字檔，不分場次貼在一起了。我分成兩段，有針對內容的回饋我分成一段，另一段單純是感謝或是稱讚的。這樣區分是希望大家能夠藉此分辨，這些內容是否適合你看錄影或是下載投影片的參考。

最後，活動結束後隔天，就看到有這麼一篇參加 .NET Conf 2020 後的心得文，作者其實不是 .NET 族群的人，不過也專程來報名參加了，而且心得文都是在講我這次安排的 session 啊! 這位朋友雖然我不認識你，不過我很感謝各位給我這些回饋，知道我們準備的內容對大家有幫助，就會有再繼續分享的動力了。
