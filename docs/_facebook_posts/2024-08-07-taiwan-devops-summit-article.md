趁著準備演講內容還記憶猶新，也有現成的簡報.. 趁還沒忘記前把文章補起來吧! 這是我七月初在 #DevOpsDays Taipei 2024 擔任 Keynote, 講的內容:

演講主題: "從 API First 到 AI First"

所謂從 API First 到 AI First, 不只是諧音梗啊，這主題主要是傳達從精準的運算架構 (API) 過渡到意圖的處理架構 (AI) 這背後的思考模式改變。我從這主軸出發，用了幾個案例 PoC，來說明巧妙的善用 AI，你可以辦到很多過去極難處理的課題。

這次演講的內容，算是把我過去半年做的各種研究與常識都整理起來，做一次完整的交代了。同時也代表 "架構師" 的角色，也開始從軟體開發的領域，跨進 AI 領域一步了。架構師被期待的是: 要知道如何 "正確" 的選擇與安置新技術的位置，這次的 DEMO 我就是從這角度來安排 AI 該如何融入你的服務之中。要做到這點，唯一的途徑就是了解 AI 對於應用程式而言本質是什麼 (我認為: 就是讓你的服務有理解使用者意圖並且做出反應的能力)，打好基礎，掌握它的本質，你才有能力做正確的安排。

> 掌握好基礎魔法，就足以對付這時代的魔法使了。

簡報中我用了動畫芙莉蓮中的這場景，雖然聽起來有點狂傲，但是不得不說作者看待專業領域真是看得透徹。我用這句話當作結尾當作對自己的期許，也感謝當天有來現場聽我場子的朋友 :D
