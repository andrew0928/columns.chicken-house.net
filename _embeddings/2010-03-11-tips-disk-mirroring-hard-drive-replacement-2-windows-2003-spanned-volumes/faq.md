# [Tips] 用 磁碟鏡像 更換硬碟 #2, Windows 2003 的跨距磁區

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是磁碟鏡像（Mirror）？
- A簡: 鏡像是軟體RAID1，將同一份資料同步寫入兩顆磁碟，提供容錯，常用於無中斷更換硬碟與提升可靠性。
- A詳: 磁碟鏡像（Mirror）是透過作業系統的動態磁碟功能，將同一個磁區或卷同步寫入兩顆實體磁碟，達到資料冗餘保護的機制。寫入時會同時寫入兩邊，讀取時可自任一顆讀取；若其中一顆故障，系統仍能持續運作。它屬於軟體RAID1，常見於Windows Server 2003/2008等版本，用於在不中斷服務情況下更換硬碟或提升資料可靠性。建立鏡像與後續的重新同步（Resync）需要時間，且必須使用動態磁碟。鏡像與跨距不同，前者強調容錯，後者強調容量聚合。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, A-Q10, B-Q1

A-Q2: 為什麼用鏡像來更換硬碟？
- A簡: 鏡像可在線搬遷資料、維持服務不中斷，待同步完成後移除舊碟，再擴大容量，風險與停機最小。
- A詳: 使用鏡像更換硬碟的核心目標，是在保持服務連續性的前提下完成資料搬移。實務流程是將新硬碟加入現有卷建立鏡像，等待系統自動重新同步，確保兩顆磁碟資料一致。同步完成後，移除舊硬碟的鏡像成員，系統實際運作即轉移到新硬碟。此流程避免了離線備份/還原或第三方影像工具的風險，並能在Windows內建工具中完成。若新硬碟較大，接著即可透過延伸/跨距功能擴大容量。此方法相較大量停機的完整還原作業更安全、可控且省時。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, B-Q2, A-Q6

A-Q3: 什麼是延伸磁區（Extend Volume）？
- A簡: 延伸磁區是將磁區容量擴大，把後方未配置空間或額外磁碟空間納入現有卷。
- A詳: 延伸磁區（Extend Volume）指的是在不刪除現有檔案系統與資料前提下，擴大既有卷的可用容量。Windows 2003中，延伸常可跨不同磁碟（動態磁碟）形成跨距，將多顆磁碟的空間串接為一個卷。自Windows Vista/2008之後，圖形介面中的「延伸」用來擴大原分割區至其後方的未配置空間（多為同一磁碟），而跨多磁碟的聚合則改以「Span Disk／Spanned Volume」來處理。延伸能保留資料，但需符合檔案系統與磁碟條件（如NTFS、動態磁碟等）。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, A-Q6, B-Q8

A-Q4: 什麼是縮小磁區（Shrink Volume）？
- A簡: 縮小磁區是減少現有卷的大小，釋放未配置空間以便建立新分割區或重新配置容量。
- A詳: 縮小磁區（Shrink Volume）讓管理者在保留既有資料的情況下，減少某卷的可用容量，將釋出的空間轉為未配置，用於新建額外分割區或其他用途。此功能自Windows Vista/2008起在內建磁碟管理中提供，搭配NTFS檔案系統使用；縮小程度受未移動檔案（如MFT、分頁檔、陰影複製）限制，必要時可先暫時關閉或移動此類檔案以增加可縮幅度。相較以往需仰賴第三方工具，內建縮小功能更直覺、安全且可控。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q4, C-Q6, D-Q2

A-Q5: 什麼是跨距磁區（Spanned Volume）與 JBOD？
- A簡: 跨距將多顆磁碟空間串接為一個卷，屬JBOD型態，重視容量聚合但不具容錯能力。
- A詳: 跨距磁區（Spanned Volume）是把多顆磁碟的未配置空間串接為單一邏輯卷，對應於非RAID的JBOD（Just a Bunch Of Disks）概念。它讓不同容量、不同硬碟的空間得以整合，提高容量利用率，最高可跨到32顆磁碟。然而跨距不具容錯能力，任一成員磁碟故障都可能導致整個卷資料不可用，因此非常依賴可靠備份。Windows 2003把「延伸」與跨距行為緊密結合；從Vista/2008起，跨距以「Span Disk／Spanned Volume」明確命名、與單一磁碟上的延伸分離。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q6, B-Q11

A-Q6: Windows 2003 與 2008 在延伸與跨距有何差異？
- A簡: 2003「延伸」可跨磁碟形成跨距；2008起延伸偏向擴大原分割區，跨磁碟則改稱Span Disk。
- A詳: 在Windows Server 2003中，「延伸」常用於把不同磁碟上的空間併成同一卷（跨距），於磁碟管理仍可看到各段來源。到了Vista/2008，微軟將功能與術語拆分：「延伸」專指擴張原分割區至其之後的可用空間，保留卷的連續性；要把多顆磁碟的空間合併，則使用「Span Disk／Spanned Volume」。同時新增「縮小」（Shrink）以釋放空間。此更動讓概念更清楚：延伸用於同卷擴張，跨距用於容量聚合，且皆倚賴動態磁碟與NTFS等條件。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q3, B-Q5, C-Q2

A-Q7: Extend 與 Span 的核心差異是什麼？
- A簡: Extend是擴大既有分割區；Span是把多顆磁碟串接為一卷。Extend重連續性，Span重聚合。
- A詳: Extend（延伸）以擴大同一個邏輯卷為目標，通常往其後方的未配置空間延展，維持卷的連續性，常見於同一顆磁碟。Span（跨距）則是把多顆磁碟的空間串接在一起，讓系統把它們視為同一卷使用，因此能靈活整合不同容量的磁碟。前者常用於升級到較大硬碟後擴大空間；後者用於需要最大化容量但不要求容錯。容錯上，Extend本身不提供保護；Span更無容錯，任一磁碟壞軌都可能波及全卷，需備份配合。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, B-Q9, B-Q11

A-Q8: 軟體鏡像與硬體RAID1有何差異？
- A簡: 軟體鏡像由作業系統管理較彈性；硬體RAID1由控制卡處理效率佳，與OS解耦且開機更單純。
- A詳: 軟體鏡像（Windows動態磁碟RAID1）由OS層實作，優點是無需額外硬體、管理彈性高、可細緻控制卷操作；缺點是吃系統資源、啟動流程較複雜、跨版本相容性需留意。硬體RAID1由控制卡協助，寫入鏡像在卡上完成，CPU負擔低，容錯/重建通常更快，對OS呈現單一邏輯磁碟，開機路徑也較單純。不過硬體RAID需成本與控制卡相容性。更換硬碟的情境下，兩者皆可達資料冗餘，軟體鏡像勝在隨手可用，硬體RAID勝在性能與封裝。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q9, D-Q3

A-Q9: 什麼是動態磁碟？與基本磁碟差異？
- A簡: 動態磁碟支援鏡像、跨距等進階卷；基本磁碟只提供傳統分割。動態更彈性但回轉需備份。
- A詳: 基本磁碟使用傳統MBR/GPT分割模型，管理主/擴充分割區。動態磁碟引入邏輯磁碟管理（LDM）層，支援簡單、跨距、鏡像、條帶化等卷類型，允許線上擴展、容錯等操作。建立鏡像或跨距必須將相關磁碟轉為動態。動態磁碟轉回基本磁碟需先備份資料、刪卷後轉回，過程可能清除資料。對於需要靈活卷管理與容錯的伺服器，動態磁碟是必要基礎。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, C-Q7, D-Q9

A-Q10: 什麼是重新同步（Resync）？
- A簡: Resync是鏡像建立或故障恢復時的資料同步過程，將兩顆磁碟內容比對並補齊一致性。
- A詳: 當建立鏡像、替換鏡像成員或故障後復原時，系統會啟動重新同步（Resync），逐區塊比對鏡像兩端資料並寫入缺失部分，以確保兩邊一致。Resync耗時取決於資料量、磁碟性能與系統負載。期間卷仍可用，但寫入作業會同時觸發同步，效能或受影響。完成Resync後，鏡像狀態恢復健康。若中途中斷（如重開機），系統會在下次上線時繼續或重新評估同步進度。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q2, D-Q3, C-Q1

A-Q11: 斷開鏡像（Break）與移除鏡像（Remove）差異？
- A簡: Break保留兩邊為獨立卷；Remove移除其中一成員，卷只留在另一顆，常用於換碟交接。
- A詳: 斷開鏡像（Break）會把鏡像拆成兩個獨立、內容相同的卷，兩者各自可掛載使用；此方式常用於需保留兩份可啟動副本的情境。移除鏡像（Remove）則是從鏡像組中移除其中一顆成員，卷仍留在另一顆磁碟上並保持線上，典型用於更換舊碟：先移除舊碟，讓新碟承接服務。選擇哪種方式取決於後續用途與風險控管，Break靈活但管理更複雜，Remove簡潔且符合換碟流程。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q8, B-Q6, D-Q5

A-Q12: 為何Vista/2008新增縮小磁區？
- A簡: 為提供內建安全分割與靈活容量管理，減少依賴第三方工具，提升系統維護可用性。
- A詳: 過去若要釋放空間或重劃分割，常須使用第三方工具，存在相容性與資料風險。Vista/2008引入Shrink，讓管理者能在系統內建工具中安全縮小NTFS卷，搭配延伸與跨距，形成完整的動態容量管理工具鏈。此舉增進IT維運的一致性與可控性，也對虛擬化與實體伺服器的即時調整帶來便利。雖功能陽春，但已涵蓋多數日常需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q4, C-Q6

A-Q13: 跨距磁區的風險與適用情境？
- A簡: 跨距無容錯，任一磁碟故障可能全毀；適用非關鍵、易重建或有良好備份的資料。
- A詳: 跨距將多顆磁碟空間串接，最大化容量利用率，但缺乏冗餘。任何一顆成員磁碟故障，都可能使整個卷的資料不可用。適用於暫存、大檔案暫置區、可重算資料或具完善備份與還原流程之非關鍵工作負載。不建議放置系統卷、資料庫主資料或高可用性要求的服務。若同時要容量與容錯，可考慮鏡像或硬體RAID，或採用其他儲存架構。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, D-Q6, A-Q7

A-Q14: Windows 跨距最多可併幾顆磁碟？
- A簡: Windows可將跨距最多併到32顆動態磁碟，但實務上受管理與風險考量限制。
- A詳: 根據Windows動態磁碟與LDM設計，跨距卷的成員上限為32顆磁碟。雖此上限提供極高的容量彈性，但每增加一顆磁碟，故障點與資料風險隨之攀升，管理複雜度與重建成本也提高。實務上建議審慎控制成員數，落實備份與監控，並評估以更可靠的RAID或集中式儲存取代大規模跨距。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q14, A-Q13, B-Q11

A-Q15: 什麼是磁碟管理（Disk Management）與DiskPart？
- A簡: 磁碟管理是GUI工具；DiskPart是命令列工具。兩者皆可操作動態卷與延伸/縮小。
- A詳: Windows提供兩種主要儲存管理工具：圖形化的「磁碟管理」與命令列的「DiskPart」。前者易用、直覺，適合日常操作；後者可自動化、批次化，且在核心版本或遠端環境特別實用。兩者皆能建立鏡像、跨距（動態磁碟）、延伸、縮小（Vista/2008起）等。對資深管理者，熟悉兩者可在GUI受限或遠端維運時保持彈性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q9, B-Q8, C-Q5

A-Q16: 鏡像換碟與「磁碟映像（影像檔）」有何不同？
- A簡: 鏡像是線上同步冗餘；磁碟映像是離線複製整碟到檔案，用於備份或部署。
- A詳: 鏡像（Mirror）在系統運作中同步兩顆磁碟，提供即時冗餘，適合無中斷換碟。磁碟映像（Disk Image）則是把整顆磁碟或分割區封裝成影像檔，常用於備份、複製佈署或跨硬體遷移，通常需要離線或停機，寫回影像也需時間與風險。換碟時若追求不中斷與保留卷狀態，鏡像較合適；若需跨硬體大量部署或完整回溯，影像較適合。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q1, C-Q3, A-Q2

A-Q17: 系統/開機分割區可以做鏡像嗎？
- A簡: Windows Server支援系統/開機卷鏡像，提升容錯；但需要動態磁碟並正確設定開機。
- A詳: Windows Server 2003/2008支援將系統或開機分割區做鏡像（RAID1），可提升開機磁碟容錯。此操作需將磁碟轉為動態，並在鏡像建立後等待Resync完成。若移除原開機碟，必須確保新碟具有正確的磁碟簽章與啟動代碼，BIOS/UEFI啟動順序也需更新。雖然支援，但務必先備份並規劃回復流程。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q12, D-Q5, C-Q8

A-Q18: 為什麼第三方分割工具風險較高？
- A簡: 相容性與中斷風險較大，誤操作恐毀損分割表或檔案系統；內建工具較一致可控。
- A詳: 第三方分割/擴容工具雖功能強大，但在不同Windows版本、動態磁碟與NTFS特性上，可能存在相容性差異；操作途中若軟體錯誤、斷電或使用者設定失誤，易導致分割表或檔案系統損壞。相較之下，Windows內建的延伸/縮小/跨距功能與系統整合度高，錯誤保護與日誌行為一致，風險較低。若必須用第三方工具，需完整備份與驗證回復計畫。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q12, C-Q6, D-Q2

### Q&A 類別 B: 技術原理類

B-Q1: Windows 軟體鏡像背後如何運作？
- A簡: 以動態磁碟LDM記錄鏡像關係，寫入時雙寫，讀取可分散；故障時依剩餘成員維持服務。
- A詳: Windows動態磁碟使用LDM（Logical Disk Manager）元資料描述卷與成員關係。建立鏡像後，卷的每個寫入I/O被複寫至兩個成員裝置；讀取可由任一端完成以降低延遲。鏡像狀態與一致性透過位圖與事件追蹤，當異常發生時可啟動Resync，僅同步差異區塊以加速恢復。管理層抽象出單一卷讓檔案系統無感，但需要OS協調I/O與錯誤處理。此機制提供容錯與線上更換硬碟能力。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q10, B-Q10

B-Q2: 鏡像建立與Resync的執行流程？
- A簡: 轉動態→建立鏡像→初始複寫→背景Resync→狀態Healthy；中斷後重啟續跑。
- A詳: 流程通常為：1) 將相關磁碟轉為動態，2) 在現有卷上建立鏡像或建立新鏡像卷，3) 進行初始資料複製（可能為全量），4) 進入背景Resync，依位圖同步差異區塊，期間卷可用，5) 完成後狀態標記為Healthy。若中途重開機，系統根據元資料恢復進度並續傳。Resync負載可受系統IO與排程調整，建議在非尖峰時段進行。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, C-Q1, D-Q3

B-Q3: Windows 2003「延伸=跨距」的實作機制是什麼？
- A簡: 以LDM在多磁碟建立一個Spanned卷，將不同區段串接，對上層呈現連續空間。
- A詳: 在2003，延伸常意指把新加入的未配置空間（可在另一顆動態磁碟）串接到既有卷之後，形成Spanned Volume。LDM會為每段（Extent）建立對應，對檔案系統呈現為單一連續空間。I/O按位址映射至對應成員磁碟的區段，不進行條帶，性能取決於後段磁碟。此模式即JBOD，成員數可多達32，但任一成員失效即影響整卷。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, A-Q6, B-Q11

B-Q4: 2008 Extend/Shrink 與NTFS如何互動？
- A簡: Extend擴至後方未配置；Shrink需移動資料，受未移動檔案限制，只支援NTFS線上縮小。
- A詳: 在Vista/2008，延伸與縮小透過NTFS的在線重配置支援：延伸需卷後方連續未配置空間，NTFS擴大其文件系統結構；縮小時，系統嘗試移動可移動資料以釋放尾端空間，但MFT、分頁檔、陰影複製等未移動檔案會限制可縮幅。縮小僅對NTFS提供線上操作；FAT32等需離線或不支援。管理者可停用相關服務、臨時移動分頁檔以提升縮小成功率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, D-Q2, C-Q6

B-Q5: 2008 之後的 Span Disk 架構？
- A簡: 將跨磁碟聚合功能明確分離為Spanned卷，由LDM維護多Extent映射，命名更清晰。
- A詳: 自Vista/2008起，微軟把跨磁碟聚合明確命名為Span Disk/Spanned Volume，與同一磁碟上的延伸語義分離。實作仍由LDM管理多個Extent的映射與狀態，對檔案系統呈現單一邏輯空間。此調整讓管理UI與命令列語意一致，降低誤解，並搭配Shrink完成更直觀的容量調整工作流。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q7, C-Q5

B-Q6: Break vs Remove鏡像在元資料上的差異？
- A簡: Break生成兩個獨立卷元資料；Remove只移除一成員，卷關係轉為單一裝置。
- A詳: 在Break時，LDM會為兩個成員各自建立獨立的卷描述，兩邊都包含完整檔案系統與卷ID（可能調整以避免衝突），因此兩卷可各自掛載。Remove則是更新鏡像卷的元資料，撤除其中一成員，將卷狀態改為在單一裝置上；被移除的成員保留資料，但系統不再視其為該卷一部分。前者利於保留兩份拷貝，後者利於換碟流程。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q11, C-Q8, D-Q5

B-Q7: 何時必須轉為動態磁碟？
- A簡: 當需鏡像、跨距或線上延伸多磁碟時需動態磁碟；僅單碟基本延伸可維持基本磁碟。
- A詳: 鏡像（RAID1）、跨距（Spanned）與條帶（RAID0）皆依賴動態磁碟的LDM元資料描述與管理。若只在同一顆磁碟後方延伸，部分情況可維持基本磁碟；但多數企業情境為了後續彈性，會轉為動態。轉換過程通常不破壞資料，但回轉需刪卷與備份還原。規劃時請評估日後需求與回復策略。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q9, C-Q7, D-Q9

B-Q8: 延伸/縮小的限制條件有哪些？
- A簡: 需NTFS、足夠連續未配置空間；縮小受未移動檔案限制；某些系統卷操作有限制。
- A詳: 延伸要求目標卷為NTFS，且在其後方有連續未配置空間（跨磁碟延伸需動態與Spanned）；縮小則僅支援NTFS線上操作，受未移動檔案影響，必要時需暫停陰影複製、移動分頁檔或重啟。系統/開機卷的延伸與縮小可能需離線操作或有更嚴格限制。USB/可攜裝置不支援動態。事前評估條件可避免GUI選項呈灰色。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q1, D-Q2, C-Q6

B-Q9: 性能：鏡像、跨距、條帶化差異？
- A簡: 鏡像寫入雙寫、讀取可分散；跨距近似單碟序列；條帶化提升吞吐但無容錯。
- A詳: 鏡像（RAID1）寫入需同時落兩顆，寫延遲略高；讀取可從任一端，於多併發有助降低讀延遲。跨距僅將空間串接，I/O依區段落在不同磁碟，無條帶並行，因此性能接近單碟序列存取。條帶化（RAID0）將I/O分散到多顆磁碟並行，能顯著提升吞吐與IOPS，但完全無容錯。依工作負載選擇：要容錯選鏡像，要容量聚合選跨距，要極致性能選條帶但務必備份。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q7, A-Q13, D-Q3

B-Q10: 鏡像的容錯與恢復原理？
- A簡: 透過冗餘成員承載服務，故障後以剩餘成員繼續，修復時Resync差異區塊恢復一致。
- A詳: 鏡像在任一成員故障時，卷可由剩餘健康成員繼續提供服務，保持線上。更換故障磁碟並加入鏡像後，系統依據位圖判定需同步的差異區塊，啟動Resync恢復一致性，完成後回到Healthy。此過程對上層檔案系統透明，但過程中性能受影響。記錄的事件與元資料讓恢復具可追溯性。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q2, D-Q3

B-Q11: 跨距磁區的資料配置與失效行為？
- A簡: 依卷位址映射到不同成員區段，任何一顆損壞都可能使整卷資料不可讀。
- A詳: Spanned卷由多個Extent串接形成，I/O按邏輯位址映射至對應磁碟的特定區段。不存在條帶或校驗，因而無冗餘保護。當任一成員磁碟故障，對應區段資料即不可得，導致檔案系統整體一致性受損，常見為整卷故障。除非另有備援（例如上層應用自帶分散冗餘），一般需自備份還原。這也是跨距僅適用非關鍵負載的原因。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q13, D-Q6, B-Q3

B-Q12: 系統/開機卷鏡像的啟動流程有何特點？
- A簡: 需正確的啟動代碼與磁碟簽章，開機管理程式能從任一成員啟動。
- A詳: 鏡像系統/開機卷時，Windows會在兩顆成員磁碟上配置啟動必要元件（MBR/引導程式），並記錄磁碟簽章。BIOS/UEFI啟動順序應包含兩顆磁碟，以便一顆失效時可由另一顆啟動。移除/更換磁碟後，需確認啟動配置（BCD/boot.ini在舊版）與磁碟簽章一致，避免無法啟動。這也是換碟流程需測試開機的關鍵。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q17, D-Q5, C-Q8

B-Q13: 磁碟簽章與卷識別如何保持一致？
- A簡: 透過LDM與磁碟簽章維護卷對應，搬移/更換後需重新掃描或匯入外來磁碟。
- A詳: Windows以磁碟簽章（Disk Signature）與LDM元資料識別磁碟與卷。更換或搬移磁碟後，系統可能標記為「外來（Foreign）」；需在磁碟管理中匯入，或使用DiskPart執行rescan/import，使元資料與系統重新建立對應。若簽章衝突，系統會調整或提示。正確流程有助維持卷ID、磁碟代號與服務穩定。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: D-Q7, C-Q10, D-Q8

B-Q14: 32顆跨距限制的考量為何？
- A簡: 出於LDM元資料與管理複雜度、風險與維護權衡，32顆提供上限但非實務建議。
- A詳: 32成員上限源於LDM元資料結構、系統管理可行性及風險控管的折衷。更多成員意味更多失效點與更難的例外處理；管理工具與測試覆蓋度也受限。雖技術上可行，但實務上應避開大規模跨距，用更可靠的儲存設計（RAID、SAN、NAS）實現容量與可用性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q14, A-Q13, B-Q11

### Q&A 類別 C: 實作應用類（10題）

C-Q1: 如何在Windows 2003用鏡像更換硬碟？
- A簡: 轉為動態→建立鏡像→等待Resync→移除舊碟成員→擴大容量，過程不中斷服務。
- A詳: 具體步驟：1) 插入新硬碟、初始化，於磁碟管理將兩顆轉為動態。2) 於舊卷上按右鍵「加入鏡像」，選新磁碟。3) 等待Resync完成（磁碟管理顯示同步中到健康）。4) 在卷上按右鍵「移除鏡像」，選舊磁碟，使卷留在新碟。5) 若新碟較大，使用「延伸」將新碟後方空間加入（2003此處會形成跨距）。注意：全程保持備份，避免在高峰期操作，Resync時效能下降。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q2, B-Q2, C-Q2

C-Q2: 2003完成鏡像後，如何移除鏡像並擴大D:？
- A簡: 先移除舊碟的鏡像成員，再以延伸把新碟剩餘空間併入D:，形成跨距擴容。
- A詳: 1) 在磁碟管理對D:按右鍵，選「移除鏡像」，選舊磁碟，讓D:留在新磁碟。2) 確認D:狀態Healthy。3) 在D:按右鍵選「延伸」，於精靈選擇新磁碟上未配置空間（可同一顆或其他動態磁碟）加入卷。4) 完成後D:容量增加；2003此步會將D:變為跨距卷。注意：跨距無容錯，請確認備份，並避免將關鍵資料置於跨距。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, A-Q13, B-Q3

C-Q3: 在2008/2008 R2如何用鏡像更換硬碟？
- A簡: 建立鏡像→Resync→移除舊碟→以延伸擴大原分割區（同碟後方），跨磁碟則建Spanned。
- A詳: 1) 新碟上線、初始化，將磁碟轉為動態。2) 對既有卷按右鍵「新增鏡像」，選新碟，等待Resync完成。3) 完成後對卷按右鍵「移除鏡像」，選舊碟。4) 擴容：若同一顆新碟後方有未配置空間，使用「延伸」直接擴大卷；如要跨多顆，改用「建立Spanned卷」。2008起將延伸與跨距語意分離，操作更直觀。全程維持備份，避開高峰期。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q5, C-Q4

C-Q4: 在2008/2008 R2如何擴大原分割區？
- A簡: 使用磁碟管理延伸至後方未配置空間；無連續空間時先調整或考慮Spanned。
- A詳: 1) 打開磁碟管理，確認目標卷為NTFS且後方有連續未配置空間。2) 右鍵卷選「延伸」，依精靈選擇要加入的空間大小。3) 完成後系統即時擴大卷與檔案系統。若後方無連續空間，可先縮小或移動其他卷（必要時第三方工具，務必備份），或改採Spanned建立跨磁碟擴張（需動態磁碟）。注意系統/開機卷限制與維護時段。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, D-Q1, C-Q5

C-Q5: 如何建立跨距磁區（Spanned Volume）？
- A簡: 將相關磁碟轉為動態，於磁碟管理建立Spanned卷，選取多顆磁碟的未配置空間。
- A詳: 1) 在磁碟管理將欲參與的磁碟轉為動態。2) 於其中一顆磁碟的未配置空間上右鍵「新建卷」→選「跨距」。3) 於精靈選取其他磁碟及各自空間大小，設定順序。4) 指定檔案系統（建議NTFS）與磁碟代號，完成建立。注意：跨距無容錯，任一成員故障可能導致全卷損毀；不建議用於關鍵資料，請備份。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q5, B-Q11, D-Q6

C-Q6: 如何縮小磁區並建立新分割區（2008+）？
- A簡: 於NTFS卷選「縮小」，釋放空間後在未配置區新建卷；必要時先移除未移動檔限制。
- A詳: 1) 右鍵NTFS卷選「縮小」，系統分析可縮幅度，輸入欲縮小MB數。2) 完成後產生未配置空間。3) 在未配置空間上新建卷，指定檔案系統與代號。若可縮幅度不如預期，先停用陰影複製、暫移分頁檔、關閉休眠並重開機，再重試；或用離線工具處理。縮小系統卷時務必備份並選在維護時段。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, D-Q2, A-Q12

C-Q7: 如何將基本磁碟轉為動態以啟用鏡像/跨距？
- A簡: 在磁碟管理對磁碟選「轉換為動態磁碟」，勾選目標磁碟；轉回基本需先備份。
- A詳: 1) 打開磁碟管理，在磁碟標頭（磁碟0/1處）右鍵選「轉換為動態磁碟」。2) 勾選目標磁碟並確認。系統會在不中斷情況下轉換（仍請備份）。完成後可建立鏡像/跨距等進階卷。注意：要轉回基本，必須先刪除動態卷並還原備份，過程會清除資料；USB/可攜式裝置不支援動態。

C-Q8: 如何安全斷開鏡像並指定保留哪顆磁碟？
- A簡: 使用「移除鏡像」，選擇要移除的那顆；或用「斷開鏡像」保留兩份獨立副本。
- A詳: 1) 在磁碟管理對鏡像卷按右鍵。2a) 若計畫讓卷留在新碟並拆下舊碟，選「移除鏡像」，勾選「舊碟」。2b) 若要保留兩份可各自使用的拷貝，選「斷開鏡像」。3) 完成後檢查卷狀態Healthy並開啟檔案確認。對系統卷，拆除舊碟前請先測試以新碟啟動。全程備份，避免同時移除電源與傳輸線。

C-Q9: 如何用DiskPart執行extend/shrink？
- A簡: 在DiskPart中選取卷後執行「extend」或「shrink desired=MB」；需NTFS與可用空間。
- A詳: 操作示例：1) 以系統管理員啟動命令提示字元，輸入「diskpart」。2) 輸入「list volume」查卷號，接著「select volume n」。3) 延伸：輸入「extend size=XXXX」（MB，省略則用全部可用）。4) 縮小：輸入「shrink desired=XXXX」。5) 完成後「exit」。注意：延伸需後方連續未配置空間（跨磁碟需動態且改採Spanned）；縮小僅支援NTFS且受未移動檔限制。可搭配「rescan」刷新裝置。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q15, B-Q8, D-Q1

C-Q10: 進行前後如何驗證資料與狀態？
- A簡: 前備份與健檢；中觀察事件與狀態；後檢查容量、跑chkdsk與抽查檔案、確認開機。
- A詳: 1) 事前：完整備份，檢查SMART、事件檢視器，確保新碟健康。2) 進行中：留意磁碟管理狀態（Resync/Healthy）、事件ID。3) 完成後：確認容量變更、執行「chkdsk /f」於次開機、抽樣比對重要檔案雜湊、檢視應用正常。若為系統卷，模擬以新碟開機（調整開機順序）確認可啟動。4) 長期：建立監控與備份排程。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, D-Q8, D-Q5

### Q&A 類別 D: 問題解決類（10題）

D-Q1: 延伸（Extend）選項是灰色的怎麼辦？
- A簡: 確認NTFS、後方連續未配置空間、動態需求與卷類型；不符時調整或改用Spanned。
- A詳: 症狀：在磁碟管理對卷按右鍵，「延伸」灰色不可選。可能原因：卷非NTFS、卷後方無連續未配置空間、目標為系統卷有額外限制、磁碟為基本無法跨碟延伸、USB裝置。解法：1) 確認檔案系統與卷狀態；2) 釋放或移動相鄰分割（必要時第三方工具，先備份）；3) 轉為動態並改採Spanned；4) 使用DiskPart「extend」與「rescan」。預防：規劃分割順序、保留後方空間。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q8, C-Q4, C-Q9

D-Q2: 縮小（Shrink）無法縮小太多或失敗？
- A簡: 受未移動檔案限制。停陰影、移分頁檔、關閉休眠、重開機後再試，或改離線工具。
- A詳: 症狀：可縮幅度遠小於可用空間或操作失敗。原因：卷尾端存在未移動檔案（MFT、分頁檔、休眠檔、陰影複製）。解法：1) 停用系統還原/陰影複製；2) 將分頁檔暫時移到其他磁碟；3) 關閉休眠；4) 重開機後重試縮小；5) 仍不行時，備份後以離線工具處理。預防：事前調整系統設定並在維護時段執行。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, C-Q6, A-Q12

D-Q3: 鏡像Resync卡住或很慢怎麼辦？
- A簡: 檢查硬碟健康、I/O負載與事件；避開尖峰、修復壞扇區、更新韌體或更換硬碟。
- A詳: 症狀：Resync長時間停滯或速度極慢。原因：硬碟壞軌/SMART警示、I/O高負載、控制器/纜線問題、背景掃描。解法：1) 查事件檢視器與磁碟狀態；2) 調整維護時段，降低負載；3) 執行硬碟診斷、更新韌體與驅動；4) 排除連線與供電問題；5) 持續異常時更換問題硬碟。預防：先健檢新碟、規劃低負載時Resync、落實監控。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q10, C-Q1

D-Q4: 無法建立鏡像該怎麼辦？
- A簡: 確認兩碟為動態且容量足夠、卷為支援類型；若為系統卷需額外條件與空間。
- A詳: 症狀：建立鏡像選項灰色或失敗。原因：磁碟為基本、空間不足、卷類型不支援、USB裝置、系統卷限制。解法：1) 轉為動態磁碟；2) 確保兩碟有相同或較大可用空間；3) 不在USB或可攜裝置上操作；4) 系統卷需保留引導空間並檢查分割對齊；5) 使用DiskPart驗證卷狀態。預防：事前規劃容量與裝置類型。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q9, C-Q7, B-Q7

D-Q5: 斷開/移除鏡像後系統無法開機？
- A簡: 可能引導設定錯誤或啟動順序不當；修復BCD/boot.ini、調整BIOS、重建MBR。
- A詳: 症狀：拆下舊碟後無法開機。原因：新碟缺引導代碼、BCD/boot.ini未更新、BIOS/UEFI啟動順序仍指向舊碟、磁碟簽章改變。解法：1) 以安裝媒體進入修復，執行啟動修復；2) 檢查/修復BCD（bcdboot）、或2003的boot.ini；3) 於BIOS/UEFI將新碟設為第一啟動；4) 必要時重建MBR/啟動程式。預防：拆碟前先以新碟測試開機、保留還原媒體。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q17, B-Q12, C-Q8

D-Q6: 跨距磁區有一顆磁碟壞了怎麼辦？
- A簡: 跨距無容錯，通常需自備份還原；可嘗試專業救援，但成功率有限，務必預防。
- A詳: 症狀：跨距卷突然無法掛載或檔案損毀。原因：任一成員磁碟故障使卷不一致。解法：1) 立即停用受影響磁碟避免二次傷害；2) 以備份還原至新卷；3) 若無備份，僅能求助專業資料救援（成功率不保）。預防：跨距僅用於非關鍵資料；落實定期備份與健康監控；關鍵工作負載改用鏡像/RAID。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q13, B-Q11, C-Q5

D-Q7: 移除舊碟後出現「Foreign」或無法上線？
- A簡: 在磁碟管理匯入外來磁碟或DiskPart「rescan」，處理簽章衝突後上線。
- A詳: 症狀：新接入的動態磁碟顯示Foreign或Offline。原因：磁碟帶有先前LDM元資料，需要匯入；或簽章衝突。解法：1) 在磁碟管理右鍵選「匯入外來磁碟」；2) 用DiskPart執行「rescan」，並「online disk」；3) 若簽章衝突，依提示變更簽章；4) 確認卷掛載並檢查資料。預防：更換流程中標記清楚、避免同簽章碰撞。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q13, C-Q10, D-Q8

D-Q8: 延伸後容量顯示異常或未反映？
- A簡: 重新掃描、重啟檔案總管、執行chkdsk；DiskPart「rescan」，必要時重開機。
- A詳: 症狀：延伸成功但容量未更新或檔案總管顯示舊值。原因：快取未更新、掛載點未刷新、檔案系統標記需修復。解法：1) 在磁碟管理執行重新掃描；2) DiskPart輸入「rescan」；3) 重啟檔案總管或登出登入；4) 執行「chkdsk /f」排程修復；5) 仍異常時重開機。預防：在低負載時操作，避免同時大量I/O。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q9, C-Q10, B-Q13

D-Q9: 想把動態磁碟轉回基本怎麼辦？
- A簡: 需先備份，刪除動態卷後轉回基本，再還原資料；評估停機與風險。
- A詳: 症狀：需回到基本磁碟以相容某工具/系統。限制：Windows不支援在線無損轉回。解法：1) 完整備份所有卷資料；2) 在磁碟管理刪除動態卷；3) 右鍵磁碟轉回「基本磁碟」；4) 重新分割並還原資料。第三方工具宣稱無損轉換，風險高不建議。預防：初期規劃評估是否真的需要動態功能。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q9, C-Q7, A-Q18

D-Q10: 卷代號（磁碟機代號）變更導致服務中斷怎辦？
- A簡: 於磁碟管理還原代號或改用掛載點；校正應用設定與相依服務。
- A詳: 症狀：更換/擴容後驅動代號改變，應用找不到路徑。解法：1) 在磁碟管理對卷右鍵「變更磁碟機代號與路徑」，還原原代號；2) 或改用NTFS資料夾掛載點以避免代號衝突；3) 檢查應用/服務設定與排程任務路徑；4) 驗證權限與共享設定。預防：操作前記錄代號與掛載點，優先採用掛載點策略。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q10, B-Q13, C-Q4
