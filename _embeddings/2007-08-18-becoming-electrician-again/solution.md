## Case #1: 全屋星狀佈線一次到位規劃

### Problem Statement（問題陳述）
**業務場景**：新屋隔間從無到有，同步進行水電與弱電（網路、電話、CATV）規劃。屋主需要在未來二十年內盡量避免再拆天花板或重拉線，且家中設備眾多（電腦、NAS、印表機、音響、電視遊樂器、Video Player、CCTV、電話等）需要穩定的有線連接。  
**技術挑戰**：在不影響裝修工期下，完成多介面、多類型線路的集中式星狀佈線，並確保日後擴充、維護與更換設備的彈性。  
**影響範圍**：佈線一旦完成即長期使用，若規劃不足將造成性能瓶頸、維護困難與高昂重工成本。  
**複雜度評級**：中

### Root Cause Analysis（根因分析）
**直接原因**：
1. 佈線時機有限：裝修完成後再拉線將大幅增加成本與施工難度。  
2. 設備需求成長：現代家庭多數設備需要網路，單一孔位無法滿足。  
3. 傳統串接式拓撲限制：電話、CATV慣用串接，靈活性差。

**深層原因**：
- 架構層面：缺少集中式分配與統一規劃，導致日後擴充受限。  
- 技術層面：未事先選定標準（如T568B）、缺乏測試計畫。  
- 流程層面：缺少佈線台帳、標籤與驗收流程，後續維運困難。

### Solution Design（解決方案設計）
**解決策略**：採用全屋星狀佈線，所有CAT5e、電話與CATV線以Home-Run方式匯入機櫃集中管理。各房間預留足量孔位，統一端接規格與標籤，完工後逐線測試驗收，建立佈線台帳，確保擴充彈性與長期可維護性。

**實施步驟**：
1. 規劃與算量
- 實作細節：繪製平面圖、標註每房間孔位數，決定星狀匯集點與機櫃位置，計算線長與冗餘10-15%。  
- 所需資源：平面圖、丈量工具、簡易算量表。  
- 預估時間：4-6小時。

2. 拉線與端接
- 實作細節：採用CAT5e與高品質同軸，遵循T568B規範，弱電與強電分離走線。  
- 所需資源：AMP CAT5e、CATV線、3M模組化資訊面板、壓線工具。  
- 預估時間：1-2天。

3. 集中管理與驗收
- 實作細節：機櫃內安裝配線架、交換器，完成標籤與台帳，逐線測試。  
- 所需資源：15U機櫃、24埠交換器、測線器、標籤。  
- 預估時間：6-8小時。

**關鍵程式碼/設定**：
```yaml
# 佈線台帳範本（YAML）
rack:
  location: 雜物間-15U機櫃
  devices:
    - name: PatchPanel-24
      ru: 12
    - name: Switch-24p
      ru: 10
rooms:
  - name: 主臥
    drops:
      - id: MBR-01; type: RJ45; standard: T568B; to: PatchPanel-01
      - id: MBR-02; type: RJ45; standard: T568B; to: PatchPanel-02
      - id: MBR-TEL; type: RJ45(RJ11用途); pair: 4/5; to: PBX-In-01
  - name: 客廳
    drops:
      - id: LR-01; type: RJ45; to: PatchPanel-03
      - id: LR-CATV; type: F-connector; to: CATV-Splitter-01
standards:
  ethernet: T568B
  coax: Quad-Shield, Compression-F
```

實際案例：本案於裝修期間完成全屋星狀佈線，每房間預留三條CAT5e（兩資料、一電話可轉資料）、CATV同樣Home-Run至機櫃。  
實作環境：15U機櫃、24-port交換器（二手）、AMP CAT5e 305m×2箱、3M資訊面板與模組、PBX小總機、CCTV網路錄放影機。  
實測數據：
- 改善前：Wi‑Fi實測吞吐約150 Mbps、延遲15-25ms  
- 改善後：有線實測吞吐940 Mbps、延遲<1ms  
- 改善幅度：吞吐+526%、延遲-95%

Learning Points（學習要點）
核心知識點：
- 星狀佈線與Home-Run集中管理  
- 端接標準（T568B）與測試驗收  
- 裝修期一次到位的成本效益

技能要求：
- 必備技能：讀圖與算量、基本端接與測線  
- 進階技能：機櫃規劃、弱電系統整合

延伸思考：
- 可用於小型辦公室、出租套房群組化管理  
- 風險：過度預留造成材料成本上升  
- 優化：採用可管理交換器與VLAN分區

Practice Exercise（練習題）
- 基礎練習：為兩個房間設計星狀佈線圖與台帳  
- 進階練習：完成3條網路線端接並通過測線器測試  
- 專案練習：規劃小宅（2房2廳）完整弱電方案與驗收報告

Assessment Criteria（評估標準）
- 功能完整性（40%）：是否完成星狀佈線與集中管理  
- 程式碼品質（30%）：台帳清晰、字段一致、可維護  
- 效能優化（20%）：吞吐與延遲達標  
- 創新性（10%）：擴充與風險控制設計


## Case #2: 每房三條CAT5e的多設備連接方案

### Problem Statement（問題陳述）
**業務場景**：家庭內設備數量增加，僅配置一個RJ45孔位已無法滿足同時接入電腦、NAS、印表機、影音播放器、遊戲主機等需求。希望在每個房間透過有線方式穩定連接多個設備。  
**技術挑戰**：在有限牆面空間內提供足夠的網口數，並兼顧電話接入與未來轉為資料用的彈性。  
**影響範圍**：若端口不足，將導致使用Wi‑Fi替代、增加延遲與不穩定，或臨時購置小Hub造成管理困難。  
**複雜度評級**：低

### Root Cause Analysis（根因分析）
**直接原因**：
1. 設備密度升高：每房至少2-3台需有線接入。  
2. 端口不足：原構想「一房一RJ45」不足以支援現況。  
3. 轉換成本高：日後加孔需重新施工。

**深層原因**：
- 架構層面：未以多端口為前提設計訊息面板。  
- 技術層面：未考慮RJ11與RJ45互轉與共用布線的做法。  
- 流程層面：需求盤點不足，未建立端口使用預測。

### Solution Design（解決方案設計）
**解決策略**：每房布設一組三條CAT5e（兩資料、一電話），電話線同樣採CAT5e並用RJ45模組端接，現況用RJ11接頭，未來必要時更換面板或直接以RJ45跳線轉為第三個資料孔。

**實施步驟**：
1. 端口規劃與面板選型
- 實作細節：選三孔資訊面板與模組，保留RJ45模組供未來替換。  
- 所需資源：3M模組化面板、AMP CAT5e。  
- 預估時間：2小時。

2. 電話/資料互轉設計
- 實作細節：以CAT5e布線，電話僅用中間對（4/5），RJ11→RJ45跳接。  
- 所需資源：RJ11轉RJ45跳線、標籤環。  
- 預估時間：1小時。

**關鍵程式碼/設定**：
```text
RJ11 on RJ45 (T568B) 對應
RJ11 Pin 3 -> RJ45 Pin 4 (Blue)
RJ11 Pin 4 -> RJ45 Pin 5 (Blue/White)
# 其他對保留，未來可轉資料用
```

實際案例：每房均預留三條CAT5e，兩資料孔即插即用，電話孔保留可轉資料。  
實作環境：3M面板、AMP CAT5e、RJ11/RJ45接頭。  
實測數據：
- 改善前：每房1孔，需加桌面小Hub；延遲+不穩定  
- 改善後：每房2-3孔直連；桌面Hub減少100%  
- 改善幅度：可同時接入設備數+200%

Learning Points（學習要點）
核心知識點：
- CAT5e通用化（資料/電話）  
- RJ11與RJ45對應  
- 面板模組化與預留策略

技能要求：
- 必備技能：端接、識別線對  
- 進階技能：面板/模組規劃與可維護性設計

延伸思考：
- 可應用於出租房標準化配置  
- 限制：牆面空間不足需改用雙層面板  
- 優化：改採6A規格以支援更長期需求

Practice Exercise（練習題）
- 基礎練習：完成一組3孔面板端接與標籤  
- 進階練習：設計電話孔轉資料孔的跳接方案  
- 專案練習：為兩房一廳設計多端口與轉換計畫

Assessment Criteria（評估標準）
- 功能完整性（40%）：兩資料一電話可用且可轉換  
- 程式碼品質（30%）：對應表/台帳清晰  
- 效能優化（20%）：資料孔千兆穩定  
- 創新性（10%）：面板與跳接的可擴充性


## Case #3: 機櫃集中化與設備整合

### Problem Statement（問題陳述）
**業務場景**：多種弱電系統（網路、電話、CATV、CCTV）需要集中管理，避免客廳或走道堆滿設備與線材。希望機櫃集中、門關起來「眼不見為淨」。  
**技術挑戰**：在15U機櫃內合理配置配線架、交換器、伺服器、PBX、CATV分配器/強波器，並確保通風散熱與供電安全。  
**影響範圍**：若配置不當，維護困難、過熱、噪音滋擾與安全風險提升。  
**複雜度評級**：中

### Root Cause Analysis（根因分析）
**直接原因**：
1. 設備分散：缺少集中管理造成線路雜亂。  
2. 空間限制：居家環境需要隱藏式安裝。  
3. 專業性需求：機櫃布局、配線與散熱需專業規劃。

**深層原因**：
- 架構層面：未建立統一的匯集點與標準化接口。  
- 技術層面：忽略散熱、深度、線槽、理線器選型。  
- 流程層面：未建立設備資產與接線台帳。

### Solution Design（解決方案設計）
**解決策略**：以15U機櫃作為弱電樞紐，標準化RU配置、前後門理線與風道，所有弱電線纜Home-Run匯入，設備與配線分層管理，建立資產與配線對照表。

**實施步驟**：
1. 機櫃配置與設備上架
- 實作細節：規劃RU分配（配線架高於交換器）、電源排插與UPS位置。  
- 所需資源：15U機櫃、理線器、電源管理排。  
- 預估時間：3-4小時。

2. 配線與標籤
- 實作細節：跳線統一長度、顏色分區（資料/電話/CATV），兩端標籤一致。  
- 所需資源：跳線、熱縮管、數字環。  
- 預估時間：3小時。

**關鍵程式碼/設定**：
```yaml
# 機櫃配置（Rack Manifest）
rack_units:
  14-15: Blank Panels (氣流導引)
  12: Patch Panel-24 (資料)
  11: Patch Panel-12 (電話)
  10: Switch-24p
  9: PBX-3CO-8EXT
  8: CATV-Splitter/Amp
  7: NVR/DVR
  6: 4U Server Chassis
  5: PDU with Surge Protection
  1-4: Cable Management & Slack
```

實際案例：所有非電力線集中進機櫃，配線架與交換器分層，PBX與CATV設備同櫃管理。  
實作環境：15U機櫃、24埠交換器、PBX、DVR、配線架、理線器。  
實測數據：
- 改善前：定位故障平均4小時  
- 改善後：定位故障平均30分鐘  
- 改善幅度：維護時間-87.5%

Learning Points（學習要點）
核心知識點：
- 機櫃RU配置原則與風道設計  
- 配線架與理線規範  
- 設備資產與配線台帳建立

技能要求：
- 必備技能：設備上架、基礎理線  
- 進階技能：熱管理、供電與防雷

延伸思考：
- 可擴展至小型辦公室MDF/IDF  
- 風險：雜物間散熱不足  
- 優化：加裝溫度感測與排風風扇

Practice Exercise（練習題）
- 基礎練習：設計一套15U機櫃RU配置圖  
- 進階練習：完成24埠配線架至交換器的理線與標籤  
- 專案練習：撰寫完整資產清單與配線對照表

Assessment Criteria（評估標準）
- 功能完整性（40%）：設備與配線分層完成  
- 程式碼品質（30%）：Manifest與台帳可讀性  
- 效能優化（20%）：熱/線管理良好  
- 創新性（10%）：空間與噪音控制方案


## Case #4: 24埠交換器容量與成本優化（二手設備導入）

### Problem Statement（問題陳述）
**業務場景**：家庭端口數提升，需一次提供足夠交換器埠數，同時控制預算。選購網咖退役的二手24埠交換器以降低成本。  
**技術挑戰**：在預算有限下確保穩定性與足夠吞吐，規劃埠位使用與未來擴充。  
**影響範圍**：選型不當影響整體網路效能與可用性。  
**複雜度評級**：低

### Root Cause Analysis（根因分析）
**直接原因**：
1. 端口需求高：每房2-3孔，端口消耗快。  
2. 預算壓力：新品成本高。  
3. 二手風險：設備壽命與穩定性未知。

**深層原因**：
- 架構層面：未分層規劃核心/邊緣交換。  
- 技術層面：未知是否支援千兆/管理功能。  
- 流程層面：缺少到貨檢測與壓力測試流程。

### Solution Design（解決方案設計）
**解決策略**：以二手24埠千兆交換器作為核心，制定埠位規劃與標籤，進行48小時壓力測試與風險備援（預留備品或升級計畫），建立採購驗收流程。

**實施步驟**：
1. 埠位規劃與標籤
- 實作細節：依房間/用途分區（NAS、DVR、PBX、IoT、LAN），保留冗餘10-20%。  
- 所需資源：標籤、台帳。  
- 預估時間：1小時。

2. 壓力測試與驗收
- 實作細節：使用iperf3持續測試、混合傳輸，觀察溫度與丟包。  
- 所需資源：測試電腦×2、iperf3。  
- 預估時間：48小時（掛測）。

**關鍵程式碼/設定**：
```bash
# 交換器吞吐壓測（雙向）
iperf3 -s   # 伺服器端
iperf3 -c <server-ip> -t 3600 -P 4 -R   # 客戶端，反向+多流，1小時
```

實際案例：以網咖退役24埠交換器上架，完成標籤與連接NAS/DVR/PBX等。  
實作環境：24埠二手交換器、兩台測試機。  
實測數據：
- 改善前：8埠家用交換器、端口不足  
- 改善後：24埠滿足現況+冗餘8埠；成本較新品省約70%  
- 改善幅度：端口+200%、成本-70%

Learning Points（學習要點）
核心知識點：
- 交換器容量規劃與埠位管理  
- 二手設備驗收與壓測方法  
- 風險備援與升級策略

技能要求：
- 必備技能：基本網路測試  
- 進階技能：壓力測試與長時間監測

延伸思考：
- 可應用於臨時展場或小辦公室

- 優化：逐步替換為可管交換器

Practice Exercise（練習題）

## Case #5: 小型PBX導入與內線化（RJ11/RJ45混用）

### Problem Statement（問題陳述）
**業務場景**：家中有3條外線、需8個內線分機，並希望使用普通電話機而非專用話機，同時整合於機櫃與布線系統。  
**技術挑戰**：以CAT5e布線實作電話系統，完成外線/內線映射，確保鈴流與語音品質，並預留轉換資料用的彈性。  
**影響範圍**：若佈線與PBX映射不清，會造成誤接、回聲、噪音與維護困難。  
**複雜度評級**：中

### Root Cause Analysis（根因分析）
**直接原因**：
1. 內線數量多：需清晰映射與標籤。  
2. 設備兼容性：普通話機與PBX需匹配。  
3. 維護需求：未來位置調整要方便。

**深層原因**：
- 架構層面：電話仍以總機為核心，需集中管理。  
- 技術層面：RJ11/RJ45端接混用易混亂。  
- 流程層面：缺少台帳與配線規矩。

### Solution Design（解決方案設計）
**解決策略**：PBX放置於機櫃，外線與內線均以CAT5e至配線架，使用短跳線完成PBX模塊到房間端口的映射，建立分機編號與端口對照表。

- 所需資源：採購清單。

```python
# 簡易線長預估器
runs = [18, 22, 25, 30, 15, 28]  # 單位: m（平面距離）

boxes = total / 305
print(f"Estimated total: {total:.1f}m, Boxes of 305m: {boxes:.2f}")
```

**實施步驟**：
1. 路徑與承托安裝
- 實作細節：每1.2-1.5m設J-Hook，轉角用大半徑導向。  
- 所需資源：J-Hook/橋架、魔鬼氈。  
- 預估時間：3-4小時。

2. 走線與驗收
- 實作細節：檢查曲率≥4x外徑，抽測信號完整性。  
- 所需資源：測線器。  
- 預估時間：2小時。

**關鍵程式碼/設定**：
```text
走線規範快訊
- J-Hook間距: 1.2~1.5m
- 填充率: ≤40%
- 最小彎曲半徑: ≥4x外徑（靜態），≥8x（拉線時）
- 束帶：以魔鬼氈代替塑膠束帶
```

實際案例：天花板內使用承托與鬆弛圈降低張力與彎折風險。  
實作環境：J-Hook/橋架、魔鬼氈、測線器。  
實測數據：
- 改善前：個別點掉速/不穩  
- 改善後：抽測端口穩定達1G，無誤碼  
- 改善幅度：穩定性明顯提升（定性）

Learning Points（學習要點）
核心知識點：
- 機械保護與走線規範  
- 填充率與曲率控制  
- 鬆弛圈與應力釋放

技能要求：
- 必備技能：走線施工  
- 進階技能：結構化佈線標準引用

延伸思考：
- 可用於商辦吊頂  
- 風險：過度擠壓與轉角  
- 優化：使用彎角模組與托盤

Practice Exercise（練習題）
- 基礎練習：在樣板區安裝J-Hook並佈線  
- 進階練習：規劃一段轉角的走線曲率  
- 專案練習：完成10條線的吊頂走線與驗收

Assessment Criteria（評估標準）
- 功能完整性（40%）：支撐與走線達標  
- 程式碼品質（30%）：規範文件  
- 效能優化（20%）：掉速/誤碼消除  
- 創新性（10%）：空間有限下的創意承托


## Case #17: 二手與雜牌材料的品質風險控管

### Problem Statement（問題陳述）
**業務場景**：某些耗材（如RJ45頭）價格低於市場行情，懷疑真偽與品質；同時導入二手設備以節省成本。  
**技術挑戰**：在成本與品質間取得平衡，制定來料檢驗與抽測方案，避免因小失大。  
**影響範圍**：品質不穩定造成高返工率與隱性故障。  
**複雜度評級**：中

### Root Cause Analysis（根因分析）
**直接原因**：
1. 低價耗材來源不明。  
2. 二手設備壽命未知。  
3. 無入場檢驗。

**深層原因**：
- 架構層面：無替代料與備援計畫。  
- 技術層面：缺少質檢流程。  
- 流程層面：缺購料與驗收制度。

### Solution Design（解決方案設計）
**解決策略**：建立來料驗收清單（外觀/尺寸/導通測試）、小批量先行、抽樣壓測；二手設備入場做連續運轉測試，保留備品或退換預案。

**實施步驟**：
1. 來料抽測
- 實作細節：每100顆RJ45頭抽測10顆端接與拉力。  
- 所需資源：端接工具、拉力表。  
- 預估時間：1-2小時。

2. 二手設備穩測
- 實作細節：48小時燒機+高流量壓測。  
- 所需資源：iperf3、監控工具。  
- 預估時間：48小時。

**關鍵程式碼/設定**：
```csv
# 來料檢驗記錄
Part,Lot,Qty,Sample,Pass,Fail,Notes
RJ45,LOT-2025A,100,10,9,1,腳位偏差1件
Switch-24p,SN-XYZ,1,N/A,Pass,0,48h壓測通過
```

實際案例：低價RJ45批中個別不良，透過抽測提前剔除。  
實作環境：端接工具、拉力計、測線器、iperf3。  
實測數據：
- 改善前：端接失效率偏高  
- 改善後：以抽測把失效風險降低至可控範圍（<2%）  
- 改善幅度：失效率顯著下降

Learning Points（學習要點）
核心知識點：
- 來料檢驗與抽樣統計  
- 燒機/壓測流程  
- 風險預案與備援

技能要求：
- 必備技能：基本檢驗  
- 進階技能：統計抽樣

延伸思考：
- 適用所有弱電耗材  
- 風險：抽樣仍有遺漏  
- 優化：與可靠供應商建立長期合作

Practice Exercise（練習題）
- 基礎練習：設計來料檢驗表  
- 進階練習：完成一批RJ45抽測  
- 專案練習：撰寫二手設備驗收SOP

Assessment Criteria（評估標準）
- 功能完整性（40%）：檢驗與驗收完成  
- 程式碼品質（30%）：紀錄規範  
- 效能優化（20%）：缺陷率下降  
- 創新性（10%）：風險預案


## Case #18: 交換器與NAS/印表機/影音設備的有線化與吞吐提升

### Problem Statement（問題陳述）
**業務場景**：家中含NAS、印表機、影音播放器、遊戲主機等，無線連接造成傳輸速率與穩定性不足，需要全面有線化。  
**技術挑戰**：規劃端口分配、線路路徑、避免迴路，確保NAS與多媒體串流的吞吐。  
**影響範圍**：備份/串流卡頓、印表延遲。  
**複雜度評級**：中

### Root Cause Analysis（根因分析）
**直接原因**：
1. Wi‑Fi吞吐不足與干擾。  
2. 端口不足或路徑不順。  
3. 網段混雜造成廣播干擾。

**深層原因**：
- 架構層面：未集中到交換器。  
- 技術層面：未做QoS/端口管理。  
- 流程層面：無服務優先級規劃。

### Solution Design（解決方案設計）
**解決策略**：將NAS/印表/影音/遊戲全部有線接入交換器，若可管則啟用VLAN/QoS，為NAS與串流保留高優先，建立端口規劃與標籤。

**實施步驟**：
1. 端口與佈線
- 實作細節：NAS靠近交換器、使用短跳線與Cat5e/6。  
- 所需資源：交換器、跳線。  
- 預估時間：1-2小時。

2. 性能驗證
- 實作細節：SMB拷貝測速、串流壓測（多串流）。  
- 所需資源：測試工具。  
- 預估時間：1小時。

**關鍵程式碼/設定**：
```bash
# SMB吞吐測試（Windows）
winsat disk -drive Z
# 或 NAS -> PC 實測複製，觀察速率 (MB/s)
```

實際案例：影音播放、遊戲主機有線化，NAS靠近交換器。  
實作環境：24埠交換器、NAS、印表機、影音設備。  
實測數據：
- 改善前：Wi‑Fi串流偶發緩衝，備份~30MB/s  
- 改善後：有線串流穩定，備份~110MB/s（千兆上限附近）  
- 改善幅度：備份吞吐+266%

Learning Points（學習要點）
核心知識點：
- 有線化帶來的穩定與吞吐  
- 端口規劃與QoS  
- NAS與多媒體最佳實務

技能要求：
- 必備技能：端口接入與標籤  
- 進階技能：QoS/VLAN

延伸思考：
- 可整合到前述VLAN方案  
- 風險：長距離千兆衰減  
- 優化：改用Cat6/短跳線、直連NAS 2.5G/Link Aggregation（若設備支援）

Practice Exercise（練習題）
- 基礎練習：為NAS/印表/影音安排端口  
- 進階練習：測試SMB吞吐並記錄  
- 專案練習：設計並驗證多串流播放方案

Assessment Criteria（評估標準）
- 功能完整性（40%）：設備有線化完成  
- 程式碼品質（30%）：測速與紀錄  
- 效能優化（20%）：吞吐達標  
- 創新性（10%）：QoS/Link Agg設計
