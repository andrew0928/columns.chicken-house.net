# Canon RAW Codec 1.3 Released！

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Canon RAW Codec？
- A簡: Canon 在 Windows 上提供的 RAW/CR2 影像解碼元件，讓系統與應用可顯示縮圖與預覽。
- A詳: Canon RAW Codec 是安裝於 Windows 的影像「解碼器」，用來讓作業系統與應用程式識別與解讀 Canon 相機的 RAW 格式（如 CR2）。其特點包含：在檔案總管顯示縮圖、在預覽窗格開圖、允許支援 WIC 的應用取用像素與中繼資料。常見應用場景包含：瀏覽器（Explorer）縮圖、相片管理工具快速檢視、WPF/.NET 應用載入 CR2 檔。對不支援原生 RAW 的系統而言，它是從「無法辨識」到「可用」的關鍵橋接元件。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q1, B-Q10

A-Q2: Canon RAW Codec 1.3 的更新重點是什麼？

- A詳: 1.3 版主要新增對 EOS Kiss X2 / EOS DIGITAL REBEL XSi / EOS 450D（文內標示含「2」）等新機型 RAW 的支援，並於說明中標註「正式」支援 Windows Vista SP1。根據文章說明，更新未提及效能優化，也未交代 x64 支援何時提供。作者實測 1.2 在 Vista SP1 亦可正常使用，顯示 1.3 在穩定性或相容性上可能包含小幅調整，但對既有使用者價值主要在新機型相容性。

A-Q3: 為什麼需要安裝 Canon RAW Codec？
- A簡: 讓 Windows 與應用能識別顯示 Canon RAW 檔縮圖、預覽與中繼資料。
- A詳: RAW 檔（如 CR2）是未經相機處理的感光資料，Windows 預設可能無法解析。安裝 Canon RAW Codec 後，系統的 WIC（Windows Imaging 架構）可透過該解碼器解析 RAW，帶來縮圖、預覽、讀取 EXIF/中繼資料的能力。對依賴檔案總管或 Pro Photo Tools 類工具進行整理的人，這是生產力關鍵；對使用 .NET/WPF 類應用的人，編解碼器則決定能否直接載入與處理 RAW。

- 學習階段: 基礎

A-Q4: 什麼是 CR2 檔案？
- A簡: Canon 相機的 RAW 檔案格式，保留感光器原始資料供後製用。
- A詳: CR2（Canon Raw Version 2）是 Canon 數位相機常見的 RAW 格式，記錄感光器原始訊號與相機拍攝時的設定（EXIF/中繼資料），未經相機機內 JPEG 處理。特點是保留更高動態範圍與色彩深度，適合專業後製；缺點是檔案較大、未必能被系統與應用原生開啟。安裝 Canon RAW Codec 後，Windows 與支援 WIC 的應用可顯示縮圖與讀取內容。

A-Q20: 「解碼器」與「編解碼器」有何差別？

### Q&A 類別 B: 技術原理類

B-Q1: Canon RAW Codec 在 Windows 中如何運作？
- A簡: 透過 WIC 註冊為解碼器，供 Explorer、.NET/WPF 與工具呼叫。
- A詳: 技術原理說明：Canon RAW Codec 以 WIC（Windows Imaging Component）編解碼器的形式註冊系統，提供 CR2 等 RAW 的解碼能力。關鍵流程：1) 安裝註冊 COM/WIC 元件；2) Explorer 或應用請求解析；3) WIC 依副檔名/MIME 交給 Canon 解碼器；4) 回傳像素與中繼資料。核心組件：WIC Runtime、Canon RAW 解碼 DLL、註冊表設定。此機制使多數應用共享同一套解碼能力。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, B-Q10, B-Q14
