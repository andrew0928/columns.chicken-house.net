## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Salling Clicker for Windows 3.5？
- A簡: 一套讓手機經由藍牙或 WiFi 遙控電腦的軟體，可控制 PowerPoint、iTunes、Windows Media Player，並把相關資訊回傳到手機顯示。
- A詳: Salling Clicker for Windows 3.5 是一款將手機變成電腦「雙向遙控器」的軟體。它支援藍牙與 WiFi 連線，能控制 PowerPoint、iTunes、Windows Media Player 等應用。不僅能傳送按鍵指令，如上一頁、下一頁，還能把投影片縮圖、標題、內文、播放曲目與專輯封面等資訊回傳到手機顯示。以簡報為例，使用者可在手機上查看備忘稿（Notes）、快速選頁（Select Slides），並設定簡報倒數計時，甚至結束後遠端關機。其核心價值在於提升簡報與多媒體操作的機動性與掌控力。

A-Q2: 為什麼要把手機當簡報器？
- A簡: 手機隨身可用，減少攜帶設備；且雙向顯示資訊與控制，增強講者流暢度與場控能力。
- A詳: 相較傳統 USB 簡報器僅有單向按鍵，手機作為簡報器的優勢在於「隨身性」與「雙向資訊」。講者可在舞台任一角落操作，同時在手機上檢視投影片縮圖、標題、內文與備忘稿，快速回應 Q&A 時的跳頁需求，並以倒數計時掌握節奏。這種整合避免頻繁走回電腦前或仰賴雙螢幕，讓演講更專注於內容與互動，特別適合重視臨場反應與流程控管的簡報場景。

A-Q3: 手機簡報器與傳統 USB 簡報器有何差異？
- A簡: USB 簡報器多為單向按鍵；手機簡報器提供雙向回饋，可看縮圖、備忘稿、快速選頁與倒數。
- A詳: 傳統 USB 簡報器通常提供上一頁、下一頁、雷射筆等單向控制，使用者仍需記憶內容或靠雙螢幕 Presenter View。手機簡報器（如 Salling Clicker）則透過藍牙或 WiFi 與電腦雙向通訊，把投影片縮圖、標題、內文、播放進度等資訊回傳到手機。講者可隨時偷看備忘稿、列出所有標題並跳轉，並使用倒數功能控時。雖無滑鼠的精準指向，但在簡報與多媒體場景，雙向回饋帶來顯著效率與掌控力提升。

- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q7, A-Q17

A-Q4: Salling Clicker 支援哪些連線方式？
- A簡: 支援藍牙與 WiFi 連線。藍牙易配對、近距離穩定；WiFi 覆蓋廣、穿牆較佳，端點需同網段。
- A詳: Salling Clicker 同時支援藍牙與 WiFi。藍牙優點是快速配對、點對點、免網路依賴，適合會議室與近距離演講；缺點是距離較短、易受干擾。WiFi 優點是範圍更大、穿牆較佳，適合大型場地或需更長距離移動的場合；但需確保手機與電腦在同一個網段，並通過防火牆允許。使用者可依場地與設備條件選擇連線方式，以取得更穩定、低延遲的控制體驗。

A-Q11: 藍牙與 WiFi 各適合哪些場景？
- A簡: 藍牙適小型會議與近距；WiFi 適大型場地與較遠距離移動，前提是同網段與網路穩定。
- A詳: 若場地較小且距離近（如會議室、教室），藍牙能迅速配對與穩定連線，操作延遲低且不依賴現場網路。若在大型場地或需較長距離移動（如演講廳、活動場），WiFi 的覆蓋更適合，但須確保手機與電腦在同一網段，並處理防火牆與 AP 壓力。選擇依據應包含距離、干擾、網路穩定度與便利性，以確保遙控流暢。

A-Q12: 為何它難以取代真正滑鼠？
- A簡: 手機遙控強於簡報與媒體控制，但不及滑鼠精準指向與細緻操作，適用面不同。
- A詳: 手機遙控聚焦在簡報與媒體的指令型操作，如頁面切換、選單選取、播放控制與資訊回顯。真正滑鼠提供像素級的指標控制，用於圖形編輯、精準點擊、拖拽等細緻操作更合適。因此，手機遙控是簡報器的升級與補強，而非滑鼠的完全替代。兩者定位不同，對應場景也不同，各自最佳發揮能提升整體效率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q7, C-Q10

### Q&A 類別 B: 技術原理類

B-Q1: Salling Clicker 的整體運作架構如何？
- A簡: 手機與電腦建立連線，手機送控制指令；電腦端控制應用並回傳狀態與內容，形成雙向通訊。
- A詳: 架構上分為手機端與電腦端兩部分。手機端負責顯示 UI、接收使用者操作、發送控制命令；電腦端負責與目標應用程式互動（如 PowerPoint、iTunes、WMP），並擷取相關狀態或內容（投影片縮圖、標題、曲目、專輯封面、進度）。兩端透過藍牙或 WiFi 建立持續的通訊通道，命令與資料雙向傳遞。這種設計把「控制」與「資訊回饋」結合，達成即時的遠端操作與確認回饋，形成完整閉環。

- 學習階段: 核心

B-Q2: PowerPoint 遙控的資訊回傳原理是什麼？

- A詳: 當手機選擇 PowerPoint 模組後，電腦端會持續監看 PowerPoint 的目前投影片索引，將該頁生成的小型預覽圖（縮圖）與文字資訊（標題、內文）封裝，透過既有連線傳給手機。每次切換頁面，電腦端更新內容並回傳，以維持手機畫面同步。此過程確保講者不必看投影幕即可掌握當前頁面要點，減少對場地視角與距離的依賴。

B-Q6: 遠端關機的流程與安全性考量是什麼？

- A詳: 症狀：配對失敗、已配對但不連、演講中斷線。可能原因：裝置不可見、未設受信任、距離過遠、2.4GHz 干擾、省電策略、藍牙驅動異常。解法：兩端設為可見並受信任；距離 5–10 公尺內；移除配對紀錄後重配；更新驅動；關閉藍牙省電與背景干擾；準備 USB 藍牙接收器備援。預防：會前測試 10 分鐘連線穩定；現場避開干擾源；有 WiFi 備線。

- 學習階段: 核心
- 關聯概念: C-Q2, B-Q7, D-Q7

D-Q2: WiFi 連不上或延遲高如何處理？
- A簡: 確保同網段、放行防火牆、換 5GHz、靠近 AP，或自建專用路由避免壅塞。
- A詳: 症狀：手機找不到電腦、已連線但卡頓。可能原因：非同網段、防火牆阻擋、網路壅塞、訊號弱。解法：確認同一子網；在防火牆允許 Salling Clicker；改連 5GHz AP；靠近路由器；高干擾環境自帶行動路由建立專用網段。預防：會前做延遲測試；關閉大型下載或雲端同步；保留網路頻寬。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q3, B-Q7, B-Q9

D-Q3: 手機不顯示投影片縮圖或清單怎麼辦？
- A簡: 先確定已載入簡報，重連一次；簡化檔案、清楚標題；演前預載縮圖避免首次延遲。
- A詳: 症狀：手機只見控制鍵無縮圖或清單空白。可能原因：簡報未正確載入、檔案過大或格式特殊、首次縮圖生成延遲。解法：先在電腦開啟目標簡報；重啟模組與連線；將檔案拆分或精簡；確保標題欄位填妥；演前打開整份簡報讓縮圖預生成。預防：避免超大多媒體內嵌；保持標題唯一且明確；提前彩排檢查同步。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q4, B-Q2, A-Q7

D-Q4: Notes 備忘稿在手機上顯示異常？
- A簡: 確認備忘稿有內容、避免長段落；重開 Notes 視圖；簡化格式，確保字體可讀。
- A詳: 症狀：Notes 空白或換行錯亂。可能原因：對應投影片無備忘稿、內容過長被截斷、含大量格式。解法：在 PowerPoint 填寫簡明備忘稿；控制長度；儘量不使用複雜格式；重開 Notes 視圖或重連；測試下一張是否正常。預防：以短句與關鍵詞撰寫；演前通篇檢查 Notes 顯示；建立簡報模板規範。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q5, B-Q3, C-Q10

D-Q5: 媒體控制無效或不回饋資訊？
- A簡: 確認播放器已啟動與曲庫可用；切換至正確模式；重啟播放器；檢查封面資料。
- A詳: 症狀：按播放無反應、曲名與封面不顯示。可能原因：播放器未啟動、非相符模式、媒體庫索引未建立、封面缺失。解法：先啟動 iTunes/WMP 並載入播放清單；在手機切到對應媒體模式；重啟播放器；檢查曲目中繼資料與封面。預防：演前建立播放清單並驗證顯示；避免多播放器同時開啟造成控制權競爭。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q9, B-Q8, A-Q16

D-Q6: 倒數計時不啟動或時間不準？
- A簡: 重新輸入時長並啟動；確保未誤觸暫停；以里程碑頁校對時間，必要時重設。
- A詳: 症狀：按開始無倒數、剩餘時間異常。可能原因：未確實啟動、誤觸暫停、畫面未更新。解法：重新輸入時長並確認開始；切換畫面再返回；重連一次檢查狀態。預防：演前測試倒數與結束提醒；在段落轉場時校對剩餘時間；必要時以場內時鐘或主持人時間做交叉確認。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q7, B-Q5, A-Q15

D-Q7: 操作延遲大或偶發卡頓怎麼優化？
- A簡: 縮短距離、避開干擾、關閉重載程式、預載縮圖；必要時改用另一種連線。
- A詳: 症狀：翻頁延遲、清單載入慢。可能原因：無線干擾或壅塞、背景程式吃資源、縮圖首次生成耗時。解法：靠近藍牙/WiFi AP；切 5GHz；關閉雲同步、掃毒與自動更新；演前打開簡報讓縮圖生成；切換藍牙與 WiFi 比較。預防：場地勘查無線品質；建立「演前關閉清單」；使用專用網與備援方案。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q9, C-Q10, A-Q11

D-Q8: 手機端顯示正常但 PowerPoint 不切頁？
- A簡: 檢查目標視窗焦點、解除鎖定、重啟模組；確認未被其他快捷鍵攔截。
- A詳: 症狀：手機顯示已操作，投影未切換。可能原因：PowerPoint 非前景視窗、簡報未播放模式、系統被鎖定、快捷鍵衝突。解法：將 PowerPoint 切到播放模式且置前；解除系統鎖定；重啟 PowerPoint 模組；關閉可能攔截快捷鍵的應用。預防：演前建立切換到播放模式的固定流程；最小化其他干擾應用。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q4, A-Q7, D-Q7

D-Q9: 遠端關機無效或遇到存檔提示？
- A簡: 檢查系統權限與電源設定；先保存並關閉檔案；必要時改用睡眠/休眠。
- A詳: 症狀：下達關機無反應、被未存檔提示阻擋。可能原因：權限不足、UAC/提示攔截、應用未關閉。解法：調整電源與關機權限；關閉或保存所有檔案；重試關機或改用睡眠/休眠。預防：演前設定允許應用關機；建立收尾清單：保存→關閉→關機；演後立即檢查是否已正確關閉。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: C-Q8, B-Q6, A-Q7

D-Q10: 手機電量消耗快如何應對？
- A簡: 降低螢幕亮度、關閉不必要無線、準備行動電源或備機，縮短高亮度顯示時間。
- A詳: 症狀：長時間簡報中手機快速掉電。可能原因：螢幕高亮、無線連線、持續資料傳輸。解法：將亮度調至可讀即可；關閉不必要的背景無線與同步；準備行動電源、短線快充或備機；在無操作時短暫鎖屏（避免斷線）。預防：演前充飽電；長場次安排中場充電；將 UI 調整為深色主題節能（若可）。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q10, A-Q2, B-Q10
