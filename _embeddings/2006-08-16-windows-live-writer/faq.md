# Windows Live Writer...

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Windows Live Writer（WLW）？
- A簡: 微軟推出的桌面部落格編輯器，離線撰寫、所見即所得，像寄信般一鍵發佈到網站。
- A詳: Windows Live Writer（WLW）是微軟於 2006 年推出的免費桌面部落格編輯器。它提供所見即所得編輯、離線撰寫與本機儲存，完成後像 e‑mail 一樣按下發佈即可上傳到部落格平台。相較於瀏覽器內的編輯器，WLW 操作順暢、快捷，可連線多平台（如 WordPress、Blogger），同步分類、標籤與圖片。核心價值在於降低摩擦、提升寫作與發佈效率。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q5, B-Q1, C-Q1

A-Q2: 什麼是桌面部落格編輯器？
- A簡: 在電腦上離線撰寫、編輯與管理文章，透過 API 發佈至部落格的應用程式。
- A詳: 桌面部落格編輯器是一種安裝於本機的應用程式，提供完整的編輯體驗（格式、圖片、連結），可離線撰寫與儲存草稿，並透過部落格 API（如 MetaWeblog、AtomPub）將內容發佈至遠端平台。它具備較佳的穩定性、鍵盤快捷與資源處理能力，特別適合不穩定網路環境或長篇內容創作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q5, B-Q1

A-Q3: 為什麼需要桌面編輯器而非網頁編輯器？
- A簡: 更穩定、可離線、編輯體驗佳，圖片與資源管理效率高，發佈更流暢。
- A詳: 網頁編輯器受瀏覽器效能與網路穩定度限制，長文、插圖、離線需求時常受影響。桌面編輯器提供：離線撰寫、存檔快取、鍵盤快捷、所見即所得、圖片壓縮與上傳管理、一鍵發佈等優勢。尤其在旅途或企業網路環境中，桌面工具能顯著提升連續性與寫作流暢度。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, B-Q1, C-Q9

A-Q4: WLW 的核心價值是什麼？
- A簡: 降低寫作與發佈摩擦，結合離線、所見即所得與一鍵發佈的流暢體驗。
- A詳: WLW 的核心價值在於「像 e‑mail 一樣」的工作流：先在本機專心寫作、排版與插圖，完成後一鍵發佈。它整合平台偵測、分類同步、圖片壓縮與上傳，並盡力呈現與網站一致的預覽。結果是更少分心、更快上稿、更穩定，讓部落格創作回到「寫作本身」而非對抗工具與網路。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, B-Q15, C-Q5

A-Q5: WLW 與 Web-based HTML 編輯器有何差異？
- A簡: WLW 本機運行、離線可用與穩定；Web 編輯器依賴瀏覽器與網路，彈性較低。
- A詳: WLW 具備本機儲存、離線撰寫、資源快取、圖片批次處理、佈景預覽與外掛擴充；Web 編輯器優勢在免安裝、隨處可用，但受瀏覽器與網路影響。對長文與大量媒體內容，WLW 的表現更穩；對臨時修改或跨裝置即時存取，Web 編輯器較方便。兩者可互補使用。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, A-Q19, B-Q8

A-Q6: 為何說 WLW 的寫作流程像 e‑mail？
- A簡: 先離線撰寫與存檔，再連線一鍵發佈，如同撰寫與寄送電子郵件。
- A詳: e‑mail 的典型流程是離線撰寫、草稿保存、連線寄送。WLW 亦然：你可在無網路時完成撰文、排版、插圖與預覽；待網路可用時一鍵發佈，系統自動登入、傳送內容與資源至伺服器。此心智模型降低學習成本，強化專注與穩定度，特別適合反覆修稿的長文寫作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, B-Q15, C-Q6

A-Q7: 直接發佈到網站有何好處與風險？
- A簡: 好處是快速與一致；風險含誤發佈、權限洩漏與格式差異需控管。
- A詳: 好處：一鍵上稿、減少步驟、保持格式一致、圖片自動上傳，縮短從完成到曝光的時間。風險：誤選目標部落格、錯誤分類、草稿誤發、公私帳密管理不當引發安全事件。建議：預設為草稿/排程、檢查預覽、啟用強密碼與應用程式專用密碼，搭配審稿與備份機制降低風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q1, D-Q2, B-Q1

A-Q8: 什麼是離線撰寫？
- A簡: 無網路時仍可完整編輯與儲存文章，稍後再連線發佈的能力。
- A詳: 離線撰寫指在無網路或網路不穩狀況下，仍可使用本機編輯器完成內容創作與排版，並以本機草稿形式保存。恢復連線後，系統會自動同步圖片、分類與內文至遠端部落格。其價值在於提升專注、不受網路干擾，並確保內容不因瀏覽器崩潰或連線中斷而流失。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q15, C-Q6

A-Q9: 誰適合使用 WLW？
- A簡: 長文創作者、圖文部落客、旅途中寫作的人與追求穩定流程的團隊。
- A詳: 若你經常撰寫長文、需大量插圖、在移動或網路不穩環境創作，或對於鍵盤快捷與穩定編輯體驗有要求，WLW 很合適。內容行銷、產品與技術寫作者、採訪記者、教育工作者也常受惠。團隊可將 WLW 做為審稿流程前端，利用草稿與排程配合 CMS 審核規範。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, C-Q9

A-Q10: WLW 與 CMS 內建編輯器比較？
- A簡: WLW 強於離線與編輯體驗；CMS 編輯器強於即時協作與雲端整合。
- A詳: WLW 提供穩定、快速的本機編輯與一鍵發佈，適合個人創作與離線情境。CMS 內建編輯器（如 Gutenberg）擁有即時預覽、雲端媒體庫、多人協作、擴充區塊等特性。若重視離線與流暢，選 WLW；若重視協作與雲端整合，選 CMS 編輯器。混用可兼顧效率與治理。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, A-Q19

A-Q11: 什麼是所見即所得（WYSIWYG）編輯？
- A簡: 編輯介面盡量呈現與發佈後一致的外觀，降低格式落差與預測成本。
- A詳: WYSIWYG 指在編輯時，即能看到接近最終發佈效果的版面與樣式。WLW 通常會下載佈景 CSS 與基本結構，在本機渲染近似效果，讓作者判斷段落、標題、圖文混排的視覺結果。雖難百分之百還原，但可大幅降低反覆試錯次數，提升排版效率與自信。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, A-Q19

A-Q12: WLW 支援哪些部落格平台？
- A簡: 支援多數主流平台，如 WordPress、Blogger、TypePad 等，透過標準 API 相容。
- A詳: WLW 透過部落格常見 API（MetaWeblog、AtomPub、Blogger API）與平台整合，涵蓋 WordPress、Blogger、TypePad 等。不同平台可能在分類、標籤、媒體上傳或認證上略有差異，WLW 以平台偵測與設定精靈降低配置成本。使用前建議確認目標平台 API 已啟用。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q22, C-Q2, C-Q3

A-Q13: 什麼是 MetaWeblog API？
- A簡: 以 XML-RPC 傳輸的部落格發佈標準，支援新文、編輯、抓取等操作。
- A詳: MetaWeblog API 是早期廣泛使用的部落格 API，透過 XML-RPC 進行遠端呼叫。常見方法包含 metaWeblog.newPost、editPost、getRecentPosts 等，能傳送標題、內容、分類與媒體資訊。WLW 及多數桌面編輯器均支援，特別適用於 WordPress 與相容平台的發文工作流。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q2, A-Q15

A-Q14: 什麼是 Atom Publishing Protocol（AtomPub）？
- A簡: 基於 HTTP 的資源導向發佈協定，用於建立、更新部落格資源。
- A詳: AtomPub 是 IETF 標準，採用 RESTful 風格，透過 HTTP 方法（POST、PUT、DELETE）對資源集合操作，傳輸格式常用 Atom XML。部分平台（如 Blogger 早期）使用 AtomPub。相較 MetaWeblog 的 RPC 風格，AtomPub 更貼近資源導向與 Web 標準，但實作細節依平台而異。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q2, A-Q12

A-Q15: 什麼是 XML-RPC？
- A簡: 一種以 XML 表示結構資料、透過 HTTP 傳輸的遠端程序呼叫協定。
- A詳: XML-RPC 將方法名稱與參數打包成 XML，經由 HTTP 送至伺服器，由伺服端執行後回傳結果。WordPress 與許多部落格平台以此實作 MetaWeblog API。其優點是簡單、跨語言；缺點是冗長與延展性有限。WLW 透過 XML-RPC 與平台互動以完成認證、發文與同步。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q13, B-Q2

A-Q16: WLW 如何處理分類與標籤？
- A簡: 透過 API 同步平台的分類與標籤，撰寫時可選取並一併發佈。
- A詳: 啟用帳號後，WLW 會呼叫平台 API 取得可用分類與標籤清單，供撰寫時指派。發佈時，系統將文章內容與所選分類/標籤一起送出，伺服器建立關聯。部分平台允許新建分類/標籤，另一些則需先於後台建立。同步失敗時可重載或手動輸入再更新。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q5, C-Q5, D-Q5

A-Q17: WLW 的圖片處理特性是什麼？
- A簡: 支援插入、壓縮、縮圖、外框與連結，並自動上傳圖片資源。
- A詳: WLW 內建圖片處理，包含大小調整、縮圖與連結大圖、陰影/邊框樣式、替代文字與對齊設定。發佈時自動上傳圖片至部落格媒體庫或指定 FTP，並更新文章內的圖片 URL。此流程降低手動壓縮與上傳成本，但需注意上傳權限與路徑設定正確。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q13, C-Q7, D-Q4

A-Q18: 草稿、發佈與排程有何差異？
- A簡: 草稿僅保存不公開；發佈立即公開；排程於指定時間自動公開。
- A詳: 草稿用於存放未完成或待審內容，僅作者可見。發佈代表文章立刻上線，對所有讀者可見。排程則設定未來時間自動公開，適合規律上稿或跨時區受眾。WLW 可選擇以草稿發佈至平台，再由後台審核與排程，建立更安全的上稿流程。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, C-Q5

A-Q19: 佈景預覽為何重要？
- A簡: 讓編輯時的排版接近實際網站，減少發佈後來回修正成本。
- A詳: 預覽會下載網站 CSS 與框架，在本機渲染接近實際的版面，讓作者在編輯時就能微調圖片尺寸、段落層級與內嵌元素位置。這有助於避免發佈後才發現跑版，尤其對圖文穿插與行動裝置觀感影響大。雖無法完全一致，但能顯著降低調整次數。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q8, B-Q19, C-Q5, D-Q6

A-Q20: WLW 有外掛擴充能力嗎？
- A簡: 有。可安裝外掛擴充編輯、發佈前處理與內容轉換等功能。
- A詳: WLW 提供外掛機制，允許開發者新增工具列按鈕、內容處理（如 Markdown 轉換）、媒體嵌入或發佈前自動流程（如加註版權、SEO 欄位）。使用者可依需求安裝外掛強化工作流。但外掛應審慎來源與相容性，避免影響穩定與安全。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q10, B-Q21, C-Q10

A-Q21: 安全性：WLW 如何保存帳密？應注意什麼？
- A簡: 以系統安全機制加密保存；建議用強密碼與平台專用應用程式密碼。
- A詳: WLW 會將帳密安全保存於系統憑證儲存（如 Windows DPAPI）。使用者應採強密碼、二階段驗證（若平台支援）、平台提供的應用程式專用密碼，並避免在共用電腦記住密碼。建議關閉自動立即發佈，預設以草稿上傳，降低誤發與帳號被誤用的風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q16, D-Q2

A-Q22: 桌面編輯器的限制與缺點有哪些？
- A簡: 安裝維護成本、跨裝置同步不便、協作弱、與新平台相容性風險。
- A詳: 桌面編輯器需安裝更新，跨裝置同步草稿較麻煩；多人即時協作較弱。平台 API 演進或安全政策變更，可能造成相容性風險。若使用過時外掛也會影響穩定。可透過雲端同步草稿資料夾、定期更新與以草稿上傳，來緩解上述問題並保留本機編輯優勢。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q6, D-Q7, C-Q8

A-Q23: 為何文章會說「用起來真舒服」？
- A簡: 因本機編輯流暢、離線可用、一鍵發佈，遠勝瀏覽器編輯摩擦感。
- A詳: 「舒服」來自低摩擦工作流：打字即回應、所見即所得、圖片拖放即上傳與壓縮、按下發佈即完成上稿。無需擔心瀏覽器頁籤遺失、連線中斷導致內容消失。這些體驗尤其在長文或多媒體文章中凸顯，讓作者專注內容而非工具本身。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q5, C-Q5

A-Q24: 桌面編輯器如何改變寫作流程？
- A簡: 從「邊上網邊編」轉為「專注離線創作，連線一鍵上稿」的節奏。
- A詳: 傳統在瀏覽器中容易分心、依賴網路。桌面編輯器讓流程變為：離線收斂資料與寫作、在本機整理媒體、檢查預覽、待連線時一鍵上稿。這種節奏降低干擾，將「創作」與「傳輸」分離，並以草稿/排程與審核補充治理需求，提升整體內容生產效率。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q9, A-Q6, A-Q18

A-Q25: 未來趨勢：離線與線上編輯如何平衡？
- A簡: 桌面離線的深度體驗，與雲端協作逐步融合，工具朝混合工作流演進。
- A詳: 未來的內容工具將融合 PWA 離線能力、雲端同步與多人協作；桌面編輯器會強化與雲端素材庫、審核流整合，同時保留本機效能與鍵盤優勢。對作者而言，選擇能在離線專注創作、上線快速協作與發佈的混合工作流，將是效率與治理的最佳平衡。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q22, B-Q15

### Q&A 類別 B: 技術原理類

B-Q1: WLW 發佈文章如何運作？
- A簡: 透過部落格 API 認證後，上傳內文與媒體，接收 postId 並更新本機狀態。
- A詳: 原理：使用 MetaWeblog/AtomPub 等 API，經 HTTP 傳送結構化文章與媒體資訊。流程：1) 平台偵測與設定 2) 認證 3) 抓取分類等字典 4) 編輯與預覽 5) 上傳圖片取得 URL 6) 送出文章內容 7) 伺服器回傳 postId 8) 本機標記同步。核心組件：編輯器、網路傳輸層、API 介接模組、媒體管理與狀態同步。

C-Q4: 如何設定圖片上傳方式與路徑？
- A簡: 優先使用平台媒體庫 API，上傳失敗時改設 FTP 並指定路徑。
- A詳: 步驟：1) 在帳號設定中選圖片上傳 2) 選「使用部落格提供者上傳」3) 測試上傳 4) 若失敗，改用 FTP：輸入 FTP 主機、帳密、網站對應 URL 與目錄 5) 套用。注意：媒體庫權限、磁碟配額與 CDN 影響。最佳實踐：啟用壓縮、合理尺寸，減少流量。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q13, D-Q4

C-Q5: 如何建立文章、預覽佈景並發佈？
- A簡: 新建文章→編輯排版→預覽佈景→選分類標籤→發佈或以草稿上傳。
- A詳: 步驟：1) 新增文章，輸入標題與內容 2) 插入圖片並設定對齊與尺寸 3) 點預覽載入佈景樣式 4) 選取分類/標籤 5) 點發佈（或發佈為草稿/排程）。注意：先預覽再發佈、連結測試、ALT 文字完整。最佳實踐：預設改為「發佈為草稿」避免誤上線。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q8, A-Q19, A-Q18

C-Q6: 如何進行離線撰寫並稍後發佈？
- A簡: 斷網或飛航下寫作存為本機草稿，恢復連線再一鍵發佈。
- A詳: 步驟：1) 開啟編輯器離線撰寫 2) 定時存檔為本機草稿 3) 加入圖片（暫存本機）4) 恢復網路後，先上傳圖片、再發佈 5) 檢查回應與 postId。注意：圖片來自本機時記得保留原始檔。最佳實踐：建立「離線待發」標籤，統一管理待發佇列。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q15, A-Q8

C-Q7: 如何插入圖片並自動產生縮圖與大圖連結？
- A簡: 透過插入圖片工具選尺寸與連結，發佈時自動上傳與替換 URL。
- A詳: 步驟：1) 插入圖片 2) 選擇顯示尺寸（如寬 800px）與品質 3) 勾選「縮圖連結至原圖」4) 設定 ALT 與對齊 5) 發佈。關鍵設定：品質 70–85、寬度依佈景內容寬。注意：避免過大檔案，行動裝置要友善。最佳實踐：使用 WebP（若平台支援）。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, A-Q17, D-Q4

C-Q8: 如何備份本機草稿與設定？
- A簡: 匯出或複製使用者資料夾中的草稿檔與設定檔，定期備份至雲端。
- A詳: 步驟：1) 找到編輯器的資料儲存路徑（使用者資料夾/AppData）2) 關閉程式 3) 複製草稿檔與設定檔至安全位置 4) 以版本化雲端同步 5) 定期演練還原。注意：勿備份明文憑證；必要時僅備份草稿內容。最佳實踐：同步到加密磁碟或零知識雲端。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q6, A-Q22, D-Q8

C-Q9: 如何從瀏覽器編輯器遷移到桌面工作流？
- A簡: 以桌面工具撰寫與草稿上傳，平台後台進行審核與排程的混合流程。
- A詳: 步驟：1) 定義新流程：桌面撰寫→草稿上傳→後台審核→排程 2) 建立分類與標籤規範 3) 設定預覽與圖片策略 4) 培訓快捷鍵與模板 5) 觀察效率指標。注意：避免同一篇同時在兩端修改。最佳實踐：以 postId 為單一真相，規定更新路徑。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q24, B-Q11, D-Q7

C-Q10: 如何安裝外掛擴充（如 Markdown 支援）？
- A簡: 下載相容外掛，放置於外掛資料夾並在編輯器啟用與測試。
- A詳: 步驟：1) 取得可信來源外掛 2) 關閉編輯器 3) 將外掛檔放入指定資料夾 4) 重開程式並在外掛管理啟用 5) 測試轉換與發佈。注意：外掛相容性與安全來源。最佳實踐：先測試在草稿站，版本管控外掛以便回退。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q10, B-Q21, A-Q20

### Q&A 類別 D: 問題解決類（10題）

D-Q1: 連線失敗或找不到端點怎麼辦？
- A簡: 檢查 URL 與端點、HTTPS 與防火牆；必要時手動指定 xmlrpc.php。
- A詳: 症狀：新增帳號時無法偵測平台或發佈失敗。原因：URL 輸入錯、XML-RPC 被關閉/阻擋、HTTP→HTTPS 重導或防火牆攔截。解法：1) 直接指定 https://站點/xmlrpc.php 2) 檢查主機與 WAF/防火牆白名單 3) 檢查憑證 4) 以草稿測試。預防：使用固定端點、記錄網路設定與代理例外。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q14, B-Q20, C-Q2

D-Q2: 驗證失敗或權限不足如何處理？
- A簡: 確認帳密、應用程式密碼與角色權限，並啟用 HTTPS。
- A詳: 症狀：401/403、提示密碼錯誤或無權發佈。原因：密碼錯、未開 API、使用者角色無發文權、未使用應用程式密碼。解法：1) 重設密碼或建立應用程式密碼 2) 檢查角色權限 3) 確保 HTTPS 4) 清除憑證快取重試。預防：最小權限原則、二步驟驗證與密碼策略。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q16, A-Q21

D-Q3: 發佈後出現亂碼怎麼辦？
- A簡: 檢查雙方編碼為 UTF‑8，資料庫與頁面宣告一致，避免混用。
- A詳: 症狀：中文顯示為問號或方塊。原因：編輯器、伺服器或資料庫編碼不一致（UTF‑8 vs Latin1）、頁面 meta 宣告錯誤。解法：1) 確認站點與資料庫皆為 UTF‑8 2) 修正頁面編碼宣告 3) 重新發佈測試。預防：新站一開始即統一 UTF‑8，禁用舊編碼插件。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q12, B-Q23

D-Q4: 圖片無法上傳或連結失效？
- A簡: 檢查媒體權限、磁碟空間與上傳端點，必要時改用 FTP。
- A詳: 症狀：圖片插入後發佈失敗或顯示破圖。原因：媒體庫權限不足、目錄不可寫、檔案超過限制、CDN/防火牆阻擋。解法：1) 檢查主機目錄權限與配額 2) 調整圖片大小與格式 3) 測試 API 上傳；失敗改 FTP 4) 清除快取。預防：設定合理壓縮、定期清理媒體庫。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q13, C-Q4

D-Q5: 無法載入分類或標籤？
- A簡: 檢查 API 權限與方法支援，重新同步或先於後台建立。
- A詳: 症狀：分類/標籤清單為空或載入失敗。原因：API 未支援標籤、權限不足、快取損壞。解法：1) 後台建立至少一個分類/標籤 2) 重新整理或重建帳號設定 3) 檢查使用者角色權限。預防：確保平台啟用分類/標籤並測試 API 方法可用。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q5, C-Q5

D-Q6: 預覽版面與網站不一致怎麼辦？
- A簡: 允許差異，檢查佈景 CSS 載入，改以瀏覽器最終預覽確認。
- A詳: 症狀：本機預覽與實際文章排版不同。原因：外掛腳本、動態 CSS、授權字體或跨網域資源無法載入。解法：1) 在瀏覽器預覽最終頁 2) 簡化內容依樣式調整 3) 於編輯器更新佈景快取 4) 測試移除衝突外掛。預防：建立內容樣式指南，避免依賴複雜腳本。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q8, B-Q19, A-Q19

D-Q7: 出現重複發佈或版本衝突？
- A簡: 以 postId 為準則，先拉回遠端版本合併，再進行更新發佈。
- A詳: 症狀：同一內容多篇、更新覆蓋。原因：脫離 postId 綁定另存新文、多人同時編輯。解法：1) 拉回遠端最新版本 2) 合併本機更動 3) 以更新而非新發佈 4) 建立團隊規範。預防：單一路徑維護、明確標示文章擁有者與狀態。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, B-Q11, C-Q9

D-Q8: 程式當機或草稿遺失如何處理？
- A簡: 啟用自動儲存與定期備份，從備份資料夾還原草稿。
- A詳: 症狀：當機後遺失未儲存內容。原因：自動儲存關閉、硬碟或外掛不穩。解法：1) 啟用自動儲存與定時快照 2) 從資料夾還原草稿 3) 移除不穩外掛 4) 更新程式。預防：版本化備份草稿與設定、避免在低電量或不穩環境長時間未存檔。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q8

D-Q9: HTML 被清洗導致格式跑掉？
- A簡: 避免危險標籤，改用平台允許的語法或短代碼，保持相容。
- A詳: 症狀：發佈後某些樣式/腳本消失。原因：伺服端安全清洗移除不允許標籤/屬性。解法：1) 以平台短代碼或區塊替代 2) 簡化 HTML、移除內嵌腳本 3) 檢查角色權限（部分角色過濾更嚴）。預防：遵循內容樣式指南，使用白名單內元素。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q9, A-Q11

D-Q10: 代理或公司網路導致無法發佈？
- A簡: 設定系統代理、申請白名單與憑證信任，必要時改用家用網路。
- A詳: 症狀：公司內網無法連線發佈。原因：代理驗證、阻擋 xmlrpc.php、SSL 檢查。解法：1) 編輯器設為使用系統代理 2) 提供端點與埠號申請白名單 3) 安裝受信憑證 4) 測試外網。預防：文件化網路需求、與 IT 預先協調佈建。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q17, D-Q1
