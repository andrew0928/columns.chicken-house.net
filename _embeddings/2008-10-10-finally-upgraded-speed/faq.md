# 終於升速了!

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 什麼是 ADSL？
- A簡: ADSL 是以銅雙絞線提供的非對稱數據接取，下載速度高於上傳，適合一般家用上網與影音串流。
- A詳: ADSL（Asymmetric Digital Subscriber Line）是一種利用既有電話銅線提供的固定寬頻技術。其特性是下載（下行）速率高於上傳（上行）速率，符合大多數使用情境的下載取向需求。ADSL 以DMT調變將頻譜切分為多個子載波，並與語音頻段分離，能同時通話與上網。常見速率如2M/256K、8M/640K等，受線路距離、雜訊與設備設定影響。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, B-Q1

Q2: 下載速率與上傳速率是什麼？有何差異？
- A簡: 下載速率是從網路取資料的速度；上傳速率是傳資料到網路的速度，兩者常不對稱。
- A詳: 下載速率（Downlink）指終端從網際網路接收資料的能力，上傳速率（Uplink）則是將資料傳出到網際網路的能力。ADSL 等家用接取常採不對稱設計，以滿足瀏覽、串流等下載導向需求。上傳對雲端備份、影音通話、遠端桌面、發佈內容影響關鍵；若上傳不足，延遲與壅塞會顯著上升，體感會劣化。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q17, D-Q4

Q3: 2M/256K 與 8M/640K 有何差異？體驗會如何改變？
- A簡: 下載增至約4倍、上傳約2.5倍；網頁與影片更順，上傳作業與視訊會議改善最明顯。
- A詳: 從2M/256K 升至8M/640K，名目下載帶寬增加4倍，上傳增加約2.5倍。實際體驗會受線路品質、尖峰壅塞與Wi‑Fi影響，但一般情況下：網頁載入更快、HD 串流更穩；雲端備份、檔案分享、視訊會議、線上遊戲語音的上傳瓶頸顯著緩解。延遲不一定改善，但排隊延遲在上傳滿載時會降低。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q13, C-Q6, D-Q1

Q4: 什麼是 FTTB？
- A簡: FTTB 是光纖到大樓配線室，再以銅線或以太網拉到住戶的接取架構。
- A詳: FTTB（Fiber To The Building）指光纖只拉到大樓機房或弱電間，樓內最後一段「最後一哩」多以乙太網、VDSL 或同軸延伸到各戶。其優點是建置成本較低、升級較快；缺點是樓內銅線品質會限制速率與穩定度。若改為 FTTH（到戶），用戶可享更高對稱速率與低延遲，但施工成本與協調門檻更高。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, B-Q8, C-Q8

Q5: FTTB、FTTH、FTTC 有何差異？
- A簡: FTTB到大樓、FTTH到用戶家、FTTC到路邊機櫃；到戶最優但施工成本最高。
- A詳: FTTC（Fiber To The Curb）光纖到路邊機櫃，後段以銅線延伸，距離較長；FTTB 到大樓機房，後段樓內佈線較短；FTTH（Fiber To The Home）光纖直達住戶終端。FTTH 在速率、對稱性與延遲最佳，FTTB 次之，FTTC 受銅線長度限制較多。選擇取決於成本、建物配線與ISP方案可用性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q8, A-Q22

Q6: 為什麼需要升速？
- A簡: 升速可縮短下載時間、緩解上傳瓶頸，改善雲端、會議、串流等日常體驗。
- A詳: 升速的核心動機包括：縮短大檔下載與更新時間、提升多裝置同時上網的順暢度、避免上傳滿載導致延遲飆升、改善視訊會議與即時互動品質。對創作者與在家工作者，上傳速率尤為關鍵。升速亦可為未來服務（4K/8K、雲端遊戲、備份同步）預留空間，但應同步檢查內網與Wi‑Fi是否成為新瓶頸。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q17, C-Q6, D-Q4

Q7: 升速的核心價值是什麼？
- A簡: 將等待時間轉為流暢體驗，並降低滿載排隊延遲，提升多工與穩定性。
- A詳: 升速不僅提升峰值吞吐，更減少在突發或持續負載時的佇列延遲（queueing delay）。當上傳、下載同時進行時，較高帶寬能縮短資料排隊時間，使瀏覽、中斷重連、雲端同步與視訊會議更穩定。同時也提高家中多裝置並行的承載能力，避免互相爭用造成卡頓，是整體體驗的複合收益。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q13, D-Q4, B-Q11

Q8: ADSL 與 VDSL 有何差異？
- A簡: VDSL 使用較高頻寬與較短銅線，能提供更高上下行速率與較低延遲。
- A詳: ADSL 以較低頻段與長距離為設計，速率受線長與雜訊影響顯著；VDSL/VDSL2 使用更高頻寬與進階調變，適合短距離銅線，常見於FTTC/FTTB 後段，能達數十至上百Mbps，且上行較有彈性。若住戶距離節點近、線路品質佳，VDSL 是良好過渡方案，直到 FTTH 普及。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, A-Q5, B-Q8

Q9: ADSL 與 Cable（同軸）有何差異？
- A簡: ADSL 為銅線點對點；Cable 共享同軸頻道，速率高但尖峰共享度較敏感。
- A詳: ADSL 透過銅雙絞線連至DSLAM，較接近點對點；Cable 以同軸與DOCSIS規範，在同一節點共享下行與上行頻寬。Cable 通常下載速率較高，上傳也逐步提升，但尖峰時段共享度高會影響體驗。ADSL 的距離與線質限制較大。兩者皆受ISP匯聚與骨幹擁塞影響。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q9, A-Q18, B-Q17

Q10: 什麼是「最後一哩」？
- A簡: 從營運商節點到用戶家間的末端接取路段，決定實際速率與穩定性。
- A詳: 最後一哩（Last Mile）指ISP接取網路到用戶終端的最後段路徑，如銅線、同軸或光纖。其介質、長度與品質直接影響可提供的速率、延遲與穩定性。FTTB/FTTH 即是改善最後一哩的升級策略。當升速體感不如預期，最後一哩的線質和佈線常是優先檢查的關鍵。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q5, D-Q8

Q11: 什麼是 DSLAM？
- A簡: DSLAM 是電信機房端匯聚多條DSL用戶線路的接取設備，配置速率檔位。
- A詳: DSLAM（Digital Subscriber Line Access Multiplexer）位於電信局端或節點機房，負責終結用戶銅線、分配頻譜、設定速率檔位（Profile），並上聯到BRAS/BNG。升速通常涉及DSLAM 參數調整，如目標SNR、最大可同步速率與Interleaving。其能力與設定直接影響線路穩定與實際可達速率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q1, D-Q6

Q12: 什麼是 PPPoE？
- A簡: PPPoE 是透過乙太網封裝PPP的撥接協定，用於寬頻認證與計費管理。
- A詳: PPPoE（Point-to-Point Protocol over Ethernet）將PPP 認證、協商與封包管理功能封裝在乙太網上，常用於ADSL、FTTx 等固定寬頻。用戶端透過帳密進行認證，建立虛擬連線，ISP 可據此分派IP、套用QoS與計費。PPPoE 會有8位元組額外負載，影響MTU/MSS，需要正確設定避免分片。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q16, D-Q2

Q13: 帶寬、吞吐量、延遲有何差異？
- A簡: 帶寬是可用容量；吞吐量是實際達成速率；延遲是往返時間，三者各自獨立。
- A詳: 帶寬（Bandwidth）是鏈路可承載的最大容量上限；吞吐量（Throughput）是應用實際達成的傳輸速率，受協定、壅塞、封包損失影響；延遲（Latency/RTT）是封包往返所需時間。升速提高帶寬與潛在吞吐，但延遲不一定下降。若上傳滿載，排隊延遲上升會讓體感變差，需配合QoS或升級上行。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q11, D-Q4, C-Q5

Q14: 什麼是 SNR 邊際（噪聲容限）？
- A簡: SNR 邊際是訊號與雜訊的安全裕度，越高越穩，能支持更高同步速率。
- A詳: SNR Margin 指實際訊雜比相對於最低要求的餘裕。較高的SNR 邊際意味線路有較多安全空間，容許更高的比特載入與速率。當升速提高調變密度時，SNR 邊際會下降；若過低易掉線。典型穩定值約6–12 dB 以上，視設備策略與噪聲環境而定。維護良好內線與縮短銅線距離可提升SNR。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q4, D-Q6, D-Q8

Q15: 什麼是線路衰減（Attenuation）？
- A簡: 衰減是訊號沿線傳輸的損耗，距離越長、線徑越細數值越高且速率受限。
- A詳: 線路衰減以dB 表示，反映訊號在銅線中逐距離遞減的程度。較高的衰減意味可用子載波減少、SNR 下降，限制可達同步速率並增加錯誤率。影響因素含線長、線徑、接頭與氧化。若衰減偏高，建議簡化家中佈線、檢修接頭、改接主插座或申請節點近移（FTTC/FTTB）。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q1, B-Q4, D-Q8

Q16: 同步速率與實際速度有何差異？
- A簡: 同步速率是物理層連線上限；實際速度受協定開銷、壅塞與Wi‑Fi影響。
- A詳: Modem 顯示的同步速率是鏈路層能力上限；應用層可見的速度會扣除協定開銷（PPPoE、TCP/IP）、加上ISP匯聚、遠端伺服器限制、Wi‑Fi干擾等因素。測速時多連線並連至近端節點較能逼近實際吞吐上限。升速後若同步提高但實測沒變，應檢查內外部瓶頸。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, C-Q6, D-Q1

Q17: 為什麼上傳速度會顯著影響體感？
- A簡: 上傳滿載會造成資料排隊，導致延遲飆升，影響網頁、語音與遊戲互動性。
- A詳: 當上傳帶寬被雲端備份或檔案分享占滿，ACK 與控制封包被延後，TCP 擁塞控制效能下降，RTT 與抖動上升，瀏覽與視訊會議體感急遽惡化。增加上傳帶寬或在路由器啟用智慧QoS/限速，能維持控制流量優先，顯著改善互動體驗。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q11, C-Q5, D-Q4

Q18: 什麼是匯聚比（Oversubscription Ratio）？
- A簡: 匯聚比是上游資源與用戶帶寬的配置比例，過高會在尖峰造成壅塞。
- A詳: ISP 在接取與匯聚層會將多用戶的帶寬疊加到有限上游頻寬以提升成本效益，這稱為匯聚。合理匯聚可兼顧體驗與成本；若匯聚比過高，尖峰時段吞吐下降、延遲上升。升速後仍受匯聚與骨幹擁塞影響，選擇用戶數較低的節點或商務級方案可改善。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q17, D-Q5, A-Q9

Q19: 什麼是流量整形與公平使用政策（FUP）？
- A簡: 流量整形與FUP 透過限速與優先級維護網路公平，可能影響特定應用。
- A詳: 流量整形（Traffic Shaping）控制特定類型流量之速率與優先級；FUP（Fair Usage Policy）在達到用量門檻後採臨時限速以維持整體品質。這些策略在尖峰時段尤為常見。若觀察到特定應用速率受限，需檢視方案條款或向ISP 查詢是否有分時、分流量政策。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q14, D-Q5, A-Q13

Q20: NAT 與公網 IP 有何差異？
- A簡: NAT 共用一個公網IP 給多設備；公網IP 直接對外，應用相容性最佳。
- A詳: NAT（Network Address Translation）將私有位址映射至公網IP，節省位址並提升安全性，但對P2P、伺服器與某些遊戲連線可能受限。公網IP（動態或固定）可直接被外部連入，適合自架服務。PPPoE 撥接後取得的位址型態影響外部可達性與連線品質。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q15, C-Q3, D-Q2

Q21: 家中 Wi‑Fi 與固定寬頻速率的關係是什麼？
- A簡: 固網升速若Wi‑Fi 規格或干擾不足，實測仍受無線連線成為新瓶頸。
- A詳: 固網側帶寬提升後，終端透過Wi‑Fi 連線的速率受制於標準（如802.11n/ac/ax）、頻段（2.4/5GHz）、通道擁擠與路由器性能。若無線吞吐低於寬頻帶寬，體感差距有限。以有線連接測速驗證，必要時升級AP、啟用5GHz、調整信道與位置，才能發揮升速效益。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q6, D-Q1, D-Q10

Q22: 光纖覆蓋率代表什麼？為何不等於可用？
- A簡: 覆蓋率是區域布建程度統計；實際可用受建物內線與施工可行性限制。
- A詳: 市政或電信商公告的光纖覆蓋率通常以路段或建物門牌可達為統計口徑，未必保証每戶能立即開通。實際可用性仍受大樓管委、弱電間容量、垂直管道、內線品質與施工同意影響。查詢時需以住戶位址進行可服務性確認，必要時推進大樓公用設施改善。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, C-Q8, B-Q18

Q23: 升速是否一定會降低延遲（Ping）？
- A簡: 不一定。延遲主要受路徑距離、路由與排隊影響，非僅帶寬大小。
- A詳: 延遲包含傳輸、佇列與處理時間。升速提升容量可降低在滿載時的佇列延遲，但空載下對純傳輸與路由延遲影響有限。若路徑繞遠、ISP 擁塞或遠端伺服器距離長，延遲不會因升速改變。透過QoS、避免上傳飽和、選擇更近節點才能實質降低體感延遲。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q13, D-Q4, B-Q17

Q24: Modem 與路由器有何差異？
- A簡: Modem 終結實體線路並同步；路由器分配網路、NAT 與Wi‑Fi 連線管理。
- A詳: DSL/光纖 Modem 負責與ISP 側設備同步、調變與連線；路由器負責IP 路由、NAT、防火牆、DHCP 與無線接入。有些設備為一體機，亦可橋接由獨立路由器管理。升速時需確認兩端裝置均支援對應速率與處理能力，避免路由器CPU 或NAT 成為瓶頸。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q4, D-Q1, B-Q10

Q25: 什麼是上行瓶頸？常見緩解方式有哪些？
- A簡: 上傳帶寬不足導致延遲與卡頓；可靠升級上行或設定QoS分流改善。
- A詳: 上行瓶頸指上傳流量接近或超過可用帶寬，使控制與互動流量被延後，體感惡化。緩解方法包括：升級方案提高上行、路由器啟用智慧QoS或SQM（如CAKE、FQ‑CoDel）、限制備份與P2P速率、分時傳輸。若常態雲端工作，應選擇上行更高或對稱的接取技術。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q5, D-Q4, B-Q11

### Q&A 類別 B: 技術原理類

Q1: ADSL 的運作原理是什麼？
- A簡: 以DMT 將頻譜切為多子載波，與語音分離，依SNR動態分配比特。
- A詳: 原理說明：ADSL 使用DMT（離散多音調）調變，將銅線頻譜切成數百個子載波，分別載送數個比特。透過分離器將語音低頻與數據高頻分開。關鍵流程：同步握手、量測SNR、比特加載、糾錯與交錯。核心組件：用戶端Modem、分離器、局端DSLAM、上聯BRAS/BNG。線質與距離決定可用子載波與速率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q14, A-Q15

Q2: PPPoE 認證與連線建立流程如何運作？
- A簡: 透過PADI/PADO/PADR/PADS 探詢與認證，建立PPP 會話並獲得IP。
- A詳: 原理說明：PPPoE 在乙太網上封裝PPP。流程：用戶送PADI 掃描接入點，伺服器回PADO；用戶以PADR 請求會話，伺服器回PADS 確認。其後透過PPP LCP 協商、PAP/CHAP 認證、IPCP 分配IP。核心組件：接取設備（BRAS/BNG）、認證伺服器、CPE 路由器/Modem。MTU 默認1492，需調整MSS 避免分片。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, B-Q16, D-Q2

Q3: 升速申請後，ISP 後端如何調整線路？
- A簡: 調整DSLAM Profile、目標SNR、上下行上限，並進行測試與開通。
- A詳: 原理說明：營運端在DSLAM 變更用戶線路Profile，包括最大同步速率、SNR 目標、交錯深度與上行/下行功率。流程：工單建立、可行性查核、設定變更、遠端測試、通知用戶重連。核心組件：DSLAM、OSS/BSS 系統、派工平台。部分升速需現場檢修或更換CPE。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q11, C-Q1, D-Q1

Q4: SNR 與衰減如何共同影響可達速率？
- A簡: 衰減限制子載波可用性；SNR 決定比特載入，兩者共同決定同步上限。
- A詳: 原理說明：每個子載波的比特載入取決於其SNR，衰減提高會降低高頻SNR，使可用子載波減少。流程：Modem 訓練期量測各子載波SNR，分配比特，設定誤碼校正。核心組件：CPE Modem、DSLAM、DMT 子載波處理。實務上若SNR Margin 低於門檻，需降速或改善線路。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q14, A-Q15, D-Q6

Q5: Interleaving 與 FastPath 有何機制差異？
- A簡: Interleaving 增強糾錯抗干擾但增加延遲；FastPath 低延遲但較敏感。
- A詳: 原理說明：Interleaving 將資料跨時間片分散，配合Reed‑Solomon FEC 抵抗突發雜訊，代價是毫秒級延遲增加。FastPath 取消交錯降低延遲，但容錯降低。流程：DSLAM 設定線路Profile 選擇模式，根據線質自動或人工調整。核心組件：DSLAM FEC 模組、CPE 解碼器。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q13, D-Q3, D-Q6

Q6: DSL DMT 子載波與比特加載如何運作？
- A簡: 依各子載波SNR 指派比特數，雜訊高者載入少，整體達成最佳吞吐。
- A詳: 原理說明：DMT 將頻譜劃分至多個4.3125 kHz 子載波；在訓練階段量測每一子載波SNR，根據水填充（waterfilling）策略分配比特。流程：訓練、量測、分配、錯誤回報與再訓練。核心組件：數位訊號處理器（DSP）、調變/解調模組、回報通道（bit swap）。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q1, B-Q4, A-Q16

Q7: 上下行頻譜如何分配？
- A簡: 以分頻將低頻給上行與語音，高頻給下行，視制式定義頻帶。
- A詳: 原理說明：ADSL 採用頻分多工（FDM），語音占低頻0–4 kHz，上行佔次低頻，多個上行與下行頻帶按制式配置。流程：分離器濾波、頻帶指派、調變。核心組件：分離器、濾波器、DSLAM 頻譜管理。不同制式（ADSL2/2+）擴展下行頻帶以提升下載速率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q2, B-Q6

Q8: FTTB 架構的核心組件與資料路徑為何？
- A簡: 光纖至大樓機房的ONU/MDU，再以乙太/VDSL/同軸分送至各戶。
- A詳: 原理說明：營運商拉光纖至大樓，終接於ONU/MDU。資料由OLT→光纖→ONU，轉為銅媒體（乙太、VDSL、同軸）。流程：入光、轉換、樓內配線、住戶CPE。核心組件：OLT、ONU/MDU、樓內交換器/分配器、住戶路由器。性能受樓內媒體種類、配線與端口速率影響。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, A-Q5, C-Q8

Q9: DOCSIS（Cable）網路的技術架構是什麼？
- A簡: CMTS 匯聚同軸頻道，Cable Modem 共享頻寬，採上行排程與下行廣播。
- A詳: 原理說明：DOCSIS 透過CMTS 管理下行廣播與上行時槽分配，Cable Modem 註冊後共享頻道。流程：掃描頻道、 ranging、註冊、授權、資料傳輸。核心組件：CM、CMTS、光節點、同軸放大器。尖峰時段共享度與排程策略影響吞吐與延遲。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q9, B-Q17, A-Q18

Q10: 路由器 NAT 性能如何影響實際速度？
- A簡: NAT 需連線追蹤與轉換，處理能力不足會限制吞吐與連線數。
- A詳: 原理說明：NAT 將內外位址映射，需維護連線表與校驗。流程：封包進出、查表、轉換、校驗、轉發。核心組件：路由器CPU/ASIC、硬體加速（如NAT offload）、韌體。升速後若路由器老舊，NAT 成為瓶頸導致測速不上去，需升級設備或啟用硬體加速。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q24, C-Q4, D-Q1

Q11: TCP 擁塞控制為何影響升速效益？
- A簡: 擁塞控制調整傳輸窗口，延遲與丟包會限制吞吐，升速需搭配低損高RTT。
- A詳: 原理說明：TCP 以慢啟動、擴增/減少控制擴大或收斂傳輸窗口。高RTT或丟包會降低有效吞吐。流程：三向交握、慢啟動、擁塞避免、快速重傳。核心組件：端點TCP 堆疊、AQM（如FQ‑CoDel）與路由排程。升速能提高上限，但需避免上傳飽和與丟包，方能達成高吞吐。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q13, A-Q17, C-Q5

Q12: DNS 解析如何影響體感速度？
- A簡: DNS 決定連線目的地解析延遲與CDN 節點選擇，影響首包時間。
- A詳: 原理說明：應用在發起連線前需DNS 解析，解析延遲與結果影響與CDN 節點距離。流程：遞迴查詢、快取、EDNS Client Subnet 影響節點選擇。核心組件：遞迴DNS、權威DNS、CDN。使用就近且穩定的DNS、保留快取，可改善首屏載入體感，但對長時間大檔下載影響有限。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q9, D-Q1, A-Q23

Q13: 速度測試的原理與準確性關鍵是什麼？
- A簡: 以多連線、近端伺服器、足夠時間測試，避免單點瓶頸影響。
- A詳: 原理說明：測速工具透過多TCP 連線飽和上下行，並量測吞吐、延遲。流程：選點、暖身、上行/下行測試、統計。核心組件：測速伺服器、客戶端、路由器/Modem。準確性仰賴伺服器距離、時段、Wi‑Fi/有線連接、背景流量。需多時段、多節點驗證升速成效。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q16, C-Q6, D-Q5

Q14: 家用路由器 QoS/SQM 的工作機制為何？
- A簡: 以排程與整形控制佇列，優先互動流量，避免上傳飽和導致高延遲。
- A詳: 原理說明：QoS/SQM 透過階層式排程、整形與AQM（如FQ‑CoDel、CAKE）限制佇列長度，並依分類給予優先。流程：流量分類、限速、佇列管理、排程。核心組件：路由器CPU、韌體實作、分類器（DSCP/port）。正確設定上/下行目標速率（略低於實測）是關鍵。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: C-Q5, D-Q4, A-Q25

Q15: IPv4/IPv6 在固定寬頻上的差異是什麼？
- A簡: IPv6 位址充足且端到端特性佳；IPv4 常需NAT，可能影響連入應用。
- A詳: 原理說明：IPv6 具龐大位址空間，減少NAT 需求；IPv4 位址稀缺常以NAT/CGNAT 提供。流程：PPPoE/ DHCPv6-PD 取得位址，路由器分派內網/前綴。核心組件：BNG、DHCPv6、NAT。對端到端與P2P 應用，IPv6 有利連通與效能；部分舊設備需注意兼容性。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q20, C-Q3, D-Q2

Q16: MTU/MSS 如何影響 PPPoE 的傳輸效率？
- A簡: PPPoE 額外負載使MTU 降至1492；需調整MSS 避免IP 分片與性能下降。
- A詳: 原理說明：PPPoE 封裝增加8位元組開銷，乙太MTU 一般需設為1492。流程：路由器在NAT/防火牆層進行MSS Clamping，將TCP MSS 調降以避免分片。核心組件：PPPoE 客戶端、路由器、端點TCP。錯誤的MTU/MSS 設定會導致重傳與飽和效率低下。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: C-Q3, D-Q7, B-Q2

Q17: ISP 匯聚與骨幹壅塞如何形成？
- A簡: 多用戶共享上游，尖峰產生排隊與丟包，降低吞吐並提高延遲。
- A詳: 原理說明：接取至匯聚層的多條用戶鏈路疊加到有限的上游連線，尖峰時段流量超出容量造成佇列與AQM 丟包。流程：用戶側→接取→匯聚→核心→外連。核心組件：匯聚交換器、上聯路由器、骨幹鏈路。觀察多節點測速與時段差異可判斷壅塞位置。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q18, D-Q5, B-Q13

Q18: 光纖覆蓋率統計與實際可用之落差機制？
- A簡: 以區域可達為口徑，不含建物內部可行性，導致名義覆蓋不等於可開通。
- A詳: 原理說明：覆蓋統計多以路網到達、端局能力計，未含大樓內管道容量與施工同意。流程：規劃→拉纖至節點→更新資料庫→住戶查詢。核心組件：GIS 覆蓋系統、資源管理。住戶層面仍需可服務性勘查與協調，可能需增建樓內設備。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q22, C-Q8, D-Q9

Q19: 固定IP 與動態IP 的運作差別？
- A簡: 固定IP 經常不變，便於架站；動態IP 會變動，成本低但連入較不穩。
- A詳: 原理說明：固定IP 由ISP 在BNG 靜態綁定；動態IP 由DHCP/PPPoE IPCP 每次撥接分派。流程：認證→位址指派→路由公告。核心組件：BNG、RADIUS/DHCP。固定IP 便於遠端連入與白名單，費用較高；動態IP 需DDNS 或穿透技術。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q20, C-Q3, D-Q2

Q20: 室內網路佈線（Cat5e/Cat6）對速率的影響？
- A簡: 有線品質影響實測吞吐與穩定；較新規格支援更高速且抗干擾較佳。
- A詳: 原理說明：乙太網纜線的頻寬與抗串擾能力決定連線介面速率與誤碼率。流程：端接壓接、測試、鏈路協商。核心組件：RJ‑45 水晶頭、交換器/路由器網口、NIC。升速驗證應優先用有線連接，確保端到端皆為1Gbps 以上介面與合格線材。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q4, C-Q6, D-Q1

### Q&A 類別 C: 實作應用類（10題）

Q1: 如何申請將 ADSL 從 2M/256K 升速到 8M/640K？
- A簡: 先線上或門市申請，通過線路可行性檢測後，等待ISP 調整與開通。
- A詳: 具體步驟：1) 上網或門市提出升速需求；2) 查詢住址線路可行性；3) 確認費率與合約；4) ISP 建立工單調整DSLAM Profile；5) 約定開通時間；6) 開通當日重啟Modem/路由器。設定：保持原PPPoE 帳密；若CPE 老舊，詢問更換。注意事項：保留申請紀錄、以有線測速驗證、若不穩定可要求回退或線檢。最佳實踐：先排除Wi‑Fi 變因。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q3, C-Q6, D-Q1

Q2: 如何檢查住家線路是否支援更高 ADSL 速率？
- A簡: 查詢ISP 可服務性，並在Modem 介面查看SNR 與衰減指標評估空間。
- A詳: 步驟：1) ISP 網站輸入住址查可服務性；2) 登入Modem 查看Line Attenuation、SNR Margin、 attainable rate；3) 以有線跑測速多時段比對；4) 若SNR Margin 高、衰減低，升速成功率高。設定：記錄當前參數值。注意：內線老化與分接會影響評估。最佳實踐：以主插座直連測試。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q14, A-Q15, D-Q8

Q3: 如何設定 PPPoE 撥接確保連線穩定？
- A簡: 在路由器輸入帳密、調整MTU 1492 與MSS，啟用自動重連與時間校時。
- A詳: 步驟：1) 路由器WAN 類型選PPPoE；2) 輸入帳號密碼；3) MTU 設1492、啟用MSS Clamping（例如1452）；4) 啟用Keep‑alive；5) 設定NTP 校時。設定片段：MTU=1492、MSS=1452。注意：避免雙重撥接；若為一體機，可橋接Modem。最佳實踐：記錄撥接日誌便於故障排查。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q12, B-Q16, D-Q2

Q4: 升速時如何選購與更換路由器/Modem？
- A簡: 選擇支援所需速率與NAT/QoS 性能、穩定韌體與足夠記憶體的機型。
- A詳: 步驟：1) 確認ISP 介面（RJ11 DSL 或RJ45 乙太）；2) 檢視路由器NAT 吞吐、CPU 與記憶體；3) 支援QoS/SQM；4) 如需Wi‑Fi，選802.11ac/ax；5) 更換後以有線測速驗證。設定：啟用硬體NAT 或Flow Acceleration。注意：保留原設定備份。最佳實踐：韌體更新到穩定版。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q10, C-Q6, D-Q1

Q5: 如何設定家用路由器 QoS/SQM 以避免上傳飽和？
- A簡: 啟用智慧QoS 或SQM，設定上/下行為實測值的90–95%，優先即時流量。
- A詳: 步驟：1) 以有線測速得穩定上下限；2) 在路由器啟用QoS/SQM（FQ‑CoDel/CAKE）；3) 設上/下行目標為實測90–95%；4) 調整分類，優先語音/會議；5) 驗證Ping 與視訊穩定。設定：上行640K 時可設為580–610K。注意：過低會浪費帶寬。最佳實踐：分時排程備份。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q14, A-Q25, D-Q4

Q6: 如何測試升速前後的效益與差異？
- A簡: 以有線多節點、多時段測速，比對吞吐、延遲、抖動與網頁載入體感。
- A詳: 步驟：1) 升速前記錄3個時段測速；2) 升速後同條件重測；3) 觀察吞吐、RTT、Jitter；4) 用相同設備與伺服器；5) 測試實際應用（雲端備份速率、視訊會議）。設定：禁用背景流量。注意：Wi‑Fi 影響大，務必有線。最佳實踐：保存圖表供日後申訴或回退依據。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, D-Q1, A-Q16

Q7: 如何改善住家內線以提升線路品質？
- A簡: 減少分接、改用主插座、檢修接頭，必要時更換為實體專線。
- A詳: 步驟：1) 將Modem 直接主插座；2) 移除過多分接與劣質延長線；3) 更換老化插座與水晶頭；4) 使用高品質雙絞線；5) 請技師測量SNR/衰減。設定：於Modem 介面比對改善前後參數。注意：避免與電力線平行長距離佈線。最佳實踐：建立獨立數據專線迴路。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q15, D-Q8, B-Q4

Q8: 如何查詢並推進 FTTB/FTTH 的可行性？
- A簡: 透過ISP 可服務性查詢、社區住戶連署與管委協調，促成入纖工程。
- A詳: 步驟：1) ISP 線上查詢住址可服務性；2) 若不可用，向營運商提出布建意願；3) 連署住戶、經管委會同意；4) 評估弱電間與管道；5) 排程施工。設定：保留申請編號。注意：施工需公區許可與電源規劃。最佳實踐：與多家ISP 詢價比較方案。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, A-Q22, D-Q9

Q9: 如何設定 DNS 以改善體感速度？
- A簡: 選用就近且穩定的DNS，或ISP 預設，並啟用快取與DoT/DoH。
- A詳: 步驟：1) 在路由器LAN 設置DNS（可測試ISP、本地公用DNS）；2) 驗證解析延遲；3) 啟用DNS 快取；4) 視需求開啟加密DoT/DoH。設定：指定主/備DNS 地址。注意：部分服務對EDNS 行為敏感，選擇能良好導向CDN 的DNS。最佳實踐：測試首包時間改善幅度。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q12, D-Q1, A-Q23

Q10: 如何監控上傳使用率避免飽和？
- A簡: 在路由器啟用流量圖與告警，設分流或限速，避免備份/P2P 壟斷上行。
- A詳: 步驟：1) 開啟路由器流量監控（per‑client/per‑app）；2) 設定限速或排程；3) 為備份軟體設上行上限；4) 設告警門檻。設定：例「備份上限300Kbps」。注意：留有餘裕給互動流量。最佳實踐：結合SQM 與排程在離峰同步。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q25, B-Q14, D-Q4

### Q&A 類別 D: 問題解決類（10題）

Q1: 升速後實測速度沒有提升怎麼辦？
- A簡: 以有線多節點重測，排除Wi‑Fi與路由器瓶頸，必要時向ISP 報修檢測。
- A詳: 症狀：測速與下載吞吐未達新方案標準。可能原因：路由器NAT 性能不足、Wi‑Fi 限制、尖峰匯聚壅塞、DSLAM 設定未更新、內線雜訊。解決：1) 有線測速；2) 檢查路由器CPU 與硬體加速；3) 多時段測試；4) 查看Modem 同步速率；5) 向ISP 申訴。預防：升速前評估設備性能。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, B-Q10, C-Q6

Q2: PPPoE 認證失敗或頻繁斷線如何處理？
- A簡: 檢查帳密、MTU/MSS 與撥接設定，觀察日誌並聯繫ISP 重置連線。
- A詳: 症狀：撥接失敗、頻繁斷開重連。原因：帳密錯誤、線路不穩、BRAS 問題、MTU 設定不當。解決：1) 重輸入帳密；2) 設MTU 1492、MSS 1452；3) 啟用Keep‑alive；4) 檢視日誌錯碼；5) ISP 側重設。預防：保存設定、避免重複撥接、橋接正確。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q16, C-Q3

Q3: 連線不穩定或常掉線，應如何診斷？
- A簡: 先看Modem SNR/衰減與錯誤計數，再檢查內線與干擾，必要時請線檢。
- A詳: 症狀：隨機斷線、頻寬忽高忽低。原因：SNR 偏低、衰減過高、突發雜訊、Interleaving 設定不當。解決：1) 查看SNR Margin/Attenuation；2) 直接主插座；3) 切換Interleaving；4) 請技師測試。預防：改善內線、使用品質分離器、避開干擾源。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q14, A-Q15, B-Q5

Q4: 上傳滿載導致延遲飆升，如何緩解？
- A簡: 設定QoS/SQM 或限制備份速率，或升級上行帶寬，確保互動流量優先。
- A詳: 症狀：開啟備份/P2P 後網頁與會議卡頓。原因：上行佇列飽和、TCP ACK 延遲。解決：1) 啟用SQM（FQ‑CoDel/CAKE）；2) 設上行目標90–95%；3) 限制備份/P2P 上限；4) 升級上行。預防：排程離峰同步、監控流量與告警。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q17, B-Q14, C-Q10

Q5: 尖峰時段速度明顯下降的原因與對策？
- A簡: 匯聚與骨幹壅塞或FUP 影響；多時段測試並與ISP 協調或更換方案。
- A詳: 症狀：晚間速率顯著低於白天。原因：匯聚比過高、節點共享壅塞、FUP 限速。解決：1) 多時段測速留證；2) 詢問ISP 調整或換節點；3) 升級商務方案；4) 觀察FUP 條款。預防：選擇口碑佳之節點與方案。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q18, B-Q17, B-Q13

Q6: Modem 顯示 SNR 低或錯誤率高怎麼辦？
- A簡: 檢查內線與分離器，請ISP 調整Profile 或線檢，必要時降速求穩。
- A詳: 症狀：SNR Margin 低於門檻、CRC/HEC 錯誤累積快。原因：線路雜訊、衰減過高、過激進Profile。解決：1) 直連主插座；2) 更換分離器；3) 請ISP 降交錯深度或調整SNR 目標；4) 降速穩定。預防：改善佈線、避開高干擾環境。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q14, B-Q4, C-Q7

Q7: MTU 設定不當導致速度慢或頁面載入異常？
- A簡: 調整PPPoE MTU=1492 並啟用MSS Clamping，避免分片與重傳。
- A詳: 症狀：部分網站載入卡住、VPN 不穩。原因：MTU 過大致IP 分片，MSS 不匹配。解決：1) 設MTU 1492；2) 啟用MSS 1452；3) 測試Path MTU；4) 重啟連線。預防：使用預設建議值，避免手動誤改。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q16, C-Q3, B-Q2

Q8: 內線老化或高衰減造成速率上不去，怎麼處理？
- A簡: 簡化佈線、換新線材與接頭，必要時申請近移節點或改用FTTx。
- A詳: 症狀：高衰減、降速才穩。原因：長距離、細線徑、接頭氧化。解決：1) 改直連主插座；2) 換新雙絞線與分離器；3) 減少分接；4) 請ISP 線檢或近移節點；5) 考慮FTTB/FTTH。預防：定期檢視接頭、避免潮濕環境。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q15, C-Q7, A-Q10

Q9: 申請升速被拒的常見原因與對應作法？
- A簡: 線質或距離不足、節點資源不足；可改善內線或改方案/等候布建。
- A詳: 症狀：工單回覆不可行。原因：距離過長、SNR 不足、節點滿載、樓內不可施工。解決：1) 改善內線；2) 申請FTTB/FTTH 可行性；3) 轉用VDSL/Cable；4) 等待擴容。預防：先做可服務性評估與技術方案比較。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q2, C-Q8, B-Q18

Q10: 家中多設備導致 Wi‑Fi 成為瓶頸，如何優化？
- A簡: 升級AP、用5GHz、調信道與位置，重點設備改用有線連接。
- A詳: 症狀：近端測速低、延遲高。原因：2.4GHz 擁擠、舊標準、AP 位置不佳。解決：1) 使用802.11ac/ax 與5GHz；2) 避開干擾信道；3) 置中與高處；4) 重要設備用有線。預防：規劃分散AP、避免過度堆疊。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q21, C-Q4, C-Q6
