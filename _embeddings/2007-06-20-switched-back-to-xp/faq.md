# 還是換回 XP 了...

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

A-Q1: 什麼是 Windows Vista？
- A簡: Windows Vista 為 2007 年推出的 Windows，強化安全與視覺，導入 UAC、Aero、SuperFetch、WDDM 與新版備份。
- A詳: Windows Vista 是微軟於 2007 年發行的主要版本，重點在安全與體驗革新。它引入 UAC（使用者帳戶控制）、Aero 圖形介面、SuperFetch 預取、搜尋索引、WDDM 顯示驅動模型與新版 VSS/備份機制。雖改善穩定性與搜尋便利，但初期驅動成熟度、相容性及資源需求引發負面體驗，常見於舊工具與周邊設備未更新的環境。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q17, B-Q14, B-Q16

A-Q2: 什麼是 Windows XP？
- A簡: Windows XP 為 2001 年發表的穩定版本，相容性佳、資源需求低，長期在企業與個人普及。
- A詳: Windows XP（NT 5.1）以高穩定性與廣泛相容著稱，驅動成熟，對舊應用支援完善。其生態包含 PowerToys 與 Resource Kit 等實用工具。相較 Vista，XP 對硬體與記憶體需求較低，反應輕快；但在安全模型、搜尋與圖形體驗上較為傳統，缺少 UAC、WDDM 與新式備份/索引等新功能。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q3

A-Q3: 為何有人從 Vista 換回 XP？
- A簡: 因相容性不足、權限阻礙與效能卡頓，影響工作流程與工具使用，故回退至 XP。
- A詳: 常見回退理由包括：1) 舊工具在 Vista 未更新而失效，如 XP PowerToys、Resource Kit 的 cdburn/dvdburn；2) 檔案系統安全更嚴謹，移動檔案常遇權限阻擋；3) 背景索引、驅動成熟度與新驅動模型導致卡頓體感；4) 個別功能（如 Media Center 錄影）受節目表標記影響而誤錄。當核心工作受影響，回退可暫保生產力。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, A-Q11, A-Q17, D-Q3

A-Q4: 什麼是 Volume Shadow Copy (VSS)？
- A簡: VSS 是 Windows 的卷影複製服務，生成快照供一致性備份與檔案回溯使用。
- A詳: VSS（Volume Shadow Copy Service）可在系統運行中建立磁碟快照，確保備份的一致性。其角色分為 Requester（備份程式）、Writer（應用協作者）與 Provider（快照提供者）。常見應用是系統保護、上一版本（Previous Versions）與線上備份。VSS 以寫入時複製（COW）降低成本，快照受限於陰影存放區容量與清除策略。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q3, C-Q2, D-Q1

A-Q5: Vista 的 VSS 與 Windows Server 2003 有何差異？
- A簡: Vista 用戶端預設策略與保留機制不同，陰影儲存行為與 2003 相較更易佔空間。
- A詳: 雖同屬 VSS，Vista 客戶端偏向使用者回溯與系統還原，預設開啟系統保護與快照保留；Windows Server 2003 側重伺服器一致性備份與受控快照排程。政策、保留邏輯與陰影存放區設定預設值不同，導致在大量變動（如相片編修）時，Vista 更可能快速耗盡空間，需透過手動上限調整。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, D-Q1

A-Q6: 什麼是 Copy-on-Write（COW）？
- A簡: COW 在資料即將改寫時保存舊區塊，讓快照引用舊資料，避免整體複製負擔。
- A詳: COW 是快照常見技術。建立快照時不複製整個卷，只有當原區塊將被寫入時，先把舊內容複製到陰影存放區，快照對應到該舊區塊，新寫入落在原卷的新位置。此機制大幅降低快照建立成本，但若寫入量大、生命周期長，陰影存放區會快速成長。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, A-Q4

A-Q7: 為何 VSS 會快速佔滿磁碟空間？
- A簡: 快照多且變動頻繁、陰影上限過大或未設，致陰影複本暴增、空間迅速耗盡。
- A詳: 影響因素包括：快照建立頻率（自動/手動）、資料修改量（如大量相片編修）、陰影存放區上限設定與資料分布。COW 在高寫入率時會大量保留舊區塊。若上限設定過大且缺乏清除策略，短時間即可吃滿磁碟。建議調整上限、刪除舊快照並降低高頻修改。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q2, D-Q1

A-Q8: 什麼是 VSS Writer？
- A簡: VSS Writer 是應用協作者，於快照前暫停 I/O 並清理，確保資料一致可還原。
- A詳: Writer 由應用供應商實作（如 SQL、Exchange、文件服務），負責在快照前進行 flush、暫停、日誌截斷等，快照後再解凍恢復。良好 Writer 可保證應用一致性，避免僅檔案層級一致卻破壞交易狀態的風險，是企業備份可靠性的關鍵。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q5, A-Q4

A-Q9: Vista 與 XP 在內建與周邊工具相容性差異？
- A簡: Vista 新 API 與安全模型致舊工具未更新而失效，需等待新版或以替代方案因應。
- A詳: Vista 引入新 API、UAC 與更嚴格的檔案/登錄存取規則。許多 XP 時代工具（如 PowerToys、Resource Kit 的 cdburn/dvdburn）若未針對新模型調整，可能無法執行或功能受限。解方包括：尋找對應新版、以第三方替代，或使用內建功能替代。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, D-Q7, B-Q20

A-Q10: 什麼是 Codec？為何 .CRW 需要編解碼器？
- A簡: Codec 讓系統能解碼特定格式；無 .CRW 編解碼器時無法在檔案總管預覽 RAW。
- A詳: Codec（編解碼器）負責解析特定媒體格式。Windows 透過 WIC 與 Shell Handler 使用相機 RAW 編解碼器生成縮圖與預覽。若缺 .CRW（Canon RAW）編解碼器，Explorer 無法顯示縮圖，需安裝廠商或微軟的攝影編解碼包，或改用支援 RAW 的看圖/轉檔軟體。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q4, D-Q8, B-Q18

A-Q11: Vista 的檔案系統安全模型有何改變？
- A簡: Vista 引入 UAC 與更嚴格 ACL 預設，需提權操作，影響檔案搬移與系統修改。
- A詳: Vista 以 UAC 實作權杖分離，預設標準權限執行，需同意才升權。NTFS ACL 預設更嚴謹，系統目錄與 Program Files 限制強化。舊應用若寫入受保護路徑將受阻或被虛擬化，造成搬移與存取行為與 XP 不同，需調整作業方式或權限設定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, A-Q12, D-Q2

A-Q12: Move 與 Copy+Delete 的權限差異為何？
- A簡: 同卷移動為改名保留原 ACL；跨卷複製繼承目錄 ACL，再刪原檔，常繞過移動限制。
- A詳: 同一磁碟卷的 Move 實質為 rename，保留原 ACL 與擁有者，若目的地受限制會被拒。跨卷 Move 實為 Copy+Delete，新檔在目標目錄建立並繼承該處 ACL，之後刪除來源，故常能成功。理解此差異有助於處理「移動失敗、複製可行」的情況。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, A-Q29, D-Q2

A-Q13: 動態磁碟與基本磁碟有何差異？
- A簡: 動態磁碟支援延伸卷、跨碟與鏡像；基本磁碟採傳統分割，相容性較佳、風險較低。
- A詳: 基本磁碟使用 MBR/GPT 分割表，卷對應分割區；動態磁碟以 LDM 描述卷，可彈性延伸、跨碟串接，並支援軟體 RAID。動態磁碟的跨版本與跨系統相容性較複雜，外接移動與回退系統時需特別留意支援矩陣。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q9, C-Q8, D-Q5

A-Q14: 為何 Vista 建立的動態磁碟在 XP 無法匯入？
- A簡: 可能使用 GPT、LDM 版本差或 OS 限制；XP 32 位不支援 GPT 與部分動態卷。
- A詳: XP 32 位不支援 GPT，對新版本 LDM 描述亦可能有限制；XP Home 更不支援動態磁碟。若在 Vista 以 GPT 或特定配置建立動態卷，XP 可能僅顯示為外來磁碟而無法匯入上線。需以支援系統讀取後備份資料，再以基本磁碟重建。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q11, C-Q8, D-Q5

A-Q15: 什麼是 GPT 與 MBR？相容性如何？
- A簡: GPT 支援 >2TB 與多分割；MBR 相容廣。XP 32 位不支援 GPT，Vista/64 位支援較完整。
- A詳: MBR 是傳統分割表，最多 2TB、四主分割。GPT 採 UEFI 架構，具主/備表頭與保護 MBR，支援大容量與更多分割。相容性上，XP 32 位不支援 GPT 磁碟；Vista、Win7 與 64 位系統支援較好，但開機需求取決於 BIOS/UEFI。跨系統移轉需留意。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q11, A-Q14

A-Q16: 什麼是 Windows Complete PC Backup？
- A簡: Vista 的整機影像備份，產生 VHD；需自修復環境還原，無法直接還原至 XP。
- A詳: Complete PC Backup 是 Vista（商用/旗艦）提供的區塊層級映像備份，將系統與資料封裝為 VHD/映像組，可用於裸機或完整復原。還原必須從 Windows 修復環境進行，無法在 XP 內直接掛載還原整機；若需抽取檔案，可在支援的系統中掛載或以工具讀取。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q12, C-Q9, D-Q6

A-Q17: Vista 為何可能感覺較慢？
- A簡: 索引/預取背景 I/O、早期驅動成熟度與 WDDM 開銷，會造成短期卡頓體感。
- A詳: Vista 引入 SuperFetch 與搜尋索引，初期會大量掃描與預載；WDDM 驅動模型改善穩定性但早期驅動成熟度不足。加上 UAC 提權、磁碟碎片與啟動程式眾多，皆可能影響流暢度。適當調整與驅動更新可改善體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q14, B-Q16, C-Q3, D-Q3

A-Q18: SuperFetch 與索引如何影響效能？
- A簡: SuperFetch 預載常用程式、索引建立全文；短期耗 I/O，長期提升啟動與搜尋速度。
- A詳: SuperFetch 觀察使用模式，在閒置時預載常用檔案至記憶體，壓力大時釋放；搜尋索引在背景抽取內容與屬性，提供即時搜尋。兩者短期會有磁碟活動，可能感到卡頓；穩定後能改善整體反應。可調整索引範圍與排程達平衡。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q14, B-Q15, C-Q3

A-Q19: 什麼是 Windows Media Center 與 EPG？
- A簡: Media Center 透過 EPG 節目表進行排程錄影，依節目 ID 與旗標決定是否錄製。
- A詳: Windows Media Center 提供電視收看與錄影功能，EPG（電子節目表）包含節目名稱、集數、重播標記等中繼資料。使用者可設定單次或劇集錄影、限定頻道與時段。錄影引擎依 EPG 標記與規則判斷是否執行，資料品質影響準確度。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q13, C-Q7, D-Q4

A-Q20: 為何 Media Center 會重複錄影？
- A簡: EPG 標記錯誤、規則過寬或頻道未限定，可能使重播被視為新集而錄下。
- A詳: 若 EPG 未正確標示重播或季/集資訊，或錄影規則設定為「全部集數」且不限定頻道/時段，系統可能多錄重播。修正方式包括：設定僅錄「新集」、限定頻道與時段、更新 EPG 或改用其他節目表來源。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q4, C-Q7

A-Q21: 什麼是 Windows PowerToys？
- A簡: XP 時代的實驗工具集，如影像調整大小；Vista 初期多未更新而不相容。
- A詳: PowerToys 提供許多便捷小工具，如 Image Resizer、RAW Viewer。這些工具多針對 XP 設計，未必符合 Vista 的新 API 與安全模型，導致執行失敗或功能受限。可尋找第三方替代或等待新版，或用新版 PowerToys（適用新系統）的同名工具。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, D-Q7

A-Q22: 什麼是 Resource Kit Tools（cdburn/dvdburn）？
- A簡: 資源工具組含命令列光碟燒錄工具；Vista 以 IMAPIv2 與內建 UI 取代舊工具。
- A詳: Resource Kit Tools 為系統管理者提供批次與自動化工具，cdburn/dvdburn 能在 XP 以命令列燒錄光碟。Vista 起改採 IMAPIv2 與新 UI，舊工具不再隨附且相容性受限；可改用 IMAPIv2 程式庫或第三方命令列工具。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q17, D-Q9, A-Q23

A-Q23: Vista 的光碟燒錄機制與 IMAPIv2 是什麼？
- A簡: IMAPIv2 提供新版燒錄 API 與內建介面，支援 DVD，取代舊有工具與流程。
- A詳: IMAPIv2 是 Vista 引進的映像母片製作 API，支援更廣的媒體型別與格式。應用可透過映像建構、資料安排與燒錄流程完成寫入。對使用者而言，內建燒錄 UI 足以取代大多數情境；需命令列時可用 IMAPIv2 SDK 或第三方工具。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q17, D-Q9

A-Q24: 什麼是 Virtual PC？有何價值？
- A簡: Virtual PC 是微軟虛擬化軟體，可隔離測試新系統與軟體，降低主系統風險。
- A詳: Virtual PC 讓你在主系統上建立虛擬機，安裝其他作業系統進行測試與驗證。對於尚未成熟或相容性存疑的版本（如早期 Vista），可先在 VM 內驗證工具、驅動與工作流程，避免直接升級導致生產力受損，並能快速回滾快照。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q19, C-Q10, A-Q30

A-Q25: 為何相容性與回溯相容性重要？
- A簡: 升級若破壞既有工具與流程，生產力大幅下降；相容性是可用性的基石。
- A詳: 多數組織倚賴穩定工具鏈與流程。升級帶來新功能，但若關鍵工具無法運作、驅動不可用或檔案權限變更干擾工作，即使新功能再好也難以落地。評估、測試與相容性策略（含替代方案）是升級成功的關鍵。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, D-Q7

A-Q26: 為何遷移前需完整備份並演練還原？
- A簡: 備份不等於可用；演練還原能驗證流程與映像有效，避免災難時才發現失效。
- A詳: 許多人僅做備份卻未測試還原，實際需要時才發現無法啟動、相容性問題或映像損毀。事前在異機或虛擬機演練還原，檢驗可啟動性與資料完整，能暴露像映像無法在 XP 還原等限制，進而調整策略與工具。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q16, C-Q9, D-Q6

A-Q27: 32 位與 64 位 Windows 有何差異？
- A簡: 64 位支援更大記憶體與安全強化；但舊驅動/軟體相容性較差，需事前驗證。
- A詳: 64 位系統支援更大位址空間與硬體 DEP、強制驅動簽章等安全機制；能更好地執行記憶體密集應用。但舊驅動與 32 位僅支援軟體可能無法運作或效能不佳。選擇時需盤點驅動與應用支援度。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, D-Q10

A-Q28: 內建工具與第三方工具的取捨？
- A簡: 內建工具整合與安全佳；第三方功能彈性高。依需求、相容性與維護成本選擇。
- A詳: 內建工具通常支援度與穩定性較好，整合安全政策與更新；第三方能快速補位功能缺口（如影像縮圖、RAW 支援）。取捨時衡量功能完整度、相容性、授權成本與長期維護，並制定替代清單以應對升級風險。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q9, C-Q5, D-Q7

A-Q29: 什麼是檔案擁有者與權限繼承？
- A簡: 檔案擁有者能變更 ACL；檔案通常繼承目錄權限，跨磁碟複製會產生新擁有者。
- A詳: NTFS 以擁有者（Owner）決定誰可修改 ACL；檔案與目錄的 DACL 可從父目錄繼承。複製到新位置時，新檔會繼承目的地權限並以執行者為擁有者；同卷移動則保留原 ACL/Owner。理解此機制有助於排除權限問題。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q8, A-Q12, D-Q2

A-Q30: 何時用虛擬化而非直接升級？
- A簡: 當相容性存疑或影響生產時，先將新系統放入 VM 測試，維持主機穩定。
- A詳: 若新系統對關鍵工具與驅動的支援尚未成熟，直接升級風險高。以虛擬機部署新系統，逐步驗證工作流程、替代方案與備援；待問題解決再轉正。此策略可降低停機風險並保有回退彈性。

- 學習階段: 基礎

### Q&A 類別 B: 技術原理類

B-Q1: VSS 如何運作？
- A簡: 由 Requester 協調 Writer 與 Provider，凍結 I/O、建立快照、解凍，供備份或回溯。
- A詳: 流程為：Requester 發起快照，VSS 通知各 Writer 準備一致性狀態（flush/hold），Provider 建立卷影複本（COW/差異），成功後解除凍結。備份應用讀取快照進行備份。核心組件含 VSS Service、Writer、Provider 與 Requester。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, B-Q5, C-Q2

B-Q2: Copy-on-Write 的技術原理是什麼？
- A簡: 寫入前將原區塊複製到陰影存放，快照參照舊內容；新寫入保留在主卷。
- A詳: 建立快照後，系統不複製整卷。當應用要改寫原區塊時，先把舊資料搬到陰影存放區，快照的邏輯位址映射到該舊資料；新資料寫回原卷。此法減少快照建立成本，但在大量寫入下，陰影存放區使用量會迅速增加。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q3

B-Q3: VSS 的儲存管理與清除策略為何？
- A簡: 陰影存放區可設上限；達上限時刪最舊快照。高變動與大檔案將加速空間消耗。
- A詳: 每個卷對應陰影存放區（常在同卷）。可用 vssadmin 檢視與調整上限。當空間不足，系統會刪除最舊快照以釋放容量；若短期變動極大，新快照也可能立即被回收。最佳化在於設定合理上限並控管變動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q2, D-Q1, A-Q5

B-Q4: Vista 與 2003 的 VSS 組件與策略差異？
- A簡: Vista 側重用戶端回溯與系統保護；2003 側重伺服器備份，排程/保留策略不同。
- A詳: Vista 預設開啟系統保護與上一版本，快照由使用者動作與排程觸發；Server 2003 的快照更傾向備份窗口控制，Writer 清單與 Provider 實作也有差異。兩者快照頻率與陰影存放區預設值不同，影響空間使用行為。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q5, B-Q3

B-Q5: VSS Writer 的一致性協調機制？
- A簡: Writer 在快照前 flush 與暫停 I/O，標記可還原點；快照完成後解凍恢復。
- A詳: Writer 接到 PrepareForSnapshot、Freeze、Thaw 等事件，於 Freeze 前完成緩衝刷新與交易落盤，確保一致性。Thaw 後恢復正常 I/O。對資料庫等應用，Writer 可執行交易截斷或記錄記憶體狀態，確保還原可用。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q8, B-Q1

B-Q6: UAC 與 ACL 的機制是什麼？
- A簡: UAC 以權杖分離控管提權；ACL 以 ACE 決定存取。兩者協作保護系統。
- A詳: UAC 將系統管理員登入後的權杖拆分為標準與提升權杖，需同意才使用高權限。NTFS 以 DACL（ACE 條目）定義允許/拒絕。存取時由安全性子系統比對權杖與 DACL 逐一決策。此設計降低惡意程式取得高權限風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q11, A-Q12

B-Q7: Move 與 Copy 的權限行為機制？
- A簡: 同卷 Move 僅改名保留原 DACL；跨卷 Copy 新建檔案，依目錄繼承產生新 DACL。
- A詳: Move 在同卷不牽涉資料搬移，僅更新目錄項，故維持原 ACL 與 Owner；跨卷需先 Copy 新檔，根據目的地權限繼承與預設擁有者建立安全描述，完成後刪除原檔。此機制解釋常見的「移動被拒、複製可行」現象。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, A-Q29

B-Q8: NTFS 權限繼承鏈如何運作？
- A簡: 父目錄的顯式/繼承 ACE 共同決定子項目 DACL；明確拒絕優先於允許。
- A詳: NTFS 允許從父目錄繼承 ACE，也可設定顯式 ACE 覆寫。有效權限取決於權杖 SIDs 與 DACL 中的 ACE 匹配，拒絕 ACE 優先於允許。管理者可透過中斷繼承與轉換顯式 ACE 精準控權。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q29, D-Q2

B-Q9: 動態磁碟的 LDM 架構是什麼？
- A簡: LDM 在磁碟保存卷組態，支援延伸、跨碟、鏡像等，提升彈性但增相容複雜。
- A詳: 動態磁碟利用邏輯磁碟管理（LDM）資料庫描述卷與磁碟群組，位於磁碟尾端並跨磁碟同步。可建立簡單/跨距/鏡像/RAID-5 卷。其跨系統移動需匯入外來磁碟，版本與 GPT/MBR 支援差異會影響上線。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q13, B-Q10

B-Q10: 外來動態磁碟的匯入流程？
- A簡: 讀取磁碟簽章與 LDM，嘗試上線卷；版本不符或 GPT 限制會造成匯入失敗。
- A詳: 新系統偵測到外來磁碟時，會比對磁碟簽章、讀取 LDM，嘗試加入磁碟群組並上線卷。若系統不支援 GPT 或 LDM 版本差異，匯入會失敗。需在支援系統存取資料後再調整配置。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: A-Q14, D-Q5

B-Q11: GPT 與 MBR 的結構與支援矩陣？
- A簡: GPT 有主/備表頭與保護 MBR，支援 >2TB；MBR 限 2TB。XP 32 位不支援 GPT。
- A詳: MBR 使用傳統 32 位位址，容量限制約 2TB。GPT 具備 GUID 分割、CRC 校驗、主/備表頭與保護 MBR，可容納更多分割與大容量。支援上，XP 32 位不支援 GPT，x64 與新系統支援較完善；啟動需求依 UEFI/BIOS 決定。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q15, A-Q14

B-Q12: Complete PC Backup 的架構如何？
- A簡: 以 WBAdmin 產生區塊層級映像封裝成 VHD，透過 WinRE 執行裸機還原。

- 學習階段: 進階

B-Q13: Media Center 錄影與 EPG 的判斷原理？
- A簡: 依節目 ID、季/集與重播旗標比對；規則含新集、頻道/時段限制共同決策。

B-Q14: SuperFetch 的工作原理？
- A簡: 分析使用模式，閒時預載常用檔案進記憶體；壓力大時釋放，提高平均反應。
- A詳: SuperFetch 監控應用啟動與檔案存取，建立預測模型，在系統閒置與 I/O 低時進行預載，提升下次啟動速度。當使用者需求上升，緩存可被回收，不與應用搶資源。短期會增加磁碟活動，長期改善體感。

B-Q15: Windows 搜尋索引器怎麼運作？

B-Q16: WDDM 顯示驅動對效能有何影響？

B-Q17: IMAPIv2 的燒錄流程如何設計？

B-Q18: Shell Codec 與 RAW 處理管線？

B-Q19: Virtual PC 的虛擬化架構與 I/O？

- A詳: Virtual PC 以 hypervisor/虛擬裝置模擬硬體，客體 OS 透過虛擬網卡、磁碟與顯示器與主機互動。I/O 由虛擬匯流排轉譯至主機驅動，提供隔離與可回滾的測試環境，但效能受限於軟體虛擬化。

B-Q20: 相容性層（ACT/Shim）如何降低不相容？

### Q&A 類別 C: 實作應用類

C-Q1: 如何規劃從 Vista 降級至 XP？
- A簡: 先完整備份與驗證還原，再蒐集驅動、導出資料、重裝 XP、還原設定與資料。

C-Q2: 如何設定 VSS 的儲存上限與刪除快照？
- A簡: 用 vssadmin 檢視/調整陰影存放區上限，刪除舊快照以回收磁碟空間。

- A詳: 1) 在索引選項中排除大型媒體資料夾；2) 更新晶片組/顯示/儲存驅動；3) 在效能選項關閉部分視覺效果；4) 使用 msconfig/工作管理員停用非必要常駐；5) 確認磁碟健康與碎片狀況；6) 設定高效能電源計畫。注意觀察改善情況再調整。

C-Q4: 如何在 Windows 安裝 Canon .CRW 編解碼器或替代？

C-Q6: 如何以正確權限搬移大量檔案避免被拒？

- 關聯概念: D-Q2, A-Q12, A-Q29

C-Q7: 如何避免 Media Center 重複錄影？
- A簡: 設定「只錄新集」、限定頻道/時段，更新 EPG，清理既有規則並重新套用。
- A詳: 1) Media Center > Tasks > Settings > TV > Recorder/Recording Defaults 將 Series 設為「新集」；2) 在節目規則中選「只此頻道」與時段；3) Settings > TV > Guide > Get Latest Guide Listings 更新 EPG；4) 刪除舊規則並重建。驗證後觀察一週。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q4, B-Q13

C-Q8: Vista 動態磁碟在 XP 無法匯入時怎辦？
- A簡: 用支援系統（Vista/WinPE）存取複製，備份後在 XP 建基本磁碟重建卷。
- A詳: 1) 以 Vista 或 WinPE 開機，將動態卷資料複製到外碟；2) 在 XP 中以磁碟管理刪除原動態卷（需先備份）、轉為基本磁碟；3) 重新建立分割與格式化；4) 將資料還原。若需保留動態功能，考慮升級 OS 或用第三方工具。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q5, A-Q13, A-Q14

C-Q9: 如何正確還原 Vista Complete PC 映像？
- A簡: 從 Vista 安裝媒體進 WinRE，選完整電腦復原，指定映像並執行還原。
- A詳: 1) 以 Vista 安裝 DVD 開機；2) 選「修復電腦」進入 WinRE；3) 選「Windows 完整電腦復原」（Complete PC Restore）；4) 掃描外接碟/網路位置映像；5) 選擇映像並還原。注意：會覆寫目標磁碟，無法直接在 XP 內完成整機還原。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q6, A-Q16, B-Q12

C-Q10: 如何在 Virtual PC 建立 Vista 測試環境？
- A簡: 新建 VM、配置 RAM/磁碟、掛載 ISO 安裝 Vista，安裝增強元件並建立快照。
- A詳: 1) 建立新 VM（≥1–2GB RAM 視硬體）；2) 建立虛擬磁碟（動態擴充）；3) 掛載 Vista ISO 安裝；4) 安裝整合元件以改善滑鼠/網卡/顯示；5) Windows Update 與驅動更新；6) 建立基準快照便於回滾。注意配置網路隔離與資源限制。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q24, A-Q30

### Q&A 類別 D: 問題解決類

D-Q1: VSS 迅速吃滿磁碟怎麼辦？
- A簡: 症狀：陰影存放區膨脹。調整上限、刪舊快照、降低變動量或移動資料。
- A詳: 症狀：磁碟忽然餘量驟減。可能原因：快照多、寫入頻繁、上限過大。解法：vssadmin 調整/刪除舊快照；將高變動資料移出受保護卷；暫停大型編修作業再建快照。預防：設定合理上限與定期維護。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q2, B-Q3

D-Q2: 移動檔案出現「沒有權限」怎麼辦？
- A簡: 症狀：移動被拒但複製可行。調整擁有權/ACL，改用複製後刪除或提權。
- A詳: 症狀：移動遇拒絕存取。原因：同卷 Move 保留舊 ACL，目的地限制導致失敗。解法：以管理者執行，takeown/ icacls 取得擁有權並賦權；以 robocopy 複製後刪除來源。預防：跨卷作業或先調整 ACL。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q6, A-Q12, B-Q7

D-Q3: Vista 操作常卡頓，如何改善？
- A簡: 症狀：偶發延遲。更新驅動、調整索引/預取、精簡啟動與效果，檢查磁碟健康。
- A詳: 可能原因：索引初建、SuperFetch 預載、WDDM 驅動不成熟、磁碟碎片或壞道。解法：更新驅動與系統；調整索引範圍；關閉不必要視覺效果；精簡啟動程式；檢查磁碟 S.M.A.R.T 與碎片。預防：定期維護與驅動更新。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q3, B-Q14, B-Q16

D-Q4: Media Center 錄影過量怎麼修正？
- A簡: 症狀：重播被錄。設定僅新集、限定頻道/時段，更新或更換 EPG 來源。
- A詳: 原因：EPG 標記錯誤或規則過寬。解法：調整為「只錄新集」、限定頻道和時段；刪除並重建規則；更新 EPG 或使用替代來源。預防：定期檢查 EPG 品質並縮小規則範圍。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q7, B-Q13

D-Q5: Vista 動態磁碟在 XP 無法匯入怎麼辦？
- A簡: 症狀：外來磁碟無法上線。用支援系統複製資料，轉為基本磁碟後重建。
- A詳: 可能原因：GPT 或 LDM 版本不支援。解法：以 Vista/WinPE 啟動，複製資料；在 XP 刪除動態卷並轉為基本磁碟，重建分割與還原。預防：跨系統前避免使用動態磁碟或採用通用檔案系統/儲存。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q8, A-Q14, B-Q10

D-Q6: Vista Complete PC 映像無法還原怎麼辦？
- A簡: 症狀：XP 中無法還原。從 Vista DVD 進 WinRE 還原，或掛載 VHD 抽取檔案。
- A詳: 原因：Complete PC 還原需 WinRE，XP 不支援整機還原。解法：用 Vista 安裝媒體開機進 WinRE 執行完整電腦復原；或在支援系統掛載 VHD 抽取檔案。預防：遷移前演練還原，清楚限制。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q9, A-Q16, B-Q12

D-Q7: XP 常用工具在 Vista 無法運作怎麼辦？
- A簡: 症狀：PowerToys/資源套件失效。找新版、替代工具或用內建功能替代。
- A詳: 原因：API 與安全模型改變。解法：搜尋針對 Vista 的新版或替代（如 Image Resizer Clone、IMAPIv2 工具）；臨時以內建功能替代；必要時以相容性模式或 Shim。預防：升級前列出替代清單並先在 VM 測試。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q5, B-Q20, A-Q9

D-Q8: 看不到 RAW 縮圖與預覽如何處理？
- A簡: 症狀：RAW 僅顯示圖示。安裝相機/官方 Codec 或改用支援 RAW 軟體。
- A詳: 原因：缺少對應 WIC 編解碼器。解法：安裝廠商 RAW Codec 或官方相機編解碼包；用第三方看圖軟體或轉為 DNG/JPEG；重新整理縮圖快取。預防：安裝可信 Codec 並維持更新。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q4, B-Q18, A-Q10

D-Q9: 找不到 cdburn/dvdburn 指令怎麼辦？
- A簡: 症狀：命令不存在。改用 IMAPIv2 程式庫、內建燒錄 UI 或第三方 CLI 工具。
- A詳: Vista 不再隨附舊工具。解法：以內建 UI 燒錄；開發/自動化則使用 IMAPIv2（C++/C# 範例可用）或第三方命令列（如 cdrtools/ImgBurn CLI）。預防：評估自動化需求，提早替換工具鏈。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q22, A-Q23, B-Q17

D-Q10: 降級/升級後驅動不相容如何診斷？
- A簡: 症狀：裝置代碼 28/未知。比對硬體 ID 下載正確驅動，必要時用通用或更換硬件。
- A詳: 1) 裝置管理員檢視硬體 ID（VEN_/DEV_）；2) 依 ID 搜尋製造商 XP/Vista 驅動；3) 若無，嘗試相容版本或通用驅動；4) BIOS/晶片組驅動先行；5) 仍不支援則評估替換硬體或維持原 OS。預防：升級前盤點相容清單。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, A-Q27
