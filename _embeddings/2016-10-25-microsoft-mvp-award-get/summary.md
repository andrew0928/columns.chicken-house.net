# 暌違多年的獎盃 - Microsoft MVP Award Get!

## 摘要提示
- 技術啟蒙: 作者原本對 Microsoft 無感，研究所時代接觸 NT 後態度大轉彎。
- 開發工具: Visual C++ 的高效率與 MSDN Library/Subscription 成為投入生態圈主因。
- 技術轉折: 自 Java 轉向 .NET 與 C#，追求「更好的 Java」理念。
- 部落格經營: 偏好長文與觀念解析，重視技術背後思維而非短期範例文。
- 社群參與: 參加競賽屢獲佳績並結識多位 MVP 前輩，累積影響力。
- MVP 評選: 早年著重論壇互動，作者因平台差異始終擦身而過。
- 意外得獎: 2016 年 10 月首度獲頒 Microsoft MVP，驚喜且感慨「來得晚」。
- MVP KIT 開箱: 分享首次收到的獎牌、證書與周邊，圓多年心願。

- 社群呼籲: 邀請讀者追蹤「安德魯的部落格」，支持未來內容。

## 全文重點
作者回顧自己與 Microsoft 技術之間近二十年的緣分。早年在 DOS 與 Windows 3.1 時期，他對 Microsoft 並不感興趣，甚至偏好在 DOS 下使用非微軟工具或改用 OS/2。直到研究所接觸 Windows NT 3.51 與 NT 4.0，他才認定 NT 系列足以與 Unix 系統抗衡，從此轉換陣營。  
Visual C++ 的高整合度、MSDN Library 的即時說明，以及 MSDN Subscription 可任意試用的軟體資源，成為 Microsoft「收買」開發者的三大誘因。作者因此建立起在 Windows 編譯、跨平台移植的流暢工作流，甚至在撰寫論文時捨 Sun JDK 改用 Microsoft Visual J++，並親歷 Microsoft 與 Sun 的 Java 官司，見證 .NET 與 C# 問世。  
退伍後，他徹底投入 .NET 技術棧，與鍵盤熱鍵為伍。隨著技術精進，他開始長期經營部落格，重視分享思路而非單純貼範例。2000 年代參與多項 Microsoft 技術比賽雖屢獲佳績，但始終與冠軍無緣；透過比賽認識多位 MVP 前輩後，更加深對社群的參與。  
早期 MVP 評選重視論壇回答度，使活躍於部落格的他多年來未能入選。直到 2016 年 10 月，他意外收到 MVP 通知信，並迎來首份 MVP KIT。雖自嘲「來得晚」且與前輩十年獎牌差距甚遠，他仍以此為新的起跑點，期許持續分享、精進與回饋社群。文章最終附上 KIT 開箱照及粉專連結，呼籲讀者繼續支持其技術內容。

## 段落重點
### 從 DOS 到 Windows：印象的起點
作者回憶 DOS/Windows 3.1 時代，對 Microsoft 缺乏好感；常以 DOS Prompt 或 OS/2 執行開發工具如 Turbo C、Borland C++。Windows 的系統穩定性與開發支援在當時皆不理想，因此他對微軟產品敬而遠之。

### NT 時代帶來的轉折
研究所時代接觸 Windows NT 3.51 與 NT 4.0，發現其穩定度足以作為工作站，遂開始改用 Microsoft 作業系統。此時他跳過 95/98/ME 系列，直接投入 NT 家族，為後續專業工作環境奠基。

### Visual C++ 與 MSDN：收買開發者的祕密武器
1.Visual C++ 提供優秀 IDE 與除錯效能；2.MSDN Library 的 F1 說明大幅提升查詢效率；3.MSDN Subscription 讓研究生能合法試用全套軟體。這三大利器說服作者長期留在 Windows 平台，也養成先在 VC++ 編譯再移植到 Solaris/gcc 的跨平台習慣。

### 從 Java 到 .NET：技術與生涯的抉擇
作者在論文研究階段使用 Visual J++，體驗到 IE JVM 的高效，親見 Microsoft 與 Sun 的訴訟導致 .NET 與 C# 誕生。退伍後，他擁抱 .NET Framework 與初代 Visual Studio.NET，兼具喜愛的物件導向與熟悉的 Windows 開發體驗。

### 部落格與社群：用文字累積影響力
相較短篇程式貼文，他偏好以長文闡述背後思考，認為觀念的價值能超越技術更迭。參加各類 Microsoft 比賽累積獎項也結識資深 MVP，並透過個人部落格持續分享，雖然論壇參與度低，影響力仍逐步擴大。

### 遲來的 MVP：十月驚喜
多年與 MVP 擦肩而過後，2016 年 10 月終於收到獲選通知與初次 MVP KIT。作者分享開箱照片，自嘲「終於有我的名字」，並以前輩十年獎盃為目標，期待繼續貢獻並邀請讀者追蹤「安德魯的部落格」，見證未來里程碑。
