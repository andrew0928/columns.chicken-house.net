# 升級 Vista ...

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 這次升級到 Vista 的核心目標是什麼？
- A簡: 以 Vista 作為桌機與媒體中心主機，伺服器服務改以虛擬機承載，兼顧日常使用與家庭伺服器功能。
- A詳: 目標是把原本低負載的家用伺服器（Windows Server 2003 x64）轉型為以 Vista Ultimate 為主的桌機與媒體中心（MCE）。同時，透過 Virtual PC（VPC）將 RRAS、IIS、SQL Express、DNS、DHCP、SMTP 等網路服務移至虛擬機執行，讓主機專注於媒體、多工與娛樂用途，並保留檔案伺服器、Volume Shadow Copy、批次備份、影片轉檔等功能。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, A-Q6

Q2: 為什麼從 Windows Server 2003 x64 換成 Vista？
- A簡: 因原機服務負載極低，轉為桌機更合適，並以虛擬化承載原伺服器角色，提升使用體驗。
- A詳: 原先主機僅提供網站、檔案分享等服務，CPU 與記憶體利用率長期低於 5%。轉用 Vista 可獲得更佳的桌面體驗與 MCE 功能，並以虛擬機承載原伺服器服務，維持功能不變。如此把高互動需求（上網、遊戲、MCE）與持續性伺服器工作隔離，兼顧體驗與穩定性，且硬體資源足夠支援此架構。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q1, A-Q6, A-Q15

Q3: 什麼是 Host OS 與 Guest OS？
- A簡: Host OS 是安裝在實體機的主系統；Guest OS 是在虛擬化平台上運行的作業系統。
- A詳: Host OS 指直接安裝在硬體上的主要作業系統（此案例為 Vista Ultimate），負責管理硬體資源與使用者日常操作。Guest OS 是在虛擬化軟體（如 Virtual PC）內運行的作業系統實例，用以承載特定服務或測試環境。此分層可實現角色隔離、降低衝突、便於備份與復原，並以較低成本延伸單機的功能。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q4, B-Q1, B-Q16

Q4: 什麼是 Virtual PC（VPC）？
- A簡: 微軟早期桌面虛擬化產品，支援 32 位元客體、單 CPU，提供簡易 VHD、網路虛擬化。
- A詳: Virtual PC 2007 是微軟的桌面型虛擬化軟體，可在 Windows 主機上建立與運行虛擬機，使用 VHD 做為虛擬磁碟。其特性包含：僅支援 32 位元客體、不支援 SMP（多處理器/多核心呈現）、提供 NAT/橋接等網路模式與簡易快照（Undo/Differencing）。適合承載輕量服務、測試與舊系統相容性需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q1, B-Q24, A-Q18

Q5: 什麼是 Media Center（MCE）？
- A簡: Windows 的多媒體中心功能，可看電視、錄影、節目表管理與影音整合。
- A詳: MCE 是整合影音娛樂的使用者介面與服務，在 Vista Ultimate（與 Home Premium）可用。它支援電視調諧卡、節目表（EPG）下載、錄影排程、媒體庫管理與遙控器操作。典型家庭客廳場景偏好靜音、睡眠喚醒、全螢幕操作，與高可用的伺服器角色在同一系統上常出現電源管理與資源競用的衝突。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, B-Q12, D-Q7

Q6: 為何網路服務與 MCE 被認為在同一 OS 上互斥？
- A簡: 功能定位、電源管理與相容性衝突，難同時滿足伺服器穩定與客廳體驗。
- A詳: 伺服器服務（RRAS、DNS、DHCP、IIS 等）需要長時穩定與網路可用性；MCE 強調睡眠喚醒、全螢幕、影音效能與周邊相容。兩者在電源策略、驅動與佈署權限上常相牴觸。再者，MCE 常見於用戶端版本，伺服器角色常見於伺服器版本，功能授權與元件也不同，因此實務上以虛擬化隔離最務實。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q1, A-Q15, B-Q19

Q7: 什麼是 RRAS？
- A簡: Windows 伺服器的路由與遠端存取服務，提供 NAT、VPN、路由等功能。
- A詳: RRAS（Routing and Remote Access Service）是 Windows Server 的網路基礎設施功能，涵蓋 LAN 路由、NAT 轉換、Site-to-Site 或遠端 VPN（PPTP/L2TP/IPsec/SSTP）、撥接等。家用場景可用於自建 VPN 回家、或內網多子網路路由；在本案例中放入 VM，避免與主機的 MCE、睡眠策略與驅動造成衝突。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q7, D-Q2

Q8: 什麼是 IIS？
- A簡: 微軟的 Web 伺服器與應用平台，支援 HTTP/HTTPS、應用程式集區與模組化管線。
- A詳: IIS（Internet Information Services）提供網站託管、應用程式管線（ISAPI/Managed Pipeline）、憑證與 TLS、應用程式集區隔離、日誌與管理工具。小型家用服務如相簿、輕量 API 或內部工具可用 IIS 承載。將 IIS 置於 VM 可控資源與安全邊界，便於備份、滾回與綁定固定 IP。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, C-Q5, D-Q5

Q9: 什麼是 SQL Server Express？
- A簡: 微軟免費的輕量級關聯式資料庫，限制資源上限，適合小型應用。
- A詳: SQL Server Express 提供 SQL 引擎、T-SQL、基本管理與本機服務，限制包含資料庫大小、CPU/記憶體使用上限。家庭或開發環境常用於小站台與工具。部署在 VM 中可易於移機與快照備份，也不會干擾主機的多媒體與遊戲工作負載。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q11, C-Q6, D-Q6

Q10: 什麼是 DNS、DHCP、SMTP，在家庭環境扮演何種角色？
- A簡: DNS 解析名稱，DHCP 派發 IP，SMTP 傳遞郵件；家用多由路由器提供前二者。
- A詳: DNS 將主機名轉為 IP，DHCP 自動配置 IP/閘道/DNS，SMTP 是郵件傳送協定。家用網多由路由器提供 DNS 轉遞與 DHCP，若自行架設需避免 DHCP 衝突並正確配置遞迴 DNS。SMTP 涉及安全與網路限制，家用多以外部郵件服務為主，內部 SMTP 僅作應用寄信中繼。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q8, B-Q9, B-Q10, D-Q3

Q11: 什麼是檔案伺服器與內建 RAID-1？
- A簡: 檔案伺服器分享資料夾；RAID-1 透過鏡像提升容錯，防單顆硬碟故障。
- A詳: 檔案伺服器以 SMB/NFS 分享文件、照片與影片。內建軟體 RAID-1（鏡像）在兩顆硬碟上同步寫入，任一顆故障時仍可存取，提升可用性而非備份。家庭資料（相片、DV 影片）典型做法是鏡像加上定期備份與 VSS「先前版本」，降低意外刪除與硬碟損壞風險。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q12, B-Q4, C-Q3

Q12: 什麼是 Volume Shadow Copy（陰影複製）？
- A簡: Windows 的快照技術，提供檔案「先前版本」，可快速還原誤刪與覆寫。
- A詳: VSS 透過寫入時複製（Copy-on-Write）在同一磁碟區保存當下狀態的差異，供系統還原、共享資料夾先前版本使用。其組件包含 Requester、Writer、Provider。對家庭檔案伺服器而言，可快速自助還原誤動的檔案；但快照不等於備份，仍需週期性離線備份以防勒索或磁碟毀損。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q5, C-Q4, D-Q8

Q13: 什麼是批次工作（Batch Job）？
- A簡: 以排程自動執行的任務，如每週備份、影片轉檔、資料清理等。
- A詳: 批次工作透過工作排程器在指定時間自動執行腳本或程式，避免手動操作遺漏。常見如每週備份到外接碟、夜間影片轉碼、日誌彙整。將其留在主機可善用閒置時間與多媒體卡編碼能力；也可放入 VM，依依賴與效能需求取捨。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q15, C-Q8, C-Q9

Q14: Vista Ultimate 32 位與 64 位差異是什麼？
- A簡: 64 位支援較大記憶體與 64 位驅動；32 位相容性佳。安裝媒體常同捆兩版本。
- A詳: 64 位 Vista 可使用更大記憶體、原生 64 位應用與強化安全（如 Kernel Patch Protection），但需 64 位驅動並可能有舊軟體相容性議題。32 位相容性普遍較佳。本文場景主機選用 Vista，虛擬機以 VPC 執行，因 VPC 不支援 64 位客體，相關服務選 32 位伺服器版本較穩妥。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q13, B-Q24, A-Q4

Q15: 為何選擇虛擬化承載網路服務？
- A簡: 隔離角色、避免衝突、便於備份與回復，兼顧主機的桌機與 MCE 體驗。
- A詳: 虛擬化讓伺服器服務與桌面工作在不同作業系統實例中執行，互不影響。若主機進行遊戲或影音，服務仍在 VM 維持可用。VM 方便做映像、差分磁碟與遷移，發生問題可快速回滾。也可限制 VM 資源，避免服務暴衝拖慢主機。此作法是家用環境成本效益高的解法。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q6, B-Q1, B-Q21

Q16: 為什麼 256MB/512MB 記憶體可跑這些服務？
- A簡: RRAS、DNS、DHCP、SMTP 等屬輕量服務，低同時併發時記憶體需求小。
- A詳: 家用或小型站台的 RRAS、DNS、DHCP、SMTP 與基本 IIS 網站屬 IO/網路導向工作，併發連線少，行程常駐但資源占用可控。相對於資料庫或大型應用，這些角色在 256–512MB 記憶體下仍可運作，前提是關閉不必要元件、精簡開機程式並監測使用量，必要時再調整。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q17, B-Q18, D-Q1

Q17: E6300 + 3GB RAM 的配置意味著什麼？
- A簡: 中階雙核與 3GB 記憶體足以同時跑桌面工作與一台輕量 VM。
- A詳: Intel Core 2 Duo E6300 提供兩核心，能分攤主機互動工作與 VM 的背景負載。3GB 記憶體讓主機保留約 2.5GB、VM 分配 0.5GB 仍可運行。雖 VPC 不支援 SMP，VM 只用單核心，但服務負載輕，整體體驗仍流暢。資源規劃要預留主機多媒體與遊戲高峰。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q16, B-Q2, B-Q17

Q18: 什麼是 SMP？VPC 不支援 SMP 的影響？
- A簡: SMP 為多處理器並行；VPC 不支援使客體僅用單核心，限制多執行緒效能。
- A詳: SMP（對稱多處理）允許作業系統調度任務到多個 CPU 核心。VPC 不支援向客體呈現多核心，故 VM 內所有工作集中於一核心，對 CPU 密集負載（如編碼、資料庫）影響明顯。對本案例的輕量網路服務影響較小，但仍需監控尖峰與延遲，必要時改用支援 SMP 的虛擬化平台。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, D-Q1, B-Q22

Q19: 什麼是橋接與 NAT 網路模式？
- A簡: 橋接讓 VM 成為實體網段成員；NAT 讓 VM 透過主機共享上網但內部位址。
- A詳: 橋接（Bridged）模式把 VM 直接接入實體 LAN，獲得同網段 IP，適合提供服務（DNS/DHCP/IIS）。NAT 模式讓 VM 使用主機私網位址，對外共享主機連線，易於隔離但不便被動連入。根據角色選擇模式，並避免 DHCP 衝突與路由混淆。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q3, C-Q2, D-Q2

Q20: 把家庭電視與桌機整合到 MCE 的價值是什麼？
- A簡: 一套介面集中影音、錄影與遙控體驗，簡化設備與操作。
- A詳: MCE 將電視、錄影、媒體庫整合於 PC，具節目表與自動錄影，亦可共用網路儲存。比起分散的機上盒與播放設備，PC 能彈性擴充與自動化（如轉檔、搬移）。不過與伺服器角色同機會牽涉電源管理與穩定性，故本案例採 VM 隔離服務以維持客廳體驗。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, B-Q12, C-Q10

Q21: Vista 作為桌機與媒體中心的核心價值是什麼？
- A簡: 驅動與多媒體相容性、MCE 功能、桌面體驗與廣泛軟體支援。
- A詳: Vista Ultimate 提供 Media Center、改良的多媒體堆疊、Aero 介面與更完整的驅動模型。對家庭使用者，能支援電視卡、遙控器與多媒體編解碼；對一般使用，應用相容性佳，搭配虛擬化承載伺服器服務，可同時達成娛樂與輕量伺服器需求。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q5, A-Q15, B-Q13

Q22: Windows Server 2003 x64 與 Vista 的定位差異？
- A簡: Server 著重伺服器穩定與角色；Vista 著重用戶端體驗與多媒體。
- A詳: Windows Server 2003 設計為伺服器 OS，提供 AD、RRAS、IIS 等角色與長期穩定運作，較少多媒體支援。Vista 是用戶端 OS，強調 UI、硬體相容性與影音功能。家用場景以 Vista 為主機、把伺服器角色放 VM，能兼顧兩者優勢，降低角色衝突。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q6, A-Q15, B-Q19

Q23: 將服務放入 VM 的風險與限制有哪些？
- A簡: 效能折損、SMP 受限、硬體直通有限、管理複雜度與備援需求增加。
- A詳: VM 帶來 CPU/IO 開銷，對高併發或密集任務可能受限；以 VPC 而言，無 SMP 與 64 位客體限制。硬體直通（如特定卡）困難，網路拓撲需謹慎。管理上多了一層映像與監控，備援要考慮主機故障的單點。權衡負載與平台能力是成功關鍵。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q18, B-Q24, D-Q1

Q24: 32 位與 64 位在 Host/Guest 選擇的考量？
- A簡: Host 盡量 64 位取資源；VPC 客體受限 32 位，選相容角色與軟體。
- A詳: 主機採 64 位可用更多記憶體與新驅動，利於多工與大型應用。VPC 僅支援 32 位客體，適合輕量角色（DNS/DHCP/IIS/SQL Express）。若客體需 64 位或多核心，應評估改用其他虛擬化平台。確保驅動相容與備援策略，避免升級後反受限。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, B-Q24, D-Q1

Q25: 為什麼把週備份與影片轉檔做成批次？
- A簡: 自動化降低遺漏，利用離峰資源，提高一致性與可預期性。
- A詳: 手動操作易遺漏且干擾日常使用。批次排程可在夜間低負載時自動備份或轉檔，並記錄日誌。這讓使用者更聚焦在主機互動工作，VM 繼續提供服務不受影響。妥善設定權限與錯誤通知可提升可靠度。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q15, C-Q8, C-Q9

Q26: 使用正版授權的重要性是什麼？
- A簡: 確保更新與安全、法遵與穩定相容，減少維護風險與功能限制。
- A詳: 正版授權確保可取得更新與修補、驅動與服務整合無障礙，亦符合法規與授權條款。對內含媒體功能與伺服器服務的環境而言，系統穩定與相容性尤為重要。升級時選購合適版本可避免功能缺失與後續重工。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q21, B-Q13

### Q&A 類別 B: 技術原理類

Q1: Virtual PC 是如何運作的？
- A簡: 以類型一之上的類型二虛擬化，模擬標準硬體，使用 VHD，提供基本網路與磁碟虛擬化。
- A詳: VPC 屬 Type-2 虛擬化，運行於 Host OS 之上，向客體提供模擬硬體（虛擬 CPU、NIC、IDE/SCSI、視訊等）。磁碟以 VHD 檔表示，支援固定、動態、差分。網路可為 NAT/橋接。若硬體支援 VT-x/AMD-V 可加速某些路徑。它不提供 SMP，故客體僅見單 CPU。核心組件：VM 管理器、VHD 子系統、虛擬網路驅動。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q4, B-Q16, B-Q24

Q2: 為何 VPC 不支援 SMP？CPU 調度如何影響效能？
- A簡: 架構限制無法向客體呈現多核心；多執行緒在單核心爭用，影響 CPU 密集工作。
- A詳: VPC 的 CPU 虛擬化僅提供一個 vCPU，簡化時間與中斷虛擬化，降低同步複雜度。結果是客體所有執行緒在單核心輪轉調度，碰上編碼、編譯或 DB 查詢時效能受限。Host 仍可用多核心分攤 VM 與其他工作。建議將 CPU 密集任務保留於 Host 或改用支援 SMP 的平台。核心組件：vCPU 模擬器、排程器。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q18, D-Q1, B-Q22

Q3: VPC 的虛擬網路如何工作（橋接/NAT/僅主機）？
- A簡: 透過虛擬 NIC 與主機網路堆疊連結，橋接直入 LAN，NAT 共享主機，僅主機隔離。
- A詳: 橋接模式將 VM 封包直接橋接到實體 NIC，取得同網段 IP，適合伺服器角色。NAT 模式由主機轉譯 VM 封包，對外隱藏 VM，適合測試。僅主機模式僅與主機互通。組件含虛擬交換器、過濾驅動、NAT 服務。關鍵步驟：建立 VM、選網路模式、配置 IP、測試連通。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q19, C-Q2, D-Q2

Q4: RAID-1 的原理是什麼？
- A簡: 寫入同時複製到兩顆碟，任一故障仍可存取；但非備份，防不了邏輯刪除。
- A詳: RAID-1 透過鏡像維持兩份一致資料，讀取可分散、寫入同步到兩碟。優點是容錯與讀取性能，缺點是磁碟成本加倍、寫入有額外負擔，且不防誤刪或勒索。Windows 透過動態磁碟提供軟體鏡像。步驟：轉動態磁碟、建立鏡像、監控重建。組件：磁碟管理、同步機制。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q11, C-Q3, D-Q8

Q5: VSS 的原理與流程為何？
- A簡: 寫入時複製差異，建立時間點快照；由 Requester、Writer、Provider 協同運作。
- A詳: VSS 流程：Requester 發起快照，協調 Writer（應用如 SQL、IIS）進行 Freeze/Thaw，Provider 執行底層快照（Copy-on-Write 或分離）。完成後提供一致性資料存取與先前版本。關鍵步驟：觸發、凍結 I/O、建立快照、解凍。組件如 vssvc、Writer 列表、存放目標。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q12, C-Q4, D-Q8

Q6: RRAS 的機制與常見模式（NAT/VPN）？
- A簡: 以內核路由與使用者模式服務提供 NAT、靜態/動態路由與 VPN 通道。
- A詳: RRAS 包含核心路由引擎、NAT 模組與 VPN 端點。NAT 將內部私網位址轉換為外部公網；VPN 以隧道（PPTP/L2TP/IPsec/SSTP）封裝內部流量。流程：啟用角色、配置介面與路由、設定憑證/金鑰、套用防火牆。組件：RemoteAccess 服務、RasMan、策略與路由表。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q7, C-Q7, D-Q2

Q7: IIS 的請求處理流程與架構？
- A簡: HTTP.SYS 接收，交由應用程式集區處理，管線模組過濾，回應由核心快取與記錄。
- A詳: 請求先經 HTTP.SYS（核心模式）排隊，路由到對應網站與應用程式集區（w3wp.exe）。Managed Pipeline 由模組處理驗證、授權、壓縮、路由等。回應可被核心快取。步驟：綁定位址/憑證、設定應用程式集區、部署站台、測試。組件：IIS 管理器、AppHost、模組/處理常式。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q8, C-Q5, D-Q5

Q8: DNS 的解析流程與快取如何運作？
- A簡: 客戶端查詢遞迴解析器，逐級詢問權威伺服器，結果快取以 TTL 控制。
- A詳: 客戶端查詢本機解析器，若無命中，詢問遞迴 DNS（路由器或本地 DNS 伺服器），自根伺服器起逐層查 NS 與 A 記錄，獲答後快取。錯誤或變更依 TTL 影響刷新。關鍵步驟：設定轉遞、區域與記錄、測試解析。組件：dnscache、DNS 服務、區域檔。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q10, C-Q2, D-Q4

Q9: DHCP 的租約分配流程是什麼？
- A簡: DHCPDISCOVER、DHCPOFFER、DHCPREQUEST、DHCPACK 四步驟完成位址租用。
- A詳: 客戶端廣播 DISCOVER，伺服器回應 OFFER；客戶端以 REQUEST 表達選擇，伺服器 ACK 確認並下發網路參數（閘道、DNS、租期）。需避免同網段多個伺服器造成衝突。步驟：設定作用範圍、排除、保留、選項。組件：DHCP 服務、租約資料庫、授權狀態。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q10, C-Q2, D-Q3

Q10: SMTP 郵件傳遞的背後機制？
- A簡: 用戶端與伺服器以 SMTP 對話，轉遞至目的域之 MX，收件伺服器再投遞。
- A詳: 寄件端以 SMTP 與中繼或權威伺服器通訊，依 MX 記錄解析目標伺服器；過程可能包含 SPF、DKIM、TLS 加密。家用 IP 常被封鎖，建議以外部 SMTP 中繼。步驟：設定中繼主機、驗證、限制轉發。組件：SMTP 服務、佇列、DNS。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q10, D-Q6

Q11: SQL Server Express 的架構與限制？
- A簡: 同核心引擎但受限資料庫大小、記憶體與 CPU 使用，多適用開發與小應用。
- A詳: Express 與完整版本共享引擎，提供 T-SQL、儲存程序、基本管理。限制如 DB 容量上限、單實例記憶體與 CPU 使用。對家用與小站足夠。步驟：安裝、設定埠/驗證模式、備份。組件：SQLSERVR 服務、SQL Browser、Management Studio。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q9, C-Q6, D-Q6

Q12: MCE 的節目表與錄影排程如何運作？
- A簡: 下載 EPG，依條件建立排程，喚醒系統錄影並管理錄影檔與衝突。
- A詳: MCE 定期更新節目表資料庫，使用者設定節目/關鍵字規則，系統於開機或喚醒後錄影。錄影檔寫入指定儲存路徑，遇時段衝突依優先順序。步驟：設定地區/訊號源、儲存位置、睡眠與喚醒。組件：eHome 服務、EPG 更新器、任務排程。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q5, C-Q10, D-Q7

Q13: Vista 32/64 位在驅動與系統組件上的差異？
- A簡: 64 位需簽章驅動與 WOW64 子系統執行 32 位程式，核心保護更嚴謹。
- A詳: 64 位 Vista 強制驅動簽章、啟用 Kernel Patch Protection、DEP 與 ASLR 更完善。32 位應用透過 WOW64 層運行。驅動相容性成關鍵，特別是影音卡與調諧器。選擇 64 位可獲更大記憶體，需確保周邊支援。組件：WOW64、驅動模型、簽章驗證。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q14, A-Q21

Q14: 檔案伺服器的 SMB 與 VSS 如何整合？
- A簡: SMB 提供分享，VSS 提供時間點快照，客戶端可檢視先前版本還原檔案。
- A詳: 伺服器端啟用影印集（Server）或系統保護（Client），在共享資料夾上可透過「先前版本」檢視快照並還原。SMB 會列出可用快照點。步驟：啟用快照、設定儲存配額、測試客戶端瀏覽先前版本。組件：LanmanServer、VSS Provider、陰影儲存。

- 學習階段: 核心
- 關聯概念: A-Q11, A-Q12, C-Q4

Q15: 工作排程器執行批次的原理？
- A簡: 以觸發條件喚起執行個體，套用安全性內容與環境，回報執行狀態。
- A詳: Task Scheduler 以觸發器（時間、事件）與條件（電源、網路）決定執行，採用指定帳戶與最高權限選項。能記錄結果與重試策略。步驟：建立工作、設定觸發/動作、測試、查看記錄。組件：Task Scheduler 服務、工作資料庫、事件記錄。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q13, C-Q8, D-Q9

Q16: 虛擬硬碟（VHD）格式與 I/O 行為？
- A簡: VHD 封裝磁碟為檔案，分固定/動態/差分，I/O 經主機檔案系統轉發。
- A詳: 固定 VHD 預先配置容量、性能穩定；動態 VHD 隨使用成長，易產生碎片；差分 VHD 依據父檔記錄變更，適合快照。I/O 先到主機檔案系統再落到實體磁碟，增加一層延遲。步驟：選擇適合類型、儲存於快速磁碟、定期壓縮/合併。組件：VHD 驅動、檔案系統。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q1, D-Q10

Q17: 主機資源分配與記憶體壓力如何影響 VM？
- A簡: VM 記憶體為保留塊，主機不足會引發分頁，影響 VM 與主機體驗。
- A詳: VM 啟動即保留設定記憶體，主機可用 RAM 下降。若主機餘裕不足，頻繁分頁會拖慢整體。規劃上需測量主機基線，留足餘裕給桌面與 MCE。步驟：監控工作管理員/資源監視器、調整 VM RAM、關閉不必要服務。組件：主機分頁檔、VM 進程工作集。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q16, D-Q1

Q18: 虛擬化下 I/O 瓶頸與最佳化機制？
- A簡: VHD 層與主機檔案系統疊加延遲；以固定 VHD、獨立磁碟與對齊改進。
- A詳: VM 的磁碟 I/O 經過多層轉譯，動態 VHD 與碎片化會惡化延遲。最佳化包含：使用固定 VHD、將 VHD 放置於獨立實體碟、避免主機防毒掃描 VHD、啟用寫入快取、對齊分割區與正確叢集大小。網路 I/O 則優化驅動與選擇橋接模式。組件：VHD 子系統、主機 IO 堆疊。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: C-Q1, D-Q10, B-Q16

Q19: 電源管理與睡眠對服務的影響？
- A簡: 睡眠會中止 VM 與服務連線，伺服器角色需避免或妥善喚醒。
- A詳: 客戶端系統預設會睡眠以省電；但睡眠會暫停 VM 進程，導致服務不可用。MCE 會自動喚醒錄影，但未必符合伺服器需求。策略：設定主機接通電源時不睡眠、關閉 USB 節電、允許定時器喚醒。組件：電源計畫、工作排程器、網路喚醒。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q6, C-Q10, D-Q7

Q20: 安全性隔離：把服務放 VM 的效益？
- A簡: 權限與網路邊界獨立，降低主機暴露面，便於快速回滾與事件鑑識。
- A詳: VM 為隔離邊界，服務在客體內運作，主機僅暴露必要埠。若 VM 受攻擊，可離線分析與還原映像。可套用不同更新節奏、權限模型與防火牆。步驟：最小化安裝、封鎖未用埠、定期映像。組件：主機防火牆、VM 虛擬交換器、VHD 快照。

- 學習階段: 進階
- 關聯概念: A-Q15, D-Q5, D-Q6

Q21: 升級與回復策略：快照/差分磁碟如何運用？
- A簡: 以差分磁碟或 Undo Disk 建立變更點，出問題可回滾至升級前狀態。

Q22: 影片編碼的工作負載特性與在 VM 的可行性？
- A簡: CPU 密集、可多執行緒；在無 SMP VM 受限，建議於主機執行。
- A詳: 影片轉碼大量使用 CPU 並具可平行性。VPC 的單 vCPU 對此不利，會拖長時間且影響服務延遲。最佳做法在 Host 以多執行緒工具（如 ffmpeg）批次處理，設定低優先序避免干擾互動。組件：編碼器、Task Scheduler、I/O 儲存。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q18, C-Q9, D-Q1

Q23: 家用環境中的 DNS/DHCP 與路由器如何協同？
- A簡: 路由器通常提供兩者；若自建伺服器，需關閉路由器 DHCP 或分段隔離。
- A詳: 多數家用路由器內建 DHCP/DNS 轉遞。若 VM 提供 DHCP，須避免同網段雙伺服器；可關閉路由器 DHCP、將 VM 設為主要 DNS/轉遞，或使用不同子網。若僅需內部名稱解析，可只架 DNS。步驟：盤點現況、明確決策、測試租約。組件：路由器設定、DHCP 作用範圍。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q2, D-Q3, B-Q9

Q24: VPC 的 32/64 位客體相容性限制？
- A簡: 僅支援 32 位客體，限制需 64 位或多核心的工作負載與新伺服器版本。
- A詳: Virtual PC 2007 不支援 64 位客體，亦不支援 SMP。因此無法安裝僅有 64 位的系統或需要多核心的角色。若需求超出（如大型 DB、AD DS 新版），應改用支援 64 位/SMP 的虛擬化（如 Hyper-V、VMware、VirtualBox）。組件：vCPU、客體能力呈現。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q4, A-Q24, D-Q1

### Q&A 類別 C: 實作應用類

Q1: 如何在 Virtual PC 建立一台 Windows Server VM 來承載服務？
- A簡: 建立新 VM、選固定或動態 VHD、安裝 Windows Server、安裝 RRAS/IIS/DNS 等角色。
- A詳: 具體步驟：1) 安裝 Virtual PC。2) 建立新 VM，分配 512MB RAM、選固定 VHD（性能較穩）。3) 設定網路為橋接。4) 開機由 ISO 安裝 Windows Server（建議 32 位版，VPC 限制）。5) 安裝角色：RRAS、IIS、DNS、DHCP、SMTP、SQL Express。6) 設定固定 IP、防火牆規則。7) 啟用 Windows Update。注意：拍攝差分磁碟做前後快照，避免變更難回復。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q1, B-Q16, C-Q2

Q2: 如何設定 VM 網路以提供 DNS/DHCP 又不干擾家用路由器？
- A簡: 使用橋接，關閉路由器 DHCP，或只啟用 DNS 並保留路由器 DHCP，避免雙重發放。
- A詳: 做法 A：橋接 VM，為其設定固定 IP，啟用 DHCP，並關閉路由器 DHCP，將 VM 設為 DNS（再轉遞 ISP DNS）。做法 B：僅在 VM 上啟用 DNS，保留路由器 DHCP，於路由器 DHCP 選項中指定 VM 為 DNS。做法 C：建立隔離子網，RRAS 路由。測試：ipconfig /all、nslookup。注意：避免兩台 DHCP 同網段運作。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q3, B-Q8, B-Q9

Q3: 如何在 Vista/Windows 版本上建立軟體 RAID-1（鏡像）？
- A簡: 在支援的版本中將磁碟轉為動態磁碟，建立鏡像，等待同步完成。
- A詳: 步驟：1) 備份資料。2) 磁碟管理中將兩顆資料碟轉為動態磁碟。3) 右鍵卷→新增鏡像，選擇另一顆磁碟。4) 等待重建完成。5) 設定電子郵件或事件通知監控故障。注意：系統磁碟鏡像與版本限制需先確認；鏡像非備份，仍需外部備份。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, C-Q8, D-Q8

Q4: 如何啟用與使用「先前版本」（VSS）保護共享檔案？
- A簡: 啟用系統保護或伺服器陰影複製，設定儲存上限，透過檔案屬性還原。
- A詳: 在 Windows Server：啟用「Shared Folders 的陰影複製」、指定磁碟與容量、設定排程。於客戶端檔案→右鍵→先前版本可還原。在 Vista 客戶端：啟用系統保護，系統自動建立還原點，也可在計劃任務觸發。注意：配置足夠空間，定期檢視快照並搭配離線備份。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q12, B-Q5, D-Q8

Q5: 如何在 VM 內安裝與設定 IIS 供內部網站使用？
- A簡: 啟用 IIS、建立網站、設定應用程式集區、綁定主機名與憑證。
- A詳: 步驟：1) 伺服器管理員勾選 IIS 角色（含管理工具）。2) 建立網站，指定根目錄。3) 設定應用程式集區（.NET 版本、管線模式）。4) 綁定主機名與埠（80/443），安裝憑證（HTTPS）。5) 防火牆放行 80/443。6) DNS 新增 A 記錄。7) 測試 http/https。注意最小權限、開日誌、限制目錄瀏覽。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, D-Q5, B-Q8

Q6: 如何安裝與設定 SQL Server Express（於 VM）？
- A簡: 安裝引擎與管理工具，啟用 TCP/IP，設定認證，建立備份排程。
- A詳: 步驟：1) 下載並安裝 SQL Server Express 與 SSMS。2) SQL Server Configuration Manager 啟用 TCP/IP，設定埠 1433。3) 設定混合驗證，建立登入。4) 防火牆開放 1433/SQL Browser。5) 建立資料庫與定期備份（T-SQL 或 SQLCMD + 排程）。6) 測試連線（sqlcmd/SSMS）。注意：限制最大記憶體、強制加密需憑證。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q11, D-Q6, C-Q8

Q7: 如何在 VM 內建置 RRAS 以提供 NAT/VPN？
- A簡: 啟用 RRAS、設定內外網介面，選擇 NAT 與 VPN 協定，配置路由與防火牆。
- A詳: 步驟：1) 安裝「路由與遠端存取」。2) RRAS 精靈選「NAT 與基本防火牆」與/或「遠端存取（VPN）」。3) 指定外網 NIC、內網 NIC。4) 設定 VPN（憑證/L2TP/IPsec）。5) 設防火牆與轉發埠。6) 測試內網上網與 VPN 撥入。注意：在家用環境需路由器埠轉發至 VM 外網 IP。
- 難度: 高級
- 學習階段: 進階
- 關聯概念: B-Q6, D-Q2, D-Q3

Q8: 如何規劃每週備份（robocopy + 排程）？
- A簡: 使用 robocopy 鏡像資料到外接碟，排程每週執行並寫入日誌，權限以最高權限。
- A詳: 腳本示例：
  robocopy D:\Data X:\Backup\Data /MIR /R:2 /W:5 /FFT /XJ /LOG:C:\Logs\backup.log
  步驟：1) 建立批次檔。2) 工作排程器新增工作，觸發「每週」，動作執行批次，勾選「以最高權限執行」。3) 條件選「喚醒電腦執行」。注意：測試還原、定期健康檢查與異地備份。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q15, D-Q9, A-Q25

Q9: 如何批次化影片轉檔（ffmpeg）並避開 VM 限制？
- A簡: 在主機以多執行緒 ffmpeg 轉檔，排程夜間執行，降低優先序避免干擾。
- A詳: 腳本示例：
  for %%f in ("D:\raw\*.ts") do ffmpeg -i "%%f" -c:v libx264 -preset slow -crf 20 -c:a aac "D:\out\%%~nf.mp4"
  步驟：1) 安裝 ffmpeg。2) 撰寫批次迴圈。3) 工作排程器排夜間執行，設定「低優先序」。4) 輸出至鏡像磁碟。注意：避免在 VM 內轉檔（VPC 單核），確保 I/O 不與錄影衝突。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q22, A-Q18, D-Q1

Q10: 如何設定 Vista MCE 的錄影儲存位置與共享？
- A簡: 在資料碟建立錄影資料夾，於 MCE 指定路徑，設定共享與權限，避免睡眠中斷。
- A詳: 步驟：1) 建立 D:\RecordedTV。2) MCE 設定→錄影→儲存位置選 D:\RecordedTV。3) 在檔案總管分享資料夾，設定唯讀/讀寫權限。4) 調整電源管理，允許喚醒錄影、連續電源。5) 如需供 VM 或他機取用，於防火牆開 SMB。注意：儲存可搭配 RAID-1 與 VSS。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q12, A-Q11, C-Q4

### Q&A 類別 D: 問題解決類

Q1: VM 僅單核心導致 CPU 滿載、服務延遲，怎麼辦？
- A簡: 降低 VM 工作負載、增加 RAM、移出 CPU 密集任務，或改用支援 SMP 的虛擬化。
- A詳: 症狀：VM CPU 長期 90%+、回應遲緩。原因：VPC 無 SMP、服務併發超出單核承受。解法：1) 把編碼等 CPU 密集任務改到 Host。2) 增 VM RAM 降低分頁。3) 限制 IIS 工作執行緒與連線。4) 拆分服務到第二台 VM。5) 若仍不足，改用 Hyper-V/VMware。預防：容量規劃、壓力測試。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: A-Q18, B-Q2, B-Q22

Q2: VM 無法上網或家用裝置互通異常，如何診斷？
- A簡: 檢查網路模式、IP 配置、DHCP 衝突與防火牆規則，逐層測試連通。
- A詳: 症狀：VM 無網路、其他裝置掉線。原因：橋接設定錯、雙 DHCP、路由/防火牆攔截。步驟：1) 確認橋接與實體 NIC。2) ipconfig /all 看 IP 來源。3) 檢查路由器 DHCP 是否與 VM 重複。4) 檢測 DNS 解析（nslookup）。5) 防火牆暫時停用測試。預防：唯一 DHCP、固定 IP、文件化拓撲。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q3, C-Q2, B-Q8

Q3: DHCP 衝突造成網路亂象，如何排除？
- A簡: 立即停用多餘 DHCP，清理租約，重啟受影響裝置並統一網段配置。
- A詳: 症狀：裝置獲得錯誤閘道或 DNS、互通不穩。原因：路由器與 VM 同時派發。解法：1) 關閉其中一個 DHCP。2) 清理 IP（ipconfig /release /renew）。3) 在保留清單設定固定裝置。4) 設定作用範圍與排除區間。預防：變更前公告、臨時關閉 Wi-Fi 以免裝置拿到錯誤租約。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q9, C-Q2, B-Q23

Q4: DNS 解析慢或失敗，如何處理？
- A簡: 檢查轉遞、根提示與快取；確認防火牆與 ISP DNS 可達，修正 TTL 與記錄。
- A詳: 症狀：開站延遲、間歇解析失敗。原因：無轉遞、快取污染、ISP DNS 不穩。解法：1) 設定可靠轉遞（如 ISP、公共 DNS）。2) 檢查區域記錄、TTL。3) 清除快取（ipconfig /flushdns、dnscmd /clearcache）。4) 檢視防火牆 53 埠。預防：監控遞迴延遲、備援轉遞清單。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q8, C-Q2, A-Q10

Q5: IIS 站台搬遷後出現 403/404，怎麼辦？
- A簡: 檢查檔案路徑、權限、應用程式集區與綁定，修正 URL Rewrite 與 MIME。
- A詳: 症狀：403 禁止或 404 找不到。原因：路徑錯、NTFS 權限不足、應用程式集區不對、主機名綁定錯。解法：1) 驗證實體路徑與應用程式集區版本。2) 調整 NTFS 與 IIS 權限。3) 確認網站綁定與 hosts/DNS。4) 檢查 URL Rewrite 與 MIME。預防：移轉前匯出設定、腳本化部署。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q7, C-Q5, B-Q21

Q6: SQL Express 連線逾時或失敗，如何排查？
- A簡: 確認服務執行、啟用 TCP/IP、開放防火牆、設定正確埠與驗證模式。
- A詳: 症狀：無法連線、逾時。原因：TCP/IP 未啟用、埠被防火牆擋、Browser 未啟用、認證錯。步驟：1) 檢查 SQLSERVR 服務。2) 啟用 TCP/IP 與固定埠。3) 開放 1433/UDP 1434。4) 測試 sqlcmd。5) 檢查連線字串。預防：監控埠狀態、限制來源 IP、定期更新。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q11, C-Q6, B-Q8

Q7: MCE 無法更新節目表或錄影失敗，怎麼解？
- A簡: 檢查網路、地區/訊號設定、儲存空間與睡眠喚醒；重建資料庫必要時重設。
- A詳: 症狀：節目表空白、錄影未觸發。原因：EPG 來源設定錯、網路不穩、空間不足、睡眠未喚醒。解法：1) 重新設定地區與訊號源。2) 測試網路與 DNS。3) 檢查錄影磁碟空間。4) 在電源選項允許計時器喚醒。5) 重建 MCE 資料庫。預防：固定錄影路徑、監控磁碟、排程更新。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q12, C-Q10, B-Q19

Q8: VSS 儲存空間不足或無法建立先前版本，如何處理？
- A簡: 增加陰影儲存空間、清理舊快照、檢查 Writer 狀態與磁碟健康。
- A詳: 症狀：無先前版本或建立失敗。原因：陰影儲存配額不足、磁碟碎片/錯誤、Writer 異常。步驟：1) 調高陰影儲存上限。2) vssadmin list writers 檢查狀態。3) chkdsk 修復。4) 刪除舊快照釋放空間。預防：合理配額、定期維護、外部備份避免只靠 VSS。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q5, C-Q4, B-Q4

Q9: 週期備份因 UAC 權限失敗或未執行，怎麼辦？
- A簡: 在排程啟用「最高權限」、使用具權限帳戶、確認路徑權限與喚醒設定。
- A詳: 症狀：任務顯示失敗、無備份輸出。原因：UAC 限制、帳戶無權限、離線磁碟。解法：1) 任務屬性勾選「使用最高權限」。2) 使用具本機系統或管理員帳戶。3) 確認目標磁碟連線與寫入權限。4) 啟用「喚醒電腦」。預防：日誌與郵件通知、定期測試還原。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q8, B-Q15, A-Q25

Q10: VHD 膨脹導致磁碟滿，如何處置？
- A簡: 清理 VM 內部檔案、壓縮或合併 VHD、改用固定 VHD 並移動至大容量磁碟。
- A詳: 症狀：主機磁碟爆滿、VM 異常。原因：動態 VHD 自動成長、日誌未清。解法：1) VM 內部清理與縮小磁碟。2) 對動態 VHD 執行壓縮（compact）。3) 合併差分 VHD。4) 改用固定 VHD。預防：定期清理、監控容量、將 VHD 放至獨立資料碟。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q16, B-Q18, C-Q1
