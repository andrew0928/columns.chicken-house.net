# 沒網路...

## 問題與答案 (FAQ)

### Q&A 類別 A: 概念理解類

Q1: 什麼是網路中斷（Outage）？
- A簡: 網路中斷指連線無法使用狀態，多因維護、故障、停電或上游異常所致。
- A詳: 網路中斷是指用戶無法透過既有連線存取網際網路或電話服務的狀態。成因包含計畫性維護、非計畫性設備故障、外力破壞、停電導致設備離線，或上游骨幹異常。特點可分全域、區域、個別用戶，或間歇性斷續。常見應對情境：先檢查供電與設備指示燈，再確認是否為公告維護，必要時撥打客服報修，並啟用行動熱點等暫時備援。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, A-Q3, D-Q1

Q2: 什麼是計畫性維護停機（Planned Maintenance）？
- A簡: 預先公告的服務暫停，為升級、割接或保養而短暫中斷。
- A詳: 計畫性維護停機是電信商為進行設備升級、線路割接、資安修補或例行保養，預先公告的短期服務中斷。通常安排於流量低的深夜時段，並提供預估開始與結束時間、影響範圍與服務類型。其價值在降低非計畫性故障風險，提升長期穩定性。用戶可提前準備備援連線與工作排程，減少影響。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q3, B-Q1, C-Q5

Q3: 計畫性與非計畫性停機有何差異？
- A簡: 計畫性可預告、可控；非計畫性突發、風險高、修復不確定。
- A詳: 計畫性停機事先規劃並公告，時間可控、風險受控，通常在夜間進行；非計畫性停機源於故障或事故，發生突然而難預測。前者目標是預防性維護、升級與可靠性改進；後者則著重於緊急處置與復舊。用戶在計畫性停機可提前安排備援，而非計畫性需即時排障與通報。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q12, D-Q9

Q4: 為什麼電信商常選在夜間施工維護？
- A簡: 夜間流量低、影響最小、維護窗口長，利於風險控制與回復。
- A詳: 夜間用戶量與流量最低，停機影響相對最小；同時夜間可取得較長維護窗口，減少趕工風險，有利於割接、搬遷與升級後的驗證回歸。夜間也便於調度資源、安排回退方案，確保即使遇到問題也能在清晨前恢復。對用戶而言，提前知悉夜間維護能妥善安排備援與工作。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q2, B-Q12, D-Q10

Q5: 沒繳費停話與故障中斷有何不同？
- A簡: 停話為帳務管制且可查詢；故障為技術異常，需排障修復。
- A詳: 未繳費停話屬帳務與合約控管，通常在到期後依約停機，可於客服或官網/APP查詢，補繳即可恢復。故障中斷由設備、線路或上游異常造成，非帳務問題，需技術檢修或等待復舊。用戶可藉通知簡訊、客服語音、帳務頁面或合約狀態分辨兩者，避免誤判延誤處理。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, D-Q1, D-Q9

Q6: 為什麼電話與網路會同時中斷？
- A簡: 常因共用本地回路或同一機房設備維護/故障所致。
- A詳: 固網電話（POTS/VoIP）與寬頻上網常共用相同的接入迴路、分纖器或同一接入機房設備（如DSLAM、OLT、MSAN）。若進行割接、停電、設備維護或出現上游鏈路故障，可能同時影響語音與數據服務。觀察兩者是否同期中斷，有助判斷故障在接入層或更上游。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, B-Q3, D-Q1

Q7: 什麼是本地回路（Local Loop）？
- A簡: 連接用戶端與電信機房的實體接入線路。
- A詳: 本地回路是用戶端（住家或辦公室）與電信商接入設備（交換局/機房）之間的實體連線，介質可能是銅線（DSL/POTS）或光纖（FTTH）。其品質、長度與施工品質會影響服務穩定性與速率。回路中斷、割接或端口故障，會造成同時影響多項服務的停機。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q3, B-Q4, B-Q5

Q8: 什麼是客戶終端設備（CPE）？
- A簡: 安裝於用戶端的網路設備，如ONU、數據機、路由器。
- A詳: 客戶終端設備（CPE）指部署在用戶端的網路硬體，包括光纖終端機（ONU/ONT）、DSL數據機、語音網關、家用路由器與交換器。其負責將電信接入轉為家中可用的數據與語音服務。CPE狀態燈、設定與韌體版本對連線穩定性影響甚鉅，是排障首要檢查點。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q8, D-Q7, D-Q8

Q9: 什麼是機房割接與線路施工？
- A簡: 於設備或纖配路進行切換/調整的工程作業。
- A詳: 割接是指將現有線路或服務切換至新設備、路徑或機房位置的作業，常伴隨光纖重熔、端口重配、交纜重接與設定遷移。施工期間會短暫中斷服務，通常列為計畫性維護並公告。割接完成後會進行驗證與回歸測試，若異常需執行回退計畫以降低風險。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q17, B-Q12, D-Q10

Q10: 什麼是服務等級協議（SLA）？
- A簡: 規範可用度、回應與修復時間的服務承諾。
- A詳: SLA是服務提供者與客戶間對可用度（如99.9%）、回應時間、修復時間（MTTR）與補償條款的約定。家用通常無嚴格SLA，企業專線則有明確指標。SLA有助對齊期望、界定責任及建立補償機制。理解SLA可協助規劃備援與風險管理。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q11, B-Q12, C-Q10

Q11: 什麼是平均修復時間（MTTR）？
- A簡: 從故障發生到修復完成的平均耗時指標。
- A詳: MTTR（Mean Time To Repair）衡量從故障發生、定位、修復到服務恢復之平均時間。它反映維運效率、備品備援程度與流程成熟度。降低MTTR需強化監測告警、標準作業、備援設計與替換零件管理。對用戶而言，理解MTTR可預估停機影響與備援需求。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, B-Q2, D-Q9

Q12: 什麼是網路可用度與停機時間？
- A簡: 可用度是正常運作比例；停機時間是不可用總時數。
- A詳: 可用度（Availability）是服務在觀察期間內維持正常運作的比例（如99.9%），停機時間則為不可用的總時數。二者相互對偶：可用度=1−停機比率。提升可用度仰賴冗餘、快速修復與預防性維護。用戶可藉此評估備援成本與業務容忍度。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q10, B-Q10, B-Q12

Q13: 什麼是事故通報與公告機制？
- A簡: 向用戶告知維護/故障情況的通知流程與渠道。
- A詳: 通報機制包含事前維護公告（時間、範圍、影響）、事中更新（進度、預估恢復）、事後報告（原因、改進）。渠道可為簡訊、Email、官網、APP推播與IVR語音。良好通報降低不確定性，提升信任與客訴防止。用戶可主動訂閱相關通知以便應變。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q5, D-Q10, B-Q14

Q14: 什麼是NOC（網路營運中心）？
- A簡: 24/7監控網路、處理異常與協調復舊的單位。
- A詳: NOC負責持續監控網路設備與鏈路，收集告警、流量與健康指標，並啟動事件處置、派工與跨團隊協作。NOC也管理維護窗口、通報與工單指派。其效能影響MTTR與用戶體驗。對外扮演資訊窗口，對內則是維運協調樞紐。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q2, B-Q13, A-Q11

Q15: 家用寬頻常見接入技術有哪些？
- A簡: DSL、Cable、FTTH/GPON、固定無線與行動4G/5G。
- A詳: 家用接入技術包含銅線的DSL（ADSL/VDSL）、同軸Cable、光纖到戶FTTH（多採GPON/XG-PON）、固定無線（FWA）與行動網路4G/5G分享。各自具不同頻寬、延遲、覆蓋與穩定性。光纖可用度與速率佳，行動網路適合作為備援或臨時連線。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q4, B-Q5, A-Q20

Q16: 光纖（FTTH）與DSL在維護上的差異？
- A簡: 光纖容量高抗干擾，DSL受線距與雜訊影響較大。
- A詳: FTTH使用光纖，具高頻寬、低延遲、抗電磁干擾，維護重點在分光器、熔接品質與OLT/ONT。DSL依賴銅線，速率受線距、線材品質與噪音影響，維護常涉及雙絞線端接、分離器與DSLAM端口。停機原因與排障方法因此有所不同。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q5, D-Q8

Q17: POTS 與 VoIP 電話有何不同？
- A簡: POTS走傳統交換語音；VoIP以IP封包傳輸語音。
- A詳: POTS（傳統固話）以電路交換於PSTN上傳輸，供電可能由局端提供；VoIP透過IP網路以SIP/RTP承載語音，需本地供電與網路品質。VoIP易受網路中斷影響，但具彈性與整合性。識別電話型態有助於分辨停電或停網下的可用性與排障重點。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q15, B-Q16, D-Q3

Q18: 停電與停網的差異與關聯？
- A簡: 停電是供電中斷；停網是連線中斷，兩者可相互影響。
- A詳: 停電指電力供應中斷，可能導致CPE、基地台或機房斷電，進而造成停網。停網則是網路連線不可用，未必伴隨停電（如上游骨幹異常）。若CPE與路由器接UPS可減少停電導致的停網，但若局端停電或光纖割接仍會中斷服務。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q11, D-Q1, C-Q9

Q19: 擁塞、封包遺失與完全中斷差別？
- A簡: 擁塞慢、遺失卡頓；完全中斷則無法連線與通話。
- A詳: 擁塞時延遲上升、速率下降但仍可連線；封包遺失造成影音卡頓、語音破裂；完全中斷則Ping無回應、PPP未連線或LOS告警。分辨型態可指引排障：擁塞關注QoS與流量；遺失排查介面錯誤與干擾；中斷則檢查供電、物理與上游。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q19, D-Q5, D-Q4

Q20: 為何需準備網路備援？
- A簡: 降低停機風險，確保關鍵工作與通訊不中斷。
- A詳: 即使最佳網路也可能遭遇維護、故障或不可抗力。準備備援（如行動熱點、雙WAN、離線流程與資料快取）能將影響降至最低，維持遠距工作、教學與關鍵服務連續性。依業務重要性決定投入，並定期演練切換確保有效。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q10, C-Q3, C-Q4

Q21: 行動數據熱點作為備援的價值？
- A簡: 快速啟用、覆蓋廣、成本低，適合短期備援。
- A詳: 行動熱點以4G/5G提供即時備援，優點是部署快、免布線、覆蓋廣，缺點是流量受限、延遲較高、受基地台負載影響。適合臨時上網、通訊與遠距會議的過渡方案。可與家用路由器搭配USB/LTE備援或手機分享以提升韌性。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q3, C-Q4, B-Q10

Q22: 客服申訴流程的核心步驟是什麼？
- A簡: 蒐集證據、報修登記、取得編號、追蹤與回報。
- A詳: 有效申訴包含：記錄異常時間與範圍、拍攝設備指示燈、測試結果；撥打客服或APP報修，取得工單編號；定期追蹤進度，必要時升級至主管或申訴單位；服務恢復後要求故障證明以便補償。完整資料能縮短排障與爭議處理時間。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q2, C-Q10, D-Q9

Q23: 停機對工作與生活的影響有哪些？
- A簡: 影響遠距會議、線上學習、交易、安防與通訊。
- A詳: 停機會中斷視訊會議、雲端協作、線上教學與金流交易，智慧家電、監控與門禁亦受影響。語音電話（特別是VoIP）可能無法撥打。瞭解影響面有助於制定優先級：確保替代通訊、資料離線可用、金融交易改走行動網路或暫緩。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q20, C-Q6, D-Q10

Q24: 使用者能做的停機前後準備有哪些？
- A簡: 前：訂閱公告、備援與離線。後：驗證、記錄與申請。
- A詳: 事前：訂閱維護通知、規劃行動備援/雙WAN、建立離線工作清單與資料快取、部署UPS。事中：切換備援、記錄證據、關閉非必要流量。事後：驗證功能、檢視帳務、記錄停機時長、申請補償、檢討改進備援與流程。持續演練可提升韌性。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q5, C-Q6, C-Q10

### Q&A 類別 B: 技術原理類

Q1: 計畫性維護的執行流程如何運作？
- A簡: 變更評估→公告→停機→執行→驗證→回報/回退。
- A詳: 原理是以變更管理控制風險：發起變更單，進行影響與回復計畫評估；核准後公告維護時窗與範圍；進入停機窗口，依步驟執行割接/升級；完成後做驗證與回歸測試（功能、性能、告警）；若失敗立即驅動回退；最後產出事後報告與知識庫更新。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q2, B-Q12, D-Q10

Q2: 電信商如何偵測並定位大範圍中斷？
- A簡: 透過監控告警、合併事件、拓撲關聯與主因分析。
- A詳: NOC透過SNMP/流量監控、主機心跳、合約SLA探測收集告警；事件關聯系統依網路拓撲與依賴關係合併多點異常，辨識同一上游節點；再以路由/光功率/端口狀態等指標定位區域或設備。最後開單派工，並同步公告與預估復舊時間。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q11, A-Q14, B-Q13

Q3: 家用網路「從牆到雲」的技術架構？
- A簡: 本地回路→CPE→路由器→ISP接入→骨幹→目標服務。
- A詳: 架構自用戶端起：本地回路（銅線/光纖）接至CPE（DSL Modem/ONT），再到家用路由器與交換器，透過PPPoE或DHCP連至ISP接入網，進入區域匯聚與骨幹，再路由至互聯網目的地（CDN/雲端）。任何一環異常都可能造成性能劣化或中断。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: A-Q7, B-Q4, B-Q5

Q4: DSL 的運作原理與關鍵組件？
- A簡: 利用銅線高頻載波，CPE與DSLAM調變解調傳輸。
- A詳: DSL在銅雙絞線上以多載波調變（如DMT）傳輸數據，透過分離器分離語音與數據頻段。用戶端CPE與局端DSLAM協商同步速率，受線距、雜訊、線材影響。關鍵組件：CPE、分離器、雙絞線、DSLAM、匯聚交換與BRAS。問題多表現為同步不穩或高錯誤率。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q16, D-Q8, B-Q6

Q5: FTTH/GPON 的運作原理與組件？
- A簡: 透過光分歧器PON架構，OLT與ONT以時分多工通訊。
- A詳: GPON採無源光網路，OLT於局端透過分光器服務多戶，與ONT/ONU以下行廣播、上行TDMA時隙通訊。關鍵組件：OLT、分光器（1:N）、光纖、ONT/ONU。光功率、LOS/LOSi指標與時隙配置影響穩定與速率。割接多涉及光纖熔接與時序重配。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q16, D-Q7, B-Q17

Q6: PPPoE 連線流程是什麼？
- A簡: PADI→PADO→PADR→PADS，驗證後建立PPP會話。
- A詳: PPPoE以以太網封裝PPP：客戶端送出PADI探測，AC回PADO；用戶端以PADR請求，AC以PADS確認並建立會話；接著進行PAP/CHAP驗證與IP配置。失敗常在AC無回應、驗證錯誤或中間阻擋。診斷可抓封包檢視互動與錯誤碼。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: D-Q6, C-Q7, B-Q7

Q7: DHCP 取得網路位址的機制？
- A簡: DORA流程：Discover、Offer、Request、Acknowledge。
- A詳: 用戶端廣播Discover，伺服器回Offer（含IP/租期）；用戶端以Request請求指定提供，伺服器以Ack確認分配。租約期滿需Renew。問題常見於伺服器不可達、位址池不足或交換器阻擋。檢視租約、伺服器可達與網段設定可排障。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q7, D-Q5, B-Q3

Q8: DNS 查詢解析如何工作？
- A簡: 遞迴解析器自根到權威查詢，回傳記錄快取。
- A詳: 客戶端向遞迴DNS請求，解析器若無快取，從根伺服器獲得TLD線索，再詢問權威DNS取得A/AAAA/CNAME等記錄，並依TTL快取以加速後續查詢。DNS異常會表現為「可連外但無法解析」。工具如nslookup/dig可協助診斷。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: D-Q5, C-Q7, B-Q9

Q9: Ping/Traceroute 診斷背後的原理？
- A簡: ICMP回應測可達與延遲；TTL遞減揭示路由跳數。
- A詳: Ping以ICMP Echo測試目標可達性與往返延遲；Traceroute藉由逐步增加TTL，觀察各跳點回應以描繪路徑。阻擋ICMP或回應率低不必然代表不可達，但對定位擁塞與斷點很有用。配合MTR可同時觀察延遲與遺失率變化。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q7, D-Q4, D-Q5

Q10: 路由器雙WAN備援的切換機制？
- A簡: 健康檢查連線，主線斷時自動切換至備線。
- A詳: 雙WAN透過連續健康檢查（Ping/DNS/HTTP）判斷主線可用性；當連續失敗達閾值即切換至備線，恢復後可自動回切或維持。機制包含負載平衡、策略路由與會話保持。關鍵在閾值設定與探測目標選擇，以避免誤判與震盪。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: C-Q4, A-Q20, B-Q12

Q11: 家用UPS如何為網路設備供電？
- A簡: 以電池逆變提供短時交流電，撐過停電斷檔。
- A詳: UPS將市電轉換為直流充電，停電時由電池逆變提供交流輸出，維持數據機、路由器供電。重點參數含額定瓦數、電池容量與切換時間。正確估算負載與續航，可讓短時停電不致停網；長時停電仍可能因局端斷電而失效。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q18, C-Q9, D-Q1

Q12: 維運變更的風險評估與回復計畫？
- A簡: 預估影響、制定回退步驟、演練與雙人覆核。
- A詳: 風險評估包含影響面、回復時間、相依服務、窗口長度與資源需求。回復計畫定義回退條件、步驟、時限與責任人，並預先驗證備援路徑。實施前進行演練與雙人覆核，可降低失誤。事後檢討納入標準作業改進。
- 難度: 中級
- 學習階段: 進階
- 關聯概念: B-Q1, D-Q10, A-Q13

Q13: 故障工單（Ticket）生命週期？
- A簡: 建立→分類→指派→處置→驗證→關單→回顧。
- A詳: 工單由告警或用戶報修建立，經分類定級與影響範圍評估，指派責任團隊處置。修復後由質保或客服驗證，與用戶確認恢復，最後關單並撰寫根因與改進。SLA監控回應/修復時間，統計用於趨勢分析與預防改進。

Q14: 公告與通知的技術通道設計？
- A簡: 多通道冗餘，依用戶偏好與事件等級彈性推送。
- A詳: 設計重點是可靠與即時：官網/狀態頁、Email、SMS、APP推播、IVR語音同步發布；根據等級控制頻率與詳盡度；支援Webhook/RSS供第三方整合。冗餘通道防單點失效，回饋機制收集用戶影響面，優化通報準確性。

Q15: 住家電話（POTS）與交換局連接機制？
- A簡: 雙絞線接入交換局，供電與語音由局端提供。
- A詳: POTS以銅雙絞線直連交換局，局端提供-48V供電與回路，話機摘機即建立電路連線。優點是停電時仍可通話（用傳統話機）。若本地回路中斷或局端設備停電，將無法撥打或接通。識別供電來源可判斷停電影響。

Q16: VoIP 電話的SIP/RTP工作流程？
- A簡: SIP建置與管理呼叫，RTP承載語音媒體封包。
- A詳: VoIP以SIP進行註冊、邀請、協商編碼與會話管理；呼叫建立後以RTP傳輸語音媒體。NAT/防火牆穿越常用STUN/TURN。網路延遲、抖動與遺失直接影響語音品質。SIP 4xx/5xx錯誤與RTP單向問題是常見故障型態。

- 學習階段: 進階

Q17: 光纖割接的技術步驟與保護？
- A簡: 熔接、清潔、測功率/OTDR，護纜與標示管理。
- A詳: 割接前標記纖芯，清潔端面；以熔接機對芯熔合；用功率計與OTDR測損耗與反射；完成後置入保護套管、走線與標示。作業需防塵、防彎折與靜電，並落實回退點設計。嚴謹流程可降低停機時長與後續隱患。

- 關聯概念: B-Q5, A-Q9, D-Q10

Q18: 復舊後的驗證與回歸測試流程？
- A簡: 功能、性能、告警、備援切換與使用者體驗驗證。
- A詳: 恢復後須檢查：基本可達（Ping/DNS/PPPoE）、性能（速率/延遲/抖動）、告警清除、日誌無新錯誤；測試備援切換與回切；抽樣撥測語音與常用應用（視訊/交易）。保留紀錄與對照基線，確認無回歸缺陷後方可結案。

Q19: 擁塞控制與QoS的基本原理？
- A簡: 排隊管理、優先權與速率限制，保障關鍵流量。
- A詳: QoS透過分類（DSCP）、排隊（WFQ/CBWFQ）、擁塞避免（RED）、整形與警戒（policing/shaping）來分配資源，確保語音視訊等低延遲流量優先。正確配置可減少高峰期卡頓，但無法彌補上游帶寬不足或中斷。

Q20: 用戶側如何區分本地與上游故障？
- A簡: 先測局域與CPE，再檢測WAN、上游與外部目標。
- A詳: 步驟：檢查電源與指示燈；LAN自測（同網段互Ping）；CPE狀態（DSL/LOS/PPPoE）；測DNS/外站；同時測多目標（ISP閘道、公共DNS、CDN）。若LAN正常、WAN異常，多為上游；若CPE告警則接入層；僅單站失效可能是目的端問題。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: C-Q7, D-Q1, D-Q5

### Q&A 類別 C: 實作應用類

Q1: 如何快速判斷是未繳費停話還是故障？
- A簡: 查帳務/簡訊/客服IVR，配合設備指示與多目標測試。
- A詳: 具體步驟：1) 登入電信APP/官網查帳務狀態；2) 檢查是否收到停話簡訊/Email；3) 撥客服IVR選「帳務查詢」聽取狀態；4) 檢視CPE指示燈（LOS/PPP/Internet）；5) 以行動網路測試外站。若帳務正常且CPE異常，多為故障，立即報修。

- 學習階段: 基礎

Q2: 如何向電信商報修並提供關鍵資訊？
- A簡: 準備時地範圍、設備燈號、測試結果與聯絡方式。
- A詳: 報修前記錄：發生時間、影響服務（網路/電話）、範圍（全家/單台）、CPE燈號（LOS/PPP/Internet/Voice）、測試（Ping/DNS/重啟）。撥客服或APP報修，提供住址、聯絡方式，取得工單編號。後續追蹤並回報任何變化，保留證據便於補償。

Q3: 如何設定手機熱點作為臨時備援？
- A簡: 開啟4G/5G熱點，設強密碼，連接電腦或路由器。
- A詳: iOS/Android：設定→個人熱點→啟用，設WPA2密碼。電腦連Wi‑Fi即可上網。進階：部分路由器支援USB Tethering，開啟「USB網路共享」後插入路由器USB口，於路由器設為備援介面。注意流量資費、耗電與熱點密碼安全。

Q4: 如何在家用路由器設置雙WAN備援？
- A簡: 啟用WAN2，設定健康檢查與切換/回切策略。
- A詳: 步驟：1) 將行動網卡/第二上游接至WAN2；2) 管理介面啟用「雙WAN/備援」；3) 設定健康檢查（Ping 8.8.8.8與ISP閘道，3/5失敗判斷斷線）；4) 選擇Failover或Load Balance；5) 開啟會話保持。範例：track 1 host 8.8.8.8 up 3 down 5。測試切換與回切。

Q5: 如何訂閱電信商維護/故障通知？

Q6: 如何建立停機應變清單與離線工作流？

### Q&A 類別 D: 問題解決類

Q1: 遇到「電話與網路全斷」怎麼辦？
- A簡: 檢查供電與CPE燈號，改用備援並立即報修。
- A詳: 症狀：無法上網、無撥號音。可能原因：局端維護、光纖/銅纜中斷、停電、帳務停話。步驟：檢查電源與UPS、觀察LOS/DSL/PPP燈；以行動網路查公告與帳務；啟用熱點備援；撥客服報修拿工單。預防：雙WAN、訂閱公告與UPS。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: C-Q1, C-Q3, A-Q6

Q2: 只有網路斷、電話正常的可能原因？
- A簡: PPPoE/DHCP故障、ISP上游異常或CPE設定錯誤。
- A詳: 症狀：語音通暢、上網失敗。原因：PPP驗證失敗、DHCP租約問題、DNS異常、路由器設定錯誤、ISP上游故障。處置：重啟CPE/路由、重設PPPoE帳密、更新DNS、檢視WAN狀態。預防：備份設定、強化健康檢查與通知。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q6, B-Q7, C-Q7

Q3: 只有電話斷、網路正常的可能原因？
- A簡: VoIP註冊失敗、話機設備故障或POTS回路異常。
- A詳: 症狀：上網正常但無法通話。原因：VoIP SIP註冊失敗、RTP單向、語音閘道壞、POTS回路接觸不良。步驟：檢查語音門號狀態、重啟語音閘道、檢視SIP註冊與錯誤碼、改測行動電話。預防：穩定供電、韌體更新與設定備份。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q16, A-Q17, C-Q2

Q4: 網路反覆斷線、時好時壞如何處理？
- A簡: 先排除供電與本地干擾，再請電信測線品質。
- A詳: 症狀：間歇斷線。原因：供電不穩、纜線鬆脫、噪音干擾、線路品質差、上游擁塞。步驟：更換電源供應器、檢查線材與端子、遠離干擾源、固件更新、監測Ping/MTR。若仍異常，要求電信測功率/噪音/誤碼率。預防：用優質線材與UPS。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: A-Q19, B-Q4, B-Q5

Q5: 速率極慢但未完全斷線的診斷方式？
- A簡: 分辨擁塞或丟包，檢測DNS/最後一哩與上游。
- A詳: 症狀：慢、卡。原因：尖峰擁塞、丟包、DNS慢、Wi‑Fi干擾。步驟：有線直連測速；Ping閘道/公共DNS/外站比較延遲；MTR觀察遺失；更換DNS；檢查Wi‑Fi干擾；停用高流量應用。預防：QoS策略、時段避尖峰、升級頻寬或備援。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: A-Q19, B-Q19, C-Q7

Q6: PPPoE 驗證失敗的原因與解法？
- A簡: 帳密錯、AC無回應、線路異常，重設與報修。
- A詳: 症狀：PPPoE連不上，錯誤651/691等。原因：帳密錯誤、AC端故障、PPPoE封包被阻、CPE異常。步驟：核對帳密、重啟CPE與路由、直連PC撥接測試、抓封包檢視PADI/PADO互動；若AC無回應即報修。預防：備份設定與定檢韌體。
- 難度: 中級
- 學習階段: 核心
- 關聯概念: B-Q6, C-Q7, C-Q2

Q7: 光纖ONU/ONT LOS燈紅或閃爍怎麼辦？
- A簡: 代表光訊號異常，檢查纖端清潔與彎折後報修。
- A詳: 症狀：LOS紅燈或閃爍，上網/語音中斷。原因：光纖斷/鬆、端面污染、過度彎折、局端割接或停電。步驟：確認CPE供電正常；輕拔插清潔光纖端面（不可觸碰鏡面）、恢復彎曲半徑≥3cm；仍異常即報修。預防：保護纖線、避免拉扯。
- 難度: 初級
- 學習階段: 基礎
- 關聯概念: B-Q5, B-Q17, C-Q8

Q8: DSL Modem 只能同步不上網怎麼辦？
- A簡: 有同步無上網，多為PPP/DHCP/DNS問題。
- A詳: 症狀：DSL燈常亮，Internet滅。原因：PPP驗證失敗、DHCP拿不到IP、DNS壞。步驟：檢查PPPoE帳密、改測DHCP動態或手動DNS（8.8.8.8）；重啟路由與Modem；直連PC測試撥接。如仍無解，報修請求檢查DSLAM/BRAS。預防：備份設定。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q4, B-Q6, C-Q8

Q9: 維護完仍未恢復服務如何追蹤？
- A簡: 先自檢與重啟，復測後持工單升級並索取說明。
- A詳: 症狀：公告已結束仍不可用。原因：個別端口未回復、設定殘留、回退失敗。步驟：重啟CPE/路由、清DNS快取、測外站；若仍故障，以原工單回報，請NOC核對端口/設定與光功率；要求預估恢復時間與後續補償。預防：完工驗證清單。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q18, A-Q13, C-Q2

Q10: 大夜維護影響遠超公告時間怎麼辦？
- A簡: 問明原因與E.T.R.，啟用備援並依合約求償。
- A詳: 症狀：維護延誤。原因：不可預期問題、回退失敗或外力。步驟：查狀態頁與客服訊息，要求提供E.T.R.（預估恢復時間）與影響說明；在業務關鍵時啟用備援；事後索取RFO（故障原因）與依合約申請補償。預防：自建多路備援與離線方案。
- 難度: 初級
- 學習階段: 核心
- 關聯概念: B-Q12, A-Q24, C-Q6
