# 困難重重的 x64

## 摘要提示
- 記憶體擴充: 加裝 4 GB 後，系統偵測與相容性問題陸續浮現。  
- BIOS Memory Remap: 開啟後雖可看到 6 GB，卻導致電視卡訊號異常。  
- TV 卡相容性: 圓剛卡在 Vista x64＋Remap 情況下無法正常收視。  
- 驅動支援不足: ASUS 有回覆但無解，圓剛則完全未回應。  
- x64 缺點: 程式需同時安裝 x64／x86 版本、Pointer 變長吃記憶體、COM 走 WOW 降效能。  
- 退回 x86: 電視卡恢復正常，但系統僅能使用 2.8 GB～3 GB RAM。  
- PAE＋RamDisk: Gavotte Ramdisk 能把 4 GB 以上「被吃掉」的記憶體轉成 RAMDisk。  
- 臨時解法: 留 2 GB 作系統記憶體，4 GB 成為 TEMP 用 RAMDisk。  
- 實務抉擇: 現階段 x64 問題多，建議待生態成熟再全面換上。  

## 全文重點
作者原本在 Vista x64 環境下運作順暢，但當主機板插滿 6 GB 記憶體並開啟 BIOS Memory Remap 後，災難接踵而來。雖然作業系統能完整識別 6 GB，卻突然發現 MCE 電視卡「訊號微弱」而無法收看。經還原設定、一一排除後，確認問題並非硬體故障，而是 TV 卡驅動在「x64＋Memory Remap」組合下無法正常運作。向 ASUS 與圓剛求助只得到「無解」或「已讀不回」，加上軟體多需同時安裝 x64 與 x86 版本、Pointer 擴大造成記憶體耗用、COM 組件多半仍走 WOW64 導致效能下滑，作者最終決定退回 Vista x86。  

回到 32 位元後，電視卡立即恢復正常，可惜因 4 GB 限制只剩 2.8 GB 可用；若再開 Memory Remap 反而只剩 2 GB。搜尋社群後，作者找到免費的 Gavotte Ramdisk。啟用 PAE 後，系統雖仍只認 2 GB RAM，但 Ramdisk 能把高位址的 4 GB「挖」出來，變成一顆大 RAMDisk。雖然把 Page File 放在 RAMDisk 顯得多此一舉，仍可將 TEMP、快取等寫入頻繁資料導向其中，多少把閒置記憶體變現。  

種種經驗讓作者認為 x64 仍有太多相容性與驅動問題，除非急需大量記憶體，否則暫時留在 x86 更省事；待記憶體普遍突破 4 GB、廠商全面移轉 x64 時，再接納 64 位元環境會是更務實的選擇。

## 段落重點
### 加裝 4 GB 後的初步喜悅與疑惑
作者將原本順暢的 Vista x64 系統升級為 6 GB RAM，初期僅偵測到 4.8 GB，開啟 BIOS Memory Remap 後終於看到完整 6 GB，滿心以為功德圓滿。然而沒過兩天，MCE 卻無法收視電視節目，明顯與加記憶體存在關聯，讓「爽度」瞬間打折。  

### BIOS Memory Remap 與電視卡衝突
透過逐步回溯設定，作者發現只要開啟 Memory Remap，圓剛 TV 卡在 Vista x64 下就會被 MCE 判定訊號微弱；關閉 Remap 又能正常收看。Device Manager 不報錯顯示問題並非硬體本身，而是驅動在 x64＋高記憶體位址環境中的缺陷。向華碩與圓剛客服求援卻得到一個「回覆無用，一個無回應」的尷尬結果。  

### Vista x86 的折衝與記憶體上限
無奈之下作者重灌 Vista x86，電視卡立即恢復正常。不過 32 位系統最多可見 3 GB 左右 RAM；更離奇的是只要打開 Memory Remap，實際可用記憶體竟降到 2 GB。網路上有不少同型板友遇到同樣情況，可見並非個案。硬體不支援、驅動不到位、作業系統限制三者交錯，讓人對 x64 生態產生遲疑。  

### Gavotte Ramdisk 與 PAE 的權宜之計
為彌補浪費掉的記憶體，作者試用免費的 Gavotte Ramdisk。啟用 PAE 後，系統雖仍只能分配 2 GB RAM，但 Ramdisk 可自動掛載高於 4 GB 的隱藏記憶體，形成一塊 4 GB 的超大 RAMDisk。雖說把 Page File 搬進 Ramdisk 顯得多此一舉，但將 TEMP、瀏覽器快取等高 I/O 檔案放入仍可提升反應速度，也算不讓昂貴的記憶體閒置。  

### 對 x64 生態的觀察與未來展望
作者總結現階段使用 x64 的阻礙：1) 軟體與 DLL 需同時備有 x86、x64 版本占空間；2) Pointer 變長導致記憶體耗用增加；3) 依賴 COM 的程式需透過 WOW64，效能略降；4) 部分硬體驅動在 Memory Remap 或高位址條件下不穩定。除非有超過 4 GB 的剛性需求，否則留在 x86 仍是最省心方案。隨著記憶體價格走高與市場需求攀升，廠商勢必加速推出完善的 x64 驅動與軟體支援，屆時再全面升級才是理性抉擇。
