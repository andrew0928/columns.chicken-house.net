# 敗家: HTPC

## 摘要提示
- 動機與痛點: 多螢幕與數位輸出需求複雜，桌機難以同時穩定驅動雙DVI+HDMI，促成另組專用HTPC。
- 平台選擇: 看上AMD 780G整合平台，整體成本較換主機板+高階顯卡便宜。
- 採購清單: 以現有零件搭配新入手M3A78-EM、AMD 4850e、4GB RAM、優質電源、MCE搖控、HDMI線與喇叭。
- 省略TV卡: 家中其他電腦已有MCE+TV卡，HTPC以播放檔案/錄影為主，不看Live TV故暫不購買。
- 耗電表現: 待機1–2W、Vista待命50–55W、影音播放60–65W、極限約80W，整體相當省電。
- MCE搖控體驗: 即插即用、支援喚醒/睡眠與常用功能，足以取代鍵鼠進行影音與相片播放。
- 手機作遙控: 以Vista SideShow + WM5/6手機，能在不開電視下瀏覽媒體庫並播放音樂。
- 補強差異: SideShow可在手機上顯示節目表與曲目信息，彌補無螢幕搖控器的資訊不足。
- 整合方案與成本: 支援SideShow的MCE搖控器售價高昂，作者改採廉價搖控器+手機組合。
- 給讀者建議: 有WM5/6手機與HTPC需求者可立即嘗試；HTPC玩家交流Tips，非玩家也可趁廉價組裝入坑。

## 全文重點
作者長期想打造HTPC，但因桌機已接雙螢幕，若再接LCD TV需同時提供雙DVI與HDMI輸出，等同更換主機板並加裝兩張中價位雙DVI顯卡，成本與複雜度過高；加上MCE有時要在桌上LCD顯示、有時要在TV顯示，操作切換繁瑣，因此認清需要一台專用HTPC。作者發現AMD 780G平台性價比佳，決定另組新機：ASUS M3A78-EM主機板、AMD 4850e低耗電CPU、DDR2 4GB、Seasonic 330W電源、Toshiba MCE搖控器、HDMI線與JungleRex喇叭，並沿用舊機殼、光碟機、HDD等，扣除奢侈配件整體可在一萬元內完成。因家中已有兩張TV卡，且很少看Live TV，HTPC暫不裝TV卡。

實際體驗上，780G平台耗電出色：開機約85W、進入Vista後50–55W、播放DVD/MPEG4時60–65W、極端負載75–80W，待命1–2W。播放DVD與MPEG4畫質令人滿意。操作上主打「無鍵鼠」：MCE搖控器即插即用（含Vista x64），綠色按鍵可喚出MCE，並以專用按鍵完成音樂、相片與影片的播放；電源鍵可控制睡眠/喚醒，使得起停快速便利。進一步，作者利用Vista SideShow預覽版與Microsoft的MCE SideShow Gadget，讓Windows Mobile 5/6手機化身SideShow裝置，能在手機上瀏覽節目表、看到播放曲目與媒體資訊，不必中斷大螢幕畫面；特別適合只聽音樂時，不開電視即可在手機上選專輯並播放，解決傳統無螢幕搖控器資訊不足的問題。雖有支援SideShow規格的高階MCE搖控器可一體化，但售價高（約USD 199），作者改用低價搖控器搭配手機與藍牙接收器的方案。最後呼籲有WM5/6手機與HTPC需求者不妨立即嘗試，並邀請HTPC族交流使用心得與技巧。

## 段落重點
### 動機與硬體考量：雙DVI+HDMI帶來的困擾
作者原本想直接以桌機充當HTPC，但桌機已接雙螢幕，要再接液晶電視就涉及第三個輸出端，而且三個輸出都需要數位訊號（兩台桌上LCD為DVI；SHARP電視僅提供HDMI或色差，不支援D-Sub），迫使系統走向兩張顯示卡、雙DVI的中價位卡組合，成本與規格大幅提高。即便硬體勉強就位，實務操作仍不便：MCE有時要在桌上LCD、有時在TV顯示，切換麻煩又不穩定。作者因此體認，若不另組專用主機，HTPC只能是短暫嘗試，無法長期穩定使用。相較之下，看到坊間對AMD 780G平台的介紹與評測，發現以整合晶片組達成影音需求，成本低於更換主機板加雙顯卡，於是萌生另組專屬HTPC的念頭，準備以此解決多輸出與使用情境切換的根本難題。

### 採購清單與成本配置：以舊帶新、萬內成軍
實際採購時，作者利用手邊備料，僅補齊關鍵元件：ASUS M3A78-EM主機板（NTD 2750）、AMD 4850e低功耗CPU（NTD 2250）、DDR2-800 2GB×2（單條NTD 990）、Seasonic 330W電源（NTD 1700）、Toshiba MCE搖控器（NTD 550）、3米HDMI線（NTD 130），以及JungleRex AF-31S喇叭（NTD 3800）。硬碟沿用Hitachi 7K250 250GB，機殼、光碟機、藍牙接收器與鍵鼠也沿用舊件。MCE常見的TV卡反而沒買：其一，市售卡價位相對偏高；其二，作者與家中長輩的桌機均已有MCE+TV卡可錄影，客廳很少看Live TV，故暫時不需要第三張。扣除較為「奢侈」的電源與喇叭，整體預算可壓在一萬元以內，展現以整合平台打造入門HTPC的高性價比。

### 使用體驗與耗電表現：省電且影音品質佳
完成組裝後，搭配既有液晶電視的觀影體驗顯著提升：不論播放DVD（MPEG2）或MPEG4/DivX影片，畫面表現都令人滿意。作者特別強調780G平台與45W TDP處理器的低功耗優勢：開機階段約85W，進入Vista後降至50–55W；播放DVD或AVI時僅60–65W；在執行高負載測試（Benchmark）時才會提升到75–80W；待命（Sleep）狀態則只需1–2W，對全天候待機的HTPC而言非常省電。此功耗曲線意味能在客廳環境中長時間待命、隨按即用，同時控制電力成本。影音播放則在不額外使用高階離散顯示卡的情況下，達成良好輸出品質，符合家庭娛樂中心對靜音、省電與穩定的需求。

### 無鍵盤滑鼠的客廳操作：MCE搖控器即插即用
為了達成客廳「遠距、簡單、一鍵直達」的體驗，作者採用Toshiba MCE搖控器，接收器插上即可用，無需額外驅動與設定，且支援Vista x64。搖控器的綠色按鍵可直接喚出MCE介面，常用功能皆有對應按鈕，足以完成影片、音樂（取代Windows Media Player）與相片（取代Windows Live Photo Gallery）的日常播放。電源鍵可控制系統的睡眠/喚醒，達到「一按即醒、再按即睡」的家電化體驗；相較完全關機與開機，睡眠只耗約1W電力，換取秒開即用的流暢性更符合客廳情境。雖然部分TV卡（如康博）可直接對應電源開關，但整體啟停時間仍不及睡眠喚醒。綜合而言，MCE搖控器已能滿足多數影音操作，實現「免鍵鼠」的核心目標。

### 用手機當遙控：Vista SideShow + WM5/WM6的妙用
作者進一步介紹以Windows Mobile 5/6手機配合Vista SideShow功能，讓手機成為SideShow裝置；再安裝Microsoft提供的MCE SideShow Gadget（Beta），即可在手機端顯示節目表、播放資訊與媒體庫內容。這彌補了傳統無螢幕搖控器的資訊缺口：例如不想中斷大螢幕播放時，手機即可顯示正在播放的曲目或節目資訊；更關鍵的是「不開電視聽音樂」的場景，使用者可在手機上直接瀏覽媒體櫃、選擇專輯與下達播放，客廳即刻播音，真正做到「離屏操控」。對已擁有WM5/WM6裝置與藍牙接收器（成本百元級）的用戶，這是一套幾乎零門檻的擴充方案。作者亦鏈接相關介紹與下載資源，說明SideShow在MCE應用的便利性與實用價值。

### 整合兩種遙控與成本權衡：高價一體 vs. 平價組合
市面上存在同時支援MCE與SideShow規格的一體化遙控器，可在單一硬體上兼具按鍵操作與資訊顯示，但售價高昂（約USD 199）。作者因此採取成本效益更高的替代組合：廉價MCE搖控器負責日常「沙發操作」的直覺與速度，WM手機透過SideShow負責資訊瀏覽與「離屏操控」（如不開電視聽音樂）。兩者優勢互補，花費卻遠低於高階一體機。最後，作者鼓勵已有WM5/WM6手機的HTPC使用者動手嘗試，也邀請HTPC玩家分享更多技巧；對尚未入門者，則以「萬內可成軍」與「體驗顯著提升」作為入坑動機。

## 資訊整理

### 知識架構圖
1. 前置知識：
- 基本 PC 組裝概念（主機板/CPU/RAM/電源/儲存/機殼）
- 顯示輸出介面差異與需求評估（DVI/HDMI/D-SUB/色差）
- Windows Vista + MCE（Media Center）基本操作
- 省電與待命（Sleep）/喚醒概念
- 藍牙配對與 Windows Mobile 5/6 裝置操作基礎（若用 SideShow）

2. 核心概念：
- HTPC 專用機的必要性：從桌機多顯示輸出困境轉向專用機，簡化操作與穩定性
- 成本效益硬體選擇：以 AMD 780G 平台與低耗電 CPU 組成，兼顧影音效能與省電
- MCE 遠端操控生態：實體 MCE 遙控器 vs. Vista SideShow（WM5/6 手機）互補
- 低功耗與待命使用模式：常態 Sleep 喚醒以換取即時可用與更佳使用體驗
- 不含 TV 卡的 HTPC 用法：聚焦檔案播放（DVD/MPEG4/MP3/相片），錄影靠其他桌機

3. 技術依賴：
- 影音輸出：主機板內顯（AMD 780G）→ HDMI 連接 LCD TV
- 系統軟體：Windows Vista + MCE → 媒體播放與 UI 操作
- 遙控層：MCE 遙控器（USB 接收器驅動免設定）→ 直接喚醒/睡眠與快鍵
- 進階遙控：Vista SideShow 驅動與 MCE Gadget（Beta）→ 依賴 WM5/6 手機與藍牙
- 聲音輸出：外接喇叭系統（如 JungleRex AF-31S）
- 省電：硬體低 TDP CPU + 高效率電源供應器 + 作業系統 Sleep

4. 應用場景：
- 客廳影音中心：DVD、DivX/MPEG4 影片播放、MP3 與相片瀏覽
- 無螢幕播放控制：不開電視以手機（SideShow）瀏覽媒體庫並播放音樂
- 單鍵使用體驗：遙控器一鍵叫出 MCE、Sleep/喚醒快速進出
- 多裝置家庭：錄影由其他 MCE + TV 卡的桌機負責，HTPC 專注播放端

### 學習路徑建議
1. 入門者路徑：
- 理解 HTPC 定位與需求（是否需要 TV 卡、輸出介面組合）
- 挑選入門硬體：APU/內顯主機板、低功耗 CPU、2×2GB RAM、可靠電源
- 安裝 Windows Vista 與啟用 MCE，測試 HDMI 連線與音訊輸出
- 購買並設定 MCE 遙控器（接收器插上即可），熟悉基本操作與 Sleep/喚醒

2. 進階者路徑：
- 優化耗電：檢視待機/播放/壓力下功耗，調整電源管理
- 建立媒體庫結構與編碼格式策略（DVD/MPEG4/MP3/相片）
- 設定 Vista SideShow + MCE Gadget，讓 WM5/6 手機成為無螢幕控制器
- 微調顯示與音訊輸出（解析度、Overscan、音量正規化）

3. 實戰路徑：
- 在客廳部署完整線路（HDMI、喇叭、遙控器接收器、藍牙接收器）
- 制定使用流程：開機→遙控進 MCE→播放→Sleep；聽音樂則改用 SideShow 手機
- 與家中其他 MCE 錄影機整合：共享錄影檔與媒體檔案
- 長期維運：定期清理媒體庫、檢查 Sleep/喚醒穩定性與遙控器相容性

### 關鍵要點清單
- 專用 HTPC 的必要性: 桌機多螢幕與操作複雜度高，專用機可大幅簡化體驗 (優先級: 高)
- 顯示輸出規劃: 同時滿足 DVI×2 與 HDMI 的難度高，HTPC 以單 HDMI 對 TV 更務實 (優先級: 高)
- 平台選擇（AMD 780G）: 內顯與成本效益佳，足以應付 DVD/MPEG4 等播放 (優先級: 高)
- 低功耗 CPU（如 4850e 45W）: 播放情境下全機約 60–65W，省電且安靜 (優先級: 高)
- 電源供應器品質: 高效率穩定度勝過大瓦數，降低整體功耗與噪音 (優先級: 中)
- 記憶體容量: 4GB 雖略超過需求，但價格帶動下具性價比 (優先級: 低)
- 不含 TV 卡策略: 若已由其他電腦負責錄影，HTPC 可專注播放功能 (優先級: 中)
- MCE 遙控器體驗: 即插即用、綠鍵直達 MCE、支援 Sleep/喚醒，日常操作最順 (優先級: 高)
- Vista SideShow + WM5/6: 手機顯示節目表/曲目並可無螢幕操控，聽音樂時特別實用 (優先級: 中)
- 遙控器與 SideShow 互補: 有螢幕的手機控制補足傳統遙控器的可視化資訊 (優先級: 中)
- 成本控管: 扣除「奢侈品」後一萬元內可成系統，聚焦剛需零件 (優先級: 高)
- Sleep 常態化: 1–2W 待命可換取即時可用，優於每次冷開機等待 (優先級: 高)
- HDMI 連線品質: 使用合適長度與品質的 HDMI 線材確保穩定顯示與音訊 (優先級: 低)
- 媒體中心一體化: 將 MP3/相片等操作也整合進 MCE 界面，統一入口 (優先級: 中)
- 升級的取捨: 合一型 SideShow 遙控器雖方便但昂貴，手機+遙控器更具成本效益 (優先級: 中)
