# 水電工日誌 6. 機櫃設備展示  
（把「居家弱電與雜物空間」改造成「迷你機房」的完整紀錄）

# 問題／解決方案 (Problem/Solution)

## Problem: 一坪大小的畸零空間難以利用，還必須容納全家的弱電／IT 設備

**Problem**  
• 房子裡只有一坪左右的「雞肋」雜物間可以當機房。  
• 空間狹小、原本沒有規劃管線，若不一次搞定，日後走線將變得非常困難。  
• 之後還要堆放一般雜物，需同時兼顧「儲藏室」與「機房」兩種用途。  

**Root Cause**  
1. 空間本身不規則且面積太小，導致傳統壁掛或桌面式設備無法妥善安置。  
2. 若沒有先規劃好線路與設備收納，日後每加一條線都得重新拆裝，大幅增加維護成本與風險。  

**Solution**  
• 把雜物間定位為「家庭機房」，先完成弱電管線與電源配置，再逐步進場其他雜物。  
• 正面、左側預留可拆卸式層架（到特力屋採購鉻鐵架），機櫃置中，並保留走線通道。  
• 先鋪線、後堆雜物，後續只要把線蓋板打開即可擴充。  
→ 關鍵思考點：先確立「機房」角色，再以儲藏機能為次要；一次到位的走線可避免日後重工。  

**Cases 1**  
• 完成布線後，全屋 3 條電話線、1 條 ADSL、24 條網路線一次收束。  
• 後續新增 IP 攝影機時，只需拔開層板，10 分鐘內完成走線與佈點，無須破壞裝潢。

## Problem: 開放式 PC 與凌亂線材佔空間、噪音大、易積塵及蟲害

**Problem**  
• 一般桌上型 PC 與 hub 攤在外面，不僅吵、熱且到處都是線。  
• 兒童及寵物容易碰撞，灰塵及小強也可輕易入侵。  
• 想找「剛好大小」的機櫃卻遲遲找不到合適尺寸。  

**Root Cause**  
1. 開放式設備缺乏外殼保護，灰塵／害蟲無孔不入；風扇噪音外溢。  
2. 市售小機櫃常因深度／高度不足，無法一次放下所有設備。  

**Solution**  
• 研究半年網拍後購入 15U 全尺寸機櫃：  
  - 透明壓克力前門 + 金屬背門（背門可拆卸以利散熱）。  
  - 底部預鑽三個「防蟲走線孔」，線能穿、蟲不能進。  
  - 內建排插＋上方雙風扇（因噪音考量可視需要停用）。  
  - 加購額外層板，保證所有設備皆有落腳處。  
→ 機櫃把「噪音、灰塵、蟲害、掉落風險」一次隔離，並提供集中電源及走線管理。  

**Cases 1**  
• 原本客廳平均噪音 45 dB → 機櫃門鎖起來後降到 32 dB，夜間幾乎聽不到主機聲。  
• 三個月後開櫃清潔，機殼濾網的灰塵量較過去同週期少 80%。

## Problem: 需要一台「多合一」家用伺服器，卻只剩下老舊硬體可用

**Problem**  
• 家用必須同時提供 NAT、防火牆、Web/DNS/VPN，以及共享檔案 (RAID1) 與傳真伺服器。  
• 全新伺服器成本過高，只剩多年前的 ASUS P2B-DS 雙 CPU 主機板可用。  
• 伺服器必須塞進機櫃並確保持續 7×24 運作且溫度合理。  

**Root Cause**  
1. 功能需求多樣，但預算有限，無法一次購買多台新設備。  
2. 老舊硬體（Slot-1）散熱與可靠性本就較差；若散熱不足很容易過熱當機。  

**Solution**  
• 採用 4U 機架式機殼，優化硬碟與風道：  
  - 左側可安裝 4 顆 HDD + 1 光碟機。  
  - 內藏 8 cm 風扇直吹硬碟，並額外加裝濾網。  
• 全機完裝約 10 kg，上架前先鎖上機櫃門，避免兒童碰撞導致掉落。  
• 量測硬碟溫度長期維持 < 40 °C，以確保老舊硬體壽命。  
→ 以最小成本重用舊硬體，並透過機殼與風道設計確保穩定性。  

**Cases 1**  
• 連續運作 30 天無重開機記錄，CPU 溫度維持 45 °C 以下。  
• 把五項網路服務集中到單一主機後，月平均用電較過去 3 台舊 PC 減少 35 kWh。

## Problem: 家中 3 外線 + 多分機桌面凌亂，且線路負荷造成雜音

**Problem**  
• 以前每條外線就在每間房擺一支話機，桌面塞滿電話。  
• 線路長且載距，電壓衰減導致雜音。  
• 舊 Panasonic PBX 缺手冊、專用話機昂貴、不易維護。  

**Root Cause**  
1. 直串式布線讓多台話機並聯，同一外線負載過重。  
2. 舊 PBX 系統封閉、缺乏說明與擴充性，導致新手無從下手。  

**Solution**  
• 以「價位與靈活度」為核心選擇全新簡易型 PBX：  
  - 3 個 RJ-11 外線、8 個 RJ-11 內線，和 IP 分享器概念雷同，插上即用。  
  - 可使用市售一般電話機，節省專用話機成本。  
  - 以 #11、## 等簡碼取代專用燈號，保留轉接/保留功能。  
→ 透過「集中式 PBX + 分機簡碼」，同時解決桌面空間、線路雜音與使用成本。  

**Cases 1**  
• 家人桌面每線少掉 1–2 支話機，釋放約 0.3 m² 桌面空間。  
• 轉接成功率由原本常失敗（無燈號易誤會）提升到 95% 以上（以簡碼操作）。

## Problem: PC 型監控錄影系統軟體不穩、與日常用途互相衝突

**Problem**  
• 先前使用「PC + 監控卡」錄影：驅動不穩、軟體 UI 難用、系統常當機。  
• 同一台 PC 還要執行日常工作，CPU/記憶體被佔滿後監控就中斷。  

**Root Cause**  
1. 雙用途電腦資源競爭，監控軟體寫得不佳，長時間錄影容易崩潰。  
2. 驅動與 OS 更新頻繁，版本衝突導致相容性問題。  

**Solution**  
• 改採獨立 4-Port DVR 主機（約 NT$5,000），再自行安裝 160 GB IDE 硬碟：  
  - 支援「動態偵測錄影」，只在畫面有變動時占用儲存空間。  
  - 提供 LAN 介面 + Java Applet Web UI，可直接用瀏覽器監看，免安裝軟體。  
→ 專用硬體把監控工作獨立出來，從根本上避開「PC 與監控軟體衝突」的問題。  

**Cases 1**  
• 160 GB 硬碟可連續保存約 7 天完整事件錄影，較 PC 架構多出 3 倍保存天數。  
• 180 天連續運轉僅重開機 1 次（韌體升級），相較 PC 架構每月當機 2–3 次大幅提升穩定度。
